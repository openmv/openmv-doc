v2.1.0¶

v2.1.0 隨附了 UVC 網路攝影機韌體、可感知連線狀態的連線按鈕，以及重新設計的 WiFi/開發板設定對話方塊，同時還有更新的範例集與升級至 3.2.0 的隨附韌體。本版本仍以 Qt Creator 4.0.2 為基礎。它重新組織了相機端 WiFi/開發板設定的儲存方式，因此現有的 WiFi 設定需要重新輸入（參見破壞性變更）。

重點¶

  • 現在為所有開發板提供 隨附 UVC 韌體，使 OpenMV Cam 可燒錄為標準 USB 網路攝影機運作。

  • 可感知連線狀態的連線按鈕 現在會根據偵測到的相機顯示不同的僅 USB、僅 WiFi、USB+WiFi 或未連線狀態。

  • 隨附相機韌體 為 OPENMV2、OPENMV3 與 OPENMV4 開發板從 3.0.0 更新至 3.2.0。

  • 重新設計的 WiFi/開發板設定 將儲存內容拆分為獨立的 BoardConfig 與 WiFiConfig 群組，並提供更長的密碼欄位與獨立的 WiFi 偵錯切換選項。

  • Thermopile Shield 範例 新增了 MLX90621、MLX90640 與 AMG8833 熱感測器示範。

  • LabVIEW 整合（LabVIEWOpenMV）現在隨安裝程式一併提供。

新功能¶

  • UVC 網路攝影機韌體：現在為所有開發板提供隨附的 UVC（USB Video Class）韌體映像檔，使 OpenMV Cam 可燒錄為標準 USB 網路攝影機運作。為 OPENMV2、OPENMV3 與 OPENMV4 新增了 uvc.bin、uvc.dfu 與 uvc.elf 檔案（3995eb570）。

  • 可感知連線狀態的連線圖示：連線工具列圖示現在會動態反映連線狀態，根據偵測到的 USB 相機與探索到的 WiFi 相機顯示不同的僅 USB、僅 WiFi、USB+WiFi 或未連線狀態（8428d9823）。

  • LabVIEW 整合：LabVIEW 整合套件（LabVIEWOpenMV）現在作為新的資料目錄隨 IDE 安裝程式一併提供（0740fbb4b、4c593eec9）。

  • 新增了 範例指令碼，涵蓋 USB HID 滑鼠/鍵盤、依需求進行 USB VCP 影像傳輸、VSYNC GPIO 輸出、Motor Shield（TB6612 馬達與步進馬達控制）以及選擇性搜尋特徵偵測（3995eb570）。

  • 70-Math 範例 現在隨附純 Python 線性代數與矩陣函式庫（ulinalg.py 與 umatrix.py），供相機端進行數學運算（3995eb570）。

其他變更與改進¶

  • WiFi/開發板設定對話方塊 已重新設計：設定被拆分為獨立的 BoardConfig 與 WiFiConfig INI 群組並採用新的鍵名，密碼欄位最多可接受 64 個字元，SSID/開發板名稱上限為 32 個字元，且有獨立的 WiFi 偵錯切換選項可不受用戶端/存取點模式影響地控制偵錯輸出（1be001c7e）。

  • WiFi 連線處理 經過強化，加入了 3 秒的 TCP 連線逾時，並將 WiFi 連接埠淘汰逾時調整為 20 秒（8428d9823），此外在序列埠抽象層中加入 DTR/RTS 控制線支援（在 TCP 上為無作用）並在開啟序列埠時觸發 DTR（c39c9c880）。

  • 深度睡眠低功耗範例 現在還會讓影像感測器進入軟睡眠並關閉電源（PWDN），並可選擇性地繞過 OV7725 穩壓器以降低耗電量，RTC 喚醒間隔由 5 秒提高至 30 秒（3995eb570）。

  • 特徵偵測範例 移除了舊的 LeNet find_numbers.py 數字辨識示範，改以新的 selective_search.py 範例取代（3995eb570）。

錯誤修正¶

  • 修正了 WiFi 相機探索的可靠性：廣播資料包現在必須以 null 結尾，位址會以 IPv4 進行比對，重複的 WiFi 連接埠不再加入清單，且在強制進入開機載入程式時會隱藏 WiFi 連接埠（47ed75b8d）。

  • 修正了相機設定中空白的開發板名稱問題，現在會預設為 OpenMV Cam 而非儲存空白名稱（66f56c420）。

平台與工具支援¶

  • Qt Creator 基礎版本： 4.0.2。

  • 隨附韌體： 為 OPENMV2、OPENMV3 與 OPENMV4 將 OpenMV 相機韌體從 3.0.0 更新至 3.2.0（3995eb570）。

  • Thermopile Shield 感測器： 新範例新增了 MLX90621、MLX90640 與 AMG8833 熱感測器的示範（相機與疊加層，可選 LCD），取代舊的通用 fir.py/fir_lcd.py 範例；fir 模組參考文件另涵蓋 MLX90620（3995eb570、57532428e、65237a257）。

  • 隨附 UVC 韌體 與 LabVIEW 整合套件 現在隨安裝程式一併提供（參見新功能）。

破壞性變更¶

v2.0.0 與 v2.1.0 之間有使用者可見的工作流程變更。相機端 WiFi/開發板設定儲存方式已重新組織：設定從單一的 BootSettings 群組移至獨立的 BoardConfig 與 WiFiConfig INI 群組並採用新的鍵名，WPA (Modern WiFi Security) 選項已從存取點安全性下拉式選單中移除（用戶端下拉式選單仍提供 Open (No password)、WPA (Modern WiFi Security) 與 WEP (Hexadecimal password)），且 WiFi 模式現在是單純的用戶端與存取點二選一，而非合併的位元遮罩。由舊版 IDE 寫入的現有相機端 openmv.config WiFi 設定不會被遷移，需要重新輸入（1be001c7e、66f56c420）。
