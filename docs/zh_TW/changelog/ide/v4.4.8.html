v4.4.8¶

v4.4.8 是一個維護版本，更新了隨附的韌體與工具。它繼續以 Qt Creator 14.0.2 為基礎，此範圍內未進行任何 rebase。隨附的 OpenMV 韌體升級至 v4.6.0，首次隨附 STM32N6（OpenMV Cam N6）韌體，新增對 FLIR Boson 熱像感測器的辨識，並更新了 Alif Security Toolkit。沒有任何面向使用者的破壞性變更，現有的使用者專案不受影響。

重點摘要¶

  • 隨附韌體 v4.6.0： 隨附的 OpenMV 韌體與範例由 v4.5.9 更新至 v4.6.0（66bfd486f）。

  • 隨附 OpenMV Cam N6 韌體： STM32N6（OpenMV Cam N6）韌體現在首次隨附於套件中（66bfd486f）。

  • FLIR Boson 熱像感測器： IDE 現在可辨識 BOSON、BOSON-320/BOSON-640 以及 BOSON-320+/BOSON-640+（Boson Plus）熱像感測器（8b795053c、d5068b286）。

  • Alif Security Toolkit v1.101.0： 隨附的 Windows 工具套件更新至 v1.101.0（7d51b00b1）。

  • 修正 OpenMV Cam AE3 SBL 燒錄： AE3 板現在在燒錄時會執行正確的安全開機載入程式（ed7c95357）。

新功能¶

  • FLIR Boson 範例： 在 01-Camera/05-Thermal-Cameras/02-FLIR-Boson 下新增了 FLIR Boson 灰階與 ironbow 串流以及色彩追蹤的隨附範例指令碼，並在 01-Camera/03-Event-Cameras/02-Genx320 下新增了 GenX320 灰階 LED 追蹤範例（66bfd486f）。

其他變更與改進¶

  • 重新整理了隨附的範例目錄結構：FLIR Lepton 範例移至新的 01-Camera/05-Thermal-Cameras/01-FLIR-Lepton 類別中，與新的 Boson 範例並列；事件相機範例則被拆分至 01-Camera/03-Event-Cameras/01-Frogeye2020 與 02-Genx320 子資料夾中（66bfd486f）。

  • 從套件中移除了未使用且無人引用的 OPENMV4_PRO 韌體資料夾；沒有任何板子項目引用它，因此不影響任何可燒錄的板子（66bfd486f）。

  • 更新了隨附的 OpenMV AE3 SBL 韌體、開機載入程式以及用於燒錄的已簽署映像檔（b8c06a8ab）。

  • 將隨附的 Alif Security Toolkit（Windows）更新至 v1.101.0，包含更新後的使用者指南（AUGD0005）與已簽署的 SP 映像檔（7d51b00b1）。

錯誤修正¶

  • 透過將其板子架構字串設為 OPENMV AE3 SBL，修正了 OpenMV Cam AE3 SBL 燒錄問題，使其能執行正確的安全開機載入程式（ed7c95357）。

  • 在燒錄 i.MX RT 板時，將 blhost 的快閃記憶體抹除逾時時間由 60 秒提高至 120 秒，避免在抹除與燒錄較大區域時發生逾時（b48e139e4）。

平台與工具支援¶

  • Qt Creator 基礎版本： 14.0.2。

  • 隨附的 OpenMV 韌體： 由 4.5.9 更新至 4.6.0（66bfd486f）。

  • OpenMV Cam N6： STM32N6 韌體（firmware.bin、network.bin、openmv.bin）現在隨附於 firmware/OPENMV_N6 下（66bfd486f）。

  • OpenMV AE3 SBL： 更新了韌體與開機載入程式的二進位檔（b8c06a8ab）。

  • Alif Security Toolkit（Windows）： 由 v1.100.0 更新至 v1.101.0（7d51b00b1）。

  • 新增可辨識的感測器： FLIR BOSON、BOSON-320、BOSON-640（8b795053c）、FLIR BOSON-320+ 與 BOSON-640+（d5068b286），以及 PixArt PS5520（66bfd486f）。

破壞性變更¶

無。沒有任何指令碼 API 變更，現有的使用者專案不受影響。被移除的 OPENMV4_PRO 韌體資料夾未被使用且無任何板子引用，因此其移除不會破壞任何燒錄流程。
