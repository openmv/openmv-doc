5.15. 形态学运算¶

形态学（Morphological） 运算作用于二值图像——即阈值化和边缘检测输出的掩膜。每个运算都遍历与平滑滤波器所用的同一种滑动邻域，但它在每个位置提出的问题是“是/否”：邻域中是否 每个 像素都为开，邻域中是否 有任何 像素为开，开/关的图案看起来像什么？这些答案能以平均滤波器无法做到的方式扩张区域、收缩区域并重新切割它们的边界。

形态学是介于 初始 二值掩膜——即阈值化、边缘检测或某种其他分类器的输出——与流水线其余部分可以使用的 干净 二值掩膜之间的环节。原始的阈值输出通常在真实前景区域中散布着孤立的噪声像素，在本应实心的区域上打着小孔，并且在阈值靠近物体边缘处切割时留下锯齿状的边界。形态学可以去除这些缺陷。

5.15.1. 四种经典运算¶

两种基本运算，以及它们的两种组合，构成了形态学工具集：

dilate() 扩张 每个前景区域。规则是：在其 (2 * size + 1) 窗口内至少有一个前景邻居的任何像素都会变成前景。可见的效果是前景区域在每个方向上扩大 size 个像素，而它们内部的 孔洞 则收缩（或消失）相同的量。

erode() 则相反。窗口内并非 每个 邻居都在前景中的任何像素都会变成背景。前景区域在每个方向上缩小 size 个像素，孤立的前景像素（它们没有前景邻居）完全消失，较大区域之间的细小连接也会被切断。

open() 是 先腐蚀后膨胀。腐蚀后的图像去除了每个孤立的噪声像素，但它也在每个方向上缩小了 size 个像素。在腐蚀之后再做相同 size 的膨胀，可以将真正的前景区域恢复到大致原来的边界，同时让噪声保持消失状态。正是这种组合使得 open 成为经典形态学中标准的“去噪”运算：孤立的前景像素消失，真实区域完好地恢复。

close() 是其镜像——先膨胀后腐蚀。膨胀填充前景区域内部的小孔洞，并连接被小间隙分隔的区域；腐蚀则将结果收缩回其原来的外边界，同时让填充好的内部保持实心。close 是标准的“填充小间隙”运算。

binary_mask.open(1)       # remove single-pixel noise
binary_mask.close(2)      # fill small holes and gaps

size 参数的含义与亮度滤波器中相同：size=1 表示 3×3 邻域，size=2 表示 5×5，依此类推。更大的 size 意味着更激进的清理——以及更长的每像素开销。

5.15.2. 顶帽与黑帽¶

另有两种组合值得了解，因为它们 恰好 提取出 open 和 close 所去除的特征：

top_hat() 返回原始图像与其开运算结果之间的 差——即 open 本会去除的那些前景像素。这实际上就是一幅由噪声像素、孤立的小前景区域以及开运算无法保留的细前景结构构成的掩膜。当那些小前景特征正是应用关心的对象时，它对于 提取 这些特征（而非去除它们）很有用。

black_hat() 返回图像的闭运算结果与原始图像之间的 差——即 close 本会填充的那些背景像素。这是一幅由前景区域内部的小孔洞以及闭运算本会桥接的区域间窄间隙构成的掩膜。

二者都不如四种基本运算常用，但这一模式值得记住——当应用需要提取标准清理流程 去除 的小特征或细特征时，顶帽和黑帽是把它们找回来的自然方法。

5.15.3. 阈值模式¶

四种基本形态学运算都接受一个整数 threshold 关键字，用于放宽每个位置上的开/关测试。不带它时，这些运算的行为如上文所述：erode() 要求 每个 邻居都为开，dilate() 要求 至少一个 邻居为开。设置 threshold 后，每个运算都能容忍这么多个邻居投相反的票。对于腐蚀，threshold 是一个像素可以拥有且仍能存活的 背景 邻居数量：threshold=4 会保留八个邻居中至少有四个为开的任何像素（在 3×3 窗口中，中心像素有八个邻居），因此腐蚀不那么激进。对于膨胀，threshold 是一个背景像素必须多拥有多少个 前景 邻居才会变为开的数量：threshold=2 要求至少三个前景邻居而非一个，因此扩张不那么激进。

阈值形式可用于在不改变窗口大小的情况下调整形态学处理的激进程度，而改变窗口大小同时会改变它所作用特征的 尺度。大多数应用都使用默认行为；阈值形式是为那些默认行为稍微过强或过弱的情况准备的。
