v3.4.0¶

v3.4.0 添加了 SSD1306 OLED 驱动，启用了 MicroPython framebuf 模块，将 H7 提升至 480 MHz，允许 通过 UART REPL 用 Ctrl-C 中断脚本，并改进了 CMSIS-NN 库和 MAVLink 光流示例。它 不引入任何重大 API 变更 —— v3.3.1 脚本无需改动即可运行。

亮点¶

  • 添加了 SSD1306 OLED 驱动。

  • 启用了 framebuf 模块。

  • H7 运行于 480 MHz —— 更高的时钟带来更强的性能。

  • 通过 UART REPL 使用 Ctrl-C —— 可从 UART REPL 中断脚本。

  • 无重大 API 变更 —— 重新刷写固件，你的 v3.3.1 脚本无需改动即可运行。

新功能¶

  • 添加了 SSD1306 OLED 显示驱动。

  • 启用了 MicroPython framebuf 模块。

  • 添加了 TV-shield、distance-shield 和 IMU-shield 示例。

其他变更和改进¶

  • 将 H7 提升至 480 MHz（更新了 H7 时钟）；更新了 CMSIS-NN 库以使用非方形 NN 函数，并更新了 NN / CIFAR-10 模型（在 NN 脚本中启用了日志记录）；编译冻结模块时包含 frozen_mpy.o。

错误修复¶

摄像头和系统：

  • 修复了 F7 和 H7 的 SCCB / I2C 时序（将 SCCB 时序移入板卡配置文件）、H7 SD 时钟以及 OpenMV 2 文件系统。

脚本和连接：

  • 允许通过 UART REPL 用 Ctrl-C 中断脚本，设置了 WINC 的 ioctl 函数，修复了 py_net_search 中缺失的 break，并修复了 MAVLink 光流示例的缩放/方向（带有绿色闪烁 LED，且无论质量如何都始终发送光流数据包）。

重大 API 变更¶

v3.4.0 不引入任何重大 Python API 变更。在 v3.3.1 上运行的脚本在 v3.4.0 上无需改动即可运行——只需重新刷写固件。（注意：随附的 MAVLink 光流示例的缩放/方向已得到修正；请重新检查其输出的下游使用者。）
