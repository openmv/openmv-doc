v1.8.0¶

v1.8.0 是一个 WiFi 与调优版本：它提升了 WINC SPI 总线速度，修复了一批 WINC 套接字缺陷，限制了最大 FPS 下降幅度，并将默认自动增益上限改为 8 倍。增益上限默认值的更改属于行为变更——请阅读下方的破坏性变更。

亮点¶

  • 更快的 WiFi —— 提升了 WINC SPI 波特率，并修复了套接字问题。

  • FPS 上限 —— 最大 FPS 下降幅度被限制在 1/2 FPS（30 FPS）。

  • 破坏性变更： 默认自动增益上限改为 8 倍——请参阅破坏性变更。

新功能¶

  • 新增了一个 FPS 测试脚本。

其他变更与改进¶

  • 提升了 WINC SPI 波特率 / 速度以加快 WiFi；将最大 FPS 下降幅度限制在 1/2 FPS（30 FPS）；整理了调试日志和 MJPEG 流式传输示例。

缺陷修复¶

  • 修复了默认帧缓冲区状态、WINC 套接字 accept 信息、WINC socket_send，并整理了套接字 / 异步请求的错误处理。

破坏性 API 变更¶

v1.7.0 与 v1.8.0 之间用户可见的 API 破坏性变更。范围：modules/ 中的 Python C 模块和 scripts/libraries/ 中的 Python 库。

该破坏性变更属于 behavior 变更（API 不变，但结果不同）——请重新检查受影响的脚本。每个提交哈希都链接到其在 GitHub 上的差异。

默认自动增益上限改为 8 倍 (behavior)¶

默认自动增益上限已改为 8 倍。依赖之前默认增益上限的脚本，在自动增益下会看到不同的图像亮度 / 噪声——如果需要旧行为，请设置显式的增益上限。

提交： bdafca5a7

迁移清单¶

如果你的脚本依赖之前的默认值，请设置显式的增益上限（增益上限变更）。其余所有脚本均可原样运行。
