5.13. 線性與鄰域濾波器¶

本章稍早的像素運算將兩張影像逐點結合在一起。濾波器 則以不同的方式做類似的工作：它根據對應位置周圍一小塊 鄰域 的輸入像素，計算出每個輸出像素的值。(x, y) 處的輸出是以 (x, y) 為中心的小方框內輸入像素的某種統計量，例如平均值、中位數或最常出現的值。

這種觀念上的小小轉變——從一次處理一個像素，轉為一次處理一整個像素視窗——正是讓一整族實用運算得以運作的關鍵。在小視窗上取簡單平均可平滑掉感測器雜訊。在相同視窗上取中位數則能去除單一像素的斑點，而不像取平均那樣大幅軟化邊緣。雙邊平均不會跨越強烈對比邊界進行平滑，因此能在清理物體內部紋理的同時保留物體的邊緣。鄰域是運算的單位；統計量的選擇則決定了濾波器的功能。

5.13.1. 核心大小¶

每個鄰域濾波器都接受一個 size 參數，用來設定視窗的 半徑（以像素為單位）。視窗本身為正方形，每邊涵蓋 (2 * size + 1) 個像素——因此 size=1 代表 3×3 的鄰域，size=2 代表 5×5，size=3 代表 7×7，依此類推。

較大的 size 代表較大的鄰域，也就代表更平滑（或更強烈）的濾波。成本隨視窗面積增長，因此 size=3 濾波器每個像素的工作量約為 size=1 濾波器的九倍。大多數清理工作的實用預設值是 size=1 或 size=2；只有在小鄰域不足以抑制應用程式想抑制的特徵時，才需要動用較大的 size。

5.13.2. 均值濾波器¶

mean() 將每個像素替換為其鄰域的 算術平均值。其結果會在視窗大小範圍內平滑掉像素之間的變化，這使它成為抑制感測器雜訊斑點最便宜的方法：高頻變化被平均掉，低頻內容則得以保留。

代價是邊緣與其他銳利特徵也會一併被平均。在濾波前寬度為一像素的明亮邊緣，經過 size=1 的均值濾波後會變成兩到三像素寬，並在兩側出現亮度漸減的斜坡。對於紋理稀少的影像（乾淨的牆面、彩色標記筆的內部）做純粹的降噪時，這樣的取捨是可以接受的。但對於邊緣很重要的繁雜場景，下列其中一種濾波器通常更為合適。

img.mean(1)        # 3x3 box average -- fast, gentle smoothing
img.mean(2)        # 5x5 box average -- stronger, slower

5.13.3. 中位數、眾數、中點¶

另外三種統計鄰域濾波器以更能抵抗離群值的統計量，取代了簡單的算術平均。

median() 回傳鄰域的 中位數——也就是視窗像素排序後位於中間的那個值。視窗中單一極亮或極暗的像素無法拉動中位數；它只會成為被捨棄的極端值之一。實際效果是中位數濾波能去除單一像素斑點與椒鹽雜訊，而不像 mean 那樣軟化邊緣。代價是每個像素需要更多運算——對視窗排序比取平均慢——而且其結果並非嚴格意義上的平均值，這在後續的數學運算中有時會造成影響。

percentile 參數（預設為 0.5）可將所選值偏離嚴格的中位數。percentile=0.0 回傳鄰域的最小值，percentile=1.0 回傳最大值；中間值則在排序後的視窗中按比例選取兩者之間的值。這讓 median 能在不失去順序統計量抗離群值能力的前提下，強調鄰域中較暗或較亮的部分。

mode() 回傳鄰域中 最常出現 的值。當雜訊模型為「大多數像素是正確的，少數被不同程度地破壞」時很有用，此時正確答案就是出現最頻繁的那個值——而當被破壞的值堆積在排序視窗的某一側時，中位數可能會錯過它。

midpoint() 回傳鄰域 最小值與最大值的加權組合——bias=0.5 給出兩者之間的中點，bias=0.0 給出最小值，bias=1.0 給出最大值。它不如其他濾波器常用，但當目標明確是要擷取較暗或較亮的特徵時，值得了解。

5.13.4. 雙邊濾波器：保留邊緣的版本¶

bilateral() 是最值得徹底理解的鄰域濾波器。它能產生 mean() 的平滑效果，但多了一個限制：鄰域像素與中心像素 差異 愈大，它在平均中所佔的權重就愈小。其結果能平滑每個均勻區域的內部，而不會跨越分隔這些區域的邊緣產生暈染，這正是大多數應用程式真正想要的效果。

有兩個參數控制濾波器折減像素權重的強度：

  • color_sigma 決定 色彩差異 如何影響權重。值愈小，濾波器對折減與中心差異較大像素的權重就愈嚴格。

  • space_sigma 決定 空間距離 如何影響權重。值愈小，靠近中心的像素獲得的權重就愈大。

預設值（color_sigma=0.1、space_sigma=1.0）是合理的起點；調整它們通常只需在一個樣本影格上執行濾波器，並反覆調整到邊緣清晰、內部乾淨為止。

雙邊濾波器比 median() 更耗費資源，更遠比 mean() 昂貴，因此 只 在應用程式確實需要保留邊緣的行為時，才值得動用它。

5.13.5. 自適應閾值化¶

均值、中位數、眾數與中點濾波器都帶有同一對關鍵字引數，可將其輸出轉變為二值閾值：

  • threshold=True 將濾波器切換為閾值化模式。

  • offset=N 在比較之前，將區域分界值偏移 N 個單位。

其機制直接建立在濾波器的一般行為之上。沒有 threshold=True 時，濾波器會在鄰域上計算統計量，並將該統計量寫入輸出像素。有 threshold=True 時，濾波器會計算相同的統計量，然後將相同位置的 來源 像素與該統計量加上偏移量進行比較，若來源較大則寫入該格式的最大值，否則寫入零。

其結果是一張二值影像，其分界值會 隨著整個影格中區域亮度的變化而移動。明亮區域得到較高的分界值，昏暗區域得到較低的分界值，而一個在區域上比鄰近像素更亮的前景像素，無論位於明亮區域或昏暗區域都能匹配——這正是單一全域閾值在光照不均的影像上無法產生的行為。

img.mean(3, threshold=True, offset=5)

offset 參數正是應用程式控制測試 嚴格程度 的地方。較小的正偏移量要求來源像素必須明顯比鄰近像素更亮才算匹配，這能以漏掉微弱前景為代價來抑制感測器雜訊造成的誤判。較小的負偏移量則能捕捉到微弱前景，但代價是讓部分雜訊通過。如何取捨取決於管線的其餘部分將如何處理這份二值輸出。

整個濾波器族都能執行自適應閾值，因此挑選 正確 的濾波器很重要：mean() 提供最便宜的自適應閾值；當輸入帶有應在計算區域分界值前先剔除的椒鹽雜訊時，則使用 median()。
