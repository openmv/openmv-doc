Gigabit PoE Shield¶

O Gigabit PoE Shield é um shield Ethernet de 10/100/1000 Mb/s com Power-over-Ethernet 802.3af para OpenMV Cams que possuem um PHY Ethernet integrado. Um único cabo até um switch PoE carrega tanto a alimentação quanto o link de rede.

Para o datasheet completo, fotos e informações de compra, consulte a página do produto Gigabit PoE Shield.

Nota

Suportado apenas na OpenMV Cam RT1062 e N6.

Destaques¶

  • Gigabit Ethernet de 10/100/1000 Mb/s com PoE IEEE 802.3af

  • Até ~6 W para a câmera via VIN de 5,6 V

  • Projeto isolado de 1500 V — empilha com shields de duplo header através de diodo de OR integrado

Pinagem¶

Referência de pinos¶

10/100 Mb/s usa apenas os pares MDI TX e MDI RX (Pares A e B). O Gigabit (1000BASE-T) é bidirecional em todos os quatro pares A/B/C/D, então as linhas MDI TX ± e MDI RX ± funcionam também como Par A e Par B em velocidades gigabit, e os Pares C e D carregam os pares adicionais exclusivos do gigabit.

Pino

Função

MDI LED

Linha do LED de link / atividade do PHY

MDI TX P / DA P

Par A positivo — MDI TX+ a 10/100, BI_DA+ a gigabit

MDI TX N / DA N

Par A negativo — MDI TX− a 10/100, BI_DA− a gigabit

MDI RX P / DB P

Par B positivo — MDI RX+ a 10/100, BI_DB+ a gigabit

MDI RX N / DB N

Par B negativo — MDI RX− a 10/100, BI_DB− a gigabit

DC P

Par C positivo (BI_DC+) — apenas gigabit

DC N

Par C negativo (BI_DC−) — apenas gigabit

DD P

Par D positivo (BI_DD+) — apenas gigabit

DD N

Par D negativo (BI_DD−) — apenas gigabit

VIN out

5,6 V a até ~1 A do regulador PoE integrado (alimenta a câmera)

trilho 3.3V

Alimenta a eletrônica integrada do shield

trilho GND

Terra comum

Nota

Os pares DC e DD estão ligados à câmera através de resistores de 0 ohm na parte traseira do shield. Remova-os para liberar P15–P18 (os pinos exclusivos do gigabit em cams como a OpenMV N6) para uso não relacionado — o shield então cai para Ethernet 10/100, já que os pares gigabit não estão mais conectados.

Uso¶

Quando o shield está conectado a um switch PoE, o PHY gigabit da câmera aparece como uma interface network.LAN. O DHCP é executado automaticamente assim que o link é estabelecido:

import network
import time

lan = network.LAN()
lan.active(True)
while not lan.isconnected():
    time.sleep(1)
print("Ethernet IP:", lan.ipconfig("addr4")[0])
