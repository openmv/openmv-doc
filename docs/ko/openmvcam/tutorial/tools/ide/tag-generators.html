13.1.15. 태그 및 바코드 생성기¶

AprilTag를 검출하는 카메라는 가리킬 인쇄된 AprilTag가 필요하므로, IDE가 이를 생성합니다. Tools → Machine Vision → AprilTag Generator는 카메라가 검출할 수 있는 여섯 가지 태그 패밀리를 패밀리 크기와 함께 괄호 안에 나열합니다:

  • TAG16H5 – 30개 태그.

  • TAG25H7 – 242개 태그.

  • TAG25H9 – 35개 태그.

  • TAG36H10 – 2,320개 태그.

  • TAG36H11 – 587개 태그. 권장 패밀리입니다: 개수, 검출 견고성, 오탐 저항성의 최적 균형을 이루며, 카메라 예제의 기본값입니다.

  • ARKTOOLKIT (ARTOOLKIT 패밀리) – 512개 태그.

패밀리는 함께 설계된 태그 패턴의 고정된 알파벳으로, 그 구성원들이 서로 혼동되기 어렵게 만들어졌습니다. 패밀리의 모든 태그는 하나의 ID 번호를 인코딩하며, 검출기는 어떤 패밀리 구성원을 보았는지 보고합니다. 프로젝트에 한 패밀리를 선택하고 프로젝트가 필요로 하는 ID만 생성하세요 – 검출은 패밀리별로 구성되며, 유효한 ID가 적을수록 오탐이 줄어듭니다.

패밀리를 선택하면 대화 상자가 생성할 태그 ID의 범위와 각 태그 아래에 패밀리 이름과 ID를 인쇄할지(기본값으로 켜져 있으며 유지할 가치가 있습니다 – 익명의 태그가 흩어진 바닥은 식별이 불가능합니다)를 묻습니다. 그런 다음 생성기는 여러분이 선택한 폴더에 태그당 하나의 인쇄 가능한 이미지를 기록합니다 – 레이블이 있는 US-Letter 페이지와 레이블이 없는 정사각형 이미지입니다. 검출 거리가 요구하는 어떤 물리적 크기로든 인쇄하고, 눈부심이 우려된다면 무광 용지에 인쇄하세요.

카메라가 읽는 다른 기호 체계 – QR 코드, Data Matrix 코드, 선형 바코드 – 의 경우, Machine Vision 메뉴의 생성기 항목은 로컬에서 생성하는 대신 온라인 생성기에 대한 웹 검색을 엽니다. 이러한 형식을 위한 무료 생성기가 어디에나 있기 때문입니다.

더 보기

find_apriltags(), find_qrcodes(), find_datamatrices(), find_barcodes() – 각 생성기의 검출 쪽입니다.
