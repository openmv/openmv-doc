v4.4.8
======

v4.4.8 is a maintenance release that refreshes the bundled firmware and tools.
It continues to build on Qt Creator 14.0.2 with no rebase in this range. The
bundled OpenMV firmware moves to v4.6.0, first-time STM32N6 (OpenMV Cam N6)
firmware is bundled, new FLIR Boson thermal sensors are recognized, and the
Alif Security Toolkit is updated. There are no user-facing breaking changes,
and existing user projects are unaffected.

.. contents:: On this page
   :local:
   :depth: 1

Highlights
----------

- **Bundled firmware v4.6.0:** The bundled OpenMV firmware and examples are
  updated from v4.5.9 to v4.6.0
  (`66bfd486f <https://github.com/openmv/qt-creator/commit/66bfd486f>`__).
- **OpenMV Cam N6 firmware bundled:** STM32N6 (OpenMV Cam N6) firmware is now
  shipped in the bundle for the first time
  (`66bfd486f <https://github.com/openmv/qt-creator/commit/66bfd486f>`__).
- **FLIR Boson thermal sensors:** ``BOSON``, ``BOSON-320``/``BOSON-640`` and
  the ``BOSON-320+``/``BOSON-640+`` (Boson Plus) thermal sensors are now
  recognized by the IDE
  (`8b795053c <https://github.com/openmv/qt-creator/commit/8b795053c>`__,
  `d5068b286 <https://github.com/openmv/qt-creator/commit/d5068b286>`__).
- **Alif Security Toolkit v1.101.0:** The bundled Windows toolkit is updated to
  v1.101.0
  (`7d51b00b1 <https://github.com/openmv/qt-creator/commit/7d51b00b1>`__).
- **OpenMV Cam AE3 SBL flashing fixed:** The AE3 board now runs the correct
  secure bootloader when flashing
  (`ed7c95357 <https://github.com/openmv/qt-creator/commit/ed7c95357>`__).

New features
------------

- **FLIR Boson examples:** Added bundled example scripts for FLIR Boson
  grayscale and ironbow streaming plus color tracking under
  ``01-Camera/05-Thermal-Cameras/02-FLIR-Boson``, and a GenX320 grayscale
  LED-tracking example under ``01-Camera/03-Event-Cameras/02-Genx320``
  (`66bfd486f <https://github.com/openmv/qt-creator/commit/66bfd486f>`__).

Other changes and improvements
------------------------------

- Reorganized the bundled example tree: the FLIR Lepton examples moved into a
  new ``01-Camera/05-Thermal-Cameras/01-FLIR-Lepton`` category alongside the
  new Boson examples, and the event-camera examples were split into
  ``01-Camera/03-Event-Cameras/01-Frogeye2020`` and ``02-Genx320`` subfolders
  (`66bfd486f <https://github.com/openmv/qt-creator/commit/66bfd486f>`__).
- Removed the unused, orphaned ``OPENMV4_PRO`` firmware folder from the bundle;
  no board entry referenced it, so no flashable board is affected
  (`66bfd486f <https://github.com/openmv/qt-creator/commit/66bfd486f>`__).
- Updated the bundled OpenMV AE3 SBL firmware, bootloader and signed images
  used for flashing
  (`b8c06a8ab <https://github.com/openmv/qt-creator/commit/b8c06a8ab>`__).
- Updated the bundled Alif Security Toolkit (Windows) to v1.101.0, including
  the refreshed user guide (AUGD0005) and signed SP images
  (`7d51b00b1 <https://github.com/openmv/qt-creator/commit/7d51b00b1>`__).

Bug fixes
---------

- Fixed OpenMV Cam AE3 SBL flashing so the correct secure bootloader is run, by
  setting its board arch string to ``OPENMV AE3 SBL``
  (`ed7c95357 <https://github.com/openmv/qt-creator/commit/ed7c95357>`__).
- Increased the ``blhost`` flash-erase timeout from 60s to 120s when
  programming i.MX RT boards, preventing timeouts during erase and flash of
  larger regions
  (`b48e139e4 <https://github.com/openmv/qt-creator/commit/b48e139e4>`__).

Platform and tool support
-------------------------

- **Qt Creator base:** 14.0.2.
- **Bundled OpenMV firmware:** updated 4.5.9 -> 4.6.0
  (`66bfd486f <https://github.com/openmv/qt-creator/commit/66bfd486f>`__).
- **OpenMV Cam N6:** STM32N6 firmware (``firmware.bin``, ``network.bin``,
  ``openmv.bin``) is now bundled under ``firmware/OPENMV_N6``
  (`66bfd486f <https://github.com/openmv/qt-creator/commit/66bfd486f>`__).
- **OpenMV AE3 SBL:** firmware and bootloader binaries updated
  (`b8c06a8ab <https://github.com/openmv/qt-creator/commit/b8c06a8ab>`__).
- **Alif Security Toolkit (Windows):** updated v1.100.0 -> v1.101.0
  (`7d51b00b1 <https://github.com/openmv/qt-creator/commit/7d51b00b1>`__).
- **New recognized sensors:** FLIR ``BOSON``, ``BOSON-320``, ``BOSON-640``
  (`8b795053c <https://github.com/openmv/qt-creator/commit/8b795053c>`__),
  FLIR ``BOSON-320+`` and ``BOSON-640+``
  (`d5068b286 <https://github.com/openmv/qt-creator/commit/d5068b286>`__), and
  the PixArt ``PS5520``
  (`66bfd486f <https://github.com/openmv/qt-creator/commit/66bfd486f>`__).

Breaking changes
----------------

None. There are no scripting API changes, and existing user projects are
unaffected. The removed ``OPENMV4_PRO`` firmware folder was unused and
referenced by no board, so its removal breaks no flashing workflow.
