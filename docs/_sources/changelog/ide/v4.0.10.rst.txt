v4.0.10
=======

v4.0.10 adds full support for the Arduino Giga (STM32H747) board with bundled
firmware, bootloader, and UVC binaries, and fixes several flashing,
recording, and localization issues. It is based on Qt Creator 10.0.1; there
are no breaking changes for existing projects.

.. contents:: On this page
   :local:
   :depth: 1

Highlights
----------

- **Arduino Giga support** is now built in, with bundled firmware,
  bootloader, and UVC binaries and DFU-based flashing.
- **Flash-safe port scanning:** serial-port scanning no longer interferes
  with an in-progress firmware flash.
- **Image-writer recording** files with zero milliseconds between frames are
  now accepted instead of being rejected.
- **Localization restored:** all Qt translation files are bundled again so
  the IDE's translations work correctly.

New features
------------

- **Arduino Giga (STM32H747):** added full board support, including bundled
  ``firmware``, ``bootloader``, and ``uvc`` binaries (``.bin`` and ``.dfu``)
  under ``share/qtcreator/firmware/ARDUINO_GIGA_H7/``, ``boards.txt`` and
  ``dfu.txt`` entries, new USB PIDs (``ARDUINOCAM_GH7_PID``, ``GIGA_LDR_PID``,
  ``GIGA_APP_PID``), and Giga detection in the port-filtering, connect, and
  license-check paths
  (`91a634714 <https://github.com/openmv/qt-creator/commit/91a634714>`__,
  `21ad38087 <https://github.com/openmv/qt-creator/commit/21ad38087>`__,
  `860aa10d6 <https://github.com/openmv/qt-creator/commit/860aa10d6>`__,
  `c27d9d7ea <https://github.com/openmv/qt-creator/commit/c27d9d7ea>`__,
  `b95a977b3 <https://github.com/openmv/qt-creator/commit/b95a977b3>`__).

Bug fixes
---------

- Fixed a JSON syntax error in the Giga DFU settings (a stray double-quote in
  the ``appvidpid`` list)
  (`c27d9d7ea <https://github.com/openmv/qt-creator/commit/c27d9d7ea>`__).
- Corrected the Giga DFU board type from ``H747`` to ``H7`` so Arduino board
  flashing works in the IDE
  (`860aa10d6 <https://github.com/openmv/qt-creator/commit/860aa10d6>`__).
- Added working locks so that serial-port scanning no longer interferes with
  an in-progress firmware flash via ``dfu-util``, ``picotool``, ``bossac``,
  or ``imx``
  (`f0de0fe00 <https://github.com/openmv/qt-creator/commit/f0de0fe00>`__,
  `da7e80b12 <https://github.com/openmv/qt-creator/commit/da7e80b12>`__).
- Image-writer recording files with zero milliseconds between frames are now
  accepted instead of being rejected as out-of-range
  (`fed86c2bb <https://github.com/openmv/qt-creator/commit/fed86c2bb>`__,
  `75bc5a848 <https://github.com/openmv/qt-creator/commit/75bc5a848>`__).
- Fixed the "No DFU settings" error dialog, which previously formatted only
  one of the two VID/PID placeholders
  (`b95a977b3 <https://github.com/openmv/qt-creator/commit/b95a977b3>`__).
- Restored bundling of all Qt translation files so the IDE's localization
  works correctly
  (`e10d65de7 <https://github.com/openmv/qt-creator/commit/e10d65de7>`__).

Platform and tool support
-------------------------

- **Qt Creator base:** 10.0.1.
- **Arduino Giga H7** firmware is now bundled
  (``ARDUINO_GIGA_H7``: ``firmware.bin``/``.dfu``,
  ``bootloader.bin``/``.dfu``, ``uvc.bin``/``.dfu``).

Breaking changes
----------------

None. This release is fully compatible with existing projects and workflows.
