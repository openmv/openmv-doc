v4.0.12
=======

v4.0.12 teaches the serial terminal to render ANSI color codes, unifies the
firmware-loader progress windows behind a shared geometry and an expanded log
panel, and refreshes the bundled firmware and example scripts. It stays on the
Qt Creator 10.0.1 base, and there are no breaking changes.

.. contents:: On this page
   :local:
   :depth: 1

Highlights
----------

- **ANSI color in the serial terminal:** the output window now interprets ANSI
  escape color codes and renders colored text, with brighter, theme-aware
  colors when a dark UI theme is active.
- **Unified firmware-loader windows:** all loader progress windows now share
  one saved geometry and open with the log panel expanded by default.
- **Bundled firmware 4.5.0:** refreshed firmware binaries for every supported
  board.
- **Modernized examples:** bundled scripts now use the current MicroPython
  ``machine`` and ``time`` API, with new ``blinky.py`` and sensor
  save/restore-settings examples.

New features
------------

- The serial terminal output window now interprets ANSI escape color codes,
  rendering colored text. When a dark UI theme is active, the renderer emits
  brighter, theme-aware colors.
  (`7f1a29ba1 <https://github.com/openmv/qt-creator/commit/7f1a29ba1>`__)

Other changes and improvements
------------------------------

- All firmware-loader progress windows (**BOSSAC**, **DfuSe**, **PyDfu**,
  **DFU Util**, **NXP IMX**, **PicoTool**) now share a single saved
  window-geometry/settings group and open with the details/log panel expanded
  by default.
  (`6bac6b7eb <https://github.com/openmv/qt-creator/commit/6bac6b7eb>`__)
- The bundled example scripts were modernized to the current MicroPython API,
  replacing ``pyb`` with the ``machine`` and ``time`` modules (for example
  ``machine.LED("LED_BLUE")`` and ``time.ticks_ms()`` / ``time.ticks_diff()``).
  A new ``00-HelloWorld/blinky.py`` example and a
  ``07-Sensor-Control/sensor_save_and_restore_settings.py`` example were added,
  and ``sensor_manual_gain_control.py`` was renamed.
  (`16fb36f8c <https://github.com/openmv/qt-creator/commit/16fb36f8c>`__)

Platform and tool support
-------------------------

- **Qt Creator base:** 10.0.1.
- **Bundled OpenMV firmware:** updated from 4.4.3 to 4.5.0, refreshing
  ``firmware`` binaries for all supported boards, plus ``uvc`` and ``openmv``
  binaries for the boards that ship them (``openmv`` binaries for
  OPENMV2/3/4/4P/PT).
  (`16fb36f8c <https://github.com/openmv/qt-creator/commit/16fb36f8c>`__)
- The Arduino Giga firmware was consolidated into a new ``ARDUINO_GIGA``
  folder (swapping its ``bin``/``dfu`` packaging and adding UVC firmware), the
  obsolete ``ARDUINO_GIGA_H7`` firmware folder was removed, and ``boards.txt``
  was repointed to ``ARDUINO_GIGA``.
  (`16fb36f8c <https://github.com/openmv/qt-creator/commit/16fb36f8c>`__)

Breaking changes
----------------

None. This release is fully compatible with existing OpenMV IDE projects and
workflows.
