v1.1.0
======

v1.1.0 makes OpenMV IDE firmware-aware: built on Qt Creator 4.0.2, it now
detects an out-of-date OpenMV Cam on connect and upgrades its firmware
automatically over the serial bootloader, adds support for the new Cortex-M7
OpenMV Cam, and turns the General Messages pane into a proper Serial Terminal.
This release changes a few default behaviors (firmware auto-upgrade and Python
file filters) that users should be aware of; see :ref:`Breaking changes
<changelog_ide_v1_1_0>` below.

.. contents:: On this page
   :local:
   :depth: 1

Highlights
----------

- **Automatic firmware upgrade.** On connect, the IDE detects an out-of-date
  OpenMV Cam and flashes the bundled firmware over the serial bootloader,
  with an option to also erase the internal filesystem.
- **Cortex-M7 OpenMV Cam support.** The new STM32F769-based board (``OMV7``)
  is supported alongside the existing Cortex-M4 board (``OMV2``), with
  per-board firmware and auto-detection.
- **Unbrick recovery.** If no camera is found, the IDE can force-flash the
  matching firmware to a bricked camera once it is reconnected.
- **Serial Terminal.** The General Messages pane is now a dark-themed
  ``Serial Terminal`` with 100000-line scrollback that pops up on output.
- **Non-blocking connect/disconnect.** Connecting, reconnecting, and shutdown
  now run asynchronously with progress dialogs so the IDE no longer hangs.
- **Live binary and grayscale preview.** The frame buffer viewer renders
  1-bit and grayscale images in addition to RGB565 and JPEG.

New features
------------

- **Automatic firmware upgrade.** When a connected OpenMV Cam reports a
  firmware version older than the bundled ``firmware.txt``, the IDE starts the
  bootloader, erases flash sectors, programs ``firmware.bin`` in chunks with
  progress dialogs, and resets the camera. The user may optionally erase the
  internal filesystem as part of the upgrade.
  (`a175bff39 <https://github.com/openmv/qt-creator/commit/a175bff39>`__,
  `50375ee95 <https://github.com/openmv/qt-creator/commit/50375ee95>`__,
  `502177f3c <https://github.com/openmv/qt-creator/commit/502177f3c>`__,
  `a871bfb48 <https://github.com/openmv/qt-creator/commit/a871bfb48>`__,
  `30eb9e0dc <https://github.com/openmv/qt-creator/commit/30eb9e0dc>`__,
  `552803018 <https://github.com/openmv/qt-creator/commit/552803018>`__,
  `889bc38fb <https://github.com/openmv/qt-creator/commit/889bc38fb>`__,
  `047581ed5 <https://github.com/openmv/qt-creator/commit/047581ed5>`__)
- **Unbrick recovery path.** If no OpenMV Cam is found, the IDE asks whether a
  bricked camera is attached, lets you pick the board type, and force-flashes
  the matching firmware once the device is reconnected.
  (`a175bff39 <https://github.com/openmv/qt-creator/commit/a175bff39>`__,
  `30eb9e0dc <https://github.com/openmv/qt-creator/commit/30eb9e0dc>`__)
- **Save script and reset actions.** New ``Tools`` menu entries save the
  current script to the OpenMV Cam's flash drive (``Save script to OpenMV
  Cam``) and reset the camera (``Reset OpenMV Cam``). ``Stop`` gained a
  ``Ctrl+T`` shortcut.
  (`9fc3dfa5a <https://github.com/openmv/qt-creator/commit/9fc3dfa5a>`__,
  `889bc38fb <https://github.com/openmv/qt-creator/commit/889bc38fb>`__)
- **Update notification and local docs.** The IDE checks openmv.io on startup
  for a newer IDE version and shows a download notification. The ``Docs``
  ``Help`` action now opens locally bundled HTML documentation
  (``html/index.html``) and the pinout action opens the bundled pinout image
  (``html/_images/pinout.png``) instead of the website.
  (`a175bff39 <https://github.com/openmv/qt-creator/commit/a175bff39>`__,
  `50375ee95 <https://github.com/openmv/qt-creator/commit/50375ee95>`__)
- **Binary and grayscale frame buffer.** The frame buffer viewer now renders
  binary (1-bit) and grayscale images in addition to RGB565 and JPEG, and the
  frame buffer can be toggled on and off live from the toolbar.
  (`a175bff39 <https://github.com/openmv/qt-creator/commit/a175bff39>`__)
- **New and updated examples.** Added a CPU overclocking example (up to
  216MHz), an improved multi-client MJPEG WiFi streamer, an FPS test, and a
  colorbar test that adapts to the OV7725 sensor.
  (`a175bff39 <https://github.com/openmv/qt-creator/commit/a175bff39>`__,
  `32dc31962 <https://github.com/openmv/qt-creator/commit/32dc31962>`__,
  `502177f3c <https://github.com/openmv/qt-creator/commit/502177f3c>`__)

Other changes and improvements
------------------------------

- **Serial Terminal.** The General Messages pane was repurposed as a
  ``Serial Terminal`` with a dark background, raised status-bar priority,
  100000-line scrollback, and no word wrap. It pops up and flashes when the
  camera prints output.
  (`9fc3dfa5a <https://github.com/openmv/qt-creator/commit/9fc3dfa5a>`__)
- **Non-blocking connect and disconnect.** Connect and disconnect were
  reworked to be non-blocking with progress dialogs and a clean asynchronous
  shutdown, so the IDE no longer hangs while the camera is busy and shows
  ``Busy... please wait...`` instead.
  (`502177f3c <https://github.com/openmv/qt-creator/commit/502177f3c>`__,
  `a175bff39 <https://github.com/openmv/qt-creator/commit/a175bff39>`__,
  `6275d8855 <https://github.com/openmv/qt-creator/commit/6275d8855>`__,
  `234f76389 <https://github.com/openmv/qt-creator/commit/234f76389>`__)
- **Smoothed FPS readout.** The FPS readout is now a smoothed running average
  over the last several frames instead of an instantaneous value, with a
  tooltip noting it may differ from the camera FPS.
  (`a175bff39 <https://github.com/openmv/qt-creator/commit/a175bff39>`__,
  `889bc38fb <https://github.com/openmv/qt-creator/commit/889bc38fb>`__)
- **Dark theme polish.** The dark theme gained custom-styled scrollbars with
  arrow images across editors and panes, dark-themed search-result and scope
  highlighting in the Inkpot style, and white text in the ``Find`` toolbar.
  (`9fc3dfa5a <https://github.com/openmv/qt-creator/commit/9fc3dfa5a>`__,
  `94d8c41f1 <https://github.com/openmv/qt-creator/commit/94d8c41f1>`__,
  `9d8683355 <https://github.com/openmv/qt-creator/commit/9d8683355>`__)
- **Python-oriented file defaults.** ``Find in Files`` now defaults its file
  filter to ``*.py`` and the file-open dialog shows all files, matching
  MicroPython development rather than C++.
  (`94d8c41f1 <https://github.com/openmv/qt-creator/commit/94d8c41f1>`__,
  `a175bff39 <https://github.com/openmv/qt-creator/commit/a175bff39>`__)

Bug fixes
---------

- Improved serial-port handling on macOS (filter to port names containing
  ``cu``) and Linux (clearer permission-denied guidance suggesting ``sudo
  adduser <user> dialout``), suppressed an SSL warning, made bootloader
  dialogs application-modal, and adjusted bootloader timing and timeouts for
  reliability.
  (`a175bff39 <https://github.com/openmv/qt-creator/commit/a175bff39>`__,
  `552803018 <https://github.com/openmv/qt-creator/commit/552803018>`__,
  `a871bfb48 <https://github.com/openmv/qt-creator/commit/a871bfb48>`__,
  `30eb9e0dc <https://github.com/openmv/qt-creator/commit/30eb9e0dc>`__)
- The IDE now opens the ``helloworld.py`` example automatically when no
  document is open on launch and reliably closes the splash screen, fixing
  edit-window and startup state issues.
  (`a175bff39 <https://github.com/openmv/qt-creator/commit/a175bff39>`__,
  `9d8683355 <https://github.com/openmv/qt-creator/commit/9d8683355>`__,
  `502177f3c <https://github.com/openmv/qt-creator/commit/502177f3c>`__)

Platform and tool support
-------------------------

- **Qt Creator base:** 4.0.2.
- **OpenMV IDE version:** bumped from 1.0.0 to 1.1.0.
  (`50375ee95 <https://github.com/openmv/qt-creator/commit/50375ee95>`__)
- **New board:** Cortex-M7 OpenMV Cam (STM32F769, ``OMV7``) is now supported
  alongside the Cortex-M4 OpenMV Cam (STM32F427, ``OMV2``), with per-board
  firmware directories and board-string auto-detection via ``boards.txt``.
  (`a175bff39 <https://github.com/openmv/qt-creator/commit/a175bff39>`__)
- **Bundled firmware:** updated to 1.7.0 and then to 1.8.0 for both ``OMV2``
  and ``OMV7`` boards; the bootloader minimum/compatible firmware version is
  1.6.0.
  (`502177f3c <https://github.com/openmv/qt-creator/commit/502177f3c>`__,
  `32dc31962 <https://github.com/openmv/qt-creator/commit/32dc31962>`__,
  `a175bff39 <https://github.com/openmv/qt-creator/commit/a175bff39>`__)
- **Windows USB drivers:** bundled digitally signed Windows USB driver files
  (``openmv`` and ``pybcdc`` ``.inf``/``.cat``) and packaged the drivers
  directory with the IDE.
  (`047581ed5 <https://github.com/openmv/qt-creator/commit/047581ed5>`__)

Breaking changes
----------------

User-visible workflow and behavior changes in v1.1.0:

- **Automatic firmware upgrade on connect.** When the connected OpenMV Cam's
  firmware is older than the bundled ``firmware.txt`` version, the IDE prompts
  you to upgrade (and optionally erase the internal filesystem) before letting
  you continue. Cancelling during the erase or program step can leave the
  camera bricked until it is reconnected and re-flashed.
- **Changed file defaults.** The ``Find in Files`` default file filter changed
  from ``*.cpp,*.h`` to ``*.py``, and the file-open dialog now shows all files
  instead of per-mimetype filters.
