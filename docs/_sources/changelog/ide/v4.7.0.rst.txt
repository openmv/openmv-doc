v4.7.0
======

v4.7.0 is a large NPU- and ROMFS-focused release built on Qt Creator 14.0.2. It
introduces an on-device ROM filesystem workflow with integrated ARM Ethos-U
(Vela) and ST Edge AI model compilers, adds STM32CubeProgrammer-based factory
programming for the OpenMV N6, and bundles OpenMV firmware 4.7.0 across every
board. Note that upgrading any camera from a pre-4.7.0 firmware now forces a FAT
filesystem erase and ROMFS reset, which wipes files stored on the device's
internal flash; see **Breaking changes** below.

.. contents:: On this page
   :local:
   :depth: 1

Highlights
----------

- **ROM File System workflow** lets you create, edit, flash, and reset an
  on-device read-only filesystem so ML models can be stored on the camera,
  integrated across the DFU, IMX, Bossac, Picotool, and Arduino-DFU
  bootloaders.
  (`efcc52fd8 <https://github.com/openmv/qt-creator/commit/efcc52fd8>`__,
  `f2cbcc802 <https://github.com/openmv/qt-creator/commit/f2cbcc802>`__,
  `a7e52424a <https://github.com/openmv/qt-creator/commit/a7e52424a>`__,
  `60b05326d <https://github.com/openmv/qt-creator/commit/60b05326d>`__,
  `7140fc2aa <https://github.com/openmv/qt-creator/commit/7140fc2aa>`__)
- **ARM Ethos-U Vela** and **ST Edge AI** model compilers are now integrated so
  ``.tflite``/``.onnx`` models are automatically compiled to the NPU when loaded
  onto Ethos-U55 (OpenMV AE3) or Neural-ART (OpenMV N6) boards.
  (`411f708ea <https://github.com/openmv/qt-creator/commit/411f708ea>`__,
  `3064411ad <https://github.com/openmv/qt-creator/commit/3064411ad>`__)
- **STM32CubeProgrammer** is now bundled to enable factory programming of the
  OpenMV N6 (STM32N657) bootloader via a ``FlashLayout.tsv``.
  (`334b04dbd <https://github.com/openmv/qt-creator/commit/334b04dbd>`__,
  `54fea4d57 <https://github.com/openmv/qt-creator/commit/54fea4d57>`__,
  `28cc94a6e <https://github.com/openmv/qt-creator/commit/28cc94a6e>`__)
- **Model Zoo browser** gains a "Filter models by board type" checkbox driven by
  a new ``models/index.csv`` table that hides models incompatible with the
  connected board.
  (`80adb2079 <https://github.com/openmv/qt-creator/commit/80adb2079>`__)
- **Per-extension ROMFS alignment**: the ROMFS image editor now sorts files
  alphabetically and packs them using byte-alignment rules from
  ``settings.json`` (e.g. ``.tflite`` models aligned to 16 bytes).
  (`e87b2bf06 <https://github.com/openmv/qt-creator/commit/e87b2bf06>`__)
- **Bundled firmware updated to 4.7.0** across all OpenMV and Arduino boards.
  (`6d04b1b50 <https://github.com/openmv/qt-creator/commit/6d04b1b50>`__,
  `d0bbb1fe7 <https://github.com/openmv/qt-creator/commit/d0bbb1fe7>`__,
  `d7d3120b3 <https://github.com/openmv/qt-creator/commit/d7d3120b3>`__)

New features

  / ``Reset ROMFS on OpenMV Cam``) replaces the old single ``Edit ROM File
  System`` action.

  `958f732b5 <https://github.com/openmv/qt-creator/commit/958f732b5>`__)

  selection-change preview, including bundled STMicroelectronics, Qualcomm,
  DeGirum, Edge Impulse, and Google model libraries.
  (`58033c5df <https://github.com/openmv/qt-creator/commit/58033c5df>`__,

  `c0959833c <https://github.com/openmv/qt-creator/commit/c0959833c>`__)

  TensorFlow Lite (``.tflite``) or ONNX (``.onnx``) model for the board's NPU
  accelerator (Vela for Ethos-U ``.tflite``, ST Edge AI for the N6 Neural-ART
  ``.tflite``/``.onnx``) and saves the converted model to disk or the camera.

  and optionally cross-compiles it to MicroPython ``.mpy`` bytecode via
  ``mpy-cross``; it lives in a new ``MicroPython Tools`` submenu under
  ``Tools``.
  (`e9d7fa143 <https://github.com/openmv/qt-creator/commit/e9d7fa143>`__,
  `47188c0e3 <https://github.com/openmv/qt-creator/commit/47188c0e3>`__)
- **Wrap Text** toggle buttons were added to the OpenMV serial terminal and the
  ``General Messages`` output pane; both word-wrap their output and persist the
  setting across sessions.
  (`8c052845a <https://github.com/openmv/qt-creator/commit/8c052845a>`__,
  `51d3e1ad6 <https://github.com/openmv/qt-creator/commit/51d3e1ad6>`__)

  (``armv7emsp``/``armv7emdp``) lets MicroPython scripts be frozen into ROMFS
  images.
  (`8c71548df <https://github.com/openmv/qt-creator/commit/8c71548df>`__)

  ``*.bin`` and ``*.dfu``, validates per-bootloader extensions, and disables the

  (`79f348b36 <https://github.com/openmv/qt-creator/commit/79f348b36>`__)
- Pre-4.7.0 upgrades now force the FAT-erase and ROMFS reset/update checkboxes
  on and warn that upgrading requires erasing the FAT filesystem.
  (`f6f3355b0 <https://github.com/openmv/qt-creator/commit/f6f3355b0>`__)

Breaking changes
----------------

User-visible workflow and behavior breaks in v4.7.0. There is no scripting API
break, but note the following:

- **FAT filesystem erased on pre-4.7.0 upgrade.** Upgrading any OpenMV Cam from
  a pre-4.7.0 firmware now forces erasing the FAT filesystem and
  resetting/updating the ROMFS during the firmware update (the dialog checkboxes
  are forced on and cannot be disabled), so user files stored on the device's
  internal flash will be wiped on upgrade.
