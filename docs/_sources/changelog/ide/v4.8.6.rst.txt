v4.8.6
======

v4.8.6 is a connectivity and tooling release focused on high-speed camera
firmware loading, smarter framebuffer streaming, and updated AI/toolchain
bundles. It remains based on Qt Creator 14.0.2. There are no breaking changes
for existing user projects.

.. contents:: On this page
   :local:
   :depth: 1

Highlights
----------

- **Reliable macOS firmware loading** for high-speed RT1062 and Alif AE3
  cameras running the V4.8.1 V1 DBG protocol, by working around a TinyUSB
  high-speed USB lockup.
- **Automatic JPEG vs raw streaming** that picks JPEG when the camera runs at
  full speed or has a hardware JPEG encoder, and raw only when the camera is
  high-speed and lacks a JPEG encoder, remembering the choice per board.
- **Alif ROM/Recovery-mode recovery** during firmware loading, driving the
  maintenance tool and prompting for a power cycle.
- **ST Edge AI tools updated to STEDGEAI 3.0.0** and the Arm GNU bare-metal
  toolchain to ``14.3.rel1``.
- **csi_ng auto-fixups** so example scripts adapt resolution and pixel format
  to the attached sensor using the new ``csi.*`` API.

New features
------------

- The IDE now automatically picks JPEG versus raw framebuffer streaming based
  on the connected board's capabilities: it prefers JPEG when the camera runs
  at full speed or has a hardware JPEG encoder, and only falls back to raw
  when the camera is high-speed and lacks a JPEG encoder. The per-board choice
  is remembered between sessions
  (`64823318a <https://github.com/openmv/qt-creator/commit/64823318a>`__).
- Alif firmware loading can now detect and recover a board that comes up in
  ROM/Recovery mode, driving the maintenance tool through recovery and
  prompting the user to power-cycle the camera afterward
  (`617724b4e <https://github.com/openmv/qt-creator/commit/617724b4e>`__).

Other changes and improvements
------------------------------

- Grayscale and HM01B0/Boson/GENX320 sensor auto-fixups now also rewrite the
  new ``csi_ng`` API calls (``.pixformat(csi.*)`` / ``.framesize(csi.*)``) so
  examples adapt resolution and pixel format to the attached sensor
  (`11a3e0d37 <https://github.com/openmv/qt-creator/commit/11a3e0d37>`__).

Bug fixes
---------

- Firmware loading for high-speed RT1062 and Alif AE3 cameras running the
  V4.8.1 V1 DBG protocol now connects reliably on macOS by splitting the
  protocol-version check packet to work around a TinyUSB high-speed USB lockup
  (`4b558274d <https://github.com/openmv/qt-creator/commit/4b558274d>`__).
- Matching the camera's USB serial drive now also accepts a byte-reversed
  serial number, fixing drive detection on boards that report the serial in
  reversed byte order
  (`633f5e559 <https://github.com/openmv/qt-creator/commit/633f5e559>`__).
- On macOS the editor tabs are now forced to align to the left of the tab bar
  instead of stretching or centering
  (`5a25d12ed <https://github.com/openmv/qt-creator/commit/5a25d12ed>`__,
  `6f4139739 <https://github.com/openmv/qt-creator/commit/6f4139739>`__).

Platform and tool support
-------------------------

- **Qt Creator base:** 14.0.2.
- **ST Edge AI tools** updated to STEDGEAI 3.0.0 (from 2.1.0) for
  Windows, macOS-arm, and Linux64, with the ``OPENMV_N6`` neuralart profile
  dropping the ``--mvei`` option; the STEDGEAI download URL was also moved to
  ``download.openmv.io``
  (`c7560debb <https://github.com/openmv/qt-creator/commit/c7560debb>`__,
  `5fd08b8b6 <https://github.com/openmv/qt-creator/commit/5fd08b8b6>`__).
- **Arm GNU bare-metal toolchain** updated to ``14.3.rel1`` (from
  ``14.2.rel1``)
  (`c7560debb <https://github.com/openmv/qt-creator/commit/c7560debb>`__).

Breaking changes
----------------

None. This release is fully compatible with existing OpenMV IDE projects and
scripts.
