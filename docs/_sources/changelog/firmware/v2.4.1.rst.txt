v2.4.1
======

v2.4.1 is a small maintenance release: it adds ``b_and()`` / ``b_or()`` /
``b_nand()`` / ``b_nor()`` / ``b_xor()`` / ``b_xnor()`` aliases for the binary
image operations and applies USB-MSC (SCSI layer) and Kanji fixes from
MicroPython upstream. It introduces **no breaking API changes** — v2.4.0
scripts run unchanged.

.. contents:: On this page
   :local:
   :depth: 1

Highlights
----------

- **Binary-op aliases** — added ``b_and()`` / ``b_or()`` / ``b_nand()`` /
  ``b_nor()`` / ``b_xor()`` / ``b_xnor()`` (the existing ``and()`` / ``or()``
  / … names still work).
- **Upstream fixes** — USB MSC (SCSI layer) and Kanji fixes from MicroPython.
- **No breaking API changes** — reflash and your v2.4.0 scripts run
  unchanged.

New features
------------

- Added ``b_and()`` / ``b_nand()`` / ``b_or()`` / ``b_nor()`` / ``b_xor()`` /
  ``b_xnor()`` method names for the binary image operations as clearer
  aliases; the original ``and()`` / ``nand()`` / ``or()`` / ``nor()`` /
  ``xor()`` / ``xnor()`` names continue to work. Updated the binary-ops
  documentation.

Bug fixes
---------

- Applied the USB mass-storage SCSI-layer fix and the Kanji fix from
  MicroPython upstream.

Breaking API changes
--------------------

v2.4.1 introduces **no breaking Python API changes**. Scripts that ran on
v2.4.0 run unchanged on v2.4.1 — just reflash the firmware.
