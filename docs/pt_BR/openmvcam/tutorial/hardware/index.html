3. Controle de Hardware¶

Cada valor Python no guia introdutório vivia na memória: um número chamado x, uma string retornada por uma função, uma lista manipulada por um laço. O controle de hardware substitui o trabalho em memória por efeitos físicos – uma tensão em um pino, um trem de pulsos em um fio, um byte enviado para outro chip por um barramento.

O módulo machine é a ponte. Suas classes (Pin, ADC, PWM, UART, SPI, I2C, CAN, WDT) mapeiam cada uma um pedaço do silício da câmera para o Python: configure uma instância, chame seus métodos e veja efeitos reais nos pinos da placa.
