v5.0.0#

v5.0.0 — це основний випуск. Він представляє OpenMV Viewer — окрему збірку лише для перегляду, призначену для демонстрації готового продукту — новий OpenMV Cam Settings Editor, протокол налагодження камери v2 для камер v5.0.0 та великий набір покращень робочих процесів редактора, термінала, прошивки й машинного зору. Його побудовано на Qt Creator 14.0.2.

Стрибок версії до 5.0.0 знаменує появу покоління камер v5.0.0 та інструментів, що його підтримують; наявні проєкти й робочі процеси v4 продовжують працювати без змін.

Основне#

  • OpenMV Viewer — новий, встановлюваний окремо варіант збірки, який запускає IDE як обмежений засіб перегляду телеметрії лише для читання. Він показує буфер кадрів камери, послідовний термінал та гістограму й може запускати фіксований скрипт, але приховує інструменти створення, AI/моделей і налаштувань, тож клієнт може демонструвати продукт, побудований на OpenMV, без повного середовища розробки.

  • OpenMV Cam Settings Editor — графічний редактор для файлу конфігурації налаштувань, що зберігається на камері, тож ви можете переглядати та змінювати налаштування плати через форму замість ручного редагування файлу.

  • Протокол налагодження камери v2 — камери v5.0.0 використовують новий, надійніший протокол налагодження з перевіркою CRC заголовка, відновленням після пропусків у послідовності та роздільним відображенням частоти кадрів, що окремо повідомляє FPS камери та FPS відображення в IDE.

  • Завантаження власної прошивки з .zip — інсталятор прошивки тепер приймає напряму комплект .zip з результатами збірки плати, що спрощує прошивання власного або локально зібраного образу прошивки.

  • Налагоджувальне логування протоколу в Serial Terminal — перемикач у Serial Terminal логує протокол налагодження камери зручними для читання рядками з позначками-емодзі для діагностики проблем із підключенням.

  • Перероблене меню Help та документація в редакторі — підменю OpenMV Docs замінює старі плоскі посилання, F1 відкриває документацію для символу Python під курсором, наведення на назву API показує її документацію вбудовано, а діалог About було модернізовано.

Нові можливості#

  • Варіант збірки OpenMV Viewer. Збірка з примусовим режимом перегляду, брендована та встановлювана як OpenMV Viewer, з власним ідентифікатором виконуваного файлу, сховищем налаштувань та ідентифікатором комплекту, тож вона співіснує з повною IDE. Вона подається як засіб перегляду телеметрії лише для читання: Run/Stop та зупинка при підключенні працюють і запускають скрипт із диска (через -auto_run), меню File/Edit та Preferences залишаються, а відновлення прошивки/ROMFS збережено, тоді як інструменти створення, інструменти машинного зору для розробників, категорії налаштувань для розробників та ресурси AI/моделей/документації вилучено зі збірки та інтерфейсу. ROMFS стає доступним лише для завантаження й переноситься до меню Tools, а діалоги завершення оновлення прошивки узагальнено (07722e78a8b, 5cd12ee260e, b97bf5447e4, 952b86de391, 1103f7988ed, f0f60261485, c743d5d67b9, b4bcdc2d71a, a37c032b6ff, 480e5668d29, 11c064cac93).

  • OpenMV Cam Settings Editor — графічний редактор для файлу конфігурації налаштувань камери, що відкривається з меню Tools (issue #205) (f55a0d7fa58, 2d7f80b03cf).

  • Протокол налагодження камери v2 для камер v5.0.0, з перевіркою CRC заголовка перед тим, як будь-якому полю заголовка можна довіряти, відновленням після пропусків у послідовності, що просувається вперед через втрачені пакети замість зупинки, та показником частоти кадрів, розділеним на частоту камери й частоту відображення в IDE (c377bdbaea8, a3f0488ddcb, 29108dd40f7, c15e6c02789).

  • Завантаження власної прошивки приймає комплект .zip — виберіть zip з результатами збірки плати, і IDE прошиває прошивку (та ROMFS), яку він містить (55e4eb2d900).

  • Налагоджувальне логування в Serial Terminal — кнопка перемикає багаторівневе налагоджувальне логування протоколу камери, де кожен рядок позначено емодзі та відображено шрифтом емодзі для зручності читання (09990c62810, 42e23c3c11e, 997705548c7, d2208de77d4).

  • Copy/Convert Current Script — дія в меню Tools, що копіює або конвертує поточний скрипт, із захистом від збереження скомпільованих main.py/boot.py (81a954f33cd).

  • Довідка з документації F1 для символа Python під курсором, що використовує спільний з підказками при наведенні механізм визначення символів (c341330bc16).

  • Підказки з документацією в редакторі — наведення на назву API OpenMV показує її документацію вбудовано, отриману зі стабів .pyi, що постачаються, та з власної документації модуля, тож ви можете дізнатися, що робить функція, метод, константа чи модуль, не залишаючи свого скрипта (7ebde92d2a6, f225824ba13, 59ad01a364a, 02c1e61a2a6).

  • Назвати камеру — локальний для IDE псевдонім, який ви можете дати підключеній камері, що відображається замість назви послідовного порту, тож ідентичні камери можна розрізнити (c600158ca34).

  • Сторінка комбінацій клавіш — сторінка комбінацій у Preferences тепер показується та відфільтрована до дій, які ви дійсно можете виконати, з чіткішим джерелом і компонуванням (4f5eae52e8e, 2d472fe5b40).

Покращення редактора, термінала та інтерфейсу#

  • Меню Help реструктуровано у підменю OpenMV Docs (прибрано старі пласкі посилання на розпіновку), із посиланнями на нотатки випусків, доданими до шляхів версій офіційних випусків (e1337ff67c4, 9eacc0994e6).

  • Діалог About модернізовано — автоматичний рік авторського права, таблиця логотипів партнерів та однорядкове посилання на нотатки випуску (d89fb74138d).

  • Панель Frame Buffer тепер показує роздільну здатність та позначку режиму буфера кадрів поруч із фіксованою кнопкою JPG; позначка режиму зроблена сірою, щоб не виглядати клікабельною (cb735f82337, 376b17d4e77, 39c6ffc3b26).

  • Графіки панелі гістограми згладжені сплайнами, панель може стискатися (приховуючи підписи та стискаючи графіки), а підпис каналу розташовано поруч зі статистикою (327324ef7e9, 9f8c7035483, d204e93d5e4).

  • Пов’язані роздільники панелей — перетягування роздільника з Ctrl/Cmd рухає роздільники редактора/термінала та буфера кадрів/гістограми разом, а маркери мініроздільників отримують ореол при наведенні з ширшою областю захоплення (379dee250d6, 96b83984342).

  • Serial Terminal — автопрокручування більше не закріплюється заново на кожній порції виводу, тож прокручування вгору під час швидкого виводу залишається на місці; термінал може запускати файл із диска, коли немає редактора (af67a076e95, bc132aef536).

  • Команди документації називаються за сторінкою замість індексу (a3201e9a4b4).

  • High-DPI / дробове масштабування — піктограми, накладення та заставка коректно відображаються за дробових коефіцієнтів масштабування (125%, 150% тощо), а головне вікно запам’ятовує свою геометрію та екран між перезапусками на конфігураціях з кількома моніторами (c60711e4950, 60901b6e13f, f241d2ea8ce, 54a1bc31fbd).

Прошивка, плати та інструменти#

  • Плата може закріпити останню випущену для неї версію прошивки, тож плата, яка більше не отримує новішої прошивки, не позначається як застаріла (bd987d32f42).

  • Текст статусу застарілої прошивки тепер забарвлено червоним, щоб виділятися (9b18d96bac5).

  • Підказка щодо відновлення завантажувача — невідповідність bootloader.bin тепер підказує перейти в режим відновлення завантажувача (d46b998c76e).

  • Відтворення відео стало надійнішим: шлях до FFmpeg показується, коли інструменти не можуть його запустити, повідомлення чіткіше, коли архітектура не підтримує відтворення, вхідні формати запитуються через ffmpeg -demuxers, а також додано запасні варіанти терміналів для Linux (0cc0e74a438, a75ec4898d0, 28e1ef10326, 51651780723).

  • Dataset Capture Script оновлено до модуля csi (41a781651e4).

  • Інсталятор для Windows — покращено вигляд майстра (250599d9d9e).

Виправлення помилок#

  • Встановлення прошивки для розробки для плат .lst (AE3) тепер розміщує двійкові файли, на які посилається лістинг, у каталог прошивки, а встановлення для розробки/власне копіює образ ROMFS з комплекту, спочатку очищаючи будь-який застарілий romfs*.img, тож прошивається завантажений ROMFS, а не залишковий дамп редактора (3cf56e98668, 43bedc8c591).

  • Попередження libjpeg для обрізаних/пошкоджених кадрів відфільтровуються з журналу, тож потік із втратами не засмічує термінал (6e2a8bed6cf, a5166abbdc1).

  • Список прикладів більше не показує внутрішній index.json (14ac8ca48a7).

  • Спливні вікна оновлення прошивки та початкового ознайомлення приховуються в режимі перегляду, тож демонстрація продукту ніколи не пропонує перезаписати прошивку клієнта (7b8528c3ad6).

  • Висячі та призначені лише для створення комбінації клавіш, що залишилися від вилучених меню, було вимкнено (be3bee8bb58, 27d1afc8705).

Локалізація#

  • Чотири нові мови — індонезійська, румунська, тайська та в’єтнамська — довели IDE до 37 перекладених мов, і кожен каталог було оновлено, перекладено та очищено для всіх мов (fcf04090843, c830e6c4de3, 5e22679923f).

Командний рядок#

  • Псевдоніми параметрів у стилі GNU ``–word`` тепер приймаються поряд із формами з одним дефісом, а вивід -help зазначає цю еквівалентність (66414213dd9, 768c5d0e159).

Підтримка платформ та інструментів#

  • Основа Qt Creator: 14.0.2.

  • OpenMV IDE оновлено до версії 5.0.0.

  • Нове: встановлюваний окремо варіант збірки OpenMV Viewer.

  • Підтримка Python 3.14 — вбудований парсер parso було оновлено до 0.8.7, що дозволяє автодоповненню коду та мовному серверу працювати з найновішим синтаксисом Python (578a4e39d97).

Критичні зміни#

Немає для наявних проєктів. Протокол налагодження v2 застосовується до камер v5.0.0; камери покоління v4 продовжують використовувати наявний протокол, а наявні скрипти й робочі процеси не зачіпаються.
