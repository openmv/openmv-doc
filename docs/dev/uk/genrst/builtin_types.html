Виняток¶

Усі винятки мають доступні для читання атрибути value та errno, а не лише StopIteration та OSError.¶

Причина: MicroPython оптимізовано для зменшення розміру коду.

Обхідний шлях: Використовуйте value лише для винятків StopIteration, а errno — лише для OSError. Не використовуйте та не покладайтесь на ці атрибути в інших винятках.

e = Exception(1)
print(e.value)
print(e.errno)

OSError¶

OSError constructor returns a plain OSError for all errno values, rather than a relevant subtype.¶

Cause: MicroPython does not include the CPython-standard OSError subclasses.

Workaround: Catch OSError and use its errno attribute to discriminate the cause.

Зразок коду:

import errno

errno_list = [  # i.e. the set implemented by micropython
    errno.EPERM,
    errno.ENOENT,
    errno.EIO,
    errno.EBADF,
    errno.EAGAIN,
    errno.ENOMEM,
    errno.EACCES,
    errno.EEXIST,
    errno.ENODEV,
    errno.EISDIR,
    errno.EINVAL,
    errno.EOPNOTSUPP,
    errno.EADDRINUSE,
    errno.ECONNABORTED,
    errno.ECONNRESET,
    errno.ENOBUFS,
    errno.ENOTCONN,
    errno.ETIMEDOUT,
    errno.ECONNREFUSED,
    errno.EHOSTUNREACH,
    errno.EALREADY,
    errno.EINPROGRESS,
]


def errno_output_type(n):
    try:
        raise OSError(n, "")
    except OSError as e:
        return f"{type(e).__name__}"
    except Exception as e:
        return f"non-OSError {type(e).__name__}"
    else:
        return "no error"


for n in errno_list:
    print(errno.errorcode[n], "=", errno_output_type(n))

bytes¶

Об’єкти bytes підтримують метод .format()¶

Причина: MicroPython прагне бути більш однорідною реалізацією, тому якщо як str, так і bytes підтримують __mod__() (оператор %), логічно також підтримувати format() для обох. Підтримку __mod__ можна виключити під час компіляції, що залишає лише format() для форматування bytes.

Обхідний шлях: Якщо вам важлива сумісність із CPython, не використовуйте .format() на об’єктах bytes.

Зразок коду:

print(b"{}".format(1))

CPython output:

MicroPython output:

Traceback (most recent call last):
  File "<stdin>", line 8, in <module>
AttributeError: 'bytes' object has no attribute 'format'

b'1'

bytes() з ключовими аргументами не реалізовано¶

Обхідний шлях: Передайте кодування як позиційний параметр, наприклад print(bytes('abc', 'utf-8'))

Зразок коду:

print(bytes("abc", encoding="utf8"))

Індексування bytes із кроком != 1 не реалізовано¶

Причина: MicroPython оптимізовано для мінімального використання пам’яті.

Обхідний шлях: Для цієї вкрай рідкісної операції використовуйте явний цикл.

complex¶

MicroPython’s complex() accepts certain incorrect values that CPython rejects¶

Причина: MicroPython оптимізовано для мінімального використання пам’яті.

Workaround: Do not use non-standard complex literals as argument to complex()

MicroPython’s complex() function accepts literals that contain a space and no sign between the real and imaginary parts, and interprets it as a plus.

Зразок коду:

Метод to_bytes не реалізує параметр signed.¶

Причина: Параметр лише за ключовим словом signed не реалізовано для int.to_bytes().

Якщо ціле число від’ємне, MicroPython поводиться так само, як CPython int.to_bytes(..., signed=True)

Якщо ціле число невід’ємне, MicroPython поводиться так само, як CPython int.to_bytes(..., signed=False).

(Різниця незначна, але в CPython додатне ціле число, перетворене з signed=True, може потребувати на один байт більше у вихідній довжині, щоб вмістити знаковий біт 0.)

Обхідний шлях: Будьте обережні під час виклику to_bytes() для значення цілого числа, яке може бути від’ємним.

Зразок коду:

x = -1
print(x.to_bytes(1, "big"))

  File "<stdin>", line 16, in <module>

memoryview¶

memoryview може стати недійсним, якщо змінити розмір його цільового об’єкта¶

Причина: CPython запобігає зміні розміру об’єкта bytearray або io.bytesIO, доки на нього посилається об’єкт memoryview. MicroPython вимагає від програміста вручну переконатися, що розмір об’єкта не змінюється, поки на нього посилається будь-який memoryview.

У найгіршому випадку зміна розміру об’єкта, на який вказує memoryview, може призвести до того, що memoryview посилатиметься на недійсну звільнену пам’ять (помилка використання після звільнення) і пошкодить середовище виконання MicroPython.

Обхідний шлях: Не змінюйте розмір жодного об’єкта bytearray або io.bytesIO, якому призначено memoryview.

Зразок коду:

b = bytearray(b"abcdefg")
m = memoryview(b)
b.extend(b"hijklmnop")
print(b, bytes(m))

CPython output:

MicroPython output:

Traceback (most recent call last):
  File "<stdin>", line 12, in <module>
BufferError: Existing exports of data: object cannot be re-sized

bytearray(b'abcdefghijklmnop') b'abcdefg'

range¶

Range objects with large start or stop arguments misbehave.¶

Cause: Intermediate calculations overflow the C mp_int_t type

Workaround: Avoid using such ranges

Зразок коду:

from sys import maxsize

# A range including `maxsize-1` cannot be created
try:
    print(range(-maxsize - 1, 0))
except OverflowError:
    print("OverflowError")

# A range with `stop-start` exceeding sys.maxsize has incorrect len(), while CPython cannot calculate len().
try:
    print(len(range(-maxsize, maxsize)))
except OverflowError:
    print("OverflowError")

# A range with `stop-start` exceeding sys.maxsize has incorrect len()
try:
    print(len(range(-maxsize, maxsize, maxsize)))
except OverflowError:
    print("OverflowError")

CPython output:

MicroPython output:

range(-9223372036854775808, 0)
OverflowError
2

OverflowError
0
0

str¶

MicroPython accepts the «,» grouping option with any radix, unlike CPython¶

Cause: To reduce code size, MicroPython does not issue an error for this combination

Workaround: Do not use a format string like {:,b} if CPython compatibility is required.

Зразок коду:

try:
    print("{:,b}".format(99))
except ValueError:
    print("ValueError")
try:
    print("{:,x}".format(99))
except ValueError:
    print("ValueError")
try:
    print("{:,o}".format(99))
except ValueError:
    print("ValueError")

CPython output:

MicroPython output:

ValueError
ValueError

110,0011
63
143

MicroPython accepts but does not properly implement the «,» or «_» grouping character for float values¶

Cause: To reduce code size, MicroPython does not implement this combination. Grouping characters will not appear in the number’s significant digits and will appear at incorrect locations in leading zeros.

Workaround: Do not use a format string like {:,f} if exact CPython compatibility is required.

Зразок коду:

print("{:,f}".format(3141.159))
print("{:_f}".format(3141.159))
print("{:011,.2f}".format(3141.159))
print("{:011_.2f}".format(3141.159))

MicroPython output:

0_003_141.16

000,3141.16
0_003141.16

Причина: MicroPython оптимізовано для мінімального використання пам’яті. Доступні прості обхідні шляхи.

Обхідний шлях: Замість s.ljust(10) використовуйте "%-10s" % s, замість s.rjust(10) — "% 10s" % s. Або "{:<10}".format(s) чи "{:>10}".format(s).
