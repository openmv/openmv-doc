Emulasi CMUcam2#

CMUcam2 memiliki kemampuan berikut:

  • Lacak blob warna yang ditentukan pengguna hingga 50 frame per detik (frame rate bergantung pada pengaturan resolusi dan ukuran jendela)

  • Lacak gerakan menggunakan frame differencing pada 26 frame per detik

  • Temukan centroid dari data pelacakan apa pun

  • Kumpulkan data warna rata-rata dan varians

  • Kumpulkan histogram 28 bin dari setiap saluran warna

  • Memproses gambar yang difilter tepi secara horizontal (tidak didukung)

  • Transfer bitmap biner real-time dari piksel yang dilacak dalam sebuah gambar

  • Windowing gambar sembarang

  • Down sampling gambar

  • Sesuaikan properti gambar kamera

  • Dump gambar mentah (saluran tunggal atau ganda)

  • Resolusi hingga 176 x 255

  • Mendukung baud rate: 115,200 57,600 38,400 19,200 9,600 4,800 2,400 1,200 BPS

  • Mengontrol 5 keluaran servo

  • Mode pemrosesan gambar paralel slave dari satu bus kamera tunggal (tidak didukung)

  • Secara otomatis menggunakan servo untuk melakukan pelacakan warna dua sumbu

  • Keluaran video analog hitam putih (PAL atau NTSC, tergantung modul kamera yang digunakan)

  • Kustomisasi paket keluaran yang fleksibel

  • Mode power down

  • Pemrosesan gambar multi-lintasan pada gambar yang di-buffer

Perbedaan Emulasi CMUcam2#

  • Rentang Input Servo

    • Servo sekarang bernilai 8-bit 0-255

  • Jumper

    • Karena tidak adanya jumper pada CMUcam3, perubahan baud rate dan arah servo perlu dilakukan pada waktu kompilasi - ini dikontrol oleh #define di bagian atas kode sumber cmucam.c

  • Perintah Set Input (SI pin )

    • Menetapkan pin menjadi input

  • Perintah Software Color Space (CS value )

    • CS 0 mengatur ruang warna ke default perangkat keras (RGB atau YCrCb)

    • CS 1 melakukan konversi perangkat lunak ke ruang warna HSV

      • Konversi mengasumsikan perangkat keras diatur ke RGB untuk hasil yang benar

      • Ruang HSV adalah ruang integer di mana semua nilai diskalakan antara 16 dan 240

    • Catatan: Ini hanya tersedia di r504 dan yang lebih tinggi

  • Set Output dapat mengontrol LED Biru dan Oranye

    • Servo 2 dan 3 akan berhenti bekerja

    • SO 2 1 -> Biru menyala

    • SO 2 0 -> Biru mati

    • SO 3 1 -> Oranye menyala

    • SO 3 0 -> Oranye mati

  • Tanpa Mode Slave

    • Ini dimungkinkan, tetapi saat ini kami tidak mendukungnya

  • Output Mask (OM)

    • Hanya mendukung paket T dan S (tipe 0 dan 1)

  • Mode Baris (LM)

    • Hanya mendukung mode track color (LM 0 1)

    • Statistik track color ditambahkan di r523 (LM 0 2)

  • Kirim JPEG (SJ)

    • Mengirim frame terkompresi jpeg, header dan semuanya

  • Mode Demo

    • Karena menahan tombol pemrograman menempatkan CMUcam3 ke mode pemrograman, Anda perlu menekan tombol setengah detik setelah startup untuk masuk ke mode demo

  • Hi-Res Frame Difference

    • Ini sekarang bekerja sama seperti frame differencing resolusi rendah, hanya pada resolusi yang lebih tinggi
