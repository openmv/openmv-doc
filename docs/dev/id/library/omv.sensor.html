sensor --- sensor kamera¶

Ditinggalkan sejak versi 4.5: Modul sensor sudah tidak digunakan lagi. Gunakan modul csi yang baru (lihat csi --- sensor kamera) sebagai gantinya. Tidak ada fitur baru yang akan ditambahkan ke modul ini dan modul ini mungkin akan dihapus pada rilis mendatang.

Modul sensor adalah antarmuka tingkat modul warisan untuk sensor kamera utama pada OpenMV Cam. Setiap panggilan adalah fungsi bebas yang beroperasi pada satu instans CSI tersembunyi, yang membatasi dukungan untuk papan dengan lebih dari satu kamera. Modul ini dipertahankan untuk kompatibilitas mundur dengan skrip OpenMV yang lebih lama; kode baru sebaiknya menggunakan modul csi yang berorientasi objek.

Nama fungsi mengikuti gaya lama set_pixformat / set_framesize. Setiap fungsi berkorespondensi satu-ke-satu dengan metode pada csi.CSI; lihat modul csi untuk rangkaian kemampuan lengkap dan deskripsi per argumen.

Contoh penggunaan:

import sensor

sensor.reset()
sensor.set_pixformat(sensor.RGB565)
sensor.set_framesize(sensor.QVGA)
sensor.skip_frames(time=2000)

while True:
    img = sensor.snapshot()

Fungsi¶

sensor.reset() → None¶

Menginisialisasi sensor kamera.

sensor.sleep(enable: bool) → None¶

Menempatkan kamera ke mode tidur jika enable adalah True. Jika tidak, membangunkannya kembali.

sensor.shutdown(enable: bool) → None¶

Menempatkan kamera ke mode daya rendah yang lebih rendah dari mode tidur. Kamera harus direset saat dibangunkan.

sensor.flush() → None¶

Menyalin apa pun yang ada di buffer bingkai ke pratinjau IDE.

sensor.snapshot() → image.Image¶

Mengambil gambar menggunakan kamera dan mengembalikan objek image.Image.

Jika sensor.set_auto_rotation() diaktifkan, metode ini mengembalikan objek image.Image baru yang sudah diputar.

sensor.skip_frames(n: int | None = None, time: int = 300) → None¶

Melewati n bingkai atau time milidetik (mana yang ditentukan) untuk membiarkan citra kamera stabil setelah mengubah pengaturan kamera.

Jika tidak ada n maupun time yang ditentukan, metode ini melewati bingkai selama 300 milidetik.

Jika keduanya ditentukan, metode ini melewati n bingkai tetapi akan habis waktu setelah time milidetik.

sensor.width() → int¶

Mengembalikan lebar resolusi sensor.

sensor.height() → int¶

Mengembalikan tinggi resolusi sensor.

sensor.get_fb() → image.Image | None¶

Mengembalikan objek citra yang dikembalikan oleh panggilan sebelumnya dari sensor.snapshot(). Mengembalikan None jika sensor.snapshot() belum pernah dipanggil sebelumnya.

sensor.get_id() → int¶

Mengembalikan ID modul kamera. Lihat konstanta sensor di bawah ini.

sensor.get_frame_available() → bool¶

Mengembalikan True jika bingkai tersedia untuk dibaca dengan memanggil sensor.snapshot().

sensor.alloc_extra_fb(width: int, height: int, pixformat: int) → image.Image¶

Ditinggalkan sejak versi 4.5: Fungsi ini sudah tidak digunakan lagi dan akan memunculkan OSError. Gunakan modul csi yang baru sebagai gantinya.

sensor.dealloc_extra_fb() → None¶

Ditinggalkan sejak versi 4.5: Fungsi ini sudah tidak digunakan lagi dan akan memunculkan OSError. Gunakan modul csi yang baru sebagai gantinya.

sensor.set_pixformat(pixformat: int) → None¶

Menetapkan format piksel untuk modul kamera. pixformat adalah salah satu dari:

  • sensor.BINARY

  • sensor.GRAYSCALE

  • sensor.RGB565

  • sensor.BAYER

  • sensor.YUV422

  • sensor.JPEG (hanya untuk OV2640/OV5640)

sensor.get_pixformat() → int¶

Mengembalikan pixformat saat ini untuk modul kamera.

sensor.set_framesize(framesize: int) → None¶

Menetapkan ukuran bingkai untuk modul kamera. Lihat konstanta ukuran bingkai di bawah ini untuk nilai yang valid.

sensor.get_framesize() → int¶

Mengembalikan ukuran bingkai saat ini untuk modul kamera.

sensor.set_framerate(rate: int) → None¶

Menetapkan laju bingkai dalam Hz untuk modul kamera.

sensor.get_framerate() → int¶

Mengembalikan laju bingkai dalam Hz untuk modul kamera.

sensor.set_windowing(roi: Tuple[int, int] | Tuple[int, int, int, int] | List[int]) → None¶

Menetapkan resolusi kamera ke sub-resolusi di dalam resolusi saat ini.

roi adalah tuple/daftar rect (x, y, w, h). Anda juga dapat meneruskan (w, h) dan roi akan dipusatkan pada bingkai. Argumen juga dapat diteruskan secara tidak terpak sebagai bilangan bulat posisional.

sensor.get_windowing() → Tuple[int, int, int, int]¶

Mengembalikan tuple roi (x, y, w, h) yang sebelumnya ditetapkan dengan sensor.set_windowing().

sensor.set_gainceiling(gainceiling: int) → bool¶

Menetapkan batas atas gain citra kamera. Nilai yang valid adalah 2, 4, 8, 16, 32, 64, atau 128. Mengembalikan True jika berhasil.

sensor.set_contrast(contrast: int) → bool¶

Menetapkan kontras citra kamera. Rentang valid adalah -3 hingga +3. Mengembalikan True jika berhasil.

sensor.set_brightness(brightness: int) → bool¶

Menetapkan kecerahan citra kamera. Rentang valid adalah -3 hingga +3. Mengembalikan True jika berhasil.

sensor.set_saturation(saturation: int) → bool¶

Menetapkan saturasi citra kamera. Rentang valid adalah -3 hingga +3. Mengembalikan True jika berhasil.

sensor.set_quality(quality: int) → bool¶

Menetapkan kualitas kompresi JPEG citra kamera. Rentang valid adalah 0 hingga 100. Mengembalikan True jika berhasil. Hanya untuk kamera OV2640/OV5640.

sensor.set_colorbar(enable: bool) → bool¶

Mengaktifkan mode uji pita warna (True) atau mematikannya (False). Mengembalikan True jika berhasil.

sensor.set_auto_gain(enable: int, gain_db: float | None = None, gain_db_ceiling: float | None = None) → None¶

enable mengaktifkan kontrol gain otomatis (1) atau mematikannya (0).

Jika enable adalah 0, Anda dapat menetapkan gain tetap dalam desibel dengan gain_db.

Jika enable bukan nol, Anda dapat menetapkan batas atas gain maksimum dalam desibel dengan gain_db_ceiling untuk algoritme kontrol gain otomatis.

gain_db dan gain_db_ceiling adalah argumen khusus kata kunci.

sensor.get_gain_db() → float¶

Mengembalikan nilai gain kamera saat ini dalam desibel.

sensor.set_auto_exposure(enable: int, exposure_us: int = -1) → None¶

enable mengaktifkan kontrol eksposur otomatis (1) atau mematikannya (0).

Jika enable adalah 0, Anda dapat menetapkan waktu eksposur tetap dalam mikrodetik dengan exposure_us. exposure_us adalah argumen khusus kata kunci.

sensor.get_exposure_us() → int¶

Mengembalikan nilai eksposur kamera saat ini dalam mikrodetik.

sensor.set_auto_whitebal(enable: int, rgb_gain_db: Tuple[float, float, float] | None = None) → None¶

enable mengaktifkan keseimbangan putih otomatis (1) atau mematikannya (0).

Jika enable adalah 0, Anda dapat menetapkan gain tetap dalam desibel untuk saluran merah, hijau, dan biru masing-masing dengan rgb_gain_db. rgb_gain_db adalah argumen khusus kata kunci.

sensor.get_rgb_gain_db() → Tuple[float, float, float]¶

Mengembalikan tuple dengan nilai gain merah, hijau, dan biru kamera saat ini dalam desibel.

sensor.set_auto_blc(enable: int, regs: List[int] | None = None) → None¶

Mengatur kontrol kalibrasi level hitam otomatis (BLC) pada kamera.

enable adalah 1 untuk mengaktifkan atau 0 untuk menonaktifkan.

regs jika dinonaktifkan, Anda dapat menetapkan nilai register BLC secara manual melalui nilai yang sebelumnya dibaca dari sensor.get_blc_regs(). Panjang daftar harus sesuai dengan jumlah register BLC sensor.

sensor.get_blc_regs() → List[int]¶

Mengembalikan register BLC sensor sebagai daftar bilangan bulat. Untuk digunakan dengan sensor.set_auto_blc().

sensor.set_hmirror(enable: bool) → None¶

Mengaktifkan mode cermin horizontal (True) atau mematikannya (False). Defaultnya adalah mati.

sensor.get_hmirror() → bool¶

Mengembalikan True jika mode cermin horizontal diaktifkan.

sensor.set_vflip(enable: bool) → None¶

Mengaktifkan mode balik vertikal (True) atau mematikannya (False). Defaultnya adalah mati.

sensor.get_vflip() → bool¶

Mengembalikan True jika mode balik vertikal diaktifkan.

sensor.set_transpose(enable: bool) → None¶

Mengaktifkan mode transpos (True) atau mematikannya (False). Defaultnya adalah mati.

  • vflip=False, hmirror=False, transpose=False -> rotasi 0 derajat

  • vflip=True, hmirror=False, transpose=True -> rotasi 90 derajat

  • vflip=True, hmirror=True, transpose=False -> rotasi 180 derajat

  • vflip=False, hmirror=True, transpose=True -> rotasi 270 derajat

sensor.get_transpose() → bool¶

Mengembalikan True jika mode transpos diaktifkan.

sensor.set_auto_rotation(enable: bool) → None¶

Mengaktifkan mode rotasi otomatis (True) atau mematikannya (False). Defaultnya adalah mati. Hanya berfungsi ketika OpenMV Cam memiliki imu yang terpasang.

sensor.get_auto_rotation() → bool¶

Mengembalikan True jika mode rotasi otomatis diaktifkan.

sensor.set_framebuffers(count: int) → None¶

Menetapkan jumlah buffer bingkai yang digunakan untuk menerima data citra.

count dapat berupa 1 (buffer tunggal), 2 (buffer ganda), 3 (buffer tiga), atau 4 atau lebih untuk menempatkan driver sensor ke mode video FIFO di mana bingkai yang diterima disimpan dalam FIFO count buffer.

sensor.get_framebuffers() → int¶

Mengembalikan jumlah buffer bingkai yang dialokasikan saat ini.

sensor.disable_delays(disable: bool | None = None) → bool | None¶

Jika disable adalah True, maka semua penundaan waktu penstabilan dalam modul sensor dinonaktifkan.

Jika dipanggil tanpa argumen, mengembalikan True jika penundaan dinonaktifkan.

sensor.disable_full_flush(disable: bool | None = None) → bool | None¶

Jika disable adalah True, maka pengosongan framebuffer otomatis saat bingkai terjatuh dinonaktifkan.

Jika dipanggil tanpa argumen, mengembalikan True jika pengosongan otomatis dinonaktifkan.

sensor.set_special_effect(sde: int) → bool¶

Menetapkan efek digital khusus (SDE) pada sensor. sde adalah salah satu dari sensor.NORMAL atau sensor.NEGATIVE. Mengembalikan True jika berhasil.

sensor.set_lens_correction(enable: bool, radi: int, coef: int) → bool¶

enable True untuk mengaktifkan, False untuk menonaktifkan. radi radius bilangan bulat piksel yang akan dikoreksi. coef daya koreksi.

Mengembalikan True jika berhasil.

sensor.set_vsync_callback(cb: Callable[[int], None] | None) → None¶

Mendaftarkan callback cb untuk dieksekusi (dalam konteks interupsi) setiap kali modul kamera menghasilkan bingkai baru (tetapi sebelum bingkai diterima).

cb mengambil satu argumen: status pin vsync saat ini setelah berubah.

Teruskan nilai yang tidak dapat dipanggil (misalnya None) untuk membatalkan pendaftaran.

sensor.set_frame_callback(cb: Callable[[], None] | None) → None¶

Mendaftarkan callback cb untuk dieksekusi (dalam konteks interupsi) setiap kali modul kamera menghasilkan bingkai baru dan bingkai siap dibaca melalui sensor.snapshot().

cb tidak mengambil argumen.

Teruskan nilai yang tidak dapat dipanggil (misalnya None) untuk membatalkan pendaftaran.

sensor.ioctl(request: int, *args: Any) → Any¶

Menjalankan permintaan khusus sensor. request adalah salah satu konstanta IOCTL_*; argumen posisional yang tersisa dan nilai kembalian bergantung pada permintaan. Permintaan yang didukung dikelompokkan berdasarkan keluarga sensor di bawah ini.

Generik (sensor apa pun):

ioctl(IOCTL_SET_READOUT_WINDOW, (x, y, w, h))

ioctl(IOCTL_SET_READOUT_WINDOW, (w, h))

Menetapkan jendela pembacaan sensor. Jendela yang lebih kecil meningkatkan laju bingkai dengan mengorbankan bidang pandang.

ioctl(IOCTL_GET_READOUT_WINDOW)

Mengembalikan jendela pembacaan saat ini sebagai tuple (x, y, w, h).

ioctl(IOCTL_SET_TRIGGERED_MODE, enable)

Mengaktifkan (True) atau menonaktifkan (False) mode terpicu pada MT9V034.

ioctl(IOCTL_GET_TRIGGERED_MODE)

Mengembalikan status mode terpicu saat ini sebagai bool.

ioctl(IOCTL_SET_FOV_WIDE, enable)

Jika True, instruksikan set_framesize() untuk mengoptimalkan bidang pandang daripada laju bingkai.

ioctl(IOCTL_GET_FOV_WIDE)

Mengembalikan status FOV-wide saat ini sebagai bool.

ioctl(IOCTL_SET_NIGHT_MODE, enable)

Mengaktifkan (True) atau menonaktifkan (False) "mode malam" cahaya rendah sensor. Hanya untuk OV7725 dan OV5640.

ioctl(IOCTL_GET_NIGHT_MODE)

Mengembalikan status mode malam saat ini sebagai bool.
