13.1.20. Penutup¶

IDE dalam satu paragraf: skrip diedit dalam editor profesional yang mengenal API kamera, dijalankan di kamera dengan satu tombol, dan diamati melalui tiga instrumen langsung -- penampil buffer bingkai untuk apa yang dilihat kamera, histogram untuk angka di baliknya, dan terminal serial untuk apa yang dikatakan skrip. Di sekeliling siklus tersebut terdapat alat pemeliharaan yang menjaga firmware, sistem file, dan ROMFS kamera tetap teratur; alat visi mesin yang menghasilkan artefak yang dikonsumsi oleh skrip -- tuple ambang batas, deskriptor yang dibersihkan, tag yang dicetak, model yang dikonversi, dataset berlabel; serta alat canggih yang melihat di balik permukaan ketika performa menjadi perhatian.

Ke mana harus pergi dari sini:

  • Menu contoh adalah jawaban tetap untuk "bagaimana cara melakukan X di kamera" -- hampir setiap fitur library memiliki contoh yang dapat dijalankan.

  • referensi library mendokumentasikan setiap modul yang ditawarkan oleh popup penyelesaian.

  • Paket openmv Python mengendalikan kamera dari skrip Python di sisi host -- protokol debug IDE tanpa IDE -- untuk rig pengujian, otomasi, dan frontend desktop khusus.

  • bab produksi melanjutkan dari tempat langkah deploy IDE berhenti: menanamkan skrip ke dalam firmware, mengirimkan aset di ROMFS, dan mengeras kamera untuk penggunaan di lapangan.
