v4.8.8#

v4.8.8 añade controles explícitos de entrada al bootloader para conectar y actualizar las OpenMV Cam, un valor predeterminado seguro por placa para forzar la entrada al bootloader desde el diálogo «Install the Latest Development Release» y un puñado de correcciones de conexión y del instalador. Esta versión sigue basándose en Qt Creator 14.0.2, sin cambio de base de Qt Creator y sin cambios incompatibles para los proyectos de usuario.

Aspectos destacados#

  • Force enter OpenMV Cam bootloader ya está disponible desde el menú Tools, forzando a la cam conectada a entrar en su bootloader al desconectarse.

  • Se añadió la casilla Force bootloader al diálogo Install the Latest Development Release para forzar la entrada al bootloader al instalar firmware de desarrollo, con un valor predeterminado seguro por placa.

  • Los dispositivos solo DFU ya no se conectan automáticamente de forma espuria, ya que la reconexión automática y la conexión automática por ajustes de placa ahora solo se activan para placas que funcionan con normalidad.

  • Los scripts de ejemplo de PAG7936 en la AE3 se reescriben automáticamente para escalar el framesize QQVGA a QVGA al cargarse.

  • Se corrigió el manejo de licencias del instalador para que la instalación continúe correctamente desde la página Ready-for-Installation.

Nuevas funciones#

  • Force enter OpenMV Cam bootloader. Un nuevo elemento en el menú Tools fuerza a la cam conectada a entrar en su bootloader al desconectarse. El diálogo Install the Latest Development Release (al que se accede desde el menú Tools) también incorpora una casilla Force bootloader para forzar la entrada al bootloader al instalar firmware de desarrollo. (350beb306, 13ca76904)

Otros cambios y mejoras#

  • Valor predeterminado de force-bootloader por placa. El valor predeterminado de force-bootloader para la instalación de versiones de desarrollo ahora se controla por placa mediante un nuevo ajuste forceBootloaderDefaultSafe, activado por defecto para las placas AE3 y N6 (96E3/9206) y desactivado para las placas DFU más antiguas, a fin de evitar que no salgan automáticamente del bootloader. (13ca76904)

  • Escalado del framesize de PAG7936 en la AE3. En la AE3 con el sensor PAG7936, los scripts de ejemplo se reescriben automáticamente para escalar el framesize QQVGA a QVGA al cargarse. (eed6325f5)

  • Conexión automática no exclusiva de DFU. La reconexión automática y la conexión automática por ajustes de placa ahora solo se activan para placas no DFU (que funcionan con normalidad) en lugar de cualquier placa presente, de modo que los dispositivos solo DFU ya no se conectan automáticamente de forma espuria. (350beb306)

Corrección de errores#

  • Se corrigió un error tipográfico en el aviso de actualización de firmware al conectar («lastest» -> «latest release firmware»). (13ca76904)

  • Se corrigió el manejo de aceptación de licencia del instalador para que el paso de instalación se ejecute en la página Ready-for-Installation en lugar de en la página de comprobación de licencia, y se eliminó el archivo de licencia incluido que tenía errores. (a9d3fb904, 716827f07)

Compatibilidad de plataforma y herramientas#

  • Base de Qt Creator: 14.0.2.

  • OpenMV IDE: versión actualizada a 4.8.8.

Cambios incompatibles#

Ninguno. Esta versión no introduce cambios visibles para el usuario en el flujo de trabajo ni en el comportamiento.
