v4.4.8¶

v4.4.8 es una versión de mantenimiento que actualiza el firmware y las herramientas incluidas. Sigue basándose en Qt Creator 14.0.2 sin ningún rebase en este rango. El firmware de OpenMV incluido pasa a la v4.6.0, se incluye por primera vez el firmware para STM32N6 (OpenMV Cam N6), se reconocen los nuevos sensores térmicos FLIR Boson y se actualiza el Alif Security Toolkit. No hay cambios incompatibles visibles para el usuario, y los proyectos de usuario existentes no se ven afectados.

Aspectos destacados¶

  • Firmware incluido v4.6.0: El firmware y los ejemplos de OpenMV incluidos se actualizan de la v4.5.9 a la v4.6.0 (66bfd486f).

  • Firmware de OpenMV Cam N6 incluido: El firmware para STM32N6 (OpenMV Cam N6) ahora se distribuye en el paquete por primera vez (66bfd486f).

  • Sensores térmicos FLIR Boson: Los sensores térmicos BOSON, BOSON-320/BOSON-640 y BOSON-320+/BOSON-640+ (Boson Plus) ahora son reconocidos por el IDE (8b795053c, d5068b286).

  • Alif Security Toolkit v1.101.0: El toolkit de Windows incluido se actualiza a la v1.101.0 (7d51b00b1).

  • Corrección del flasheo del SBL de OpenMV Cam AE3: La placa AE3 ahora ejecuta el gestor de arranque (bootloader) seguro correcto al flashear (ed7c95357).

Nuevas funciones¶

  • Ejemplos de FLIR Boson: Se añadieron scripts de ejemplo incluidos para la transmisión en escala de grises e ironbow del FLIR Boson, además del seguimiento por color, en 01-Camera/05-Thermal-Cameras/02-FLIR-Boson, y un ejemplo de seguimiento de LED en escala de grises del GenX320 en 01-Camera/03-Event-Cameras/02-Genx320 (66bfd486f).

Otros cambios y mejoras¶

  • Se reorganizó el árbol de ejemplos incluido: los ejemplos de FLIR Lepton se movieron a una nueva categoría 01-Camera/05-Thermal-Cameras/01-FLIR-Lepton junto a los nuevos ejemplos de Boson, y los ejemplos de cámaras de eventos se dividieron en las subcarpetas 01-Camera/03-Event-Cameras/01-Frogeye2020 y 02-Genx320 (66bfd486f).

  • Se eliminó del paquete la carpeta de firmware OPENMV4_PRO, que estaba sin uso y huérfana; ninguna entrada de placa la referenciaba, por lo que no se ve afectada ninguna placa flasheable (66bfd486f).

  • Se actualizaron el firmware del SBL de OpenMV AE3, el gestor de arranque (bootloader) y las imágenes firmadas que se utilizan para el flasheo (b8c06a8ab).

  • Se actualizó el Alif Security Toolkit (Windows) incluido a la v1.101.0, incluyendo la guía de usuario renovada (AUGD0005) y las imágenes SP firmadas (7d51b00b1).

Correcciones de errores¶

  • Se corrigió el flasheo del SBL de OpenMV Cam AE3 para que se ejecute el gestor de arranque (bootloader) seguro correcto, estableciendo su cadena de arquitectura de placa en OPENMV AE3 SBL (ed7c95357).

  • Se aumentó el tiempo de espera del borrado de memoria flash de blhost de 60 s a 120 s al programar placas i.MX RT, evitando tiempos de espera agotados durante el borrado y el flasheo de regiones más grandes (b48e139e4).

Compatibilidad de plataformas y herramientas¶

  • Base de Qt Creator: 14.0.2.

  • Firmware de OpenMV incluido: actualizado de 4.5.9 -> 4.6.0 (66bfd486f).

  • OpenMV Cam N6: El firmware para STM32N6 (firmware.bin, network.bin, openmv.bin) ahora se incluye en firmware/OPENMV_N6 (66bfd486f).

  • OpenMV AE3 SBL: binarios del firmware y del gestor de arranque (bootloader) actualizados (b8c06a8ab).

  • Alif Security Toolkit (Windows): actualizado de v1.100.0 -> v1.101.0 (7d51b00b1).

  • Nuevos sensores reconocidos: FLIR BOSON, BOSON-320, BOSON-640 (8b795053c), FLIR BOSON-320+ y BOSON-640+ (d5068b286), y el PixArt PS5520 (66bfd486f).

Cambios incompatibles¶

Ninguno. No hay cambios en la API de scripting, y los proyectos de usuario existentes no se ven afectados. La carpeta de firmware OPENMV4_PRO eliminada estaba sin uso y no era referenciada por ninguna placa, por lo que su eliminación no rompe ningún flujo de trabajo de flasheo.
