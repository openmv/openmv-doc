Pixy (CMUcam5)#

Nota

El sitio de Pixy se ha trasladado a pixycam.com, donde Pixy y sus sucesores siguen teniendo soporte. Esta página conserva la descripción general original de la CMUcam5.

Pixy es un sistema de visión pequeño, rápido, fácil de usar, de bajo costo y fácilmente disponible:

  • Aprende a detectar objetos que usted le enseña

  • Emite lo que detecta 50 veces por segundo

  • Se conecta a Arduino con el cable incluido, y también funciona con Raspberry Pi, BeagleBone y controladores similares

  • Se proporcionan todas las bibliotecas para Arduino, Raspberry Pi, etc.

  • Se admiten C/C++ y Python

  • Se comunica mediante una de varias interfaces: SPI, I2C, UART, USB o salida analógica/digital

  • La utilidad de configuración se ejecuta en Windows, macOS y Linux

  • Todo el software y firmware es de código abierto, con licencia GNU

  • Toda la documentación de hardware, incluidos esquemas, lista de materiales, diseño de PCB, etc., se proporciona

Cómo comenzó Pixy#

Pixy (CMUcam5) es una asociación entre el Carnegie Mellon Robotics Institute y Charmed Labs. Pixy proviene de un largo linaje de CMUcams, pero Pixy tuvo su verdadero comienzo como una campaña de Kickstarter. Empezó a enviarse por primera vez en marzo de 2014, y rápidamente se convirtió en uno de los sistemas de visión más populares jamás fabricados. Pixy se financia exclusivamente a través de las ventas.

La visión como sensor#

Si desea que su robot realice una tarea como recoger un objeto, perseguir una pelota o localizar una estación de carga, y quiere un solo sensor que ayude a lograr todas estas tareas, entonces la visión es su sensor. Los sensores de visión (imagen) son útiles porque son muy flexibles: con el algoritmo adecuado, un sensor de imagen puede detectar prácticamente cualquier cosa.

Pero hay dos inconvenientes con los sensores de imagen. Primero, generan muchos datos – decenas de megabytes por segundo – y segundo, procesar esta cantidad de datos puede saturar muchos procesadores. Y si el procesador puede seguir el ritmo de los datos, gran parte de su potencia de procesamiento no estará disponible para otras tareas.

Pixy aborda estos problemas emparejando un potente procesador dedicado con el sensor de imagen. Pixy procesa las imágenes del sensor de imagen y solo envía la información útil (por ejemplo, «dinosaurio morado detectado en x=54, y=103») a su microcontrolador, y lo hace a la velocidad de fotogramas (50 Hz).

Pixy sigue siendo un producto activo. Su documentación, software y soporte actuales se encuentran en pixycam.com.
