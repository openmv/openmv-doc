13.1.14. El editor de puntos clave¶

La coincidencia de puntos clave con find_keypoints() funciona a partir de un archivo de descriptor guardado – el archivo .orb que un script escribe con image.save_descriptor() tras extraer puntos clave de un objeto de referencia (los ejemplos de puntos clave en el menú Examples muestran el flujo de captura). Los descriptores reales suelen recoger basura: esquinas del fondo, reflejos especulares, características que pertenecen a la escena en lugar de al objeto. El editor de puntos clave, en Tools → Machine Vision → Keypoints Editor, es la herramienta de limpieza.

Edit File abre un descriptor (.orb o .lbp) junto con la imagen de la que se extrajo – el editor encuentra el archivo de imagen correspondiente guardado junto al descriptor – y dibuja cada punto clave sobre la imagen. Selecciona los valores atípicos con el ratón y elimínalos con la tecla de borrado; las casillas alternan la visibilidad de cada octava de detección para que puedas trabajar las escalas una a una. Guarda, y el descriptor recortado se vuelve a escribir – el descriptor y la imagen anteriores se copian primero en archivos .bak (un .bak anterior se reemplaza).

Merge Files combina varios archivos de descriptor en uno. Captura descriptores del mismo objeto desde unos cuantos ángulos y distancias, fusiónalos, y el resultado coincide con el objeto en más poses de las que lo haría cualquier captura individual.
