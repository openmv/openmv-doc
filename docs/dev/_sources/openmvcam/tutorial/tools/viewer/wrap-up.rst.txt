Wrap up
=======

That is OpenMV Viewer: connect a camera, run a product's
script, and watch its frame buffer, serial output, and
histogram live -- the parts of OpenMV IDE that show what
a camera is doing, with the development tools taken away.
It is the application to reach for when the job is to
*present* a finished OpenMV product rather than build
one.

When you do need to change the application -- edit the
script, tune a threshold, retrain a model, adjust
settings -- reach for the full :doc:`OpenMV IDE
<../ide/index>`. The two install side by side, so a
development machine can carry both: build in the IDE,
demonstrate in the Viewer.
