sensor — 카메라 센서#

버전 4.5부터 폐지됨: sensor 모듈은 더 이상 사용되지 않습니다. 대신 새로운 csi 모듈(csi — 카메라 센서 참조)을 사용하십시오. 이 모듈에는 새 기능이 추가되지 않으며 향후 릴리스에서 제거될 수 있습니다.

sensor 모듈은 OpenMV Cam의 기본 카메라 센서에 대한 레거시 모듈 수준 인터페이스입니다. 모든 호출은 숨겨진 단일 CSI 인스턴스에서 동작하는 자유 함수이므로 카메라가 두 개 이상인 보드에 대한 지원이 제한됩니다. 이는 이전 OpenMV 스크립트와의 하위 호환성을 위해 유지되며, 새 코드는 객체 지향적인 csi 모듈을 대신 사용해야 합니다.

함수 이름은 이전의 set_pixformat / set_framesize 스타일을 따릅니다. 각 함수는 csi.CSI의 메서드와 일대일로 대응됩니다. 전체 기능 집합과 인수별 설명은 csi 모듈을 참조하십시오.

사용 예:

import sensor

sensor.reset()
sensor.set_pixformat(sensor.RGB565)
sensor.set_framesize(sensor.QVGA)
sensor.skip_frames(time=2000)

while True:
    img = sensor.snapshot()

함수#

sensor.reset() → None#

카메라 센서를 초기화합니다.

sensor.sleep(enable: bool) → None#

enable이 True이면 카메라를 절전 상태로 전환합니다. 그렇지 않으면 다시 깨웁니다.

sensor.shutdown(enable: bool) → None#

카메라를 절전보다 더 낮은 전력 모드로 전환합니다. 카메라는 깨어날 때 재설정되어야 합니다.

sensor.flush() → None#

프레임 버퍼에 있던 내용을 IDE 미리보기로 복사합니다.

sensor.snapshot() → image.Image#

카메라로 사진을 찍어 image.Image 객체를 반환합니다.

sensor.set_auto_rotation()이 활성화되어 있으면 이 메서드는 이미 회전된 새 image.Image 객체를 반환합니다.

sensor.skip_frames(n: int | None = None, time: int = 300) → None#

카메라 설정 변경 후 카메라 이미지가 안정화되도록 n 프레임 또는 time 밀리초(지정된 쪽)만큼 건너뜁니다.

n과 time 중 어느 것도 지정하지 않으면 이 메서드는 300밀리초 동안 프레임을 건너뜁니다.

둘 다 지정하면 이 메서드는 n 프레임을 건너뛰되 time 밀리초 후 타임아웃됩니다.

sensor.width() → int#

센서 해상도 너비를 반환합니다.

sensor.height() → int#

센서 해상도 높이를 반환합니다.

sensor.get_fb() → image.Image | None#

이전 sensor.snapshot() 호출에서 반환된 이미지 객체를 반환합니다. sensor.snapshot()이 이전에 호출된 적이 없으면 None을 반환합니다.

sensor.get_id() → int#

카메라 모듈 ID를 반환합니다. 아래의 센서 상수를 참조하십시오.

sensor.get_frame_available() → bool#

sensor.snapshot()을 호출하여 읽을 수 있는 프레임이 있으면 True를 반환합니다.

sensor.alloc_extra_fb(width: int, height: int, pixformat: int) → image.Image#

버전 4.5부터 폐지됨: 이 함수는 더 이상 사용되지 않으며 OSError를 발생시킵니다. 대신 새로운 csi 모듈을 사용하십시오.

sensor.dealloc_extra_fb() → None#

버전 4.5부터 폐지됨: 이 함수는 더 이상 사용되지 않으며 OSError를 발생시킵니다. 대신 새로운 csi 모듈을 사용하십시오.

sensor.set_pixformat(pixformat: int) → None#

카메라 모듈의 픽셀 형식을 설정합니다. pixformat은 다음 중 하나입니다:

  • sensor.BINARY

  • sensor.GRAYSCALE

  • sensor.RGB565

  • sensor.BAYER

  • sensor.YUV422

  • sensor.JPEG (OV2640/OV5640 전용)

sensor.get_pixformat() → int#

카메라 모듈의 현재 픽셀 형식을 반환합니다.

sensor.set_framesize(framesize: int) → None#

카메라 모듈의 프레임 크기를 설정합니다. 유효한 값은 아래의 framesize 상수를 참조하십시오.

sensor.get_framesize() → int#

카메라 모듈의 현재 프레임 크기를 반환합니다.

sensor.set_framerate(rate: int) → None#

카메라 모듈의 프레임 속도를 Hz 단위로 설정합니다.

sensor.get_framerate() → int#

카메라 모듈의 프레임 속도를 Hz 단위로 반환합니다.

sensor.set_windowing(roi: Tuple[int, int] | Tuple[int, int, int, int] | List[int]) → None#

카메라의 해상도를 현재 해상도 내부의 하위 해상도로 설정합니다.

roi는 사각형 튜플/리스트 (x, y, w, h)입니다. (w, h)를 전달할 수도 있으며 이 경우 roi는 프레임 중앙에 배치됩니다. 인수는 위치 정수로 풀어서 전달할 수도 있습니다.

sensor.get_windowing() → Tuple[int, int, int, int]#

이전에 sensor.set_windowing()으로 설정한 roi 튜플 (x, y, w, h)을 반환합니다.

sensor.set_gainceiling(gainceiling: int) → bool#

카메라 이미지 게인 상한을 설정합니다. 유효한 값은 2, 4, 8, 16, 32, 64, 128입니다. 성공 시 True를 반환합니다.

sensor.set_contrast(contrast: int) → bool#

카메라 이미지 대비를 설정합니다. 유효 범위는 -3부터 +3까지입니다. 성공 시 True를 반환합니다.

sensor.set_brightness(brightness: int) → bool#

카메라 이미지 밝기를 설정합니다. 유효 범위는 -3부터 +3까지입니다. 성공 시 True를 반환합니다.

sensor.set_saturation(saturation: int) → bool#

카메라 이미지 채도를 설정합니다. 유효 범위는 -3부터 +3까지입니다. 성공 시 True를 반환합니다.

sensor.set_quality(quality: int) → bool#

카메라 이미지 JPEG 압축 품질을 설정합니다. 유효 범위는 0부터 100까지입니다. 성공 시 True를 반환합니다. OV2640/OV5640 카메라 전용입니다.

sensor.set_colorbar(enable: bool) → bool#

컬러 바 테스트 모드를 켜거나(True) 끕니다(False). 성공 시 True를 반환합니다.

sensor.set_auto_gain(enable: int, gain_db: float | None = None, gain_db_ceiling: float | None = None) → None#

enable은 자동 게인 제어를 켜거나(1) 끕니다(0).

enable이 0이면 gain_db로 고정 게인을 데시벨 단위로 설정할 수 있습니다.

enable이 0이 아니면 자동 게인 제어 알고리즘을 위해 gain_db_ceiling으로 최대 게인 상한을 데시벨 단위로 설정할 수 있습니다.

gain_db와 gain_db_ceiling은 키워드 전용 인수입니다.

sensor.get_gain_db() → float#

현재 카메라 게인 값을 데시벨 단위로 반환합니다.

sensor.set_auto_exposure(enable: int, exposure_us: int = -1) → None#

전치(transpose) 모드를 켜거나(True) 끕니다(False). 기본값은 끄기입니다.

  • vflip=False, hmirror=False, transpose=False -> 0도 회전

  • vflip=True, hmirror=False, transpose=True -> 90도 회전

  • vflip=True, hmirror=True, transpose=False -> 180도 회전

  • vflip=False, hmirror=True, transpose=True -> 270도 회전

sensor.get_auto_rotation() → bool#

count는 1(단일 버퍼), 2(이중 버퍼), 3(삼중 버퍼)일 수 있으며, 4 이상이면 수신된 프레임을 count개의 버퍼로 구성된 FIFO에 저장하는 비디오 FIFO 모드로 센서 드라이버를 전환합니다.

cb는 인수 하나를 받습니다: 변경된 후의 vsync 핀의 현재 상태입니다.

등록을 해제하려면 호출 불가능한 값(예: None)을 전달하십시오.

sensor.set_frame_callback(cb: Callable[[], None] | None) → None#

카메라 모듈이 새 프레임을 생성하고 sensor.snapshot()을 통해 프레임을 읽을 준비가 될 때마다 (인터럽트 컨텍스트에서) 실행될 콜백 cb를 등록합니다.

cb는 인수를 받지 않습니다.

등록을 해제하려면 호출 불가능한 값(예: None)을 전달하십시오.

센서별 요청을 실행합니다. request는 IOCTL_* 상수 중 하나입니다. 나머지 위치 인수와 반환 값은 요청에 따라 다릅니다. 지원되는 요청은 아래에 센서 계열별로 그룹화되어 있습니다.

일반 (모든 센서):

ioctl(IOCTL_GET_READOUT_WINDOW)

현재 판독 창을 (x, y, w, h) 튜플로 반환합니다.

현재 트리거 모드 상태를 bool로 반환합니다.

ioctl(IOCTL_SET_FOV_WIDE, enable)

True이면 set_framesize()가 프레임 속도보다 시야각에 최적화되도록 지시합니다.

ioctl(IOCTL_GET_FOV_WIDE)

현재 FOV-wide 상태를 bool로 반환합니다.

ioctl(IOCTL_SET_NIGHT_MODE, enable)

센서의 저조도 “야간 모드”를 활성화(True) 또는 비활성화(False)합니다. OV7725 및 OV5640 전용입니다.

ioctl(IOCTL_GET_NIGHT_MODE)

현재 야간 모드 상태를 bool로 반환합니다.

ioctl(IOCTL_GET_RGB_STATS)

센서에서 읽은 원시 RGB 채널 통계의 4-튜플 (r, gb, gr, b)을 반환합니다(일반적으로 화이트 밸런스 조정에 사용됨).
