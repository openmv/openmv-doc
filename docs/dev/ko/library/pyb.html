pyb — 보드 관련 함수#

경고

pyb 모듈은 더 이상 사용되지 않습니다(deprecated). 새 코드에서는 크로스 포트(cross-port) 모듈인 machine 모듈을 사용하세요. 이 모듈은 MCU 계열과 무관하게 모든 OpenMV Cam에서 동일한 기능을 제공하는 반면, pyb 는 STM32 기반 보드에만 존재합니다. pyb 는 이전 스크립트와의 하위 호환성을 위해 유지되며, 새로운 기능은 추가되지 않고 향후 릴리스에서 제거될 수 있습니다.

pyb 모듈에는 보드와 관련된 STM32 전용 함수가 포함되어 있습니다.

시간 관련 함수#

pyb.delay(ms: int) → None#

지정한 밀리초만큼 지연합니다.

pyb.udelay(us: int) → None#

지정한 마이크로초만큼 지연합니다.

pyb.millis() → int#

보드가 마지막으로 리셋된 이후 경과한 밀리초 수를 반환합니다.

결과는 항상 MicroPython의 smallint(31비트 부호 있는 숫자)이므로, 2^30 밀리초(약 12.4일) 이후에는 음수를 반환하기 시작합니다.

pyb.stop() 이 실행되면 이 함수를 지원하는 하드웨어 카운터가 “sleeping” 상태가 지속되는 동안 일시 중지됩니다. 이는 pyb.elapsed_millis() 의 결과에 영향을 미칩니다.

pyb.micros() → int#

보드가 마지막으로 리셋된 이후 경과한 마이크로초 수를 반환합니다.

결과는 항상 MicroPython의 smallint(31비트 부호 있는 숫자)이므로, 2^30 마이크로초(약 17.8분) 이후에는 음수를 반환하기 시작합니다.

pyb.stop() 이 실행되면 이 함수를 지원하는 하드웨어 카운터가 “sleeping” 상태가 지속되는 동안 일시 중지됩니다. 이는 pyb.elapsed_micros() 의 결과에 영향을 미칩니다.

pyb.elapsed_millis(start: int) → int#

start 이후 경과한 밀리초 수를 반환합니다.

이 함수는 카운터 랩어라운드(wrap)를 처리하며 항상 양수를 반환합니다. 따라서 약 12.4일까지의 기간을 측정하는 데 사용할 수 있습니다.

예시:

start = pyb.millis()
while pyb.elapsed_millis(start) < 1000:
    # Perform some operation
pyb.elapsed_micros(start: int) → int#

start 이후 경과한 마이크로초 수를 반환합니다.

이 함수는 카운터 랩어라운드(wrap)를 처리하며 항상 양수를 반환합니다. 따라서 약 17.8분까지의 기간을 측정하는 데 사용할 수 있습니다.

예시:

start = pyb.micros()
while pyb.elapsed_micros(start) < 1000:
    # Perform some operation
    pass

리셋 관련 함수#

pyb.hard_reset() → None#

외부 RESET 버튼을 누르는 것과 유사한 방식으로 OpenMV Cam을 리셋합니다.

pyb.bootloader() → None#

펌웨어 업데이트를 위해 OpenMV 자체 USB DFU 부트로더로 리셋합니다. 이는 실행 중인 코드에서 DFU를 호출하는 권장 방법으로, 리셋 시 BOOT 핀을 어서트할 필요가 없습니다.

OpenMV Cam N6에서는 사용할 수 없습니다(해당 보드에서는 pyb 레거시 호환성 계층이 비활성화되어 있습니다). 대신 machine.bootloader() 를 사용하세요.

pyb.fault_debug(value: bool) → None#

하드 폴트(hard-fault) 디버깅을 활성화하거나 비활성화합니다. 하드 폴트는 잘못된 메모리 접근과 같이 기반 시스템에서 치명적인 오류가 발생하는 경우를 말합니다.

value 인수가 False 이면 하드 폴트가 발생할 때 보드가 자동으로 리셋됩니다.

value 가 True 이면 보드에 하드 폴트가 발생할 때 레지스터와 스택 트레이스를 출력한 다음 LED를 무한히 깜빡입니다.

기본값은 비활성화, 즉 자동으로 리셋하는 것입니다.

인터럽트 관련 함수#

pyb.disable_irq() → bool#

인터럽트 요청을 비활성화합니다. 이전 IRQ 상태를 반환합니다. 즉 비활성화/활성화된 IRQ에 대해 각각 False/True 를 반환합니다. 이 반환값을 enable_irq에 전달하여 IRQ를 원래 상태로 복원할 수 있습니다.

pyb.enable_irq(state: bool = True) → None#

인터럽트 요청을 활성화합니다. state 가 True (기본값)이면 IRQ가 활성화됩니다. state 가 False 이면 IRQ가 비활성화됩니다. 이 함수의 가장 일반적인 용도는 크리티컬 섹션을 빠져나오기 위해 disable_irq 가 반환한 값을 전달하는 것입니다.

pyb.freq(sysclk: int | None = None, hclk: int | None = None, pclk1: int | None = None, pclk2: int | None = None) → Tuple[int, int, int, int] | None#

인수 없이 호출하면 클럭 주파수의 튜플 (sysclk, hclk, pclk1, pclk2) 를 헤르츠 단위로 반환합니다:

필드

의미

sysclk

CPU 주파수

hclk

AHB 버스, 코어 메모리 및 DMA 주파수

pclk1

APB1 버스 주파수

pclk2

APB2 버스 주파수

인수를 지정하면 함수가 CPU 주파수를 설정합니다(추가 인수가 제공되면 버스 주파수도 설정합니다). 주파수는 헤르츠 단위이며, 지정한 값을 초과하지 않는 가장 큰 지원 주파수가 선택됩니다. 유효한 sysclk 주파수 집합과 최대 hclk / pclk1 / pclk2 값은 MCU에 따라 다릅니다. 사용 중인 OpenMV Cam에 해당하는 STM32 레퍼런스 매뉴얼을 참조하세요. 버스 주파수는 드라이버가 요청된 값에 가장 잘 맞도록 선택한 프리스케일러를 통해 sysclk 에서 파생됩니다.

경고

USB가 활성화된 상태에서 주파수를 변경하면 USB가 불안정해질 수 있습니다. USB 주변장치가 시작되기 전인 boot.py 에서 변경하고, MCU의 USB 클럭 요구 사항에 비해 너무 낮은 sysclk 값을 선택하지 마세요.

전원 관련 함수#

pyb.wfi() → None#

내부 또는 외부 인터럽트를 대기합니다.

이는 인터럽트(내부든 외부든)가 발생할 때까지 MCU의 전력 소비를 줄이는 wfi 명령을 실행하며, 인터럽트가 발생하면 실행이 계속됩니다. 시스템 틱 인터럽트는 1밀리초마다 한 번(1000Hz) 발생하므로 이 함수는 최대 1ms 동안 블록됩니다.

pyb.stop() → None#

OpenMV Cam을 STM32 stop 저전력 상태로 전환합니다. 깨어날 때 이 지점부터 실행이 계속됩니다. 깨어나려면 외부 인터럽트나 실시간 클럭 이벤트가 필요합니다. 이를 구성하려면 pyb.RTC.wakeup() 을 참조하세요. 실제로 달성 가능한 전류 소비량은 보드와 어떤 주변장치에 클럭이 계속 공급되는지에 따라 달라집니다.

pyb.standby() → None#

OpenMV Cam을 STM32 standby 딥 슬립 상태로 전환합니다. 이는 가장 낮은 전력 차단 수준으로, 깨어날 때 장치가 하드 리셋을 통해 빠져나오므로 실행이 제자리에서 재개되지 않습니다. 깨어나려면 실시간 클럭 이벤트가 필요합니다. 이를 구성하려면 pyb.RTC.wakeup() 을 참조하세요. 실제로 달성 가능한 전류 소비량은 보드에 따라 달라집니다.

기타 함수#

pyb.have_cdc() → bool#

USB가 시리얼 장치로 연결되어 있으면 True를, 그렇지 않으면 False를 반환합니다.

참고

이 함수는 더 이상 사용되지 않습니다. 대신 pyb.USB_VCP().isconnected()를 사용하세요.

pyb.hid(data: Tuple[int, int, int, int]) → None#

4개 요소로 구성된 튜플(또는 리스트)을 받아 USB 호스트(PC)로 전송하여 HID 마우스 이동 이벤트를 신호로 보냅니다.

참고

이 함수는 더 이상 사용되지 않습니다. 대신 pyb.USB_HID.send() 를 사용하세요.

pyb.info(dump_alloc_table: bool | None = None) → None#

보드에 대한 많은 정보를 출력합니다.

pyb.main(filename: str) → None#

boot.py 가 완료된 후 실행할 메인 스크립트의 파일명을 설정합니다. 이 함수를 호출하지 않으면 기본 파일 main.py 가 실행됩니다.

이 함수는 boot.py 내부에서 호출하는 것만이 의미가 있습니다.

pyb.mount(device: Any, mountpoint: str, *, readonly: bool = False, mkfs: bool = False) → None#

참고

이 함수는 더 이상 사용되지 않습니다. 대신 vfs.mount() / vfs.umount() 와 vfs.AbstractBlockDev 파생 블록 장치를 사용하세요.

블록 장치를 mountpoint 아래에 마운트합니다. device 는 레거시 pyb 블록 장치 프로토콜을 구현해야 합니다(이 또한 더 이상 사용되지 않습니다. 최신 인터페이스는 vfs.AbstractBlockDev 를 참조하세요):

메서드

용도

readblocks(self, blocknum, buf)

블록 blocknum 부터 시작하여 장치에서 buf 크기만큼의 바이트를 복사합니다. buf 길이는 512의 배수입니다.

writeblocks(self, blocknum, buf) (선택 사항)

블록 blocknum 부터 시작하여 buf 를 장치에 씁니다. 생략하면 장치가 읽기 전용으로 마운트됩니다.

count(self)

장치의 512바이트 블록 수를 반환합니다.

sync(self) (선택 사항)

캐시된 쓰기를 모두 플러시합니다.

mountpoint 은 장치를 마운트할 파일 시스템 루트의 경로이며, 슬래시(/)로 시작해야 합니다. readonly 는 읽기 전용 마운트를 강제합니다. mkfs 는 파일 시스템이 없는 경우 새 파일 시스템을 생성합니다.

pyb.repl_uart(uart: UART | None = None) → UART | None#

REPL이 반복 표시되는 UART 객체를 가져오거나 설정합니다.

pyb.rng() → int#

30비트 하드웨어 생성 난수를 반환합니다.

pyb.sync() → None#

모든 파일 시스템을 동기화합니다.

pyb.unique_id() → bytes#

MCU의 고유 ID인 12바이트(96비트) 문자열을 반환합니다.

pyb.usb_mode(modestr: str | None = None, port: int = -1, vid: int = 0xf055, pid: int = -1, msc: Tuple = (), hid: Tuple = pyb.hid_mouse, high_speed: bool = False) → str | None#

인수 없이 호출하면 현재 USB 모드를 문자열로 반환합니다.

modestr 가 제공되어 호출되면 USB 모드 구성을 시도합니다. modestr 에 사용할 수 있는 값은 다음과 같습니다:

modestr

구성 내용

None

USB를 비활성화합니다.

'VCP'

VCP(Virtual COM Port)만 사용합니다.

'MSC'

MSC(USB 대용량 저장 장치 클래스)만 사용합니다.

'VCP+MSC'

VCP와 MSC를 사용합니다.

'VCP+HID'

VCP와 HID(휴먼 인터페이스 장치)를 사용합니다.

'VCP+MSC+HID'

VCP, MSC, HID를 함께 사용합니다. 모든 OpenMV Cam에서 지원되지는 않습니다.

하위 호환성을 위해 'CDC' 는 'VCP' 를 의미하는 것으로 이해됩니다('CDC+MSC' 와 'CDC+HID' 도 마찬가지입니다).

port 매개변수는 정수(0, 1, …)여야 하며, 보드가 여러 포트를 지원하는 경우 어떤 USB 포트를 사용할지 선택합니다. 값이 -1이면 기본 또는 자동으로 선택된 포트를 사용합니다.

vid 및 pid 매개변수를 사용하면 VID(공급업체 id)와 PID(제품 id)를 지정할 수 있습니다. pid 값이 -1이면 modestr 값을 기반으로 PID가 선택됩니다.

MSC 모드를 활성화하는 경우 msc 매개변수를 사용하여 대용량 저장 인터페이스에 노출할 SCSI LUN 리스트를 지정할 수 있습니다. 예를 들어 msc=(pyb.Flash(), pyb.SDCard()) 와 같습니다.

HID 모드를 활성화하는 경우 hid 키워드 매개변수를 전달하여 HID 세부 정보를 지정할 수도 있습니다. 이는 (subclass, protocol, max packet length, polling interval, report descriptor) 튜플을 받습니다. 기본적으로 USB 마우스에 적합한 값을 설정합니다. 또한 USB 키보드에 적합한 튜플인 pyb.hid_keyboard 상수도 있습니다.

high_speed 매개변수를 True 로 설정하면 하드웨어가 지원하는 경우 USB HS 모드가 활성화됩니다.

상수#

아래의 두 상수는 모두 다음 형식의 미리 만들어진 5개 요소 튜플입니다

(subclass, protocol, max_packet_size, polling_interval_ms, report_descriptor)

이는 OpenMV Cam이 호스트에 USB HID 장치로 표시되도록 usb_mode() 의 hid 인수로 전달하기에 적합합니다. subclass = 1 은 “부트 인터페이스”를 의미하며 protocol 은 부트 장치 클래스를 선택합니다(1 = 키보드, 2 = 마우스). 다섯 번째 요소는 호스트가 장치를 열거할 때 사용하는 HID 리포트 디스크립터를 담은 bytes 객체입니다.

pyb.hid_mouse: tuple#

상대 X/Y 이동을 지원하는 3버튼 부트 마우스용 미리 만들어진 HID 디스크립터입니다. 튜플은 (1, 2, 4, 8, <mouse report descriptor>) 입니다: 부트 서브클래스, 마우스 프로토콜, 4바이트 입력 리포트(버튼 마스크 + X + Y + 휠), 8ms마다 폴링. 내장된 리포트 디스크립터는 pyb.USB_HID().send((buttons, dx, dy, wheel)) 에서 사용하는 것과 동일합니다.

pyb.hid_keyboard: tuple#

USB 부트 키보드용 미리 만들어진 HID 디스크립터입니다. 튜플은 (1, 1, 8, 8, <keyboard report descriptor>) 입니다: 부트 서브클래스, 키보드 프로토콜, 8바이트 입력 리포트(수정자 바이트, 예약된 바이트 1개, 동시 키 코드 6개), 8ms마다 폴링. 내장된 리포트 디스크립터는 표준 8바이트 HID 부트 키보드 레이아웃과 일치하므로, USB_HID.send() 를 통해 전송되는 리포트는 (modifiers, 0, key1, key2, key3, key4, key5, key6) 형식의 bytes 여야 합니다.
