v4.8.1¶

v4.8.1은 Qt Creator 14.0.2를 기반으로 하며, 새로운 고신뢰성 OpenMV V2 카메라 프로토콜, 내장 Code Profiler, 그리고 카메라를 분리하지 않고 마운트 상태로 유지하는 저장 워크플로를 중심으로 합니다. 번들 펌웨어와 전체 예제 모음이 4.8.1로 업데이트되었습니다. 스크립팅 API의 호환성 변경은 없지만, 저장 시 더 이상 분리되지 않는 동작과 일부 실행 시점 시리얼 플래그의 제거는 사용자에게 보이는 워크플로 변경입니다.

주요 사항¶

  • OpenMV Protocol V2는 CRC32, 시퀀싱, ACK/NAK, 재전송, 이벤트, 단편화를 갖춘 새로운 고신뢰성 카메라 스택(OMVCamera/OMVTransport/OMVDebug)입니다. 연결 시점에 자동 협상되며 리셋, 부팅, 스트리밍, 스크립트 실행, 통계를 구동합니다.

  • Code Profiler는 카메라로부터 함수별 프로파일링 레코드를 폴링하여 호출 횟수, 최소/최대/총/평균 마이크로초(μs), 평균 사이클, 백분율, 이벤트 카운터를 정렬 가능한 표로 표시하는 새 창(Window > Show Code Profiler)입니다.

  • Save in place는 더 이상 카메라 드라이브를 분리하지 않습니다. 이제 IDE는 main.py를 기록하고 볼륨을 디스크로 플러시하여 장치가 마운트된 상태로 유지됩니다.

  • 다중 센서 감지는 보드의 모든 카메라를 보고하며, 상태 표시줄에 Sensors: X, Y 형식으로 메인 센서를 먼저 표시합니다.

  • Dynamic Frame Reading은 픽셀 형식별로 최적의 원시 스트리밍 해상도를 다시 계산하고 스트림을 자동으로 재구성합니다.

  • 번들 펌웨어 및 예제가 모든 보드에서 4.8.1로 업데이트되어 새로운 GenX320 이벤트 카메라, TensorFlow ML, Alif OLED, LCD/Touch-LCD 실드 예제가 추가되었습니다.

새로운 기능¶

  • OpenMV V2 통신 프로토콜을 추가했습니다. CRC32, 시퀀싱, ACK/NAK, 재전송, 이벤트, 단편화를 갖춘 완전한 전송 및 트랜잭션 계층과, 연결 시 자동 협상되어 시리얼 드라이버에 통합된 OMVCamera 클래스를 포함합니다 (2fde9eb3f, fff1bf2ba, 8c41fcfad, 7fc7ed4bc, 84c3956af, 669039adf, a49259852, 07af1922a, 85f0c7c87, 5070cc213, e314459cc, e0367ca95, 0da2f79d0, 2b6fc9866, 534ecffbe).

  • Code Profiler 창(Window > Show Code Profiler)을 추가했습니다. 함수별 프로파일링 레코드를 폴링하여 호출 횟수, 최소/최대/총/평균 마이크로초(μs), 평균 사이클, 백분율, 이벤트 카운터를 표시하며, 번들된 ELFIO ELF 파서를 통해 주소를 심볼로 변환합니다 (019d3cd2c, 57cd9e1cc).

  • Debug Protocol Settings 대화상자를 추가했습니다. 매초 갱신되는 실시간 System Info, Host Stats, Device Stats를 표시하고, 채널별 폴링 속도와 결합/분리 폴링 컨트롤을 노출합니다 (5220a6125, 0da2f79d0).

  • 다중 센서 감지를 추가했습니다. 카메라가 둘 이상인 보드는 상태 표시줄에 모든 카메라를 Sensors: X, Y 형식으로 메인 센서를 먼저 표시합니다 (cebc79ce7).

  • Dynamic Frame Reading을 추가했습니다(기본 켜짐). 픽셀 형식(BINARY, GRAY, RGB565, ARGB8, JPEG, PNG)별로 최적의 원시 스트리밍 해상도를 다시 계산하고 스트림을 재구성합니다 (75c073bdb, 0da2f79d0).

  • BINARY(1bpp 모노) 및 ARGB8(32비트 알파) 원시 픽셀 형식에 대한 프레임 버퍼 디코딩을 추가하여 이제 해당 이미지 유형이 뷰어에서 렌더링됩니다 (dabb82062).

  • Windows에서 Connect가 이제 OS에 의해 문제로 표시된 USB 장치를 검색하고, 연결을 차단할 수 있는 불량 장치 목록과 함께 경고합니다 (1a5beb081, 1b59cd78e).

  • 상태 표시줄 등록 표시기를 추가했습니다. 연결된 카메라에 대해 녹색 Registered 또는 산호색 Unregistered 버튼이 표시되며, 클릭하면 보드를 등록할 수 있습니다 (a8c98a4b9).

  • 프레임버퍼 뷰어가 이제 새로운 FB_MESSAGE 이스케이프 코드를 통해 실행 중인 스크립트에서 전송된 가운데 정렬 텍스트 메시지를 이미지 대신 표시할 수 있습니다 (530048201).

  • 새로운 model-zoo 항목을 추가했습니다: ST FastDepth 깊이 추정 모델(224/256/320), ST 머리 랜드마크 포즈 모델, 손바닥 검출 및 손 랜드마크 모델, 그리고 BlazeFace 정면 얼굴 검출 모델 (551668410, 4d64556d4, fadd6c4ed, b8277bb88, a8c98a4b9).

  • 공장 빌드 전용 자체 테스트 스크립트(카메라, IMU, ToF, WiFi, LAN)를 추가했습니다. 이 스크립트는 공장용 IDE 변형에 번들되어 실행 시 자동으로 열립니다 (0db7fbbb0). 테스트의 RT1060 LAN 부분은 현재 비활성화되어 있습니다 (48370d022).

기타 변경 및 개선¶

  • Save in place: 스크립트를 저장해도 더 이상 카메라 드라이브가 분리되지 않습니다. IDE가 main.py를 기록하고 볼륨을 디스크로 플러시하여(Windows에서는 FlushFileBuffers, Linux에서는 syncfs, macOS에서는 sync_volume_np/F_FULLFSYNC) 장치가 마운트된 상태로 유지됩니다 (1614c572f, 6928b51ea, 35ed53967).

  • 카메라가 사용 중일 때 클릭된 툴바 동작(프레임버퍼 비활성화, JPEG 압축, 스크립트 저장, 설정 구성)이 이제 Busy 오류 대화상자를 표시하는 대신 큐에 들어가 장치가 사용 가능해지면 자동으로 실행됩니다 (f4315f0a4).

  • 프레임버퍼 뷰어가 이제 기본적으로 fit-in-view로 설정되어 시작 시 이미지가 창에 맞게 자동 조정되며, 시리얼 터미널 출력 창도 이제 시작 시 기본으로 표시됩니다 (f3ce4dc75).

  • .lite 확장자를 가진 컴파일된 모델 파일이 이제 모든 곳에서 인식됩니다: model-zoo 브라우저 필터링 및 표시, 펌웨어 ROMFS 정렬, 그리고 Vela/STEdge AI 컴파일러가 이제 .lite 출력을 생성합니다 (f3ce4dc75).

  • STEdge AI(N6) 모델 컴파일 대화상자가 이제 NPU RAM 및 hyperRAM 사용률 백분율을 파싱하여 보고하고, 재배치된 네트워크 출력을 모델의 확장자와 일치하는 파일로 복사합니다 (f3ce4dc75).

  • model-zoo 브라우저가 이제 크기를 숨기는 대신 파일 Size 열을 표시하고 이름 열을 맞게 늘립니다 (f3ce4dc75).

  • 새 스크립트 템플릿이 새로운 csi 모듈 API로 업데이트되고 기본적으로 VGA 프레임 크기를 사용하도록 변경되었으며, 번들된 Hello World 예제는 QVGA에서 VGA 프레임 크기로 전환되었습니다 (f3ce4dc75).

  • Convert Video 대화상자가 공유 로더 대화상자를 사용하도록 재작성되어, 색상 출력, 성공/실패 메시지, OK로 닫기 버튼, 그리고 플랫폼에서 FFMPEG를 사용할 수 없을 때의 명확한 오류 표시를 제공합니다 (f56efc7be).

  • 프레임버퍼 뷰어가 이제 JPEG/PNG 이미지가 버퍼링하여 전송하기에 너무 클 때 새로운 FB_BUFFER_ERROR 이스케이프 코드를 통해 깜박이는 경고를 표시합니다 (fda826126).

  • DFU 부트로더 진입이 이제 V2 프로토콜 펌웨어에서 부트로더가 상주 상태를 유지하도록 강제하며, 강제 시 멈출 수 있는 구형 부트로더(<1.0.2)에서는 일반 리셋으로 폴백하는 버전 감지를 갖췄습니다 (975857221).

  • ST 객체 검출 모델 폴더의 이름을 변경하고 재구성했으며(st_yolo_lc_v1을 yolo_lc로, tiny_yolo_v2를 yolo_v2로, yolov8n 사람 모델을 단순 크기 이름 파일로), 대용량 tiny_yolo_v2 int8 변형을 제거했습니다 (34240cc3c).

  • 시리얼 타이밍이 이제 실행 시점 명령줄 플래그 대신 펌웨어 settings.json의 protocol 재정의 키(overrideReadTimeout, overrideReadStallTimeout, overridePerCommandWait, 그리고 V2의 overrideCRC/overrideSEQ/overrideACK 키)를 통해 조정됩니다 (019d8bf1d).

  • flake8과 pycodestyle의 Python 언어 서버 줄 길이 제한을 120열로 설정하여 긴 줄이 더 이상 기본값 79에서 표시되지 않도록 했습니다 (dabb82062).

  • Qualcomm 모델 모음과 google/mobilenet_v1을 models_unused로 이동하여 번들된 머신러닝 모델 세트를 줄여 설치 프로그램 크기를 축소했습니다 (c44ddba20).

버그 수정¶

  • 펌웨어 다운로드 및 리소스 업데이트 진행 대화상자를 QPointer 가드를 사용해 충돌에 안전하게 만들어, 다운로드 중간에 닫히더라도 삭제된 대화상자를 사용하지 않도록 했습니다 (f3ce4dc75).

  • 드라이브 스캔이 이제 카메라의 USB 드라이브를 찾을 때까지 계속 재스캔하고 연결 해제 시 오래된 드라이브 항목을 지우며, 이전의 일회성 타이머 핵을 제거했습니다 (a31d41b4e).

  • 여러 연결 및 시리얼 수정: 숨겨진 센서(예: SoftCSI)가 더 이상 표시되는 센서 유형을 어지럽히지 않고, Windows 시리얼 keep-alive 타이머가 USB 정지를 방지하며, Start가 더 이상 이미 실행 중인 스크립트를 다시 시작하지 않고, 포트가 닫힐 때 카메라 채널이 깔끔하게 연결 해제됩니다 (a8e467be6).

  • 프로파일러 뷰가 일시적으로 비활성화되어 있던 열별 PMU 이벤트 카운터 선택(체크 가능한 이벤트 메뉴)과 저장된 카운터/헤더 상태를 복원했습니다 (40c679e90).

플랫폼 및 도구 지원¶

  • Qt Creator 기반: 14.0.2.

  • 번들 펌웨어: 4.7.0에서 4.8.1로 업데이트되어 OPENMV2/3/4/4P/PT, OPENMV_N6, OPENMV_AE3, OPENMV_RT1060, 그리고 Arduino GIGA/Nicla Vision/Portenta H7 보드용 펌웨어를 다시 빌드했습니다 (2657818ca).

  • RT1062 및 AE3 보드의 macOS 프로토콜 버전 핸드셰이크를 다중 명령 USB 전송을 해당 보드가 처리할 수 있는 USBDBG_LEN 크기 조각으로 분할하여 수정했습니다 (6b9d5f842).

  • 번들된 Windows 드라이버 패키지를 재작업했습니다: 보드별 서명된 .cat/.inf 드라이버 파일(AE3, H7, H7Plus, M4, M7, N6, Pico/PT, RT1062)을 추가하고 CDC 시리얼 드라이버를 dpinst 기반 자동 설치 프로그램이 있는 새 openmvcdc 폴더로 이동했습니다 (439557829, ba3b01bfb).

  • 센서 ID 0x1311 및 0x1312(기존 0x1313과 함께)를 MT9V0X2로 등록하여 구형 MT9V032 카메라 센서 리비전에 대한 지원을 추가했습니다 (eaaa624e3).

호환성에 영향을 주는 변경¶

이 릴리스에는 스크립팅 API 호환성 변경은 없지만, 사용자에게 보이는 몇 가지 워크플로 변경이 있습니다:

  • 스크립트를 저장해도 더 이상 카메라의 USB 드라이브가 분리되거나 마운트 해제되지 않습니다. 이제 IDE가 볼륨 버퍼를 제자리에서 플러시하므로 저장 후에도 장치가 마운트된 상태로 유지됩니다.

  • 실행 시점 시리얼 플래그 -override_read_timeout(기본 5000 ms), -override_read_stall_timeout(기본 1000 ms), -override_per_command_wait(기본 1 ms, macOS에서는 2 ms)가 제거되었습니다. 시리얼 타이밍은 이제 펌웨어 settings.json의 protocol 재정의 키를 통해 구성됩니다 (019d8bf1d).

  • 번들된 부트로더가 이제 .bin 형식으로만 제공됩니다. 보드별 .dfu 펌웨어 및 부트로더 파일은 제거되었습니다.
