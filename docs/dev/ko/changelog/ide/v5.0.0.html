v5.0.0#

v5.0.0은 메이저 릴리스입니다. 완성된 제품을 선보이기 위한 별도의 읽기 전용 빌드인 OpenMV Viewer, 새로운 OpenMV Cam Settings Editor, v5.0.0 카메라를 위한 v2 카메라 디버그 프로토콜, 그리고 편집기, 터미널, 펌웨어, 머신 비전 워크플로 전반에 걸친 대규모 개선 사항을 도입합니다. 이 릴리스는 Qt Creator 14.0.2를 기반으로 합니다.

5.0.0으로의 버전 도약은 v5.0.0 카메라 세대와 이를 지원하는 도구의 등장을 의미합니다. 기존 v4 프로젝트와 워크플로는 변경 없이 계속 동작합니다.

주요 사항#

  • OpenMV Viewer — IDE를 잠긴 읽기 전용 텔레메트리 뷰어로 실행하는, 별도로 설치되는 새로운 빌드 변형입니다. 카메라의 프레임 버퍼, 시리얼 터미널, 히스토그램을 표시하고 고정된 스크립트를 실행할 수 있지만, 작성, AI/모델, 설정 도구는 숨기므로, 고객이 전체 개발 환경 없이도 OpenMV 기반으로 만든 제품을 시연할 수 있습니다.

  • OpenMV Cam Settings Editor — 카메라의 온보드 설정 구성 파일을 위한 그래픽 편집기로, 파일을 직접 편집하는 대신 양식에서 보드 설정을 보고 변경할 수 있습니다.

  • v2 카메라 디버그 프로토콜 — v5.0.0 카메라는 헤더 CRC 검증, 시퀀스 간격 복구, 그리고 카메라 내 FPS와 IDE 디스플레이 FPS를 별도로 보고하는 분할 프레임 속도 표시를 갖춘 더 견고한 새 디버그 프로토콜을 사용합니다.

  • .zip에서 사용자 지정 펌웨어 로드 — 이제 펌웨어 설치 관리자가 보드의 빌드 출력 .zip 번들을 직접 받아들여, 사용자 지정 또는 로컬에서 빌드한 펌웨어 이미지를 손쉽게 플래시할 수 있습니다.

  • 시리얼 터미널 프로토콜 디버그 로깅 — 시리얼 터미널의 토글이 연결 문제를 진단하기 위해 읽기 쉬운 이모지 태그가 붙은 줄로 카메라 디버그 프로토콜을 기록합니다.

  • 개편된 도움말 메뉴 및 편집기 내 문서 — 기존의 단순 링크 목록을 OpenMV Docs 하위 메뉴가 대체하며, F1 키는 커서 아래의 Python 심볼에 대한 문서를 열고, API 이름 위에 마우스를 올리면 해당 문서가 인라인으로 표시되며, 정보 대화 상자가 현대적으로 개선되었습니다.

새로운 기능#

  • OpenMV Viewer 빌드 변형. OpenMV Viewer로 브랜딩되어 설치되는 강제 뷰어 모드 빌드로, 자체 실행 파일 id, 설정 저장소, 번들 id를 가지므로 전체 IDE와 공존합니다. 읽기 전용 텔레메트리 뷰어로 제공됩니다. 실행/정지와 연결 시 정지가 동작하며 디스크의 스크립트를 실행하고(-auto_run을 통해), 파일/편집 메뉴와 환경 설정이 유지되며, 펌웨어/ROMFS 복구가 보존되는 반면, 작성 도구, 개발자 머신 비전 도구, 개발자 설정 범주, AI/모델/문서 리소스는 빌드와 UI에서 제거됩니다. ROMFS는 로드 전용이 되어 도구 메뉴로 승격되고, 펌웨어 업데이트 완료 대화 상자는 일반화됩니다 (07722e78a8b, 5cd12ee260e, b97bf5447e4, 952b86de391, 1103f7988ed, f0f60261485, c743d5d67b9, b4bcdc2d71a, a37c032b6ff, 480e5668d29, 11c064cac93).

  • OpenMV Cam Settings Editor — 도구 메뉴에서 열리는 카메라 설정 구성 파일용 GUI 편집기입니다 (이슈 #205) (f55a0d7fa58, 2d7f80b03cf).

  • v5.0.0 카메라를 위한 v2 카메라 디버그 프로토콜로, 어떤 헤더 필드도 신뢰하기 전에 헤더 CRC 검증을 수행하고, 멈추는 대신 손실된 패킷을 건너뛰며 전진하는 시퀀스 간격 복구를 제공하며, 프레임 속도 표시를 카메라 내 속도와 IDE 디스플레이 속도로 분할합니다 (c377bdbaea8, a3f0488ddcb, 29108dd40f7, c15e6c02789).

  • 사용자 지정 펌웨어 로드가 .zip 번들을 받아들입니다 — 보드의 빌드 출력 zip을 선택하면 IDE가 그 안에 포함된 펌웨어(및 ROMFS)를 플래시합니다 (55e4eb2d900).

  • 시리얼 터미널 디버그 로깅 — 버튼이 카메라 프로토콜의 단계별 디버그 로깅을 토글하며, 각 줄에는 이모지 태그가 붙고 가독성을 위해 이모지 글꼴로 렌더링됩니다 (09990c62810, 42e23c3c11e, 997705548c7, d2208de77d4).

  • 현재 스크립트 복사/변환 — 현재 스크립트를 복사하거나 변환하는 도구 동작으로, 컴파일된 main.py/boot.py 저장을 방지하는 보호 장치를 갖추고 있습니다 (81a954f33cd).

  • 커서 아래 Python 심볼에 대한 F1 문서 도움말로, 마우스 오버 툴팁과 심볼 확인기를 공유합니다 (c341330bc16).

  • 편집기 내 문서 툴팁 — OpenMV API 이름 위에 마우스를 올리면 함께 제공되는 .pyi 스텁과 모듈 자체 문서에서 해석된 문서가 인라인으로 표시되므로, 스크립트를 벗어나지 않고도 함수, 메서드, 상수, 모듈의 동작을 확인할 수 있습니다 (7ebde92d2a6, f225824ba13, 59ad01a364a, 02c1e61a2a6).

  • 카메라 이름 지정 — 연결된 카메라에 부여할 수 있는 IDE 로컬 별칭으로, 시리얼 포트 이름 대신 표시되어 동일한 카메라들을 구별할 수 있습니다 (c600158ca34).

  • 키보드 단축키 페이지 — 이제 환경 설정 단축키 페이지가 표시되며 실제로 도달할 수 있는 동작으로 필터링되고, 출처와 레이아웃이 더 명확해졌습니다 (4f5eae52e8e, 2d472fe5b40).

편집기, 터미널, UI 개선 사항#

  • 도움말 메뉴가 재구성되어 OpenMV Docs 하위 메뉴로 바뀌었으며(기존의 평면 핀아웃 링크는 제거됨), 공식 릴리스 버전 경로에 릴리스 노트 링크가 추가되었습니다 (e1337ff67c4, 9eacc0994e6).

  • 정보 대화 상자 현대화 — 자동 저작권 연도, 파트너 로고 표, 그리고 한 줄짜리 릴리스 노트 링크 (d89fb74138d).

  • 프레임 버퍼 패널이 이제 고정된 JPG 버튼 옆에 해상도 표시와 프레임 버퍼 모드 레이블을 보여줍니다. 모드 레이블은 클릭할 수 있는 것처럼 보이지 않도록 회색으로 처리됩니다 (cb735f82337, 376b17d4e77, 39c6ffc3b26).

  • 히스토그램 창 그래프가 스플라인으로 부드럽게 처리되고, 창을 압축할 수 있으며(레이블 생략과 축소 가능한 그래프), 채널 레이블이 통계 옆에 위치합니다 (327324ef7e9, 9f8c7035483, d204e93d5e4).

  • 연동된 패널 분할선 — Ctrl/Cmd로 분할선을 드래그하면 편집기/터미널과 프레임 버퍼/히스토그램 분할선이 함께 이동하며, 미니 분할선 핸들에는 더 넓은 잡기 영역을 갖춘 마우스 오버 후광이 표시됩니다 (379dee250d6, 96b83984342).

  • 시리얼 터미널 — 자동 스크롤이 더 이상 출력 청크마다 다시 고정되지 않으므로, 빠른 출력 중에 위로 스크롤한 상태가 유지됩니다. 편집기가 없을 때 터미널이 디스크의 파일을 실행할 수 있습니다 (af67a076e95, bc132aef536).

  • 문서 명령이 인덱스 대신 페이지 이름으로 지정됩니다 (a3201e9a4b4).

  • 고해상도(High-DPI) / 분수 배율 — 아이콘, 오버레이, 스플래시 화면이 분수 배율(125%, 150% 등)에서 올바르게 렌더링되며, 다중 모니터 환경에서 다시 시작해도 메인 창이 위치와 화면을 기억합니다 (c60711e4950, 60901b6e13f, f241d2ea8ce, 54a1bc31fbd).

펌웨어, 보드, 도구#

  • 보드가 마지막으로 출하된 펌웨어 버전을 고정할 수 있어, 더 이상 새 펌웨어를 받지 않는 보드가 구버전으로 표시되지 않습니다 (bd987d32f42).

  • 구버전 펌웨어 상태 텍스트가 이제 눈에 띄도록 빨간색으로 표시됩니다 (9b18d96bac5).

  • 부트로더 복구 힌트 — 이제 bootloader.bin 불일치 시 부트로더 복구 모드로 진입하도록 안내합니다 (d46b998c76e).

  • 비디오 재생이 더 견고해졌습니다. 도구가 FFmpeg를 실행할 수 없을 때 그 경로가 표시되고, 아키텍처가 재생을 지원하지 않을 때 메시지가 더 명확해지며, 입력 형식은 ffmpeg -demuxers로 조회되고, Linux 터미널 대체 방안이 추가되었습니다 (0cc0e74a438, a75ec4898d0, 28e1ef10326, 51651780723).

  • 데이터셋 캡처 스크립트가 csi 모듈로 업데이트되었습니다 (41a781651e4).

  • Windows 설치 관리자 마법사 외관이 개선되었습니다 (250599d9d9e).

버그 수정#

  • .lst 보드(AE3)를 위한 개발 펌웨어 설치가 이제 리스팅이 참조하는 바이너리를 플래시 디렉터리로 준비하며, 개발/사용자 지정 설치는 먼저 오래된 romfs*.img를 모두 지운 뒤 번들의 ROMFS 이미지를 준비하므로, 남아 있던 편집기 덤프 대신 다운로드된 ROMFS가 플래시됩니다 (3cf56e98668, 43bedc8c591).

  • 잘리거나 손상된 프레임에 대한 libjpeg 경고가 로그에서 필터링되어, 손실이 있는 스트림이 터미널을 도배하지 않습니다 (6e2a8bed6cf, a5166abbdc1).

  • 예제 목록에 더 이상 내부 index.json이 표시되지 않습니다 (14ac8ca48a7).

  • 뷰어 모드에서는 펌웨어 업그레이드와 온보딩 팝업이 억제되어, 제품 데모가 고객의 펌웨어를 덮어쓰도록 요청하는 일이 결코 발생하지 않습니다 (7b8528c3ad6).

  • 제거된 메뉴에서 남은, 연결이 끊긴 작성 전용 키보드 단축키가 비활성화되었습니다 (be3bee8bb58, 27d1afc8705).

지역화#

  • 네 개의 새로운 언어 — 인도네시아어, 루마니아어, 태국어, 베트남어 — 가 추가되어 IDE가 37개 번역 언어를 지원하게 되었으며, 모든 언어에서 모든 카탈로그가 업데이트, 번역, 정리되었습니다 (fcf04090843, c830e6c4de3, 5e22679923f).

명령줄#

  • 이제 단일 대시 형식과 함께 GNU 스타일 ``–word`` 옵션 별칭이 허용되며, -help 출력이 그 동등성을 안내합니다 (66414213dd9, 768c5d0e159).

플랫폼 및 도구 지원#

  • Qt Creator 기반: 14.0.2.

  • OpenMV IDE 버전이 5.0.0으로 올라갔습니다.

  • 신규: 별도로 설치되는 OpenMV Viewer 빌드 변형.

  • Python 3.14 지원 — 함께 제공되는 parso 파서가 0.8.7로 업데이트되어, 최신 Python 문법에서도 코드 자동 완성과 언어 서버가 정상적으로 작동합니다 (578a4e39d97).

호환성 파괴 변경 사항#

기존 프로젝트에 대해서는 없음. v2 디버그 프로토콜은 v5.0.0 카메라에 적용됩니다. v4 세대 카메라는 기존 프로토콜을 계속 사용하며, 기존 스크립트와 워크플로에는 영향을 주지 않습니다.
