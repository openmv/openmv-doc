v2.6.7¶

v2.6.7은 DFU 펌웨어 플래싱과 비디오 변환 워크플로를 내장된 IDE 터미널 대화 상자로 옮기고, Arduino Portenta H7용 CYW4343 WiFi 보조 프로세서 프로그래밍을 추가하며, 그레이스케일 센서 처리와 DFU 모드 프롬프트를 개선합니다. IDE는 여전히 Qt Creator 4.0.2 기반이며, 번들 OpenMV 펌웨어는 3.6.9로 올라가고, 호환성을 깨는 변경 사항은 없습니다.

주요 사항¶

  • 내장 DFU 플래싱 터미널은 외부 OS 터미널 창 대신 라이브 출력(오류는 빨간색으로 강조 표시)과 Cancel 버튼이 있는 크기 조절 가능한 IDE 대화 상자 안에서 dfu-util을 실행합니다.

  • CYW4343 WiFi 펌웨어가 이제 Arduino Portenta H7을 DFU로 플래싱할 때 메인 펌웨어와 함께 프로그래밍됩니다.

  • 내장 비디오 변환 터미널은 라이브 출력과 Cancel이 있는 동일한 크기 조절 가능한 IDE 대화 상자에서 FFmpeg Convert Video 도구를 실행합니다.

  • 재설계된 DFU 모드 프롬프트는 펌웨어 버전 드롭다운과 마지막 선택을 기억하는 선택적 Erase internal file system 체크박스를 사용합니다.

  • 그레이스케일 센서 지원은 자동 RGB565에서 GRAYSCALE으로의 재작성을 HM01B0에 더해 MT9V034로 확장합니다.

새로운 기능¶

  • 내장 DFU 플래싱 터미널. dfu-util이 이제 외부 OS 터미널 창을 대체하여, 라이브 stdout을 일반 텍스트로 스트리밍하고 stderr을 빨간색으로 강조 표시하며 Cancel 버튼과 저장된 창 지오메트리를 갖춘 QDialog 터미널로 직접 실행됩니다. (c2875d871)

  • CYW4343 WiFi 보조 프로세서 프로그래밍. Arduino Portenta H7에서 DFU를 통해 펌웨어를 로드하면 이제 dfu.txt의 extraProgramCommands로 구동되어 온보드 CYW4343 WiFi 펌웨어(0x90F00000 위치의 CYW4343/cyw4343.bin)를 플래싱하는 추가 프로그래밍 단계가 실행됩니다. (62e0ba771, ba31998ce)

  • 내장 비디오 변환 터미널. FFmpeg 기반 Convert Video 도구는 이제 외부 터미널을 실행하는 대신 라이브 출력과 Cancel이 있는 크기 조절 가능한 IDE 터미널 대화 상자에서 ffmpeg을 실행합니다. (5987bb152)

  • 공장 프로비저닝 옵션. 숨겨진 -form_key 명령줄 옵션을 제공하면 수동 등록 프롬프트를 표시하는 대신 연결된 OpenMV Cam의 보드/ID를 OpenMV 서버에 자동으로 등록합니다. (d5740f71e)

기타 변경 사항 및 개선 사항¶

  • 재설계된 DFU 모드 프롬프트. board detected in DFU mode 프롬프트는 이제 정확한 최신 릴리스 펌웨어 버전을 보여주는 드롭다운과 선택적 Erase internal file system 체크박스가 있는 대화 상자이며, 마지막으로 선택한 작업과 지우기 상태를 기억합니다. (0a4d6258c)

  • 지우기 체크박스 툴팁. 부트로더/DFU 파일 시스템 지우기 체크박스에 이제 내부 플래시 드라이브를 지워도 분리형 SD 카드는 지워지지 않음을 명확히 하는 툴팁이 표시됩니다. (0a4d6258c)

  • 열린 예제의 인플레이스 그레이스케일 재작성. 이미 열려 있는 helloworld 예제를 HM01B0 또는 MT9V034 센서에서 실행하면, IDE는 그레이스케일 전용 센서에서도 동작하도록 예제를 GRAYSCALE을 사용하도록 그 자리에서 재작성합니다. (4448edb38)

  • 더 명확한 플래시 후 메시지. DFU/펌웨어 업데이트 후 확인 메시지는 이제 디스크를 지우지 않고 main.py를 덮어쓴 경우를 설명하므로, 카메라는 단순히 main.py를 실행하고 드라이브를 마운트합니다. (62e0ba771)

  • 업데이트된 예제 스크립트. ulab.py는 이제 평균/표준편차에 ulab.numerical을 사용하고, deep_sleep.py는 OV7725 레귤레이터 바이패스 전에 센서 소프트 슬립의 순서를 재정렬하며, 새로운 19-Low-Power/extint_wakeup.py는 외부 인터럽트를 통해 stop 모드에서 깨어나는 방법을 보여줍니다. (c30fcb7c3)

  • Connect 대화 상자 정돈. Connecting... 진행률 텍스트가 가독성을 위해 줄바꿈으로 다시 포맷되었습니다. (0d3619245)

버그 수정¶

  • 모든 다른 이름으로 저장 대화 상자(이미지, 템플릿, 디스크립터, 키포인트, 네트워크 모델, 데이터셋 내보내기, 터미널 로그, 비디오 저장, 비디오 변환 출력 저장)가 확장자가 없는 파일을 저장하는 대신 확장자가 없는 파일 이름을 거부하고 사용자에게 확장자를 추가하도록 안내하도록 수정했습니다. (3a3ba3c5e)

  • macOS 전용 UI 렌더링을 수정했습니다. 메뉴 텍스트/버튼 색상 반전(메뉴 배경 밝기에 따라 결정됨)과 사용자 지정 다크 스크롤바 스타일링이 더 이상 macOS에 적용되지 않습니다. (39179b6cb, 8b11fbcab)

플랫폼 및 도구 지원¶

  • Qt Creator 기반: 4.0.2.

  • 번들 OpenMV 펌웨어: 3.6.8에서 3.6.9로.

  • CYW4343 WiFi 펌웨어: Arduino Portenta H7용으로 새로운 CYW4343/cyw4343.bin 블롭이 번들로 제공됩니다.

  • 그레이스케일 센서 스크립트: New File, 예제, 데이터셋 스크립트가 이제 HM01B0에 더해 MT9V034에서도 RGB565를 GRAYSCALE으로 전환합니다(HM01B0의 경우 VGA에서 QVGA로만 전환). (ef0c22efb, d5bdd8890)

호환성에 영향을 주는 변경 사항¶

없음. 이 릴리스는 하위 호환되며 기존 프로젝트와 스크립트는 변경이 필요하지 않습니다.
