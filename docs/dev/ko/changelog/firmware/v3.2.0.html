v3.2.0#

v3.2.0은 FLIR Lepton 드라이버를 대폭 개선하고(DMA, FLIR 1/3을 위한 스케일링, 더 빠른 UVC 스트리밍), OpenMV 3 및 4에서 exFAT를 활성화하며, CMSIS-NN을 5.4.0으로 업데이트하고, VSYNC GPIO 출력 및 USB-VCP 예제를 추가합니다. 호환성을 깨는 API 변경은 없으며 — v3.1.0 스크립트는 변경 없이 실행됩니다.

주요 사항#

  • FLIR Lepton — DMA 기반 캡처, FLIR 1/3을 위한 정상 동작하는 스케일링, 그리고 더 빠른 UVC 스트리밍.

  • exFAT OpenMV 3 및 4에서 활성화.

  • CMSIS-NN 5.4.0 업데이트.

  • 예제 — VSYNC GPIO 출력 및 USB VCP.

  • 호환성을 깨는 API 변경 없음 — 다시 플래시하면 v3.1.0 스크립트가 변경 없이 실행됩니다.

새로운 기능#

  • exFAT — OpenMV 3 및 4에서 exFAT 파일시스템을 활성화했습니다.

  • 예제 — VSYNC GPIO 출력 예제와 USB-VCP 예제를 추가했습니다.

기타 변경 및 개선 사항#

  • CMSIS-NN을 5.4.0으로 업데이트했고(갱신된 NN 학습/테스트 스크립트 및 레이블이 지정된 이미지 생성 스크립트 포함), IDE가 이제 숨김 마커 파일을 통해 OpenMV 디스크를 감지하며, 소프트 리셋 시 구성을 파싱하고, FLIR Lepton이 이제 UVC를 통해 더 빠르게 스트리밍하며 MT9V034에 대해 올바른 그레이스케일 UVC 디스크립터를 전송합니다.

버그 수정#

카메라 및 센서:

  • FLIR Lepton 드라이버가 스케일링을 통해 FLIR 1/3에서 잘 동작하도록 했고, Lepton 초기화를 MSP로 옮겼으며, Lepton 1과 3에 DMA를 사용했고(전용 VOSPI 메모리 영역 포함), MT9V034 스냅샷 경로를 수정했습니다(스트리밍 콜백 우회책 제거).

시스템 및 연결:

  • WINC 패치 세트를 추가하고 WINC 초기화 타임아웃, H7 DMA 영역 크기(MPU를 통해 캐시 불가능하게 설정), H7 SPI 클럭 소스, 그리고 CMSIS-NN 컬럼 버퍼 크기를 수정했습니다.

하드웨어 및 보드 지원#

  • OpenMV 3 / 4 — exFAT 파일시스템 지원.

호환성을 깨는 API 변경#

v3.2.0은 호환성을 깨는 Python API 변경이 없습니다. v3.1.0에서 실행되던 스크립트는 v3.2.0에서 변경 없이 실행됩니다 — 펌웨어만 다시 플래시하세요.
