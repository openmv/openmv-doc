CamScripter 시작하기#

개요#

그것은 무엇입니까?#

CamScripter는 CMUcam3용 사용자 정의 스크립트를 작성하고 디버깅하는 데 도움을 주는 도구입니다. Lua 프로그래밍 언어를 사용하여 스크립트를 작성할 수 있게 합니다. 이 언어와 CMUcam3용으로 제공되는 Lua API를 사용하여, 생각할 수 있는 거의 모든 용도로 카메라를 사용자 정의할 수 있습니다. 이 튜토리얼에서는 설치 과정을 안내한 다음 카메라에서 실행할 샘플 스크립트를 개발합니다.

시작하기 전에#

이 튜토리얼은 다음을 가정합니다:

  • 이미 CMUcam3를 가지고 있습니다

  • 빠른 시작 가이드 또는 SDK 설치 가이드를 사용하여 이미 카메라를 작동시켰습니다

  • 카메라를 컴퓨터에 연결하는 데 필요한 모든 것을 가지고 있습니다

설치, 1부: Lua, 시리얼 포트 통신#

Lua#

CamScripter를 사용하면 Lua 프로그래밍 언어로 CMUcam3를 프로그래밍할 수 있습니다. 그렇게 하기 전에, 이 언어를 컴퓨터에 설치해야 합니다.

  1. 적절한 ZIP 바이너리를 다운로드하십시오. (Windows의 경우: Lua-Windows(아카이브에 보존되지 않음), 다른 사용자의 경우 http://luabinaries.luaforge.net/download.html)

  2. Lua 바이너리를 C:\lua에 압축 해제하십시오

  3. 명령 프롬프트를 여십시오 (Windows에서는 시작 메뉴에서 “Run…”을 선택하고 ‘cmd’를 입력하십시오).

  4. Lua 파일의 압축을 푼 위치로 cd 하십시오. 저는, ‘cd C:lua’를 입력합니다

  5. x가 Lua 버전인 ‘luax.x.exe -v’를 입력하십시오

설치가 순조롭게 진행되었다면, 아래 스크린샷(아카이브에 보존되지 않음)처럼 화면에 Lua 버전 번호가 출력되는 것을 볼 수 있습니다.

가능하면 CMUcam3 웹사이트에서 lua를 설치하는 것이 좋습니다. 직접 재컴파일하려면 먼저 일반 lua 소스가 부동 소수점을 사용하지 않도록 패치해야 합니다(lua-5.1-no-fp.patch).

windows의 cygwin에서, 컴파일하려면 다음 줄을 사용하십시오:

cd src
make LUAC_T=luac.exe MYCFLAGS=-mno-cygwin MYLDFLAGS=-mno-cygwin luac.exe
...
make LUA_T=lua.exe MYCFLAGS=-mno-cygwin MYLDFLAGS=-mno-cygwin lua.exe

Java 런타임 환경(JRE)#

CamScripter에는 Java 런타임 환경(버전 6 이상)이 필요합니다. JRE가 아직 설치되어 있지 않다면 여기에서 다운로드하여 설치할 수 있습니다.

RXTX#

CamScripter는 RXTX 시리얼 통신 라이브러리를 사용하여 컴퓨터의 시리얼 포트를 통해 CMUcam3와 통신합니다. 이 라이브러리는 컴퓨터에 설치해야 합니다. 먼저, RXTX (최소 버전 2.1)를 다운로드하십시오. 버전 2.1.7은 여기 있습니다: ftp://ftp.qbang.org/pub/rxtx/rxtx-2.1-7-bins-r2.zip.

설치하려면 (Windows 전용 - 다른 OS의 경우, rxtx INSTALL 파일의 지침을 따르십시오):

  1. RXTX 다운로드의 압축을 풀고, 결과 폴더로 이동하십시오

  2. Windows를 더블 클릭하십시오

  3. i368-mingw32를 더블 클릭하십시오

  4. 이제 Java 런타임 환경이 어디에 위치하는지 알아내야 합니다. 아마 C:\Program Files\Java\jre<version number>에 있을 것입니다. 예를 들어, 제 것은 C:\Program Files\Java\jre1.6.0_01에 있습니다. RXTX i368-mingw32 폴더에서 rxtxSerial.dll 및 rxtxParallel.dll 파일을 jre/bin 폴더로 복사하십시오. 그래서 제 예에서는, 그 두 파일을 C:\Program Files\Java\jre1.6.0_01\bin 폴더에 넣었습니다.

설치, 2부: Eclipse#

좋습니다, 거의 준비가 되었습니다! 할 일이 몇 가지만 남았습니다.

아직 없다면, Eclipse (버전 3.2 이상)를 설치하십시오. http://eclipse.org/downloads에서 설치할 수 있습니다. Eclipse Classic이 안전한 선택이지만, 나열된 다른 패키지 중 어느 것이든 마찬가지로 작동합니다. 설치하려면, 선택한 패키지의 압축을 C:\Program Files\eclipse에 풀기만 하면 됩니다.

이제 Eclipse가 설치되었으니, CamScripter를 Eclipse 플러그인으로 설치해야 합니다:

  1. C:\Program Files\eclipse\에서 eclipse.exe를 더블 클릭하여 Eclipse를 여십시오

  2. 기본 작업 공간을 수락하려면 Enter를 누르십시오

  3. Help -> Software Updates -> Find and Install을 선택하여 설치/업데이트 마법사를 여십시오

  4. “Search for new features to install” 라디오 버튼을 선택하고, Next를 클릭하십시오.

  5. “New Remote Site…” 버튼을 클릭하십시오

  6. name에는 Camscripter를 입력하고, URL에는 http://cmucam.org/~juanpablo/를 입력하십시오. OK를 클릭하십시오.

  7. Camscripter 옆의 상자가 체크되어 있는지 확인하고, “Finish” 버튼을 클릭하십시오.

  8. 설치할 기능으로 “Camscripter”를 선택하고, Next를 클릭하십시오.

  9. 라이선스 계약에 동의하고 Next를 클릭하십시오

  10. Finish를 클릭하십시오 - Eclipse가 플러그인을 다운로드합니다

  11. Camscripter 플러그인을 설치하려면 “Install”을 클릭하십시오.

  12. Eclipse 플랫폼을 다시 시작하려면 Yes를 클릭하십시오

그러면 Eclipse가 Camscripter 플러그인이 설치된 상태로 다시 시작됩니다. 설치되었는지 확인하려면 Window -> Show View -> Other…를 선택하십시오.

CMUcam3 옆의 더하기 기호를 클릭하고 “CamScripter”를 선택한 다음 OK를 클릭하십시오. Eclipse 하단에 CamScripter 뷰가 보일 것입니다(아래 이미지에서 빨간색으로 둘러싸인 영역, 아카이브에 보존되지 않음). 하단 영역이 너무 작으면 확장할 수 있습니다.

  1. CamScripter 뷰

  2. 텍스트 메시지가 표시되는 콘솔 창

  3. 이미지 및 그래픽 메시지가 표시되는 피드백 창

  4. Lua 코드를 실행합니다. 전체 파일을 실행하거나 선택한 코드 줄만 실행할 수 있습니다.

  5. CMUcam3의 SD/MMC 카드에 Lua 코드 설치

설정#

모든 것을 설치했습니다! 몇 가지 설정 단계만 거치면 CamScripter를 프로그래밍하게 됩니다.

camscripter 프로젝트를 카메라에 로드#

이제 모든 것이 설치되었으니, 카메라에 CamScripter 프로젝트를 로드해야 합니다. projects/camscripter에 위치한 camscripter 프로젝트에서 make를 실행하고(빠른 시작 가이드에서 체크아웃한 소스 코드 트리에서), 그런 다음 hex 파일을 카메라에 로드하십시오. 파일이 로드되면 카메라를 끄십시오.

CamScripter 환경설정 설정#

이제 Eclipse로 돌아가서, CamScripter 환경설정 화면을 여십시오: Window –> Preferences –> CamScripter Setup.

  • 설치, 1부: Lua, 시리얼 포트 통신에서 Lua를 설치한 위치에 있는 luacx.x.exe(여기서 x는 Lua 버전) 파일을 가리키도록 Luac 위치를 설정하십시오(‘c’에 유의).

    • 관심 있는 분들을 위해, luac은 단순히 lua 코드를 컴파일하는 반면 luax.x.exe는 실제로 코드를 해석하고 실행합니다. 우리는 Eclipse에서 코드를 컴파일한 다음 카메라에서 실행합니다 – 그래서 luac입니다.

  • 카메라를 연결하는 시리얼 포트의 이름을 제공하십시오. “See Serial Ports” 버튼을 클릭하여 사용 가능한 시리얼 포트 목록을 볼 수 있습니다.

CamScripter 환경설정 스크린샷은 아카이브에 보존되지 않았습니다.

Hello, World!#

이제 모든 것이 설치되고 제대로 구성되었으니, 필수적인 Hello, World 프로그램을 다뤄 봅시다. 먼저, 새 프로젝트를 만들어야 합니다. File –> New –> Project…를 선택하십시오. 만들려는 프로젝트 유형을 선택할 수 있는 상자가 나타납니다(아래 참조). “General”을 더블 클릭하고, “Project”를 선택한 다음, Next 버튼을 클릭하십시오.

프로젝트 이름을 “Hello”로 지정하고, Finish 버튼을 클릭하십시오. 그런 다음 File –> New –> File을 클릭하여 새 파일을 만드십시오(Eclipse 3.3에서는 File –> New –> Other –> General –> File입니다). 파일 이름 상자에, 상위 폴더를 Hello로 하여 “hello_world.lua”를 입력하십시오. Finish 버튼을 클릭하십시오.

카메라가 컴퓨터의 시리얼 포트에 연결되어 있는지 다시 한 번 확인하십시오. 카메라를 켜기 전에 카메라가 지원하는 두 가지 모드를 이해하는 것이 중요합니다:

  • 독립 실행형: 카메라는 PC와의 어떠한 상호 작용도 기대하지 않습니다. 기본적으로 카메라는 이 모드에서 시작됩니다.

  • 대화형: 카메라는 Eclipse의 CamScripter로부터 컴파일된 Lua 코드를 수신하고 피드백을 반환합니다. 카메라를 대화형 모드로 전환하려면, 카메라를 켜고 주황색 LED가 켜지는 것이 보이면 카메라 버튼을 누르십시오(버튼을 누를 시간이 3초 있습니다). 그 시점에 주황색 LED가 여러 번 깜박이며 대화형 모드로 진입하고 있음을 알려줍니다. 모든 것이 잘 진행되면 Welcome to CamScripter Interactive mode. Click the execute button to run a script. 라는 메시지가 표시됩니다.

첫 번째 Lua 프로그램을 실행할 수 있도록 대화형 모드에서 카메라를 시작해 봅시다.

편집기 창에서, 코드 줄을 입력하십시오:

print("Hello, World!");

파일을 저장하고 CamScripter 뷰에서 Execute 버튼을 클릭하십시오. 그러면 짜잔!! 코드가 유선으로 카메라로 전송되고, 카메라가 이를 실행한 다음, 출력한 메시지를 다시 보냅니다.

이제 좀 더 흥미로운 것을 시도해 봅시다. 세 줄의 코드를 추가하여, hello_world.lua 파일이 다음을 포함하도록 하십시오:

print("Hello, World!");
take_picture("pic_name");
print_picture("pic_name");
print("It's a picture!");

이제 Execute 버튼을 다시 클릭하십시오. 시리얼 포트를 통해 이미지를 보내는 것은 시간이 꽤 걸리므로, 이번에는 실행하는 데 시간이 더 오래 걸립니다. 하지만 완료되면, CamScripter의 콘솔 영역에 두 메시지가 보이고, 피드백 영역에 카메라로 찍은 이미지가 보일 것입니다(렌즈 캡을 제거하셨죠?).

꽤 저해상도 이미지라는 것을 알 수 있습니다. 시리얼 포트를 통해 고해상도 이미지를 보내는 데 시간이 꽤 오래 걸리기 때문에, CamScripter에서는 카메라가 기본적으로 저해상도 사진을 찍도록 설정되어 있습니다. 하지만, camera_set_resolution 함수를 사용하여 카메라를 고해상도로 변경할 수 있습니다.

이 섹션의 Eclipse 프로젝트, 파일, camera-on 및 Hello-World 스크린샷은 아카이브에 보존되지 않았습니다.

유용한 링크#

이제 CamScripter가 완전히 설치되고 구성되었으며, 그 기능을 보여주는 작은 예제를 보았습니다. Camscripter를 사용하는 더 큰 예제를 보고 싶다면, camscripter/examples 디렉터리에 포함된 샘플 프로그램을 살펴보십시오. 도움이 될 수 있는 몇 가지 다른 참고 사항과 링크:

CamScripter Lua API는 CMUcam3의 동작을 제어하기 위해 Lua 프로그램에서 사용할 수 있는 CMUcam3 함수를 설명합니다.

다음은 Lua 사용을 시작하는 데 도움이 되는 몇 가지 링크입니다. string 및 math 라이브러리와 같은 외부 라이브러리는 CMUcam3에 설치된 Lua 버전에 포함되어 있지 않다는 점에 유의하십시오. Lua 인터프리터를 CMUcam3의 제한된 공간에 맞추기 위해 제거해야 했습니다.

  • Lua 튜토리얼 시작하는 데 도움이 되는 튜토리얼.

  • Lua 참조 매뉴얼: Lua의 완전한 문서.

  • Lua 위키 답을 찾기에 유용한 곳. 샘플 코드, FAQ 및 다른 리소스에 대한 링크.

  • LuaEclipse는 유용하다고 느낄 수 있는 유용한 Eclipse 플러그인입니다. 구문 강조 표시와 Lua 코드 작성을 더 쉽게 만드는 다른 기능을 제공합니다.
