CMUcam2 에뮬레이션#

CMUcam2에는 다음과 같은 기능이 있습니다:

  • 초당 최대 50프레임으로 사용자 정의 컬러 블롭 트래킹(프레임 속도는 해상도 및 윈도우 크기 설정에 따라 다름)

  • 초당 26프레임으로 프레임 차분을 사용하여 움직임 트래킹

  • 임의의 트래킹 데이터의 중심(centroid) 찾기

  • 평균 색상 및 분산 데이터 수집

  • 각 색상 채널의 28 구간 히스토그램 수집

  • 수평 에지 필터링된 이미지 처리 (지원되지 않음)

  • 이미지에서 트래킹된 픽셀의 실시간 이진 비트맵 전송

  • 임의 이미지 윈도잉

  • 이미지 다운 샘플링

  • 카메라의 이미지 속성 조정

  • 원시 이미지 덤프(단일 또는 다중 채널)

  • 최대 176 x 255 해상도

  • 지원 보드 레이트: 115,200 57,600 38,400 19,200 9,600 4,800 2,400 1,200 BPS

  • 5개의 서보 출력 제어

  • 단일 카메라 버스에서의 슬레이브 병렬 이미지 처리 모드 (지원되지 않음)

  • 서보를 자동으로 사용하여 2축 컬러 트래킹 수행

  • 흑백 아날로그 비디오 출력 (사용하는 카메라 모듈에 따라 PAL 또는 NTSC)

  • 유연한 출력 패킷 커스터마이징

  • 전원 차단 모드

  • 버퍼링된 이미지에 대한 다중 패스 이미지 처리

CMUcam2 에뮬레이션 차이점#

  • 서보 입력 범위

    • 서보는 이제 0-255 8비트 값입니다

  • 점퍼

    • CMUcam3에는 점퍼가 없으므로 보드 레이트와 서보 방향 변경은 컴파일 시에 수행해야 합니다 - 이는 cmucam.c 소스 코드 상단의 #define으로 제어됩니다

  • Set Input (SI pin ) 명령

    • pin을 입력으로 설정합니다

  • Software Color Space (CS value ) 명령

    • CS 0은 색 공간을 하드웨어 기본값(RGB 또는 YCrCb)으로 설정합니다

    • CS 1은 HSV 색 공간으로 소프트웨어 변환을 수행합니다

      • 변환은 올바른 결과를 위해 하드웨어가 RGB로 설정되어 있다고 가정합니다

      • HSV 공간은 모든 값이 16에서 240 사이로 스케일링되는 정수 공간입니다

    • 참고: 이것은 r504 이상에서만 사용할 수 있습니다

  • Set Output으로 파란색 및 주황색 LED를 제어할 수 있습니다

    • 서보 2와 3이 작동을 멈춥니다

    • SO 2 1 -> 파란색 켜짐

    • SO 2 0 -> 파란색 꺼짐

    • SO 3 1 -> 주황색 켜짐

    • SO 3 0 -> 주황색 꺼짐

  • 슬레이브 모드 없음

    • 가능하지만, 현재는 지원하지 않습니다

  • 출력 마스크 (OM)

    • T 및 S 패킷(유형 0과 1)만 지원

  • 라인 모드 (LM)

    • 컬러 트래킹 모드(LM 0 1)만 지원

    • r523에서 추가된 트랙 컬러 통계 (LM 0 2)

  • JPEG 전송 (SJ)

    • 헤더를 포함한 jpeg 압축 프레임을 전송합니다

  • 데모 모드

    • 프로그래밍 버튼을 누르고 있으면 CMUcam3가 프로그래밍 모드로 들어가므로, 데모 모드에 진입하려면 시작 후 0.5초 후에 버튼을 눌러야 합니다

  • 고해상도 프레임 차분

    • 이것은 이제 더 높은 해상도에서 저해상도 프레임 차분과 동일하게 작동합니다
