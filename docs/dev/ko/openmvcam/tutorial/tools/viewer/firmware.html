13.2.6. 펌웨어 업데이트 및 복구#

Viewer는 카메라의 펌웨어 관리자를 유지하므로, 전체 IDE 없이도 현장에서 장치를 복구하거나 다시 프로비저닝할 수 있습니다. 모든 보드에 맞는 펌웨어 릴리스와 모든 보드 제품군의 부트로더용 로더를 번들로 포함하며, 별도의 지정 없이 보드를 감지하고 올바른 프로토콜로 통신합니다.

IDE와 달리 Viewer는 업데이트를 재촉하지 않습니다: 따라서 데모가 오래된 버전 프롬프트로 인해 중단되는 일이 없으며, 제품이 출하하는 펌웨어가 무엇이든 그대로 유지됩니다. 펌웨어 작업이 필요할 때는 Tools 메뉴에서 필요에 따라 사용할 수 있습니다.

13.2.6.1. 타사 보드 및 차량 업데이트#

Viewer는 third-party repositories을 지원하므로 자체 Viewer에 자체 보드를 배송하는 회사는 해당 카메라를 현장에서 최신 상태로 유지할 수 있습니다. 이는 잔소리 금지 규칙의 한 가지 예외입니다. 타사 보드 does는 뷰어를 통해 오래된 메시지와 업데이트를 받습니다. 업데이트는 공급업체 자체의 것이므로 OpenMV가 아닌 저장소 매니페스트의 URL에서 제공됩니다. OpenMV의 자체 보드는 여전히 메시지를 표시하지 않습니다. 뷰어는 시작 시 저장소의 펌웨어 채널을 확인하고 공급업체가 이를 게시할 때 최신 펌웨어를 제공합니다. 뷰어에는 이에 대한 인터페이스가 없기 때문에 저장소에 있을 수 있는 예제, 모델 또는 스텁을 풀다운하지 않습니다.

13.2.6.2. 펌웨어 로드#

Tools → Load Custom Firmware는 디스크에서 펌웨어 바이너리 – 또는 보드의 빌드 출력 .zip 번들 – 를 플래시하며, 교체 카메라에 제품 자체 펌웨어를 넣는 방법입니다. 대화 상자는 IDE와 동일한 옵션을 제공합니다: 내부 플래시 파일 시스템 지우기, ROM 파일 시스템 재설정.

13.2.6.3. ROM 파일 시스템 로드#

Tools → Load ROMFS onto OpenMV Cam은 .img ROM 파일 시스템 이미지 – 제품의 모델과 자산 – 를 카메라에 기록하므로, Viewer는 장치의 펌웨어뿐만 아니라 데이터도 복원할 수 있습니다.

13.2.6.4. 벽돌이 된 카메라 복구#

펌웨어가 손상되어 더 이상 시리얼 포트로 나타나지 않는 카메라도 여전히 복구할 수 있습니다. 부트로더는 펌웨어 아래에 위치하며 하드웨어 손상이 아닌 한 어떤 상황에서도 살아남기 때문입니다. 연결 시 카메라를 찾지 못하면 Viewer는 벽돌이 된 카메라가 연결되어 있는지 묻고 복구 과정을 안내합니다: 보드 유형을 선택하면 Viewer가 보드의 부트로더 인터페이스를 구동하고 번들된 릴리스 펌웨어를 설치합니다. Tools → Force enter OpenMV Cam bootloader는 여전히 연결되는 카메라에 대해 부트로더로 재부팅하는 단계를 명시적으로 수행합니다.
