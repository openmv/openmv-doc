MicroPython in C erweitern¶

Dieses Kapitel beschreibt Optionen zur Implementierung zusätzlicher Funktionalität in C, jedoch aus Code, der außerhalb des Haupt-Repositorys von MicroPython geschrieben wurde. Der erste Ansatz ist nützlich, um Ihre eigene benutzerdefinierte Firmware mit einigen projektspezifischen zusätzlichen Modulen oder Funktionen zu erstellen, auf die von Python aus zugegriffen werden kann. Der zweite Ansatz dient zum Erstellen von Modulen, die zur Laufzeit geladen werden können.

Weitere Informationen zum Erstellen von Kernmodulen, die im Haupt-Repository von MicroPython leben, finden Sie im Bibliotheksabschnitt.
