Schnellstart#

Erste Schritte#

Bitte befolgen Sie die folgenden Schritte:

  1. Sie benötigen die folgenden zwei Elemente, um die CMUcam4 zu testen

    • Ein externes 4V- bis 9V-DC-Netzteil, das mindestens 250 mA liefern kann

      • Dies kann von einem Arduino, einem FTDI Breakout Board, einem FTDI-Kabel oder einem Steckernetzteil stammen

    • Ein NTSC-Fernseher (oder ein kompatibler Fernsehmonitor) und ein RCA-Kabel

  2. Aufbau zum Testen der CMUcam4

    • Verbinden Sie das NTSC-TV über das RCA-Kabel mit der RCA-Koaxialbuchse der CMUcam4

    • Verbinden Sie das externe Netzteil mit der DC-Hohlbuchse der CMUcam4 (oder anderen Stromanschlüssen)

Die grüne Betriebs-LED sollte aufleuchten, sobald Sie das externe Netzteil mit der CMUcam4 verbinden. Nach etwa 2 Sekunden sollte die rote Hilfs-LED aufleuchten, um anzuzeigen, dass die CMUcam4 einsatzbereit ist. Wenn die grüne Betriebs-LED nicht aufleuchtet, überprüfen Sie bitte die Verbindung zum externen Netzteil.

Testverfahren#

Bitte befolgen Sie die folgenden Schritte:

  1. Drücken und halten Sie die Reset-Taste an der CMUcam4

  2. Drücken und halten Sie die Benutzertaste an der CMUcam4

  3. Lassen Sie die Reset-Taste los (lassen Sie nicht die Benutzertaste los)

  4. Warten Sie, bis die rote Hilfs-LED aufleuchtet (2 Sekunden)

  5. Warten Sie, bis die rote Hilfs-LED mit 10 Hz zu blinken beginnt, und lassen Sie dann die Benutzertaste los

    • Der Fernseher sollte sich einschalten (Sie sollten einen Startbildschirm auf dem Fernseher sehen)

  6. Die CMUcam4 wird sich nun für die nächsten 5 Sekunden an die Beleuchtungsbedingungen anpassen

    • Platzieren Sie das Objekt, das Sie verfolgen möchten, in den nächsten 5 Sekunden nicht vor der CMUcam4

  7. Warten Sie, bis die rote Hilfs-LED aufhört, mit 10 Hz zu blinken

    • Die CMUcam4 ist jetzt mit der Anpassung an die Beleuchtungsbedingungen fertig

    • Die Schwenk- und Neigeservo-Pins sollten 1500 μs Impulse bei 50 Hz ausgeben

  8. Platzieren Sie das Objekt, das Sie verfolgen möchten, vor der CMUcam4 und drücken Sie die Benutzertaste

    • Wenn die rote Hilfs-LED mit 10 Hz zu blinken beginnt, überprüfen Sie die Verbindung des OV9665-Kameramoduls

      • Das OV9665-Kameramodul ist möglicherweise beschädigt und muss höchstwahrscheinlich ersetzt werden

  9. Sie sollten jetzt das verfolgte Objekt (oder ähnliches) auf dem Fernseher angezeigt sehen.

    • Die Schwenk- und Neigeservos werden, sofern angeschlossen, ebenfalls versuchen, die Kamera zum verfolgten Objekt hin zu bewegen

  10. Bitte probieren Sie dieses Verfahren mit verschiedenen Objekten in unterschiedlichen Umgebungen aus, um zu sehen, was am besten funktioniert

Laden Sie die Testanleitung im PDF-Format hier herunter.

Kommunikationswerkzeuge#

Sie benötigen einen der folgenden (oder ähnlichen) USB-zu-Seriell-Konverter, um mit der CMUcam4 zu kommunizieren:

  • Ein FTDI 5V Breakout Board

    • Sie benötigen außerdem eine externe Stromversorgung, die die CMUcam4 mit Strom versorgen kann

    • Bitte verbinden Sie das FTDI 5V Breakout Board mit dem 6-poligen Anschluss an der CMUcam4

  • Ein FTDI 3.3V Breakout Board

    • Sie benötigen außerdem eine externe Stromversorgung, die die CMUcam4 mit Strom versorgen kann

    • Bitte verbinden Sie das FTDI 3.3V Breakout Board mit dem 6-poligen Anschluss an der CMUcam4

  • Ein Prop Clip

    • Sie benötigen außerdem eine externe Stromversorgung, die die CMUcam4 mit Strom versorgen kann

    • Bitte verbinden Sie den Prop Clip mit dem 4-poligen Anschluss an der CMUcam4

  • Ein Prop Plug

    • Sie benötigen außerdem eine externe Stromversorgung, die die CMUcam4 mit Strom versorgen kann

    • Bitte verbinden Sie den Prop Plug mit dem 4-poligen Anschluss an der CMUcam4

  • Ein FTDI-5V-Kabel mit 5V-E/A

    • Empfohlen für 5V-tolerante Systeme

    • Bitte verbinden Sie das FTDI 5V Kabel mit 5V I/O mit dem 6-poligen Anschluss an der CMUcam4

  • Ein FTDI-5V-Kabel mit 3.3V-E/A

    • Empfohlen für 3.3V-tolerante Systeme

    • Bitte verbinden Sie das FTDI 5V Kabel mit 3.3V I/O mit dem 6-poligen Anschluss an der CMUcam4

Empfohlene serielle Terminalprogramme#

Das Parallax Serial Terminal ist ein praktisches Werkzeug für die Kommunikation mit seriell-basierten Mikrocontrollern wie dem Parallax Propeller-Chip. Es ist das empfohlene serielle Terminal für die Kommunikation mit der Kameraplatine. Das Parallax Serial Terminal (PST) ist eine eigenständige Anwendung mit einer Größe von weniger als 1 MB und erfordert für die Verwendung keine Installation. PST steht bei Parallax Inc. hier zum Download zur Verfügung. Befolgen Sie die folgenden Schritte, um PST einzurichten:

  1. Führen Sie PST aus

  2. Gehen Sie zu Echo On und stellen Sie sicher, dass es aktiviert ist

  3. Gehen Sie zu Com Port und wählen Sie aus der Dropdown-Liste den COM-Port, an den die CMUcam4 angeschlossen ist

  4. Gehen Sie zu Baud Rate und wählen Sie 19200 aus der Dropdown-Liste

    • Klicken Sie bei Bedarf auf Enable

Für Nicht-Windows-Benutzer wird Brad’s SPIN Tool (BST) empfohlen. BST kann hier heruntergeladen werden. BST ist eine eigenständige Anwendung mit grafischer Benutzeroberfläche (GUI) und integrierter Entwicklungsumgebung (IDE), die weniger als 10 MB groß ist, für den Parallax Propeller Chip konzipiert wurde und keine Installation für die Nutzung erfordert. BST enthält ein integriertes, einfach zu bedienendes serielles Terminal. Befolgen Sie die folgenden Schritte, um das integrierte, einfach zu bedienende serielle Terminal von BST einzurichten:

  1. Führen Sie BST aus

  2. Gehen Sie zu View und wählen Sie Serial Terminal aus der Dropdown-Liste

    • Das bst Terminal sollte aufspringen – bitte klicken Sie darauf

  3. Gehen Sie zu Baud und wählen Sie 19200 aus der Dropdown-Liste

  4. Gehen Sie zu Format und wählen Sie 8 Bits und Parity None aus der Dropdown-Liste

  5. Gehen Sie zu Port und wählen Sie aus der Dropdown-Liste den COM-Port, an den die CMUcam4 angeschlossen ist

    • Suchen Sie nach Ports mit dem Namen /dev/tty/USB###

  6. Gehen Sie zu Communicate und wählen Sie Connect, Terminal Echo und Reset Propeller aus dem Dropdown-Menü

    • Gehen Sie zu Communicate und wählen Sie Display ASCII für ASCII-Ausgabe oder Display Hex für Hex-Ausgabe

HINWEIS: Sie können die FTDI-5V-Kabel nicht mit BST verwenden!

Das liegt daran, dass BST die grüne RTS-Leitung am FTDI 5V Cable auf Low zieht, wenn Sie auf Connect klicken. Dies hält die CMUcam4 auf unbestimmte Zeit an. Sie können die grüne RTS-Leitung vom Stecker entfernen, um dieses Problem zu beheben.
