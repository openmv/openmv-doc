Typische Verwendungen#

Die CMUcam4 kann verwendet werden, um Farben zu verfolgen oder grundlegende Bildstatistiken zu erfassen. Die beste Leistung kann erzielt werden, wenn stark kontrastierende und intensive Farben vorhanden sind. Zum Beispiel kann sie leicht einen roten Ball auf weißem Hintergrund verfolgen, aber es wäre schwierig, zwischen verschiedenen Brauntönen bei wechselndem Licht zu unterscheiden. Das Verfolgen farbiger Objekte kann verwendet werden, um Orientierungspunkte zu lokalisieren, Linien zu folgen oder sich bewegende Baken zu verfolgen. Mithilfe von Farbstatistiken ist es der CMUcam4 möglich, eine Szene zu überwachen, eine bestimmte Farbe zu erkennen oder eine primitive Bewegungserkennung durchzuführen. Wenn die CMUcam4 eine drastische Farbänderung erkennt, hat sich wahrscheinlich etwas in der Szene verändert. Im „Line Mode“ kann die CMUcam4 niedrig aufgelöste Binärbilder farbiger Objekte erzeugen. Dies kann verwendet werden, um anspruchsvollere Bildverarbeitung durchzuführen, die Linienverfolgung mit Abzweigungserkennung oder sogar einfache Formerkennung umfasst. Diese fortgeschritteneren Operationen erfordern benutzerdefinierte Algorithmen zur Nachbearbeitung der von der CMUcam4 gesendeten Binärbilder. Wie bei einer normalen Digitalkamera könnte diese Art der Verarbeitung einen Computer oder zumindest einen schnellen Mikrocontroller erfordern.

Typische Konfiguration#

Die häufigste Konfiguration für die CMUcam4 ist, dass sie über eine standardmäßige serielle TTL-Schnittstelle mit einem Master-Prozessor kommuniziert. Dieser „Master-Prozessor“ könnte ein Computer (über USB oder RS232), ein Arduino, eine Basic Stamp, ein PIC oder ein ähnlicher Mikrocontroller sein. Die CMUcam4 ist klein genug, um eingebetteten Systemen einfache Bildverarbeitung hinzuzufügen, die sich die Größe oder Leistung eines standardmäßigen computerbasierten Bildverarbeitungssystems nicht leisten können. Ihr Kommunikationsprotokoll ist so konzipiert, dass es selbst den langsamsten Prozessoren gerecht wird. Die CMUcam4 unterstützt verschiedene Baudraten, um langsameren Prozessoren gerecht zu werden. Für noch langsamere Prozessoren kann die CMUcam4 im „Poll Mode“ arbeiten. In diesem Modus kann der Host-Prozessor die CMUcam4 um nur ein einzelnes Datenpaket bitten. Dies gibt langsameren Prozessoren die Möglichkeit, leichter mit den Daten synchronisiert zu bleiben. Es ist auch möglich, mit dem „Delay Mode“-Befehl eine Verzögerung zwischen einzelnen seriellen Datenzeichen hinzuzufügen. Aufgrund von Kommunikationsverzögerungen verringern sowohl der Poll Mode als auch der Delay Mode die Gesamtzahl der Frames, die in einer Sekunde verarbeitet werden können.
