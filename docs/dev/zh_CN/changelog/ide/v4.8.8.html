v4.8.8¶

v4.8.8 为连接和更新 OpenMV Cam 新增了显式的引导加载程序进入控制，为从“Install the Latest Development Release”对话框强制进入引导加载程序提供了按板卡的安全默认值，并修复了若干连接和安装程序问题。本次发布仍基于 Qt Creator 14.0.2，未变更 Qt Creator 基础版本，也不会对用户项目造成破坏性变更。

亮点¶

  • 现在可从 Tools 菜单使用 Force enter OpenMV Cam bootloader，在断开连接时强制将已连接的摄像头进入其引导加载程序。

  • 在 Install the Latest Development Release 对话框中新增了 Force bootloader 复选框，可在安装开发版固件时强制进入引导加载程序，并提供按板卡的安全默认值。

  • 仅支持 DFU 的设备不再被误判而自动连接，因为自动重连和板卡设置自动连接现在仅对正常运行的板卡触发。

  • AE3 上的 PAG7936 示例脚本 在加载时会被自动改写，将 QQVGA 帧尺寸升级为 QVGA。

  • 修复了安装程序的许可证处理，使安装能够从 Ready-for-Installation 页面正确进行。

新功能¶

  • 强制进入 OpenMV Cam 引导加载程序。 Tools 菜单中新增了一项，可在断开连接时强制将已连接的摄像头进入其引导加载程序。Install the Latest Development Release 对话框（从 Tools 菜单进入）也新增了 Force bootloader 复选框，可在安装开发版固件时强制进入引导加载程序。（350beb306，13ca76904）

其他变更与改进¶

  • 按板卡的强制引导加载程序默认值。 开发版安装的强制引导加载程序默认值现在通过新的 forceBootloaderDefaultSafe 设置按板卡驱动，对 AE3 和 N6（96E3/9206）板卡默认开启，对较旧的 DFU 板卡默认关闭，以避免它们无法自动退出引导加载程序。（13ca76904）

  • AE3 上的 PAG7936 帧尺寸升级。 在搭载 PAG7936 传感器的 AE3 上，示例脚本在加载时会被自动改写，将 QQVGA 帧尺寸升级为 QVGA。（eed6325f5）

  • 非仅 DFU 设备的自动连接。 自动重连和板卡设置自动连接现在仅对非 DFU（正常运行）的板卡触发，而非任何在场的板卡，因此仅支持 DFU 的设备不再被误判而自动连接。（350beb306）

错误修复¶

  • 修复了连接时固件更新提示中的拼写错误（“lastest”->“latest release firmware”）。（13ca76904）

  • 修复了安装程序的许可证接受处理，使安装步骤在 Ready-for-Installation 页面而非许可证检查页面运行，并移除了存在问题的捆绑许可证文件。（a9d3fb904，716827f07）

平台与工具支持¶

  • Qt Creator 基础版本： 14.0.2。

  • OpenMV IDE： 版本升级至 4.8.8。

破坏性变更¶

无。本次发布不引入任何用户可见的工作流或行为破坏。
