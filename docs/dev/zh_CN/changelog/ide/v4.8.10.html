v4.8.10¶

v4.8.10 新增了对 OpenMV AE3 板的支持，并刷新了捆绑的 AI 工具链，将 Ethos-U Vela NPU 编译器更新至 5.0.0，将 ST Edge AI 工具更新至 4.0.0。本次发布仍基于 Qt Creator 14.0.2，未进行 rebase，且不包含任何重大变更。

亮点¶

  • 现在支持 OpenMV AE3 Alif Ensemble E3 板，包括 alif_tools 引导加载程序/DFU 编程以及每核 Ethos-U55 NPU 加速。

  • Ethos-U Vela NPU 模型编译器从 4.2.0 更新至 5.0.0，并针对 Python 3.13 ABI 重新构建。

  • ST Edge AI（stedgeai）模型转换工具在所有平台上从 3.0.0 更新至 4.0.0。

  • 更快的连接回退： IDE 现在在 2 次重连尝试后（而非 10 次）放弃失败的摄像头连接。

新功能¶

  • OpenMV AE3 Alif Ensemble E3 板支持（内部代号 “AE3 classic”）。新增 AE3 / OPENMV AE3 SBL2 板类型，IDE 显示名称为 OpenMV AE3 SBL2（VID:PID 37C5:16E3），支持 alif_tools 引导加载程序/DFU 编程、双核（HP/HE）romfs 布局，以及每核 Ethos-U55 Vela NPU 加速配置（高性能核 ethos-u55-256，高效核 ethos-u55-128）。该板是一个隐藏/通过 DFU 编程的目标，通过其引导加载程序的 VID:PID 映射访问，而非作为通常列出的可选连接目标（adbed3825）。

其他更改和改进¶

  • 将自动摄像头重连尝试的最大次数从 10 次减少到 2 次，使 IDE 在连接失败时更快放弃（2ddecec53）。

  • 在 Windows、Linux 和 macOS arm64 上将捆绑的 Ethos-U Vela NPU 模型编译器从 4.2.0 更新至 5.0.0，针对 Python 3.13 ABI 重新构建，重新编译了 regor 和 mlw_codec 二进制文件，并在所有平台上将捆绑的 numpy 更新至 2.4.4（be9d29d9d、04c73f187、93260af21、a58458cfb、a6e1ac7ed）。

  • 在 Windows、Linux 和 macOS arm64 上将捆绑的 ST Edge AI（stedgeai）模型转换工具从 3.0.0 更新至 4.0.0（1a727567c、8dcfd8472、27aabc740）。

平台和工具支持¶

  • Qt Creator 基础版本： 14.0.2。

  • 新板： OpenMV AE3 SBL2（Alif Ensemble E3），一个隐藏/通过 DFU 编程的目标，通过 alif_tools 编程，具有双核 Ethos-U55 NPU 加速。

  • Ethos-U Vela NPU 编译器：4.2.0 -> 5.0.0（Windows、Linux、macOS arm64），针对 Python 3.13 ABI 重新安装。

  • ST Edge AI（stedgeai）工具：3.0.0 -> 4.0.0（Windows、Linux、macOS arm64）。

  • 捆绑的 numpy（Vela 环境）：在所有平台上更新至 2.4.4。

重大变更¶

无。本次发布不引入任何用户可见的工作流程或行为不兼容改动，现有项目可继续保持不变地工作。
