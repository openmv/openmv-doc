4.6. 卷帘快门与全局快门¶

传感器逐个单元读出其二维像素栅格。这种读出过程中有两点会影响所记录的图像：像素被扫描的顺序，以及每行的曝光窗口在时间上如何与该扫描对齐。前者由硅片固定下来；后者则有两种成熟的形式，它们对运动场景的影响很大。

4.6.1. 读出顺序¶

典型的传感器从左下角的像素开始，沿该行向右扫描，然后向上推进到下一行再次向右扫描，依此类推，直到在右上角结束。

这一顺序并非偶然。镜头在将场景投射到传感器上时会将其水平镜像并垂直翻转——场景顶部落在传感器底部，场景左侧落在传感器右侧——而从左下角向上的读出方式正是按照能够同时撤销这两种翻转的顺序遍历传感器，从而将像素以正立方向存入内存。

4.6.2. 卷帘快门¶

在卷帘快门（rolling-shutter）传感器中，每一行依次被曝光和读出。当某一行正在被读出时，下一行仍在完成它的曝光，再下一行才刚刚开始，依此类推——每一行的曝光窗口在时间上相对于下一行略有偏移。传感器的积分窗口按扫描顺序在整帧上滚动，完整一次扫描需要一整个帧周期。

对于静止场景，这是不可见的。对于快速运动的场景，这种偏移会表现为倾斜（skew）——在第一行被捕获到最后一行被捕获之间发生移动的物体，会在同一帧的不同行中出现在不同的位置。

卷帘快门是更廉价的设计。由于每一行在完成曝光后会被迅速读出，像素电路不需要每像素的屏蔽存储单元来在整个传感器范围的读出过程中保持其值。省下来的晶体管使光电二极管占据更大比例的像素面积，这直接转化为在相同物理像素尺寸下更高的灵敏度和更低的噪声。出于这个原因，大多数消费级图像传感器都采用卷帘快门。

4.6.3. 全局快门¶

在全局快门（global-shutter）传感器中，每个像素都在同一瞬间开始曝光，并在同一瞬间结束曝光。捕获到的电荷随后被转移到像素上的屏蔽存储区，逐行读出便从那里进行。无论场景移动得多快，捕获到的帧都代表同一时刻。

全局快门需要更多的硅片成本，而代价落在光电二极管上。要在整个传感器范围的读出过程中保持每一行的值，每个像素都需要一个额外的屏蔽存储单元，以及将其与光电二极管隔离的晶体管——这些面积本应属于光电二极管本身。更小的光电二极管在单位时间内捕获的光子更少，因此全局快门像素的灵敏度低于同等尺寸的卷帘快门像素。要以相同的亮度记录同一场景，就需要更长的曝光时间或更高的增益，而额外的电路还会在此基础上略微提高读出噪声。

另一项代价在于曝光预算。在卷帘快门传感器中，每一行的曝光与相邻行的读出相重叠，因此每一行几乎都能在整个帧周期内积分光线。在全局快门中，读出必须等到每一行都完成曝光后才能开始，因此在给定帧率下，最大曝光时间是帧周期减去完整读出时间。在相同帧率下，卷帘快门像素每帧最终会获得更多的光线。

这些代价会叠加：与卷帘快门同类产品相比，全局快门传感器的像素数更少、噪声更大、灵敏度更低，且每像素成本更高。只有在必须干净地捕获快速运动时，这种权衡才值得。

4.6.4. 何时使用哪种¶

快门类型是传感器的硬件属性，而非软件设置。这一选择在摄像头设计时就已确定。

在以下情况下，卷帘快门是合适的：

  • 场景静止或移动缓慢；

  • 应用能够容忍一定程度的倾斜（大多数摄影和大多数用户界面相关工作）；

  • 成本和每美元分辨率是首要考虑因素。

在以下情况下，全局快门是正确的选择：

  • 场景包含必须干净捕获的快速运动（机器人、无人机、传送带检测）；

  • 摄像头本身相对于静态场景在振动或移动；

  • 图像被送入一个假设每帧都是单一时刻的视觉算法（大多数姿态估计和运动恢复结构流水线）。

备注

OpenMV Cam 系列默认采用全局快门传感器用于机器视觉用途，在这类用途中，运动主体（或运动摄像头）上的运动模糊会破坏下游的检测和跟踪。对于慢速或静态场景的图像质量比冻结快速运动更重要的应用，也提供卷帘快门传感器模块——即经典的摄影式拍摄。
