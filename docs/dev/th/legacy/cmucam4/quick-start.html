เริ่มต้นอย่างรวดเร็ว#

เริ่มต้นใช้งาน#

โปรดทำตามขั้นตอนด้านล่าง:

  1. คุณจะต้องมีสองรายการต่อไปนี้เพื่อทดสอบ CMUcam4

    • แหล่งจ่ายไฟภายนอก 4V ถึง 9V DC ที่สามารถจ่ายได้อย่างน้อย 250 mA

      • สิ่งนี้อาจมาจาก Arduino, FTDI Breakout Board, FTDI Cable หรือ Wall Wart

    • ทีวี NTSC (หรือจอโทรทัศน์ที่เข้ากันได้) และ สาย RCA

  2. การตั้งค่าเพื่อทดสอบ CMUcam4

    • เชื่อมต่อ ทีวี NTSC เข้ากับ แจ็คโคแอกเชียล RCA ของ CMUcam4 โดยใช้ สาย RCA

    • เชื่อมต่อแหล่งจ่ายไฟภายนอกเข้ากับ แจ็ค DC Barrel ของ CMUcam4 (หรือพอร์ตพลังงานอื่นๆ)

LED กำลังไฟสีเขียวควรสว่างขึ้นเมื่อคุณเชื่อมต่อแหล่งจ่ายไฟภายนอกกับ CMUcam4 หลังจากประมาณ 2 วินาที LED เสริมสีแดงควรสว่างขึ้นเพื่อบ่งชี้ว่า CMUcam4 พร้อมทำงาน หาก LED กำลังไฟสีเขียวไม่สว่าง โปรดตรวจสอบการเชื่อมต่อแหล่งจ่ายไฟภายนอกอีกครั้ง

ขั้นตอนการทดสอบ#

โปรดทำตามขั้นตอนด้านล่าง:

  1. กดค้าง ปุ่ม reset บน CMUcam4

  2. กดค้าง ปุ่ม user บน CMUcam4

  3. ปล่อย ปุ่ม reset (อย่าปล่อย ปุ่ม user)

  4. รอจนกว่า LED เสริมสีแดงจะติด (2 วินาที)

  5. รอจนกว่า LED เสริมสีแดงจะเริ่มกะพริบที่ 10 Hz แล้วปล่อย ปุ่มผู้ใช้ (user button)

    • ทีวีควรจะเปิดขึ้น (คุณควรเห็นหน้าจอสแปลชแสดงบนทีวี)

  6. ตอนนี้ CMUcam4 จะปรับตัวเข้ากับสภาพแสงในอีก 5 วินาที ข้างหน้า

    • อย่าวางวัตถุที่คุณต้องการติดตามไว้หน้า CMUcam4 ในช่วง 5 วินาที ถัดไป

  7. รอจนกว่า LED เสริมสีแดงจะหยุดกะพริบที่ 10 Hz

    • ตอนนี้ CMUcam4 ปรับตัวเข้ากับสภาพแสงเสร็จแล้ว

    • พินเซอร์โวแพนและก้มควรส่งออกพัลส์ 1500 μs ที่ 50 Hz

  8. วางวัตถุที่คุณต้องการติดตามไว้หน้า CMUcam4 แล้วกด ปุ่ม user

    • หาก LED เสริมสีแดงเริ่มกะพริบที่ 10 Hz ให้ตรวจสอบการเชื่อมต่อโมดูลกล้อง OV9665

      • โมดูลกล้อง OV9665 อาจเสียหายและมีแนวโน้มว่าต้องเปลี่ยนใหม่

  9. ตอนนี้คุณควรเห็นวัตถุที่ถูกติดตาม (หรือคล้ายกัน) แสดงบนทีวี

    • เซอร์โวแพนและก้ม หากเชื่อมต่ออยู่ ก็จะพยายามขับกล้องไปยังวัตถุที่ถูกติดตามด้วย

  10. โปรดลองขั้นตอนนี้กับวัตถุต่างๆ ในสภาพแวดล้อมต่างๆ เพื่อดูว่าอะไรใช้งานได้ดีที่สุด

เครื่องมือสื่อสาร#

คุณจะต้องมีหนึ่งใน ตัวแปลง USB เป็น Serial ต่อไปนี้ (หรือคล้ายกัน) เพื่อสื่อสารกับ CMUcam4:

  • บอร์ด Breakout FTDI 5V

    • คุณยังต้องการแหล่งจ่ายไฟภายนอกที่สามารถจ่ายไฟให้ CMUcam4 ได้

    • โปรดเชื่อมต่อ บอร์ด Breakout FTDI 5V เข้ากับขั้วต่อ 6-pin บน CMUcam4

  • บอร์ด Breakout FTDI 3.3V

    • คุณยังต้องการแหล่งจ่ายไฟภายนอกที่สามารถจ่ายไฟให้ CMUcam4 ได้

    • โปรดเชื่อมต่อ บอร์ด Breakout FTDI 3.3V เข้ากับขั้วต่อ 6-pin บน CMUcam4

  • Prop Clip หนึ่งอัน

    • คุณยังต้องการแหล่งจ่ายไฟภายนอกที่สามารถจ่ายไฟให้ CMUcam4 ได้

    • โปรดเชื่อมต่อ Prop Clip เข้ากับขั้วต่อ 4-pin บน CMUcam4

  • Prop Plug หนึ่งอัน

    • คุณยังต้องการแหล่งจ่ายไฟภายนอกที่สามารถจ่ายไฟให้ CMUcam4 ได้

    • โปรดเชื่อมต่อ Prop Plug เข้ากับขั้วต่อ 4-pin บน CMUcam4

  • สาย FTDI 5V พร้อม I/O 5V

    • แนะนำสำหรับระบบที่รองรับ 5V

    • โปรดเชื่อมต่อ สาย FTDI 5V พร้อม I/O 5V เข้ากับขั้วต่อ 6-pin บน CMUcam4

  • สาย FTDI 5V พร้อม I/O 3.3V

    • แนะนำสำหรับระบบที่รองรับ 3.3V

    • โปรดเชื่อมต่อ FTDI สาย 5V พร้อม I/O 3.3V เข้ากับขั้วต่อ 6-pin บน CMUcam4

โปรแกรมเทอร์มินัลซีเรียลที่แนะนำ#

Parallax Serial Terminal เป็นเครื่องมือที่สะดวกสำหรับการสื่อสารกับไมโครคอนโทรลเลอร์ที่ใช้ซีเรียลเช่นชิป Parallax Propeller มันคือเทอร์มินัลอนุกรมที่แนะนำให้ใช้ในการสื่อสารกับบอร์ดกล้อง Parallax Serial Terminal (PST) เป็นแอปพลิเคชันแบบสแตนด์อโลนที่มีขนาดน้อยกว่า 1 MB และไม่ต้องติดตั้งเพื่อใช้งาน PST สามารถดาวน์โหลดได้จาก Parallax Inc. ที่นี่ ทำตามขั้นตอนด้านล่างเพื่อตั้งค่า PST:

  1. รัน PST

  2. ไปที่ Echo On และตรวจสอบให้แน่ใจว่าถูกเลือกไว้

  3. ไปที่ Com Port แล้วเลือกพอร์ต COM ที่ CMUcam4 เชื่อมต่ออยู่จากรายการดรอปดาวน์

  4. ไปที่ Baud Rate แล้วเลือก 19200 จากรายการดรอปดาวน์

    • คลิก Enable หากจำเป็น

สำหรับผู้ใช้ที่ไม่ใช่ Windows แนะนำให้ใช้ Brad's SPIN Tool (BST) สามารถดาวน์โหลด BST ได้ ที่นี่ BST เป็นแอปพลิเคชันแบบสแตนด์อโลนที่เป็นสภาพแวดล้อมการพัฒนาแบบรวม (IDE) พร้อมส่วนติดต่อผู้ใช้แบบกราฟิก (GUI) ขนาดน้อยกว่า 10 MB ที่ออกแบบมาสำหรับชิป Parallax Propeller และไม่ต้องติดตั้งเพื่อใช้งาน BST มีเทอร์มินัลซีเรียลที่ใช้งานง่ายในตัว ทำตามขั้นตอนด้านล่างเพื่อตั้งค่าเทอร์มินัลซีเรียลที่ใช้งานง่ายในตัวของ BST:

  1. รัน BST

  2. ไปที่ View แล้วเลือก Serial Terminal จากรายการดรอปดาวน์

    • bst Terminal ควรจะปรากฏขึ้น -- โปรดคลิกที่มัน

  3. ไปที่ Baud แล้วเลือก 19200 จากรายการดรอปดาวน์

  4. ไปที่ Format แล้วเลือก 8 Bits และ Parity None จากรายการดรอปดาวน์

  5. ไปที่ Port แล้วเลือกพอร์ต COM ที่ CMUcam4 เชื่อมต่ออยู่จากรายการดรอปดาวน์

    • มองหาพอร์ตที่ชื่อ /dev/tty/USB###

  6. ไปที่ Communicate แล้วเลือก Connect, Terminal Echo และ Reset Propeller จากเมนูดรอปดาวน์

    • ไปที่ Communicate แล้วเลือก Display ASCII สำหรับเอาต์พุต ASCII หรือ Display Hex สำหรับเอาต์พุต hex

หมายเหตุ: คุณไม่สามารถใช้สาย FTDI 5V กับ BST ได้!

นี่เป็นเพราะ BST ดึงสาย RTS สีเขียวลงต่ำบน FTDI 5V Cable เมื่อคุณคลิก Connect. สิ่งนี้ทำให้ CMUcam4 หยุดทำงานอย่างไม่มีกำหนด คุณสามารถถอดสาย RTS สีเขียวออกจากขั้วต่อเพื่อแก้ปัญหานี้ได้

ขั้นตอนถัดไป#

ขั้นตอนถัดไปของคุณ (หลังจากดู แกลเลอรี) คือไปที่เว็บเพจเอกสาร นอกจากนี้ ตรวจสอบให้แน่ใจว่าได้ไปที่เว็บเพจ คำอธิบายการติดตามสี, เคล็ดลับและเทคนิค, คำถามที่พบบ่อย และ การแก้ไขปัญหา ของเราสำหรับข้อมูลเพิ่มเติม สุดท้าย ไปที่เว็บเพจ Arduino Interface Library และลองดู! จากนั้นคุณก็พร้อมที่จะสร้างหุ่นยนต์การมองเห็นของคุณเอง!
