ไม่มี

บันทึกภาพที่อยู่ในบัฟเฟอร์ภาพปัจจุบันไปยังชื่อไฟล์ที่ส่งเข้ามา ชื่อไฟล์ต้องเป็นชื่อไฟล์ FAT16 ที่ถูกต้อง <8.3> ไม่เช่นนั้นจะส่งคืนข้อผิดพลาด หากมีไฟล์อยู่แล้วด้วยชื่อไฟล์ที่กำหนด ไฟล์นั้นจะถูกเขียนทับ โปรดทราบว่าฟังก์ชันนี้ไม่ได้ถ่ายภาพเอง - คุณต้องเรียก take_picture ก่อนที่จะเรียก save_picture

pixel_new#

pixel_new()

พารามิเตอร์:

ไม่มี

ค่าที่ส่งคืน:

โครงสร้างข้อมูล pixel ใหม่

สร้างและส่งคืนโครงสร้างข้อมูล pixel ใหม่ ดู เอกสาร pixel สำหรับรายละเอียดเกี่ยวกับโครงสร้าง ดู pixbuf_get_pixel และ get_pixel ด้วยสำหรับการใช้งานโครงสร้าง pixel

get_pixel#

get_pixel(pixel pxl, integer x, integer y)

พารามิเตอร์:

pxl โครงสร้างข้อมูล pixel ที่จะเก็บค่าพิกเซล x, y พิกัด x และ y ของพิกเซลที่จะดึงภายในบัฟเฟอร์ภาพ

ค่าที่ส่งคืน:

ไม่มี ค่าพิกเซลถูกเก็บใน pixel ที่ส่งเข้ามา

รับ pixel เดียวจากภายในบัฟเฟอร์ภาพของ CMUcam3 (pixbuf) x และ y ต้องน้อยกว่า pixbuf_get_width และ pixbuf_get_height ตามลำดับ หากยังไม่ได้เรียก take_picture เพื่อเก็บภาพไว้ใน pixbuf ฟังก์ชันนี้จะไม่ล้มเหลว แต่ค่าที่ใส่ลงใน pixel จะเป็นข้อมูลขยะ

image_new#

image_new(integer width, integer height)

พารามิเตอร์:

width ความกว้างของภาพ, height ความสูงของภาพ

ค่าที่ส่งคืน:

โครงสร้างข้อมูล image ใหม่

รับโครงสร้างข้อมูล image ใหม่ พร้อมความกว้าง ความสูง และ pixsize ที่ส่งเข้ามา ดูเอกสาร image สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการใช้โครงสร้างข้อมูล image

รับ pixel จากโครงสร้างข้อมูล image ที่ส่งเข้ามา ค่า RGB ของพิกเซลจะถูกเก็บไว้ในโครงสร้างข้อมูล pixel ที่ส่งเข้ามา พารามิเตอร์ x ต้องน้อยกว่าความกว้างของ image และ y ต้องน้อยกว่าความสูงของ image

framediff_new#

framediff_new(integer template_width, integer template_height)

พารามิเตอร์:

template_width ความกว้างของเทมเพลตที่เรากำลัง diff, template_height ความสูงของเทมเพลต

ค่าที่ส่งคืน:

โครงสร้างข้อมูล framediff ใหม่

รับโครงสร้างข้อมูล framediff ใหม่ พร้อมความกว้างและความสูงของเทมเพลตที่ส่งเข้ามา โครงสร้าง framediff มักใช้เพื่อประเมินความแตกต่างระหว่างเฟรมภาพสองเฟรมที่ถ่ายด้วย CMUcam3 แต่โปรดดูเอกสาร framediff สำหรับข้อมูลเพิ่มเติม

framediff_scanline_start#

framediff_scanline_start(framediff fd)

พารามิเตอร์:

framediff_scanline#

framediff_scanline(image img, framediff fd)

พารามิเตอร์:

รับพิกเซลทั้งหมดจาก img และโหลดลงในช่องที่เหมาะสมในโครงสร้าง framediff เพื่อให้สามารถเปรียบเทียบเทมเพลตทั้งสอง (current_template และ previous_template) ได้ หาก fd:get_load_frame เป็นจริง จะโหลดพิกเซลใน img ลงในช่องเทมเพลตก่อนหน้า หาก fd:get_load_frame เป็นเท็จ จะโหลดพิกเซลจาก img ลงใน current_template

พารามิเตอร์:

ออบเจกต์ tracker ที่ใช้ในการติดตามสี ซึ่งตั้งค่าด้วยค่า RGB ที่ต้องการตั้ง

ค่าที่ส่งคืน:

บูลีนที่บ่งชี้ว่าการติดตามสามารถเกิดขึ้นได้หรือไม่ ส่งกลับ true แม้ว่าจะไม่พบสี -- ส่งกลับ false ก็ต่อเมื่อเกิดข้อผิดพลาดเท่านั้น

ติดตามสีใน CMUcam3 โดยใช้ออบเจกต์ tracker ที่ส่งเข้ามา ต้องเรียก take_picture() ก่อนพยายามติดตามสี วิเคราะห์บัฟเฟอร์ภาพและตั้งค่าในออบเจกต์ tracker เพื่อให้ได้กล่องขอบเขตสำหรับสีและค่า (x, y) ที่แสดงถึงเซนทรอยด์ของสี ดู เอกสาร tracker สำหรับรายละเอียดเพิ่มเติม
