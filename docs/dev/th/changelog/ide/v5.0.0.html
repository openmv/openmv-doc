v5.0.0#

v5.0.0 เป็นรุ่นใหญ่ นำเสนอ OpenMV Viewer — บิลด์แบบอ่านอย่างเดียวที่แยกต่างหากสำหรับการแสดงผลิตภัณฑ์ที่เสร็จสมบูรณ์ — OpenMV Cam Settings Editor ใหม่ โปรโตคอลดีบักกล้อง v2 สำหรับกล้อง v5.0.0 และชุดการปรับปรุงเวิร์กโฟลว์ตัวแก้ไข เทอร์มินัล เฟิร์มแวร์ และแมชชีนวิชันจำนวนมาก สร้างบน Qt Creator 14.0.2

การกระโดดเวอร์ชันไปที่ 5.0.0 บ่งบอกถึงการมาถึงของกล้องรุ่น v5.0.0 และเครื่องมือที่รองรับ โปรเจกต์และเวิร์กโฟลว์ v4 ที่มีอยู่ยังคงทำงานได้เหมือนเดิม

จุดเด่น#

  • OpenMV Viewer — บิลด์แบบใหม่ที่ติดตั้งแยกต่างหาก ซึ่งรัน IDE เป็นตัวแสดงข้อมูลเทเลเมทรีแบบอ่านอย่างเดียวที่ถูกล็อก แสดงเฟรมบัฟเฟอร์ เทอร์มินัลอนุกรม และฮิสโทแกรมของกล้อง และสามารถรันสคริปต์ที่กำหนดไว้ได้ แต่ซ่อนเครื่องมือสร้างงาน AI/โมเดล และการตั้งค่า ลูกค้าจึงสามารถสาธิตผลิตภัณฑ์ที่สร้างบน OpenMV ได้โดยไม่ต้องใช้สภาพแวดล้อมการพัฒนาแบบเต็ม

  • OpenMV Cam Settings Editor — ตัวแก้ไขแบบกราฟิกสำหรับไฟล์กำหนดค่าการตั้งค่าบนบอร์ดของกล้อง คุณจึงดูและเปลี่ยนการตั้งค่าบอร์ดจากฟอร์มแทนการแก้ไขไฟล์ด้วยมือ

  • โปรโตคอลดีบักกล้อง v2 — กล้อง v5.0.0 ใช้โปรโตคอลดีบักใหม่ที่ทนทานยิ่งขึ้น พร้อมการตรวจสอบ CRC ส่วนหัว การกู้คืนช่องว่างลำดับ และการแสดงอัตราเฟรมแบบแยกที่รายงาน FPS บนกล้องและ FPS การแสดงผลของ IDE แยกกัน

  • โหลดเฟิร์มแวร์กำหนดเองจาก .zip — ตัวติดตั้งเฟิร์มแวร์ตอนนี้รับบันเดิล .zip ที่เป็นผลลัพธ์การบิลด์ของบอร์ดได้โดยตรง ทำให้แฟลชอิมเมจเฟิร์มแวร์แบบกำหนดเองหรือที่บิลด์ในเครื่องได้ง่าย

  • การบันทึกดีบักโปรโตคอลในเทอร์มินัลอนุกรม — ปุ่มสลับในเทอร์มินัลอนุกรมจะบันทึกโปรโตคอลดีบักกล้องด้วยบรรทัดที่อ่านได้และติดแท็กอีโมจิ สำหรับวินิจฉัยปัญหาการเชื่อมต่อ

  • ปรับปรุงเมนู Help และเอกสารในตัวแก้ไข — เมนูย่อย OpenMV Docs เข้ามาแทนที่ลิงก์แบบเรียบเดิม F1 เปิดเอกสารสำหรับสัญลักษณ์ Python ที่อยู่ใต้เคอร์เซอร์ การวางเมาส์เหนือชื่อ API จะแสดงเอกสารแบบอินไลน์ และกล่องโต้ตอบ About ได้รับการปรับให้ทันสมัย

คุณสมบัติใหม่#

  • บิลด์แบบ OpenMV Viewer บิลด์ที่บังคับโหมด viewer สร้างแบรนด์และติดตั้งเป็น OpenMV Viewer พร้อม id ของไฟล์ปฏิบัติการ ที่เก็บการตั้งค่า และ bundle id ของตัวเอง จึงอยู่ร่วมกับ IDE แบบเต็มได้ นำเสนอเป็นตัวแสดงเทเลเมทรีแบบอ่านอย่างเดียว: Run/Stop และ stop-on-connect ทำงานและรันสคริปต์จากดิสก์ (ผ่าน -auto_run) เมนู File/Edit และ Preferences ยังคงอยู่ และการกู้คืนเฟิร์มแวร์/ROMFS ถูกเก็บไว้ ในขณะที่เครื่องมือสร้างงาน เครื่องมือแมชชีนวิชันสำหรับนักพัฒนา หมวดการตั้งค่าสำหรับนักพัฒนา และทรัพยากร AI/โมเดล/เอกสารถูกตัดออกจากบิลด์และ UI ROMFS กลายเป็นโหลดอย่างเดียวและถูกเลื่อนไปยังเมนู Tools และกล่องโต้ตอบการอัปเดตเฟิร์มแวร์เสร็จสมบูรณ์ถูกทำให้เป็นแบบทั่วไป (07722e78a8b, 5cd12ee260e, b97bf5447e4, 952b86de391, 1103f7988ed, f0f60261485, c743d5d67b9, b4bcdc2d71a, a37c032b6ff, 480e5668d29, 11c064cac93)

  • OpenMV Cam Settings Editor — ตัวแก้ไข GUI สำหรับไฟล์กำหนดค่าการตั้งค่าของกล้อง เปิดจากเมนู Tools (issue #205) (f55a0d7fa58, 2d7f80b03cf)

  • โปรโตคอลดีบักกล้อง v2 สำหรับกล้อง v5.0.0 พร้อมการตรวจสอบ CRC ส่วนหัวก่อนที่จะเชื่อถือฟิลด์ส่วนหัวใด ๆ การกู้คืนช่องว่างลำดับที่เลื่อนไปข้างหน้าข้ามแพ็กเก็ตที่หายไปแทนการค้าง และการอ่านอัตราเฟรมที่แยกเป็นอัตราบนกล้องและอัตราการแสดงผลของ IDE (c377bdbaea8, a3f0488ddcb, 29108dd40f7, c15e6c02789)

  • โหลดเฟิร์มแวร์กำหนดเองรับบันเดิล .zip — เลือก zip ที่เป็นผลลัพธ์การบิลด์ของบอร์ด แล้ว IDE จะแฟลชเฟิร์มแวร์ (และ ROMFS) ที่อยู่ในนั้น (55e4eb2d900)

  • การบันทึกดีบักในเทอร์มินัลอนุกรม — ปุ่มสลับการบันทึกดีบักแบบหลายระดับของโปรโตคอลกล้อง โดยแต่ละบรรทัดติดแท็กอีโมจิและแสดงในฟอนต์อีโมจิเพื่อให้อ่านง่าย (09990c62810, 42e23c3c11e, 997705548c7, d2208de77d4)

  • Copy/Convert Current Script — การกระทำในเมนู Tools ที่คัดลอกหรือแปลงสคริปต์ปัจจุบัน พร้อมการป้องกันการบันทึก main.py/boot.py ที่คอมไพล์แล้ว (81a954f33cd)

  • ความช่วยเหลือเอกสาร F1 บนสัญลักษณ์ Python ใต้เคอร์เซอร์ ใช้ตัวแก้ปัญหาสัญลักษณ์ร่วมกับคำแนะนำเมื่อวางเมาส์ (c341330bc16)

  • คำแนะนำเอกสารในตัวแก้ไข — การวางเมาส์เหนือชื่อ API ของ OpenMV จะแสดงเอกสารแบบอินไลน์ ซึ่งดึงมาจากสตับ .pyi ที่มาพร้อมกันและเอกสารของโมดูลเอง ทำให้คุณอ่านได้ว่าฟังก์ชัน เมธอด ค่าคงที่ หรือโมดูลทำอะไรได้โดยไม่ต้องออกจากสคริปต์ (7ebde92d2a6, f225824ba13, 59ad01a364a, 02c1e61a2a6).

  • ตั้งชื่อกล้อง — ชื่อแทนเฉพาะใน IDE ที่คุณตั้งให้กล้องที่เชื่อมต่อได้ แสดงแทนชื่อพอร์ตอนุกรมเพื่อให้แยกกล้องที่เหมือนกันออกจากกันได้ (c600158ca34)

  • หน้าแป้นพิมพ์ลัด — หน้าแป้นพิมพ์ลัดใน Preferences ตอนนี้แสดงและกรองเฉพาะการกระทำที่คุณเข้าถึงได้จริง พร้อมแหล่งที่มาและเลย์เอาต์ที่ชัดเจนขึ้น (4f5eae52e8e, 2d472fe5b40)

การปรับปรุงตัวแก้ไข เทอร์มินัล และ UI#

  • เมนู Help ได้รับการจัดโครงสร้างใหม่ เป็นเมนูย่อย OpenMV Docs (ลบลิงก์ pinout แบบเรียบเดิมออก) พร้อมเพิ่มลิงก์บันทึกรุ่นในเส้นทางเวอร์ชันของรุ่นที่เผยแพร่อย่างเป็นทางการ (e1337ff67c4, 9eacc0994e6)

  • กล่องโต้ตอบ About ได้รับการปรับให้ทันสมัย — ปีลิขสิทธิ์อัตโนมัติ ตารางโลโก้พันธมิตร และลิงก์บันทึกรุ่นแบบบรรทัดเดียว (d89fb74138d)

  • แผง Frame Buffer ตอนนี้แสดงการอ่านความละเอียดและป้ายโหมดเฟรมบัฟเฟอร์ข้างปุ่ม JPG ที่ตายตัว ป้ายโหมดเป็นสีเทาจึงดูไม่เหมือนคลิกได้ (cb735f82337, 376b17d4e77, 39c6ffc3b26)

  • บานหน้าต่างฮิสโทแกรม พล็อตถูกปรับให้เรียบแบบสไปลน์ บานหน้าต่างสามารถบีบอัดได้ (ย่อป้ายและพล็อตที่บีบได้) และป้ายช่องสัญญาณอยู่ข้างสถิติ (327324ef7e9, 9f8c7035483, d204e93d5e4)

  • ตัวแบ่งแผงที่เชื่อมกัน — การลากตัวแบ่งด้วย Ctrl/Cmd จะเลื่อนตัวแบ่งตัวแก้ไข/เทอร์มินัลและเฟรมบัฟเฟอร์/ฮิสโทแกรมไปพร้อมกัน และที่จับตัวแบ่งขนาดเล็กจะมีรัศมีเมื่อวางเมาส์พร้อมพื้นที่จับที่กว้างขึ้น (379dee250d6, 96b83984342)

  • เทอร์มินัลอนุกรม — การเลื่อนอัตโนมัติจะไม่ถูกตรึงใหม่ทุกครั้งที่มีชิ้นเอาต์พุต การเลื่อนขึ้นระหว่างเอาต์พุตที่เร็วจึงอยู่กับที่ เทอร์มินัลสามารถรันไฟล์จากดิสก์ได้เมื่อไม่มีตัวแก้ไข (af67a076e95, bc132aef536)

  • คำสั่งเอกสาร ถูกตั้งชื่อตามหน้าแทนตามดัชนี (a3201e9a4b4)

  • High-DPI / การปรับขนาดแบบเศษส่วน — ไอคอน โอเวอร์เลย์ และหน้าจอเริ่มต้นแสดงผลได้อย่างถูกต้องที่ตัวคูณขนาดแบบเศษส่วน (125%, 150% และอื่น ๆ ทำนองนี้) และหน้าต่างหลักจะจดจำเรขาคณิตและหน้าจอของมันเมื่อรีสตาร์ทในการตั้งค่าแบบหลายจอภาพ (c60711e4950, 60901b6e13f, f241d2ea8ce, 54a1bc31fbd).

เฟิร์มแวร์ บอร์ด และเครื่องมือ#

  • บอร์ดสามารถตรึงเวอร์ชันเฟิร์มแวร์ที่จัดส่งล่าสุดได้ ดังนั้นบอร์ดที่ไม่ได้รับเฟิร์มแวร์ใหม่กว่าอีกต่อไปจะไม่ถูกทำเครื่องหมายว่าล้าสมัย (bd987d32f42)

  • เฟิร์มแวร์ที่ล้าสมัย ข้อความสถานะตอนนี้เป็นสีแดงเพื่อให้โดดเด่น (9b18d96bac5)

  • คำแนะนำการกู้คืน bootloader — ความไม่ตรงกันของ bootloader.bin ตอนนี้จะแนะนำให้เข้าสู่โหมดกู้คืน bootloader (d46b998c76e)

  • การเล่นวิดีโอ ทนทานยิ่งขึ้น: เส้นทาง FFmpeg จะแสดงเมื่อเครื่องมือเปิดใช้ไม่ได้ ข้อความชัดเจนขึ้นเมื่อสถาปัตยกรรมไม่รองรับการเล่น รูปแบบอินพุตถูกสอบถามด้วย ffmpeg -demuxers และเพิ่มการถอยกลับของเทอร์มินัล Linux (0cc0e74a438, a75ec4898d0, 28e1ef10326, 51651780723)

  • Dataset Capture Script อัปเดตเป็นโมดูล csi (41a781651e4)

  • ตัวติดตั้ง Windows ปรับปรุงรูปลักษณ์ของวิซาร์ด (250599d9d9e)

การแก้ไขข้อบกพร่อง#

  • การติดตั้งเฟิร์มแวร์สำหรับการพัฒนา สำหรับบอร์ด .lst (AE3) ตอนนี้จะจัดเตรียมไบนารีที่รายการอ้างอิงลงในไดเรกทอรีแฟลช และการติดตั้งแบบ dev/custom จะจัดเตรียมอิมเมจ ROMFS ของบันเดิลพร้อมล้าง romfs*.img ที่ค้างอยู่ก่อน ดังนั้น ROMFS ที่ดาวน์โหลดจะถูกแฟลชแทนดัมป์ของตัวแก้ไขที่หลงเหลืออยู่ (3cf56e98668, 43bedc8c591)

  • คำเตือน libjpeg สำหรับเฟรมที่ถูกตัดทอน/เสียหายถูกกรองออกจากบันทึก ดังนั้นสตรีมที่มีการสูญเสียจะไม่สแปมเทอร์มินัล (6e2a8bed6cf, a5166abbdc1)

  • รายการตัวอย่างจะไม่แสดง index.json ภายในอีกต่อไป (14ac8ca48a7)

  • ป๊อปอัปการอัปเกรดเฟิร์มแวร์และการเริ่มต้นใช้งานถูกระงับในโหมด viewer ดังนั้นการสาธิตผลิตภัณฑ์จะไม่แจ้งให้เขียนทับเฟิร์มแวร์ของลูกค้า (7b8528c3ad6)

  • แป้นพิมพ์ลัดที่ค้างอยู่และเฉพาะการสร้างงานที่หลงเหลือจากเมนูที่ถูกตัดออกถูกปิดใช้งาน (be3bee8bb58, 27d1afc8705)

การแปลภาษา#

  • สี่ภาษาใหม่ — อินโดนีเซีย โรมาเนีย ไทย และเวียดนาม — ทำให้ IDE มี 37 ภาษาที่แปลแล้ว และทุกแคตตาล็อกได้รับการอัปเดต แปล และทำความสะอาดในทุกภาษา (fcf04090843, c830e6c4de3, 5e22679923f)

บรรทัดคำสั่ง#

  • ชื่อแทนตัวเลือกแบบ GNU ``--word`` ตอนนี้ยอมรับควบคู่กับรูปแบบขีดเดียว และเอาต์พุต -help ระบุความเทียบเท่า (66414213dd9, 768c5d0e159)

การรองรับแพลตฟอร์มและเครื่องมือ#

  • ฐาน Qt Creator: 14.0.2.

  • OpenMV IDE เพิ่มเวอร์ชันเป็น 5.0.0

  • ใหม่: บิลด์ OpenMV Viewer ที่ติดตั้งแยกต่างหาก

  • รองรับ Python 3.14 — ตัวแยกวิเคราะห์ parso ที่มาพร้อมกันได้รับการอัปเดตเป็น 0.8.7 เพื่อให้การเติมโค้ดอัตโนมัติและ language server ทำงานได้กับไวยากรณ์ Python ล่าสุด (578a4e39d97).

การเปลี่ยนแปลงที่ทำให้เข้ากันไม่ได้#

ไม่มีสำหรับโปรเจกต์ที่มีอยู่ โปรโตคอลดีบัก v2 ใช้กับกล้อง v5.0.0 กล้องรุ่น v4 ยังคงใช้โปรโตคอลที่มีอยู่ และสคริปต์และเวิร์กโฟลว์ที่มีอยู่ไม่ได้รับผลกระทบ
