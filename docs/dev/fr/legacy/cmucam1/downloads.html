Téléchargements#

Manuel de l’utilisateur#

Le manuel d'utilisation de la CMUcam1 v2.00 pour le micrologiciel v1.12 est disponible au format PDF. Il contient une liste des pièces, le schéma de la carte et une description du protocole logiciel. Les manuels des versions plus anciennes du micrologiciel se trouvent ici : v1.11, v1.1, v1.0.

Si vous décidez d’expérimenter la modification des réglages des registres de la caméra, vous pourriez aussi être intéressé par le téléchargement du manuel de la caméra OV6620 depuis OmniVision.

Construction de la carte#

Si vous la construisez vous-même à partir de zéro, vous devrez télécharger ce fichier de code HEX (contenant le dernier micrologiciel v1.12) pour programmer la puce SX28. C’est le code qui implémente la fonctionnalité et le protocole série décrits dans le manuel. Vous pouvez obtenir le téléchargeur et l’application SX-Key auprès de Parallax Inc. Les versions plus anciennes du micrologiciel se trouvent ici : v1.11, v1.1, v1.0.

Programme d’interface Java#

Nous avons un programme java open source que vous pouvez télécharger pour communiquer avec votre CMUcam1 et la tester. Ce programme d’interface vous permet de suivre des objets, d’analyser des données et de capturer des images complètes de la CMUcam1. Voici des captures d’écran de cette interface.

Voici le projet Java compressé écrit pour Linux.

Voici la version de l'interface graphique qui détecte automatiquement votre système d’exploitation. Elle fonctionne sous les systèmes UNIX qui prennent en charge les E/S série comme s’il s’agissait d’un fichier (tels que Linux), ainsi que sous Windows 95/98/2000/NT/XP. Nous recommandons d’utiliser au moins un processeur Pentium 2 400 MHz exécutant Java 1.2.2 ou une version supérieure.

Pour voir des exemples de code C pour le microprocesseur Cerebellum et la Handyboard BotBall communiquant avec la CMUcam1, rendez-vous sur cette page web.
