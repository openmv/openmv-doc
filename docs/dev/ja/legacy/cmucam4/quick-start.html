クイックスタート#

はじめに#

以下の手順に従ってください：

  1. CMUcam4 をテストするには、以下の2つのアイテムが必要です

    • 少なくとも 250 mA を供給できる 4V～9V DC の外部電源

      • これは Arduino、FTDI Breakout Board、FTDI Cable、または Wall Wart からのものでかまいません

    • NTSC TV（または互換のテレビモニタ）と RCAケーブル

  2. CMUcam4 をテストするためのセットアップ

    • RCAケーブル を使用して NTSC TV をCMUcam4の RCA Coaxial Jack に接続します

    • 外部電源をCMUcam4の DC Barrel Jack（または他の電源ポート）に接続します

外部電源を CMUcam4 に接続すると、緑の電源 LED が点灯するはずです。約 2秒 後、CMUcam4 が動作可能であることを示すために、赤い補助 LED が点灯するはずです。緑の電源 LED が点灯しない場合は、外部電源の接続をもう一度確認してください。

テスト手順#

以下の手順に従ってください：

  1. CMUcam4の reset button を押し続けます

  2. CMUcam4の user button を押し続けます

  3. reset button を離します（user button は離さないでください）

  4. 赤い補助 LED が点灯するまで待ちます（2秒）

  5. 赤い補助 LED が 10 Hz で点滅し始めるまで待ってから、ユーザーボタン を離します

    • テレビがオンになるはずです（テレビにスプラッシュ画面が表示されるはずです）

  6. CMUcam4 はこれから 5秒間 照明条件に調整します

    • 次の 5秒間 は、トラッキングしたい物体をCMUcam4の前に置かないでください

  7. 赤い補助 LED が 10 Hz での点滅を停止するまで待ちます

    • CMUcam4 は照明条件への調整を完了しました

    • パンとチルトのサーボピンは 50 Hz で 1500 μs のパルスを出力するはずです

  8. トラッキングしたい物体をCMUcam4の前に置き、user button を押します

    • 赤色の補助LEDが 10 Hz で点滅し始めたら、OV9665 カメラモジュールの接続を調べます

      • OV9665 カメラモジュールが破損している可能性があり、おそらく交換が必要です

  9. 追跡された物体（または類似のもの）がテレビに表示されるはずです。

    • 接続されていれば、パンとチルトのサーボも追跡対象の物体に向かってカメラを駆動しようとします

  10. さまざまな環境でさまざまな物体を使ってこの手順を試し、何が最もうまく機能するか確認してください

テストガイドをPDF形式で こちら からダウンロードします。

通信ツール#

CMUcam4 と通信するために、以下（または類似）の USB-シリアルコンバータ のいずれかが必要です:

  • FTDI 5V Breakout Board

    • また、CMUcam4 に電力を供給できる外部電源も必要になります

    • FTDI 5V Breakout Board をCMUcam4の 6-pin コネクタに接続してください

  • FTDI 3.3V Breakout Board

    • また、CMUcam4 に電力を供給できる外部電源も必要になります

    • FTDI 3.3V Breakout Board をCMUcam4の 6-pin コネクタに接続してください

  • Prop Clip 1つ

    • また、CMUcam4 に電力を供給できる外部電源も必要になります

    • Prop Clip をCMUcam4の 4-pin コネクタに接続してください

  • Prop Plug 1つ

    • また、CMUcam4 に電力を供給できる外部電源も必要になります

    • Prop Plug をCMUcam4の 4-pin コネクタに接続してください

  • FTDI 5V Cable w/ 5V I/O

    • 5Vトレラントシステムに推奨

    • FTDI 5V Cable w/ 5V I/O をCMUcam4の 6-pin コネクタに接続してください

  • FTDI 5V Cable w/ 3.3V I/O

    • 3.3Vトレラントシステムに推奨

    • FTDI 5V Cable w/ 3.3V I/O をCMUcam4の 6-pin コネクタに接続してください

推奨シリアルターミナルプログラム#

Parallax Serial Terminal は、Parallax Propeller チップなどのシリアルベースのマイクロコントローラと通信するための便利なツールです。カメラボードとの通信に使用する推奨シリアルターミナルです。Parallax Serial Terminal（PST）は 1 MB 未満のスタンドアロンアプリケーションで、使用にインストールは不要です。PST は Parallax Inc. から こちら でダウンロードできます。PST をセットアップするには、以下の手順に従ってください:

  1. PST を実行します

  2. Echo On に移動し、チェックされていることを確認します

  3. Com Port に移動し、ドロップダウンリストから CMUcam4 が接続されているCOMポートを選択します

  4. Baud Rate に移動し、ドロップダウンリストから 19200 を選択します

    • 必要に応じて Enable をクリックします

Windows以外のユーザーには、Brad's SPIN Tool (BST)を推奨します。BSTは こちら からダウンロードできます。BSTは、Parallax Propellerチップ向けに設計された10 MB未満のスタンドアロンアプリケーションで、グラフィカルユーザーインターフェース (GUI) 統合開発環境 (IDE) であり、使用にインストールを必要としません。BSTには、使いやすいシリアルターミナルが組み込まれています。BSTの組み込みの使いやすいシリアルターミナルをセットアップするには、以下の手順に従ってください：

  1. BST を実行します

  2. View に移動し、ドロップダウンリストから Serial Terminal を選択します

    • bst ターミナル がポップアップするはずです — それをクリックしてください

  3. Baud に移動し、ドロップダウンリストから 19200 を選択します

  4. Format に移動し、ドロップダウンリストから 8 Bits と Parity None を選択します

  5. Port に移動し、ドロップダウンリストから CMUcam4 が接続されているCOMポートを選択します

    • /dev/tty/USB### という名前のポートを探します

  6. Communicate に移動し、ドロップダウンメニューから Connect、Terminal Echo、Reset Propeller を選択します

    • Communicate に移動し、ASCII出力の場合は Display ASCII を、hex出力の場合は Display Hex を選択します

注意：FTDI 5VケーブルはBSTと一緒に使用できません！

これは、 Connect をクリックしたときに BST が FTDI 5V Cable の緑のRTS線をLowに引き下げるためです。 これにより CMUcam4 が無期限に停止します。この問題を解決するには、コネクタから緑のRTS線を取り外してください。
