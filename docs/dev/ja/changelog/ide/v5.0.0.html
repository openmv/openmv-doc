v5.0.0#

v5.0.0 はメジャーリリースです。OpenMV Viewer**（完成した製品を披露するための、独立した読み取り専用ビルド）、新しい **OpenMV Cam Settings Editor、v5.0.0 カメラ向けの v2 カメラデバッグプロトコル、そしてエディタ・ターミナル・ファームウェア・マシンビジョンのワークフローに関する多数の改善を導入します。Qt Creator 14.0.2 をベースに構築されています。

5.0.0 へのバージョンアップは、v5.0.0 世代のカメラとそれをサポートするツール群の登場を示します。既存の v4 プロジェクトやワークフローはこれまでどおり動作し続けます。

ハイライト#

  • OpenMV Viewer — IDE をロックダウンされた読み取り専用のテレメトリビューアとして実行する、新しい別インストールのビルドバリアントです。カメラのフレームバッファ、シリアルターミナル、ヒストグラムを表示し、固定のスクリプトを実行できますが、オーサリング、AI/モデル、設定のツールは隠されているため、顧客は完全な開発環境なしで OpenMV 上に構築された製品をデモできます。

  • OpenMV Cam Settings Editor — カメラのオンボード設定構成ファイル用のグラフィカルエディタで、ファイルを手作業で編集する代わりにフォームからボード設定を表示・変更できます。

  • v2 カメラデバッグプロトコル — v5.0.0 カメラは、ヘッダー CRC 検証、シーケンスギャップ回復、オンカメラの FPS と IDE 表示の FPS を別々に報告する分割フレームレート表示を備えた、より堅牢な新しいデバッグプロトコルを使用します。

  • .zip からのカスタムファームウェア読み込み — ファームウェアインストーラがボードのビルド出力 .zip バンドルを直接受け付けるようになり、カスタムまたはローカルでビルドしたファームウェアイメージを簡単に書き込めます。

  • シリアルターミナルのプロトコルデバッグログ — シリアルターミナルのトグルで、接続の問題を診断するために、読みやすい絵文字タグ付きの行でカメラデバッグプロトコルをログに記録します。

  • ヘルプメニューとエディター内ドキュメントの刷新 — OpenMV Docs サブメニューが従来のフラットなリンクを置き換え、F1 でカーソル位置の Python シンボルのドキュメントが開き、API 名にカーソルを合わせるとそのドキュメントがインラインで表示され、About ダイアログが刷新されました。

新機能#

  • OpenMV Viewer ビルドバリアント。 OpenMV Viewer としてブランド化・インストールされる強制ビューアモードのビルドで、独自の実行ファイル ID、設定ストア、バンドル ID を持つため、完全な IDE と共存します。読み取り専用のテレメトリビューアとして表示されます: 実行/停止と接続時停止が機能し、ディスクからスクリプトを実行し（-auto_run 経由）、ファイル/編集メニューと設定は残り、ファームウェア/ROMFS の回復も維持されます。一方で、オーサリングツール、開発者向けマシンビジョンツール、開発者向け設定カテゴリ、AI/モデル/ドキュメントのリソースはビルドと UI から除去されます。ROMFS は読み込み専用となり Tools メニューに移動され、ファームウェア更新完了ダイアログは汎用化されます (07722e78a8b, 5cd12ee260e, b97bf5447e4, 952b86de391, 1103f7988ed, f0f60261485, c743d5d67b9, b4bcdc2d71a, a37c032b6ff, 480e5668d29, 11c064cac93)。

  • OpenMV Cam Settings Editor — カメラの設定構成ファイル用の GUI エディタで、Tools メニューから開きます（issue #205）(f55a0d7fa58, 2d7f80b03cf)。

  • v2 カメラデバッグプロトコル — v5.0.0 カメラ向けで、いずれのヘッダーフィールドも信頼する前にヘッダー CRC を検証し、パケットのドロップ時に停止せず前方へロールフォワードするシーケンスギャップ回復を備え、フレームレート表示をオンカメラのレートと IDE 表示のレートに分割します (c377bdbaea8, a3f0488ddcb, 29108dd40f7, c15e6c02789)。

  • カスタムファームウェア読み込みが .zip バンドルに対応 — ボードのビルド出力 zip を選択すると、IDE はそこに含まれるファームウェア（と ROMFS）を書き込みます (55e4eb2d900)。

  • シリアルターミナルのデバッグログ — ボタンでカメラプロトコルの段階的なデバッグログを切り替えます。各行は絵文字タグ付きで、読みやすさのために絵文字フォントで描画されます (09990c62810, 42e23c3c11e, 997705548c7, d2208de77d4)。

  • 現在のスクリプトのコピー/変換 — 現在のスクリプトをコピーまたは変換する Tools アクションで、コンパイル済みの main.py/boot.py の保存を防ぐガードを備えています (81a954f33cd)。

  • F1 ドキュメントヘルプ — カーソル下の Python シンボルに対して、ホバーツールチップとシンボルリゾルバを共有します (c341330bc16)。

  • エディター内ドキュメントツールチップ — OpenMV の API 名にカーソルを合わせると、同梱の .pyi スタブとモジュール自身のドキュメントから解決されたドキュメントがインラインで表示され、スクリプトを離れることなく関数、メソッド、定数、モジュールの機能を確認できます (7ebde92d2a6, f225824ba13, 59ad01a364a, 02c1e61a2a6)。

  • カメラに名前を付ける — 接続したカメラに付けられる IDE ローカルの別名で、シリアルポート名の代わりに表示されるため、同一のカメラを区別できます (c600158ca34)。

  • キーボードショートカットページ — 設定のショートカットページが表示されるようになり、実際に到達できるアクションに絞り込まれ、出所とレイアウトがより明確になりました (4f5eae52e8e, 2d472fe5b40)。

エディタ、ターミナル、UI の改善#

  • ヘルプメニューを再構成 して OpenMV Docs サブメニューにまとめ（従来のフラットなピンアウトリンクを廃止）、公式リリースのバージョンパスにリリースノートへのリンクを追加しました (e1337ff67c4, 9eacc0994e6)。

  • バージョン情報ダイアログの刷新 — 自動の著作権年、パートナーロゴの表、そして 1 行のリリースノートリンク (d89fb74138d)。

  • フレームバッファパネル は、固定の JPG ボタンの横に解像度表示とフレームバッファモードのラベルを表示するようになりました。モードラベルはクリック可能に見えないようグレー表示されます (cb735f82337, 376b17d4e77, 39c6ffc3b26)。

  • ヒストグラムペイン のプロットはスプライン平滑化され、ペインは圧縮でき（ラベルを省略し、プロットを縮小可能）、チャンネルラベルは統計値の横に配置されます (327324ef7e9, 9f8c7035483, d204e93d5e4)。

  • 連動するパネル分割線 — Ctrl/Cmd を押しながら分割線をドラッグすると、エディタ/ターミナルとフレームバッファ/ヒストグラムのスプリッタが一緒に動き、ミニスプリッタのハンドルはより広いつかみ領域を持つホバーハローが付きます (379dee250d6, 96b83984342)。

  • シリアルターミナル — 自動スクロールが出力チャンクごとに再固定されなくなったため、高速出力中に上へスクロールしてもその位置が保たれます。エディタがない場合、ターミナルはディスクからファイルを実行できます (af67a076e95, bc132aef536)。

  • ドキュメントコマンド がインデックスではなくページ名で名付けられるようになりました (a3201e9a4b4)。

  • 高 DPI / 分数スケーリング — アイコン、オーバーレイ、スプラッシュ画面が分数スケール係数 (125%、150% など) で正しく描画され、マルチモニター構成では再起動後もメインウィンドウがジオメトリと表示画面を記憶します (c60711e4950, 60901b6e13f, f241d2ea8ce, 54a1bc31fbd)。

ファームウェア、ボード、ツール#

  • ボードは最後に出荷されたファームウェアバージョンを固定できる ようになり、新しいファームウェアが提供されなくなったボードが期限切れとしてフラグ付けされなくなりました (bd987d32f42)。

  • 期限切れのファームウェア のステータステキストが、目立つように赤色で表示されるようになりました (9b18d96bac5)。

  • ブートローダー回復のヒント — bootloader.bin の不一致時に、ブートローダー回復モードへの移行を示唆するようになりました (d46b998c76e)。

  • 動画再生 がより堅牢になりました: ツールが FFmpeg を起動できない場合にそのパスが表示され、アーキテクチャが再生に対応していない場合のメッセージが明確になり、入力フォーマットは ffmpeg -demuxers で照会され、Linux ターミナルのフォールバックが追加されました (0cc0e74a438, a75ec4898d0, 28e1ef10326, 51651780723)。

  • データセットキャプチャスクリプト が csi モジュールに更新されました (41a781651e4)。

  • Windows インストーラ のウィザードの外観が改善されました (250599d9d9e)。

バグ修正#

  • 開発ファームウェアのインストール — .lst ボード（AE3）では、リストが参照するバイナリをフラッシュディレクトリにステージングするようになりました。また、開発/カスタムインストールは、まず古い romfs*.img を消去してからバンドルの ROMFS イメージをステージングするため、残っていたエディタのダンプではなくダウンロードした ROMFS が書き込まれます (3cf56e98668, 43bedc8c591)。

  • libjpeg の警告 — 切り詰められた/破損したフレームに対する警告がログから除外され、ロスの多いストリームがターミナルを埋め尽くさないようになりました (6e2a8bed6cf, a5166abbdc1)。

  • サンプル一覧に内部の index.json が表示されなくなりました (14ac8ca48a7)。

  • ビューアモードではファームウェアアップグレードとオンボーディングのポップアップが抑制されるため、製品デモが顧客のファームウェアを上書きするよう促すことはありません (7b8528c3ad6)。

  • 除去されたメニューから残された、宙に浮いたショートカットやオーサリング専用のキーボードショートカットが無効化されました (be3bee8bb58, 27d1afc8705)。

ローカライゼーション#

  • 4 つの新しい言語 — インドネシア語、ルーマニア語、タイ語、ベトナム語 — により IDE は 37 の翻訳言語に対応し、すべての言語でカタログが更新・翻訳・クリーンアップされました (fcf04090843, c830e6c4de3, 5e22679923f)。

コマンドライン#

  • GNU スタイルの ``--word`` オプションエイリアス が、シングルダッシュ形式とともに受け付けられるようになり、-help 出力にその同等性が記載されます (66414213dd9, 768c5d0e159)。

プラットフォームとツールのサポート#

  • Qt Creator ベース: 14.0.2。

  • OpenMV IDE のバージョンを 5.0.0 に更新。

  • 新規: 別途インストールされる OpenMV Viewer ビルドバリアント。

  • Python 3.14 対応 — 同梱の parso パーサーが 0.8.7 に更新され、最新の Python 構文でコード補完と言語サーバーが引き続き動作します (578a4e39d97)。

破壊的変更#

既存のプロジェクトに対してはなし。v2 デバッグプロトコルは v5.0.0 カメラに適用されます。v4 世代のカメラは引き続き既存のプロトコルを使用し、既存のスクリプトやワークフローには影響しません。
