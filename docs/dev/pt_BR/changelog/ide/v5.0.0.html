v5.0.0#

v5.0.0 é uma versão principal. Ela introduz o OpenMV Viewer — uma compilação separada, somente leitura, para exibir um produto finalizado — um novo OpenMV Cam Settings Editor, o protocolo de depuração de câmera v2 para câmeras v5.0.0, e um grande conjunto de melhorias no editor, no terminal, no firmware e no fluxo de trabalho de visão de máquina. Ele é construído sobre o Qt Creator 14.0.2.

O salto de versão para 5.0.0 marca a chegada da geração de câmeras v5.0.0 e das ferramentas que a suportam; projetos e fluxos de trabalho v4 existentes continuam funcionando sem alterações.

Destaques#

  • OpenMV Viewer — uma nova variante de compilação, instalada separadamente, que executa o IDE como um visualizador de telemetria bloqueado e somente leitura. Ele mostra o buffer de quadros da câmera, o terminal serial e o histograma e pode executar um script fixo, mas oculta as ferramentas de autoria, de IA/modelo e de configurações, para que um cliente possa demonstrar um produto construído sobre o OpenMV sem o ambiente de desenvolvimento completo.

  • OpenMV Cam Settings Editor — um editor gráfico para o arquivo de configuração de ajustes embarcado da câmera, para que você possa visualizar e alterar as configurações da placa a partir de um formulário em vez de editar o arquivo manualmente.

  • Protocolo de depuração de câmera v2 — as câmeras v5.0.0 usam um novo protocolo de depuração mais robusto, com validação de CRC de cabeçalho, recuperação de lacunas de sequência e uma exibição de taxa de quadros dividida que reporta separadamente o FPS na câmera e o FPS de exibição do IDE.

  • Carregar Firmware Personalizado a partir de um .zip — o instalador de firmware agora aceita diretamente o pacote .zip de saída de compilação de uma placa, facilitando a gravação de uma imagem de firmware personalizada ou compilada localmente.

  • Registro de depuração do protocolo no Terminal Serial — uma opção no Terminal Serial registra o protocolo de depuração da câmera com linhas legíveis, marcadas com emoji, para diagnosticar problemas de conexão.

  • Menu de Ajuda reformulado e documentação no editor — um submenu OpenMV Docs substitui os antigos links planos, F1 abre a documentação do símbolo Python sob o cursor, passar o mouse sobre o nome de uma API mostra sua documentação embutida, e a caixa de diálogo Sobre foi modernizada.

Novos recursos#

  • Variante de compilação OpenMV Viewer. Uma compilação em modo visualizador forçado, com a marca e instalada como OpenMV Viewer, com seu próprio id de executável, armazenamento de configurações e id de pacote, para que coexista com o IDE completo. Ela se apresenta como um visualizador de telemetria somente leitura: Executar/Parar e parar ao conectar funcionam e executam um script a partir do disco (via -auto_run), os menus Arquivo/Editar e as Preferências permanecem, e a recuperação de firmware/ROMFS é mantida, enquanto as ferramentas de autoria, as ferramentas de visão de máquina para desenvolvedores, as categorias de configurações de desenvolvedor e os recursos de IA/modelo/documentação são removidos da compilação e da interface. O ROMFS torna-se somente carregamento e é promovido ao menu Ferramentas, e as caixas de diálogo de conclusão de atualização de firmware são genericizadas (07722e78a8b, 5cd12ee260e, b97bf5447e4, 952b86de391, 1103f7988ed, f0f60261485, c743d5d67b9, b4bcdc2d71a, a37c032b6ff, 480e5668d29, 11c064cac93).

  • OpenMV Cam Settings Editor — um editor de GUI para o arquivo de configuração de ajustes da câmera, aberto a partir do menu Ferramentas (issue #205) (f55a0d7fa58, 2d7f80b03cf).

  • Protocolo de depuração de câmera v2 para câmeras v5.0.0, com validação de CRC de cabeçalho antes que qualquer campo do cabeçalho seja confiável, recuperação de lacunas de sequência que avança sobre pacotes perdidos em vez de travar, e uma leitura de taxa de quadros dividida na taxa na câmera e na taxa de exibição do IDE (c377bdbaea8, a3f0488ddcb, 29108dd40f7, c15e6c02789).

  • Carregar Firmware Personalizado aceita um pacote .zip — escolha o zip de saída de compilação de uma placa e o IDE grava o firmware (e o ROMFS) que ele contém (55e4eb2d900).

  • Registro de depuração no Terminal Serial — um botão alterna o registro de depuração em níveis do protocolo da câmera, com cada linha marcada com emoji e renderizada na fonte de emoji para facilitar a leitura (09990c62810, 42e23c3c11e, 997705548c7, d2208de77d4).

  • Copiar/Converter Script Atual — uma ação de Ferramentas que copia ou converte o script atual, com uma proteção contra salvar um main.py/boot.py compilado (81a954f33cd).

  • Ajuda de documentação com F1 sobre o símbolo Python sob o cursor, compartilhando o resolvedor de símbolos com as dicas exibidas ao passar o cursor (c341330bc16).

  • Dicas de documentação no editor — passar o mouse sobre o nome de uma API OpenMV mostra sua documentação embutida, resolvida a partir dos stubs .pyi incluídos e da própria documentação do módulo, para que você possa ler o que uma função, método, constante ou módulo faz sem sair do seu script (7ebde92d2a6, f225824ba13, 59ad01a364a, 02c1e61a2a6).

  • Nomear uma câmera — um alias local do IDE que você pode dar a uma câmera conectada, mostrado no lugar do nome da porta serial para que câmeras idênticas possam ser distinguidas (c600158ca34).

  • Página de Atalhos de Teclado — a página de atalhos das Preferências agora é exibida e filtrada para as ações que você realmente pode acessar, com origem e layout mais claros (4f5eae52e8e, 2d472fe5b40).

Melhorias no editor, no terminal e na interface#

  • Menu Ajuda reestruturado em um submenu OpenMV Docs (removendo os antigos links planos de pinagem), com links de notas de versão adicionados aos caminhos de versão de lançamento oficial (e1337ff67c4, 9eacc0994e6).

  • Caixa de diálogo Sobre modernizada — ano de copyright automático, uma tabela de logotipos de parceiros e um link de notas de versão em linha única (d89fb74138d).

  • O painel do Buffer de Quadros agora mostra a leitura de resolução e um rótulo de modo do buffer de quadros ao lado do botão fixo JPG; o rótulo de modo fica acinzentado para que não pareça clicável (cb735f82337, 376b17d4e77, 39c6ffc3b26).

  • Os gráficos do painel do Histograma são suavizados por spline, o painel pode ser comprimido (elidindo rótulos e comprimindo os gráficos), e o rótulo do canal fica ao lado das estatísticas (327324ef7e9, 9f8c7035483, d204e93d5e4).

  • Divisores de painéis acoplados — arrastar um divisor com Ctrl/Cmd move os divisores do editor/terminal e do buffer de quadros/histograma juntos, e as alças do minidivisor ganham um halo ao passar o cursor com uma área de captura mais ampla (379dee250d6, 96b83984342).

  • Terminal Serial — a rolagem automática não é mais reafixada a cada bloco de saída, então rolar para cima durante uma saída rápida permanece no lugar; o terminal pode executar um arquivo do disco quando não há editor (af67a076e95, bc132aef536).

  • Os comandos de documentação são nomeados por página em vez de por índice (a3201e9a4b4).

  • Alta DPI / escala fracionária — ícones, sobreposições e a tela de abertura são renderizados corretamente em fatores de escala fracionários (125%, 150% e similares), e a janela principal lembra sua geometria e tela entre reinicializações em configurações com múltiplos monitores (c60711e4950, 60901b6e13f, f241d2ea8ce, 54a1bc31fbd).

Firmware, placas e ferramentas#

  • Uma placa pode fixar sua última versão de firmware lançada, para que uma placa que não recebe mais firmware mais recente não seja sinalizada como desatualizada (bd987d32f42).

  • O texto de status de firmware desatualizado agora é colorido em vermelho para se destacar (9b18d96bac5).

  • Dica de recuperação do bootloader — uma incompatibilidade de bootloader.bin agora sugere entrar no modo de recuperação do bootloader (d46b998c76e).

  • A reprodução de vídeo é mais robusta: o caminho do FFmpeg é exibido quando as ferramentas não conseguem iniciá-lo, a mensagem é mais clara quando uma arquitetura não suporta a reprodução, os formatos de entrada são consultados com ffmpeg -demuxers, e foram adicionados fallbacks de terminal para Linux (0cc0e74a438, a75ec4898d0, 28e1ef10326, 51651780723).

  • Script de Captura de Dataset atualizado para o módulo csi (41a781651e4).

  • Aparência do assistente do instalador do Windows melhorada (250599d9d9e).

Correções de bugs#

  • A instalação de firmware de desenvolvimento para placas .lst (AE3) agora prepara na pasta de flash os binários que a listagem referencia, e a instalação de desenvolvimento/personalizada prepara a imagem ROMFS do pacote enquanto primeiro limpa qualquer romfs*.img obsoleto, para que o ROMFS baixado seja gravado em vez de um resíduo do editor (3cf56e98668, 43bedc8c591).

  • Os avisos do libjpeg para quadros truncados/corrompidos são filtrados do log para que um fluxo com perdas não inunde o terminal (6e2a8bed6cf, a5166abbdc1).

  • A lista de exemplos não mostra mais o index.json interno (14ac8ca48a7).

  • As janelas pop-up de atualização de firmware e de integração inicial são suprimidas no modo visualizador para que uma demonstração de produto nunca solicite a substituição do firmware do cliente (7b8528c3ad6).

  • Atalhos de teclado pendentes e exclusivos de autoria que sobraram de menus removidos foram desativados (be3bee8bb58, 27d1afc8705).

Localização#

  • Quatro novos idiomas — indonésio, romeno, tailandês e vietnamita — levando o IDE a 37 idiomas traduzidos, e cada catálogo foi atualizado, traduzido e limpo em todos os idiomas (fcf04090843, c830e6c4de3, 5e22679923f).

Linha de comando#

  • Aliases de opção no estilo GNU ``–word`` agora são aceitos junto às formas com traço único, e a saída de -help indica a equivalência (66414213dd9, 768c5d0e159).

Suporte de plataforma e ferramentas#

  • Base do Qt Creator: 14.0.2.

  • Versão do OpenMV IDE elevada para 5.0.0.

  • Novo: a variante de compilação OpenMV Viewer instalada separadamente.

  • Suporte ao Python 3.14 — o analisador parso incluído foi atualizado para 0.8.7, mantendo o preenchimento de código e o servidor de linguagem funcionando com a sintaxe mais recente do Python (578a4e39d97).

Alterações incompatíveis#

Nenhuma para projetos existentes. O protocolo de depuração v2 se aplica às câmeras v5.0.0; câmeras da geração v4 continuam usando o protocolo existente, e scripts e fluxos de trabalho existentes não são afetados.
