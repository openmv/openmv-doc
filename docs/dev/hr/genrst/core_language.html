Posebna metoda __del__ nije implementirana za korisnički definirane klase¶

Primjer koda:

import gc


class Foo:
    def __del__(self):
        print("__del__")


f = Foo()
del f

gc.collect()

CPython output:

MicroPython output:

__del__

__init_subclass__ isn’t automatically called.¶

Cause: MicroPython does not currently implement PEP 487.

Workaround: Manually call __init_subclass__ after class creation if needed. e.g.:

class A(Base):
    pass
A.__init_subclass__()

Primjer koda:

class Base:
    @classmethod
    def __init_subclass__(cls):
        print(f"Base.__init_subclass__({cls.__name__})")


class A(Base):
    pass

CPython output:

MicroPython output:

Base.__init_subclass__(A)

__init_subclass__ isn’t an implicit classmethod.¶

Cause: MicroPython does not currently implement PEP 487. __init_subclass__ is not currently in the list of special-cased class/static methods.

Workaround: Decorate declarations of __init_subclass__ with @classmethod.

Primjer koda:

def regularize_spelling(text, prefix="bound_"):
    # for regularizing across the CPython "method" vs MicroPython "bound_method" spelling for the type of a bound classmethod
    if text.startswith(prefix):
        return text[len(prefix) :]
    return text


class A:
    def __init_subclass__(cls):
        pass

    @classmethod
    def manual_decorated(cls):
        pass


a = type(A.__init_subclass__).__name__
b = type(A.manual_decorated).__name__

print(regularize_spelling(a))
print(regularize_spelling(b))
if a != b:
    print("FAIL")

CPython output:

MicroPython output:

method

function
method
FAIL

MicroPython doesn’t support parameterized __init_subclass__ class customization.¶

Cause: MicroPython does not currently implement PEP 487. The MicroPython syntax tree does not include a kwargs node after the class inheritance list.

Workaround: Use class variables or another mechanism to specify base-class customizations.

Primjer koda:

class Base:
    @classmethod
    def __init_subclass__(cls, arg=None, **kwargs):
        cls.init_subclass_was_called = True
        print(f"Base.__init_subclass__({cls.__name__}, {arg=!r}, {kwargs=!r})")


class A(Base, arg="arg"):
    pass


# Regularize across MicroPython not automatically calling __init_subclass__ either.
if not getattr(A, "init_subclass_was_called", False):
    A.__init_subclass__()

CPython output:

MicroPython output:

Base.__init_subclass__(A, arg='arg', kwargs={})

Traceback (most recent call last):
  File "<stdin>", line 16, in <module>
TypeError: function doesn't take keyword arguments

__init_subclass__ can’t be defined a cooperatively-recursive way.¶

Cause: MicroPython does not currently implement PEP 487. The base object type does not have an __init_subclass__ implementation.

Workaround: Omit the recursive __init_subclass__ call unless it’s known that the grandparent also defines it.

Primjer koda:

class Base:
    @classmethod
    def __init_subclass__(cls, **kwargs):
        cls.init_subclass_was_called = True
        super().__init_subclass__(**kwargs)


class A(Base):
    pass


# Regularize across MicroPython not automatically calling __init_subclass__ either.
if not getattr(A, "init_subclass_was_called", False):
    A.__init_subclass__()

CPython output:

MicroPython output:

Traceback (most recent call last):
  File "<stdin>", line 22, in <module>
  File "<stdin>", line 13, in __init_subclass__
AttributeError: 'super' object has no attribute '__init_subclass__'

Redoslijed razrješavanja metoda (MRO) nije usklađen s CPythonom¶

Uzrok: Redoslijed razrješavanja metoda koji ide u dubinu i nije iscrpan

Zaobilazno rješenje: Izbjegavajte složene hijerarhije klasa s višestrukim nasljeđivanjem i složenim prekoračenjima metoda. Imajte na umu da mnogi jezici uopće ne podržavaju višestruko nasljeđivanje.

Primjer koda:

class Foo:
    def __str__(self):
        return "Foo"


class C(tuple, Foo):
    pass


t = C((1, 2, 3))
print(t)

Promjena naziva privatnih članova klase (name mangling) nije implementirana¶

Uzrok: MicroPython prevodilac ne implementira promjenu naziva za privatne članove klase.

Zaobilazno rješenje: Izbjegavajte korištenje ili sukob s globalnim imenima ručnim dodavanjem jedinstvenog prefiksa imenu privatnog člana klase.

Primjer koda:

def __print_string(string):
    print(string)


class Foo:
    def __init__(self, string):
        self.string = string

    def do_print(self):
        __print_string(self.string)


example_string = "Example String to print."

class_item = Foo(example_string)
print(class_item.string)

class_item.do_print()

Example String to print.

NameError: name '_Foo__print_string' is not defined. Did you mean: '__print_string'?

Example String to print.
Example String to print.

Pri nasljeđivanju nativnih tipova, pozivanje metode u __init__(self, ...) prije super().__init__() uzrokuje AttributeError (ili segfault ako MICROPY_BUILTIN_METHOD_CHECK_SELF_ARG nije omogućen).¶

Uzrok: MicroPython nema odvojene metode __new__ i __init__ u nativnim tipovima.

Zaobilazno rješenje: Prvo pozovite super().__init__().

Primjer koda:

class L1(list):
    def __init__(self, a):
        self.append(a)


try:
    L1(1)
    print("OK")
except AttributeError:
    print("AttributeError")


class L2(list):
    def __init__(self, a):
        super().__init__()
        self.append(a)


try:
    L2(1)
    print("OK")
except AttributeError:
    print("AttributeError")

CPython output:

MicroPython output:

OK
OK

AttributeError
OK

Exceptions¶

Throwing a derived exception class instance in its __init__ without first calling super().__init__ is a TypeError¶

Cause: In MicroPython, an object is incompletely constructed if it does not call its superclass init function or return normally from its __init__. This prevents its usage in some circumstances.

Workaround: Call the superclass __init__ method before raising the exception.

Primjer koda:

class C(Exception):
    def __init__(self):
        raise self


class C1(Exception):
    def __init__(self):
        super().__init__()
        raise self


try:
    C()
except Exception as e:
    print(type(e).__name__)

try:
    C1()
except Exception as e:
    print(type(e).__name__)

CPython output:

MicroPython output:

C
C1

TypeError
C1

Uzrok: MicroPython broji „self” kao argument.

Zaobilazno rješenje: Tumačite poruke o pogreškama imajući na umu gore navedene informacije.

try:
    [].append()

Objekti funkcija nemaju atribut __module__¶

Uzrok: MicroPython je optimiziran za smanjenu veličinu koda i upotrebu RAM-a.

Zaobilazno rješenje: Za neugrađene module koristite sys.modules[function.__globals__['__name__']].

Primjer koda:

def f():
    pass


print(f.__module__)

MicroPython output:

  File "<stdin>", line 13, in <module>

Korisnički definirani atributi za funkcije nisu podržani¶

Uzrok: MicroPython je visoko optimiziran za upotrebu memorije.

Zaobilazno rješenje: Koristite vanjski rječnik, npr. FUNC_X[f] = 0.

Primjer koda:

def f():
    pass

Generator¶

Upravljač konteksta __exit__() ne poziva se u generatoru koji se ne izvodi do kraja¶

Primjer koda:

class foo(object):
    def __enter__(self):
        print("Enter")

    def __exit__(self, *args):
        print("Exit")


def bar(x):
    with foo():
        while True:
            x += 1
            yield x


def func():
    g = bar(0)
    for _ in range(3):
        print(next(g))


func()

Izvođenje¶

Lokalne varijable nisu uključene u rezultat locals()¶

Uzrok: MicroPython ne održava simboličko lokalno okruženje, optimizirano je u niz utora. Stoga se lokalnim varijablama ne može pristupiti po imenu.

Primjer koda:

def test():
    val = 2
    print(locals())


test()

CPython output:

MicroPython output:

{'val': 2}

{'test': <function test at 0x7f79e4e05260>, '__name__': '__main__', '__file__': '<stdin>'}

Kod koji se izvodi u funkciji eval() nema pristup lokalnim varijablama¶

Uzrok: MicroPython ne održava simboličko lokalno okruženje, optimizirano je u niz utora. Stoga se lokalnim varijablama ne može pristupiti po imenu. Efektivno, eval(expr) u MicroPythonu ekvivalentan je eval(expr, globals(), globals()).

Primjer koda:

val = 1


def test():
    val = 2
    print(val)
    eval("print(val)")


test()

f-strings¶

f-stringovi ne podržavaju spajanje s susjednim literalima ako ti literali sadrže vitičaste zagrade¶

Uzrok: MicroPython je optimiziran za prostor koda.

Zaobilazno rješenje: Koristite operator + između literalnih nizova kada oba nisu f-stringovi

Primjer koda:

x, y = 1, 2
print("aa" f"{x}")  # works
print(f"{x}" "ab")  # works
print("a{}a" f"{x}")  # fails
print(f"{x}" "a{}b")  # fails

Zaobilazno rješenje: Uvijek koristite uravnotežene vitičaste i uglate zagrade u izrazima unutar f-stringova

import¶

Atribut __path__ paketa ima drugačiji tip (jedan string umjesto liste stringova) u MicroPythonu¶

Uzrok: MicroPython ne podržava pakete imenskog prostora raspoređene po datotečnom sustavu. Osim toga, MicroPythonov sustav uvoza visoko je optimiziran za minimalnu upotrebu memorije.

Zaobilazno rješenje: Detalji rukovanja uvozom inherentno ovise o implementaciji. Ne oslanjajte se na takve detalje u prenosivim aplikacijama.

Primjer koda:

import modules

print(modules.__path__)

CPython output:

MicroPython output:

['/home/runner/work/openmv-doc/openmv-doc/micropython/tests/cpydiff/modules']

../tests/cpydiff/modules

MicroPython ne podržava pakete imenskog prostora raspoređene po datotečnom sustavu.¶

Uzrok: MicroPythonov sustav uvoza visoko je optimiziran za jednostavnost, minimalnu upotrebu memorije i minimalne troškove pretraživanja datotečnog sustava.

Zaobilazno rješenje: Ne instalirajte module koji pripadaju istom paketu imenskog prostora u različitim direktorijima. Za MicroPython preporučuje se imati najviše 3-komponentne putanje za pretraživanje modula: za trenutnu aplikaciju, po korisniku (za pisanje), za cijeli sustav (samo za čitanje).

Primjer koda:

import sys

sys.path.append(sys.path[1] + "/modules")
sys.path.append(sys.path[1] + "/modules2")

import subpkg.foo
import subpkg.bar

print("Two modules of a split namespace package imported")

CPython output:

MicroPython output:

Two modules of a split namespace package imported

Traceback (most recent call last):
  File "<stdin>", line 14, in <module>
ImportError: no module named 'subpkg.bar'
