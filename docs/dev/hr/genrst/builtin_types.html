Iznimka¶

Sve iznimke imaju čitljive atribute value i errno, ne samo StopIteration i OSError.¶

Uzrok: MicroPython je optimiziran za smanjenje veličine koda.

Zaobilazno rješenje: Koristite value samo na iznimkama StopIteration, a errno samo na iznimkama OSError. Ne koristite se ovim atributima na ostalim iznimkama niti se na njih oslanjajte.

Primjer koda:

e = Exception(1)
print(e.value)
print(e.errno)

CPython output:

MicroPython output:

 
Traceback (most recent call last):
  File "<stdin>", line 9, in <module>
AttributeError: 'Exception' object has no attribute 'value'

Korisnički definirani atributi za ugrađene iznimke nisu podržani¶

Uzrok: MicroPython je visoko optimiziran za upotrebu memorije.

Zaobilazno rješenje: Koristite korisnički definirane podklase iznimki.

Metoda Exception.__init__ ne postoji.¶

Uzrok: Nasljeđivanje nativnih klasa nije potpuno podržano u MicroPythonu.

Zaobilazno rješenje: Umjesto toga koristite poziv putem super():

    def __init__(self):
        super().__init__()

OSError¶

OSError constructor returns a plain OSError for all errno values, rather than a relevant subtype.¶

Cause: MicroPython does not include the CPython-standard OSError subclasses.

Workaround: Catch OSError and use its errno attribute to discriminate the cause.

Primjer koda:

import errno

errno_list = [  # i.e. the set implemented by micropython
    errno.EPERM,
    errno.ENOENT,
    errno.EIO,
    errno.EBADF,
    errno.EAGAIN,
    errno.ENOMEM,
    errno.EACCES,
    errno.EEXIST,
    errno.ENODEV,
    errno.EISDIR,
    errno.EINVAL,
    errno.EOPNOTSUPP,
    errno.EADDRINUSE,
    errno.ECONNABORTED,
    errno.ECONNRESET,
    errno.ENOBUFS,
    errno.ENOTCONN,
    errno.ETIMEDOUT,
    errno.ECONNREFUSED,
    errno.EHOSTUNREACH,
    errno.EALREADY,
    errno.EINPROGRESS,
]


def errno_output_type(n):
    try:
        raise OSError(n, "")
    except OSError as e:
        return f"{type(e).__name__}"
    except Exception as e:
        return f"non-OSError {type(e).__name__}"
    else:
        return "no error"


for n in errno_list:
    print(errno.errorcode[n], "=", errno_output_type(n))

bytes¶

Objekti bytes podržavaju metodu .format()¶

Uzrok: MicroPython nastoji biti dosljednija implementacija, pa ako i str i bytes podržavaju __mod__() (operator %), ima smisla podržati format() za oba. Podrška za __mod__ može se i isključiti pri kompajliranju, što ostavlja samo format() za formatiranje bajtova.

Zaobilazno rješenje: Ako vam je važna kompatibilnost s CPythonom, ne koristite .format() na objektima bytes.

Primjer koda:

print(b"{}".format(1))

CPython output:

MicroPython output:

 
Traceback (most recent call last):
  File "<stdin>", line 8, in <module>
AttributeError: 'bytes' object has no attribute 'format'

b'1'

Zaobilazno rješenje: Prenesite kodiranje kao pozicijski parametar, npr. print(bytes('abc', 'utf-8'))

complex¶

MicroPython’s complex() accepts certain incorrect values that CPython rejects¶

Uzrok: MicroPython je visoko optimiziran za upotrebu memorije.

Workaround: Do not use non-standard complex literals as argument to complex()

MicroPython’s complex() function accepts literals that contain a space and no sign between the real and imaginary parts, and interprets it as a plus.

Primjer koda:

try:
    print(complex("1 1j"))
except ValueError:
    print("ValueError")

CPython output:

MicroPython output:

 
ValueError

(1+1j)

Metoda to_bytes ne implementira parametar signed.¶

Uzrok: Parametar isključivo ključnom riječju signed nije implementiran za int.to_bytes().

Kada je cijeli broj negativan, MicroPython se ponaša isto kao CPython int.to_bytes(..., signed=True)

Kada je cijeli broj nenegativan, MicroPython se ponaša isto kao CPython int.to_bytes(..., signed=False).

(Razlika je suptilna, ali u CPythonu pozitivni cijeli broj konvertiran s signed=True može zahtijevati jedan bajt više u duljini izlaza, kako bi se smjestio bit predznaka 0.)

Zaobilazno rješenje: Budite pažljivi pri pozivu to_bytes() na vrijednosti cijelog broja koji može biti negativan.

Primjer koda:

x = -1
print(x.to_bytes(1, "big"))

CPython output:

MicroPython output:

 
Traceback (most recent call last):
  File "<stdin>", line 16, in <module>
OverflowError: can't convert negative int to unsigned

b'\xff'

memoryview¶

memoryview može postati nevažeći ako se promijeni veličina cilja¶

Uzrok: CPython sprječava promjenu veličine objekta bytearray ili io.bytesIO dok postoji objekt memoryview koji ga referencira. MicroPython zahtijeva da programer ručno osigura da se objekt ne promijeni u veličini dok ga bilo koji memoryview referencira.

U najgorem slučaju, promjena veličine objekta koji je cilj memoryview može uzrokovati da memoryview(i) referenciraju nevažeću oslobođenu memoriju (greška use-after-free) i oštete MicroPython runtime.

Zaobilazno rješenje: Ne mijenjajte veličinu nijednog objekta bytearray ili io.bytesIO koji mu ima dodijeljen memoryview.

Primjer koda:

b = bytearray(b"abcdefg")
m = memoryview(b)
b.extend(b"hijklmnop")
print(b, bytes(m))

CPython output:

MicroPython output:

 
Traceback (most recent call last):
  File "<stdin>", line 12, in <module>
BufferError: Existing exports of data: object cannot be re-sized

bytearray(b'abcdefghijklmnop') b'abcdefg'

range¶

Range objects with large start or stop arguments misbehave.¶

Cause: Intermediate calculations overflow the C mp_int_t type

Workaround: Avoid using such ranges

Primjer koda:

from sys import maxsize

# A range including `maxsize-1` cannot be created
try:
    print(range(-maxsize - 1, 0))
except OverflowError:
    print("OverflowError")

# A range with `stop-start` exceeding sys.maxsize has incorrect len(), while CPython cannot calculate len().
try:
    print(len(range(-maxsize, maxsize)))
except OverflowError:
    print("OverflowError")

# A range with `stop-start` exceeding sys.maxsize has incorrect len()
try:
    print(len(range(-maxsize, maxsize, maxsize)))
except OverflowError:
    print("OverflowError")

CPython output:

MicroPython output:

 
range(-9223372036854775808, 0)
OverflowError
2

OverflowError
0
0

str¶

MicroPython accepts the „,” grouping option with any radix, unlike CPython¶

Cause: To reduce code size, MicroPython does not issue an error for this combination

Workaround: Do not use a format string like {:,b} if CPython compatibility is required.

Primjer koda:

try:
    print("{:,b}".format(99))
except ValueError:
    print("ValueError")
try:
    print("{:,x}".format(99))
except ValueError:
    print("ValueError")
try:
    print("{:,o}".format(99))
except ValueError:
    print("ValueError")

CPython output:

MicroPython output:

 
ValueError
ValueError
ValueError

110,0011
63
143

MicroPython accepts but does not properly implement the „,” or „_” grouping character for float values¶

Cause: To reduce code size, MicroPython does not implement this combination. Grouping characters will not appear in the number’s significant digits and will appear at incorrect locations in leading zeros.

Workaround: Do not use a format string like {:,f} if exact CPython compatibility is required.

Primjer koda:

print("{:,f}".format(3141.159))
print("{:_f}".format(3141.159))
print("{:011,.2f}".format(3141.159))
print("{:011_.2f}".format(3141.159))

CPython output:

MicroPython output:

 
3,141.159000
3_141.159000
0,003,141.16
0_003_141.16

3141.159000
3141.159000
000,3141.16
0_003141.16

str.ljust() i str.rjust() nisu implementirani¶

Uzrok: MicroPython je visoko optimiziran za upotrebu memorije. Dostupna su jednostavna zaobilazna rješenja.

Zaobilazno rješenje: Umjesto s.ljust(10) koristite "%-10s" % s, umjesto s.rjust(10) koristite "% 10s" % s. Alternativno, "{:<10}".format(s) ili "{:>10}".format(s).

Primjer koda:
