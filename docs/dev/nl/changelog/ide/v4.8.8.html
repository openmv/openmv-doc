v4.8.8#

v4.8.8 voegt expliciete bedieningselementen voor het binnengaan van de bootloader toe voor het verbinden en bijwerken van OpenMV Cams, een veilige standaardwaarde per board voor het geforceerd binnengaan van de bootloader vanuit het dialoogvenster “Install the Latest Development Release”, en een handvol verbindings- en installerfixes. Deze release blijft gebaseerd op Qt Creator 14.0.2, zonder wijziging van de Qt Creator-basis en zonder brekende wijzigingen voor gebruikersprojecten.

Hoogtepunten#

  • Force enter OpenMV Cam bootloader is nu beschikbaar via het menu Tools en dwingt de verbonden cam bij het verbreken van de verbinding in zijn bootloader.

  • Het selectievakje Force bootloader is toegevoegd aan het dialoogvenster Install the Latest Development Release om het binnengaan van de bootloader af te dwingen bij het installeren van ontwikkelingsfirmware, met een veilige standaardwaarde per board.

  • DFU-only-apparaten maken niet langer ten onrechte automatisch verbinding, aangezien automatisch opnieuw verbinden en automatisch verbinden via de board-instellingen nu alleen worden geactiveerd voor normaal draaiende boards.

  • PAG7936-voorbeeldscripts op de AE3 worden automatisch herschreven om bij het laden de QQVGA-framegrootte op te schalen naar QVGA.

  • Afhandeling van de installerlicentie hersteld zodat de installatie correct verloopt vanaf de Ready-for-Installation-pagina.

Nieuwe functies#

  • Force enter OpenMV Cam bootloader. Een nieuw item in het menu Tools dwingt de verbonden cam bij het verbreken van de verbinding in zijn bootloader. Het dialoogvenster Install the Latest Development Release (te bereiken via het menu Tools) krijgt ook een selectievakje Force bootloader om het binnengaan van de bootloader af te dwingen bij het installeren van ontwikkelingsfirmware. (350beb306, 13ca76904)

Overige wijzigingen en verbeteringen#

  • Force-bootloader-standaard per board. De force-bootloader-standaard voor de installatie van ontwikkelingsreleases wordt nu per board aangestuurd via een nieuwe instelling forceBootloaderDefaultSafe, standaard ingeschakeld voor de AE3- en N6-boards (96E3/9206) en uitgeschakeld voor oudere DFU-boards om te voorkomen dat zij de bootloader niet automatisch verlaten. (13ca76904)

  • Opschaling van framegrootte voor PAG7936 op de AE3. Op de AE3 met de PAG7936-sensor worden voorbeeldscripts bij het laden automatisch herschreven om de QQVGA-framegrootte op te schalen naar QVGA. (eed6325f5)

  • Automatisch verbinden voor niet-DFU-only. Automatisch opnieuw verbinden en automatisch verbinden via de board-instellingen worden nu alleen geactiveerd voor niet-DFU-boards (die normaal draaien) in plaats van voor elk aanwezig board, zodat DFU-only-apparaten niet langer ten onrechte automatisch verbinding maken. (350beb306)

Bugfixes#

  • Een typefout in de firmware-updateprompt bij het verbinden hersteld (“lastest” -> “latest release firmware”). (13ca76904)

  • De afhandeling van de licentieacceptatie van de installer hersteld zodat de installatiestap wordt uitgevoerd op de Ready-for-Installation-pagina in plaats van op de licentiecontrolepagina, en het foutieve meegeleverde licentiebestand verwijderd. (a9d3fb904, 716827f07)

Platform- en toolondersteuning#

  • Qt Creator-basis: 14.0.2.

  • OpenMV IDE: versie verhoogd naar 4.8.8.

Brekende wijzigingen#

Geen. Deze release introduceert geen voor de gebruiker zichtbare onderbrekingen in workflow of gedrag.
