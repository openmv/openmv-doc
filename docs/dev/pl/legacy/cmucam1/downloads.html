Pliki do pobrania#

Podręcznik użytkownika#

Dostępny jest w formacie PDF Podręcznik użytkownika CMUcam1 v2.00 dla firmware v1.12. Zawiera listę części, schemat płytki i opis protokołu programowego. Podręczniki dla starszych wersji firmware można znaleźć tutaj: v1.11, v1.1, v1.0.

Jeśli zdecydujesz się eksperymentować ze zmianą ustawień rejestrów kamery, możesz być również zainteresowany pobraniem Podręcznika kamery OV6620 z OmniVision.

Budowa płytki#

Jeśli budujesz to samodzielnie od podstaw, musisz pobrać ten plik z kodem HEX (zawierający najnowszy firmware v1.12), aby zaprogramować układ SX28. Jest to kod, który implementuje funkcjonalność i protokół szeregowy opisany w podręczniku. Program pobierający SX-Key oraz aplikację możesz uzyskać od Parallax Inc. Starsze wersje firmware można znaleźć tutaj: v1.11, v1.1, v1.0.

Program interfejsu w Javie#

Mamy program w Javie o otwartym kodzie źródłowym, który możesz pobrać, aby połączyć się i przetestować swój CMUcam1. Ten program interfejsu pozwala śledzić obiekty, analizować dane i przechwytywać kompletne obrazy CMUcam1. Oto Zrzuty ekranu programu w Javie tego interfejsu.

To jest spakowany projekt Java napisany dla Linuksa.

To jest wersja GUI, która automatycznie wykrywa Twój system operacyjny. Działa w systemach UNIX obsługujących szeregowe wejście/wyjście tak, jakby było ono plikiem (takich jak Linux), a także w Windows 95/98/2000/NT/XP. Zalecamy używanie co najmniej procesora Pentium 2 400 MHz z Java 1.2.2 lub nowszą.

Aby zobaczyć przykłady kodu C dla mikroprocesora Cerebellum i BotBall Handyboard komunikujących się z CMUcam1, przejdź na tę stronę internetową.

Schemat#

Oto JPEG w wysokiej rozdzielczości bieżącego schematu CMUcam1 (wersja płytki 1.23).

Zanim zapytasz o kod źródłowy, możesz być zainteresowany pobraniem Podręcznika procesora SX28 z Ubicom.
