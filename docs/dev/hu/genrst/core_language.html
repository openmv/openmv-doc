Osztályok¶

A __del__ speciális metódus nincs megvalósítva felhasználó által definiált osztályokhoz¶

Mintakód:

import gc


class Foo:
    def __del__(self):
        print("__del__")


f = Foo()
del f

gc.collect()

CPython output:

MicroPython output:

 
__del__

__init_subclass__ isn’t automatically called.¶

Cause: MicroPython does not currently implement PEP 487.

Workaround: Manually call __init_subclass__ after class creation if needed. e.g.:

class A(Base):
    pass
A.__init_subclass__()

Mintakód:

class Base:
    @classmethod
    def __init_subclass__(cls):
        print(f"Base.__init_subclass__({cls.__name__})")


class A(Base):
    pass

CPython output:

MicroPython output:

 
Base.__init_subclass__(A)

__init_subclass__ isn’t an implicit classmethod.¶

Cause: MicroPython does not currently implement PEP 487. __init_subclass__ is not currently in the list of special-cased class/static methods.

Workaround: Decorate declarations of __init_subclass__ with @classmethod.

Mintakód:

def regularize_spelling(text, prefix="bound_"):
    # for regularizing across the CPython "method" vs MicroPython "bound_method" spelling for the type of a bound classmethod
    if text.startswith(prefix):
        return text[len(prefix) :]
    return text


class A:
    def __init_subclass__(cls):
        pass

    @classmethod
    def manual_decorated(cls):
        pass


a = type(A.__init_subclass__).__name__
b = type(A.manual_decorated).__name__

print(regularize_spelling(a))
print(regularize_spelling(b))
if a != b:
    print("FAIL")

CPython output:

MicroPython output:

 
method
method

function
method
FAIL

MicroPython doesn’t support parameterized __init_subclass__ class customization.¶

Cause: MicroPython does not currently implement PEP 487. The MicroPython syntax tree does not include a kwargs node after the class inheritance list.

Workaround: Use class variables or another mechanism to specify base-class customizations.

Mintakód:

class Base:
    @classmethod
    def __init_subclass__(cls, arg=None, **kwargs):
        cls.init_subclass_was_called = True
        print(f"Base.__init_subclass__({cls.__name__}, {arg=!r}, {kwargs=!r})")


class A(Base, arg="arg"):
    pass


# Regularize across MicroPython not automatically calling __init_subclass__ either.
if not getattr(A, "init_subclass_was_called", False):
    A.__init_subclass__()

CPython output:

MicroPython output:

 
Base.__init_subclass__(A, arg='arg', kwargs={})

Traceback (most recent call last):
  File "<stdin>", line 16, in <module>
TypeError: function doesn't take keyword arguments

__init_subclass__ can’t be defined a cooperatively-recursive way.¶

Cause: MicroPython does not currently implement PEP 487. The base object type does not have an __init_subclass__ implementation.

Workaround: Omit the recursive __init_subclass__ call unless it’s known that the grandparent also defines it.

Mintakód:

class Base:
    @classmethod
    def __init_subclass__(cls, **kwargs):
        cls.init_subclass_was_called = True
        super().__init_subclass__(**kwargs)


class A(Base):
    pass


# Regularize across MicroPython not automatically calling __init_subclass__ either.
if not getattr(A, "init_subclass_was_called", False):
    A.__init_subclass__()

CPython output:

MicroPython output:

 
Traceback (most recent call last):
  File "<stdin>", line 22, in <module>
  File "<stdin>", line 13, in __init_subclass__
AttributeError: 'super' object has no attribute '__init_subclass__'

A metódusfeloldási sorrend (MRO) nem felel meg a CPython-nak¶

Ok: Mélység-először, nem kimerítő metódusfeloldási sorrend

Megkerülés: Kerülje a többszörös öröklődéssel és összetett metódus-felüldefiniálással rendelkező bonyolult osztályhierarchiákat. Tartsa szem előtt, hogy sok nyelv egyáltalán nem támogatja a többszörös öröklődést.

Mintakód:

class Foo:
    def __str__(self):
        return "Foo"


class C(tuple, Foo):
    pass


t = C((1, 2, 3))
print(t)

CPython output:

MicroPython output:

 
Foo

(1, 2, 3)

A privát osztálytagok névcsúfítása nincs megvalósítva¶

Ok: A MicroPython fordító nem valósítja meg a névcsúfítást a privát osztálytagokhoz.

Megkerülés: Kerülje a globális nevekkel való ütközést úgy, hogy manuálisan egyedi előtagot ad a privát osztálytag nevéhez.

Mintakód:

def __print_string(string):
    print(string)


class Foo:
    def __init__(self, string):
        self.string = string

    def do_print(self):
        __print_string(self.string)


example_string = "Example String to print."

class_item = Foo(example_string)
print(class_item.string)

class_item.do_print()

  File "<stdin>", line 18, in do_print
NameError: name '_Foo__print_string' is not defined. Did you mean: '__print_string'?

Example String to print.
Example String to print.

Natív típusok öröklésekor egy metódus hívása a __init__(self, ...)-ban a super().__init__() előtt AttributeError-t vált ki (vagy segfault-ot okoz, ha a MICROPY_BUILTIN_METHOD_CHECK_SELF_ARG nincs engedélyezve).¶

Ok: A MicroPython natív típusaiban nincs különálló __new__ és __init__ metódus.

Megkerülés: Először hívja a super().__init__()-t.

Mintakód:

class L1(list):
    def __init__(self, a):
        self.append(a)


try:
    L1(1)
    print("OK")
except AttributeError:
    print("AttributeError")


class L2(list):
    def __init__(self, a):
        super().__init__()
        self.append(a)


try:
    L2(1)
    print("OK")
except AttributeError:
    print("AttributeError")

CPython output:

MicroPython output:

 
OK
OK

AttributeError
OK

Exceptions¶

Throwing a derived exception class instance in its __init__ without first calling super().__init__ is a TypeError¶

Cause: In MicroPython, an object is incompletely constructed if it does not call its superclass init function or return normally from its __init__. This prevents its usage in some circumstances.

Workaround: Call the superclass __init__ method before raising the exception.

Mintakód:

class C(Exception):
    def __init__(self):
        raise self


class C1(Exception):
    def __init__(self):
        super().__init__()
        raise self


try:
    C()
except Exception as e:
    print(type(e).__name__)

try:
    C1()
except Exception as e:
    print(type(e).__name__)

CPython output:

MicroPython output:

 
C
C1

TypeError
C1

A metódusok hibaüzenetei váratlan argumentumszámot jeleníthetnek meg¶

Ok: A MicroPython a „self”-et argumentumként számolja.

Megkerülés: A hibaüzeneteket a fenti információ figyelembevételével értelmezze.

Mintakód:

try:

A függvényobjektumoknak nincs __module__ attribútumuk¶

Ok: A MicroPython a kisebb kódméret és RAM-használat érdekében van optimalizálva.

Megkerülés: Nem beépített moduloknál használja a sys.modules[function.__globals__['__name__']] kifejezést.

Mintakód:

def f():
    pass


print(f.__module__)

A függvények felhasználó által definiált attribútumai nem támogatottak¶

Ok: A MicroPython erősen optimalizált a memóriahasználat szempontjából.

Megkerülés: Használjon külső szótárt, pl. FUNC_X[f] = 0.

Generátor¶

A kontextuskezelő __exit__()-ja nem hívódik meg olyan generátorban, amely nem fut le teljesen¶

Mintakód:

class foo(object):
    def __enter__(self):
        print("Enter")

    def __exit__(self, *args):
        print("Exit")


def bar(x):
    with foo():
        while True:
            x += 1
            yield x


def func():
    g = bar(0)
    for _ in range(3):
        print(next(g))


func()

Futtatókörnyezet¶

A helyi változók nem szerepelnek a locals() eredményében¶

Ok: A MicroPython nem tart fenn szimbolikus helyi környezetet, hanem egy résidőkeret-tömbre van optimalizálva. Ezért a helyi változók névvel nem érhetők el.

Mintakód:

def test():
    val = 2
    print(locals())


test()

CPython output:

MicroPython output:

 
{'val': 2}

{'test': <function test at 0x7fddda005260>, '__name__': '__main__', '__file__': '<stdin>'}

Az eval() függvényben futó kód nem fér hozzá a helyi változókhoz¶

Ok: A MicroPython nem tart fenn szimbolikus helyi környezetet, hanem egy résidőkeret-tömbre van optimalizálva. Ezért a helyi változók névvel nem érhetők el. A MicroPythonban az eval(expr) gyakorlatilag egyenértékű az eval(expr, globals(), globals()) hívással.

Mintakód:

val = 1


def test():
    val = 2
    print(val)
    eval("print(val)")


test()

f-strings¶

Az f-stringek nem támogatják a szomszédos literálokkal való összefűzést, ha a szomszédos literálok kapcsos zárójeleket tartalmaznak¶

Ok: A MicroPython a kódterület optimalizálására van tervezve.

Megkerülés: Használja a + operátort literál stringek között, ha nem mindkettő f-string

Mintakód:

x, y = 1, 2
print("aa" f"{x}")  # works
print(f"{x}" "ab")  # works
print("a{}a" f"{x}")  # fails
print(f"{x}" "a{}b")  # fails

import¶

Egy csomag __path__ attribútuma eltérő típusú MicroPythonban (egyetlen string a stringek listája helyett)¶

Ok: A MicroPython nem támogatja a fájlrendszerben szétterülő névtércsomagokat. Ezen túlmenően a MicroPython importrendszere erősen optimalizált a minimális memóriahasználat érdekében.

Megkerülés: Az importkezelés részletei alapvetően implementációfüggők. Ne támaszkodjon ilyen részletekre hordozható alkalmazásokban.

Mintakód:

import modules

print(modules.__path__)

CPython output:

MicroPython output:

 
['/home/runner/work/openmv-doc/openmv-doc/micropython/tests/cpydiff/modules']

../tests/cpydiff/modules

A MicroPython nem támogatja a fájlrendszerben szétterülő névtércsomagokat.¶

Ok: A MicroPython importrendszere erősen optimalizált az egyszerűség, a minimális memóriahasználat és a minimális fájlrendszer-keresési terhelés érdekében.

Megkerülés: Ne telepítsen ugyanazon névtércsomaghoz tartozó modulokat különböző könyvtárakba. MicroPythonhoz ajánlott legfeljebb 3 összetevőjű modulkeresési útvonalat megadni: a jelenlegi alkalmazáshoz, felhasználónként (írható) és rendszerszinten (nem írható).

Mintakód:

import sys

sys.path.append(sys.path[1] + "/modules")
sys.path.append(sys.path[1] + "/modules2")

import subpkg.foo
import subpkg.bar

print("Two modules of a split namespace package imported")

CPython output:

MicroPython output:

 
Two modules of a split namespace package imported

Traceback (most recent call last):
  File "<stdin>", line 14, in <module>
ImportError: no module named 'subpkg.bar'
