Эмуляция CMUcam2#

CMUcam2 обладает следующими возможностями:

  • Отслеживание определённых пользователем цветовых пятен со скоростью до 50 кадров в секунду (частота кадров зависит от настроек разрешения и размера окна)

  • Отслеживание движения с помощью покадрового вычитания со скоростью 26 кадров в секунду

  • Найти центроид любых данных отслеживания

  • Собрать данные о среднем цвете и дисперсии

  • Собрать 28-столбцовую гистограмму каждого цветового канала

  • Обработка изображений с горизонтальной фильтрацией краёв (не поддерживается)

  • Передача бинарного растрового изображения отслеживаемых пикселей на изображении в реальном времени

  • Произвольный оконный режим изображения

  • Понижение дискретизации изображения

  • Настройте свойства изображения камеры

  • Вывести необработанное изображение (один или несколько каналов)

  • Разрешение до 176 x 255

  • Поддерживает скорости передачи: 115 200 57 600 38 400 19 200 9 600 4 800 2 400 1 200 бит/с

  • Управление 5 выходами сервоприводов

  • Режим ведомой параллельной обработки изображений от единой шины камеры (не поддерживается)

  • Автоматически использовать сервоприводы для двухосевого отслеживания цвета

  • Ч/б аналоговый видеовыход (PAL или NTSC, в зависимости от используемого модуля камеры)

  • Гибкая настройка выходных пакетов

  • Режим пониженного энергопотребления

  • Многопроходная обработка буферизованного изображения

Различия эмуляции CMUcam2#

  • Диапазон входа сервопривода

    • Сервоприводы теперь принимают 8-битные значения 0-255

  • Перемычки

    • Из-за отсутствия перемычек на CMUcam3 изменения скорости передачи и направления сервоприводов необходимо выполнять на этапе компиляции — они управляются директивами #define в начале исходного кода cmucam.c

  • Команда Set Input (SI pin )

    • Устанавливает pin как вход

  • Команда Software Color Space (CS value )

    • CS 0 устанавливает цветовое пространство по умолчанию для аппаратуры (RGB или YCrCb)

    • CS 1 выполняет программное преобразование в цветовое пространство HSV

      • Преобразование предполагает, что аппаратура настроена на RGB для корректных результатов

      • Пространство HSV — это целочисленное пространство, где все значения масштабированы в диапазоне от 16 до 240

    • Примечание: это доступно только в r504 и выше

  • Set Output может управлять синим и оранжевым светодиодами

    • Сервоприводы 2 и 3 перестанут работать

    • SO 2 1 -> Синий вкл

    • SO 2 0 -> Синий выкл

    • SO 3 1 -> Оранжевый вкл

    • SO 3 0 -> Оранжевый выкл

  • Нет режима ведомого

    • Это возможно, но в настоящее время мы это не поддерживаем

  • Маска вывода (OM)

    • Поддерживает только пакеты T и S (тип 0 и 1)

  • Построчный режим (LM)

    • Поддерживает только режим отслеживания цвета (LM 0 1)

    • Статистика отслеживания цвета добавлена в r523 (LM 0 2)

  • Send JPEG (SJ)

    • Отправляет jpeg-сжатый кадр, заголовок и всё остальное

  • Демонстрационный режим

    • Поскольку удержание кнопки программирования переводит CMUcam3 в режим программирования, вам нужно нажать кнопку через полсекунды после запуска, чтобы войти в демонстрационный режим

  • Разность кадров высокого разрешения

    • Теперь это работает так же, как покадровое вычитание низкого разрешения, только при более высоком разрешении
