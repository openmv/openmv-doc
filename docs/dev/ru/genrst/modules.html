Только позиционные параметры¶

Для экономии размера кода многие функции, принимающие именованные аргументы в CPython, в MicroPython принимают только позиционные аргументы.

MicroPython обозначает только позиционные параметры так же, как CPython: вставляет / для обозначения конца позиционных параметров. Любая функция, сигнатура которой заканчивается на /, принимает только позиционные аргументы. Подробнее см. PEP 570.

Пример¶

Например, в CPython 3.4 сигнатура конструктора socket.socket выглядит так:

socket.socket(family=AF_INET, type=SOCK_STREAM, proto=0, fileno=None)

Однако сигнатура, задокументированная в MicroPython, выглядит так:

socket(af=AF_INET, type=SOCK_STREAM, proto=IPPROTO_TCP, /)

Символ / в конце параметров означает, что все они являются только позиционными в MicroPython. Следующий код работает в CPython, но не в большинстве портов MicroPython:

import socket
s = socket.socket(type=socket.SOCK_DGRAM)

MicroPython вызовет исключение:

TypeError: function doesn't take keyword arguments

Следующий код будет работать как в CPython, так и в MicroPython:

import socket
s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

array¶

Сравнение между разными кодами типов не поддерживается¶

Причина: Размер кода

Обходное решение: Сравнивайте отдельные элементы

Пример кода:

import array

array.array("b", [1, 2]) == array.array("i", [1, 2])

CPython output:

MicroPython output:

 
Traceback (most recent call last):
  File "<stdin>", line 10, in <module>
NotImplementedError:

Проверка переполнения не реализована¶

Причина: MicroPython реализует неявное усечение для уменьшения размера кода и времени выполнения

Обходное решение: Если необходима совместимость с CPython, явно маскируйте значение

Пример кода:

import array

a = array.array("b", [257])
print(a)

CPython output:

MicroPython output:

 
Traceback (most recent call last):
  File "<stdin>", line 10, in <module>
OverflowError: signed char is greater than maximum

array('b', [1])

Поиск целого числа не реализован¶

Пример кода:

import array

print(1 in array.array("B", b"12"))

CPython output:

MicroPython output:

 
False

Traceback (most recent call last):
  File "<stdin>", line 10, in <module>
NotImplementedError:

Удаление из массива не реализовано¶

Пример кода:

import array

a = array.array("b", (1, 2, 3))
del a[1]
print(a)

CPython output:

MicroPython output:

 
array('b', [1, 3])

Traceback (most recent call last):
  File "<stdin>", line 11, in <module>
TypeError: 'array' object doesn't support item deletion

Индексирование с шагом != 1 пока не реализовано¶

Пример кода:

import array

a = array.array("b", (1, 2, 3))
print(a[3:2:2])

CPython output:

MicroPython output:

 
array('b')

Traceback (most recent call last):
  File "<stdin>", line 11, in <module>
NotImplementedError: only slices with step=1 (aka None) are supported

errno¶

MicroPython does not include ENOTSUP as a name for errno 95.¶

Cause: MicroPython does not implement the ENOTSUP canonical alias for EOPNOTSUPP added in CPython 3.2.

Workaround: Use errno.EOPNOTSUPP in place of errno.ENOTSUP.

Пример кода:

import errno

print(f"{errno.errorcode[errno.EOPNOTSUPP]=!s}")

CPython output:

MicroPython output:

 
errno.errorcode[errno.EOPNOTSUPP]=ENOTSUP

errno.errorcode[errno.EOPNOTSUPP]=EOPNOTSUPP

json¶

Модуль JSON не выбрасывает исключение, когда объект не поддаётся сериализации¶

Пример кода:

import json

try:
    print(json.dumps(b"shouldn't be able to serialise bytes"))
except TypeError:
    print("TypeError")

CPython output:

MicroPython output:

 
TypeError

"shouldn't be able to serialise bytes"

os¶

Атрибут environ не реализован¶

Обходное решение: Используйте getenv, putenv и unsetenv

Пример кода:

import os

try:
    print(os.environ.get("NEW_VARIABLE"))
    os.environ["NEW_VARIABLE"] = "VALUE"
    print(os.environ["NEW_VARIABLE"])
except AttributeError:
    print("should not get here")
    print(os.getenv("NEW_VARIABLE"))
    os.putenv("NEW_VARIABLE", "VALUE")
    print(os.getenv("NEW_VARIABLE"))

CPython output:

MicroPython output:

 
None
VALUE

should not get here
None
VALUE

getenv возвращает фактическое значение вместо кэшированного¶

Причина: Атрибут environ не реализован

Пример кода:

import os

print(os.getenv("NEW_VARIABLE"))
os.putenv("NEW_VARIABLE", "VALUE")
print(os.getenv("NEW_VARIABLE"))

CPython output:

MicroPython output:

 
None
None

None
VALUE

random¶

Метод getrandbits может возвращать не более 32 бит за раз.¶

Причина: Внутреннее состояние ГПСЧ составляет только 32 бита, поэтому он может возвращать не более 32 бит данных за раз.

Обходное решение: Если вам нужно число, содержащее более 32 бит, используйте модуль random из micropython-lib.

Пример кода:

import random


x = random.getrandbits(64)
print("{}".format(x))

CPython output:

MicroPython output:

 
5732323987643404715

Traceback (most recent call last):
  File "<stdin>", line 11, in <module>
ValueError: bits must be 32 or less

Метод randint может возвращать целое число, не превышающее размер машинного слова.¶

Причина: ГПСЧ способен генерировать только 32 бита состояния за раз. Результат приводится к целому числу размера машинного слова, а не к полному объекту int.

Обходное решение: Если вам нужны целые числа, превышающие размер машинного слова, используйте модуль random из micropython-lib.

Пример кода:

import random


x = random.randint(2**128 - 1, 2**128)
print("x={}".format(x))

CPython output:

MicroPython output:

 
x=340282366920938463463374607431768211456

Traceback (most recent call last):
  File "<stdin>", line 11, in <module>
OverflowError: overflow converting long int to machine word

struct¶

Struct pack with too few args, not checked by MicroPython¶

Пример кода:

import struct

try:
    print(struct.pack("bb", 1))
    print("Should not get here")
except:
    print("struct.error")

CPython output:

MicroPython output:

 
struct.error

b'\x01\x00'
Should not get here

Struct pack with too many args, not checked by MicroPython¶

Пример кода:

import struct

try:
    print(struct.pack("bb", 1, 2, 3))
    print("Should not get here")
except:
    print("struct.error")

CPython output:

MicroPython output:

 
struct.error

b'\x01\x02'
Should not get here

Struct pack with whitespace in format, whitespace ignored by CPython, error on MicroPython¶

Причина: MicroPython оптимизирован для уменьшения размера кода.

Обходное решение: Не используйте пробелы в строках формата.

Пример кода:

import struct

try:
    print(struct.pack("b b", 1, 2))
    print("Should have worked")
except:
    print("struct.error")

CPython output:

MicroPython output:

 
b'\x01\x02'
Should have worked

struct.error

sys¶

Переопределение sys.stdin, sys.stdout и sys.stderr невозможно¶

Причина: Они хранятся в памяти только для чтения.

Пример кода:

import sys

sys.stdin = None
print(sys.stdin)

CPython output:

MicroPython output:

 
None

Traceback (most recent call last):
  File "<stdin>", line 10, in <module>
AttributeError: 'module' object has no attribute 'stdin'
