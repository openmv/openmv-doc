13.1.16. Генераторы меток и штрихкодов¶

Камере, которая обнаруживает AprilTags, нужны напечатанные AprilTags, на которые её можно навести, поэтому IDE их генерирует. Tools → Machine Vision → AprilTag Generator перечисляет шесть семейств меток, которые камера может обнаруживать, с указанием размера семейства в скобках:

  • TAG16H5 – 30 меток.

  • TAG25H7 – 242 метки.

  • TAG25H9 – 35 меток.

  • TAG36H10 – 2320 меток.

  • TAG36H11 – 587 меток. Рекомендуемое семейство: наилучший баланс количества, устойчивости обнаружения и сопротивления ложным срабатываниям, и значение по умолчанию в примерах камеры.

  • ARKTOOLKIT (семейство ARTOOLKIT) – 512 меток.

Семейство – это фиксированный алфавит шаблонов меток, спроектированных вместе так, чтобы его члены было трудно спутать друг с другом; каждая метка в семействе кодирует один номер ID, и детектор сообщает, какого члена семейства он увидел. Выберите для проекта одно семейство и генерируйте только те ID, которые нужны проекту – обнаружение настраивается для каждого семейства, а меньшее число допустимых ID означает меньше ложных срабатываний.

Выберите семейство, и диалог спросит диапазон ID меток для генерации и нужно ли печатать имя семейства и ID под каждой меткой (включено по умолчанию и стоит того, чтобы оставить – пол, усеянный анонимными метками, невозможно идентифицировать). Затем генератор записывает по одному пригодному для печати изображению на метку в выбранную вами папку – страница US-Letter с подписью, квадратное изображение без неё. Печатайте их в любом физическом размере, которого требует ваше расстояние обнаружения, на матовой бумаге, если важны блики.

Для других символик, которые читает камера – QR-коды, коды Data Matrix и линейные штрихкоды – пункты генератора в меню Machine Vision открывают веб-поиск онлайн-генераторов, а не генерируют локально, поскольку бесплатные генераторы для этих форматов повсеместны.

См. также

find_apriltags(), find_qrcodes(), find_datamatrices() и find_barcodes() – сторона обнаружения для каждого генератора.
