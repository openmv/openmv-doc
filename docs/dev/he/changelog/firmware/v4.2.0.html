v4.2.0#

v4.2.0 מוסיפה תמיכה בזיהוי עצמים לצינור ה-ML, מעצבת מחדש את סקריפטי האתחול / pyexec, ומתקנת טעינת JPEG ואת USB במהירות גבוהה (ULPI). העיצוב מחדש של האתחול/הביצוע הוא שינוי התנהגות — קראו את השינוי שובר התאימות שלהלן.

עיקרים#

  • תמיכת זיהוי עצמים נוספה לצינור TensorFlow/ML.

  • סקריפטי אתחול ו-pyexec עוצבו מחדש.

  • תוקנו טעינת JPEG ופין ה-DIR של USB במהירות גבוהה (ULPI).

  • שובר תאימות: העיצוב מחדש של סקריפט האתחול / pyexec משנה את התנהגות האתחול — ראו את השינוי שובר התאימות.

תכונות חדשות#

  • זיהוי עצמים — נוספה תמיכת מודלי זיהוי עצמים לצינור ההסקה של TensorFlow/ML.

שינויים ושיפורים נוספים#

  • תת-המודול המצורף של MicroPython עודכן.

תיקוני באגים#

  • תוקנו טעינת תמונת JPEG והגדרת פין ה-DIR של USB במהירות גבוהה ULPI.

תמיכת חומרה ולוחות#

  • Arduino Nano RP2040 Connect — הוגדרו פיני ה-I2C ברירת המחדל.

שינויי API שוברי תאימות#

שינויי API גלויים למשתמש בין v4.1.4 ל-v4.2.0.

השינוי שובר התאימות היחיד הוא שינוי התנהגות ברצף האתחול/הביצוע.

סקריפטי האתחול ו-pyexec עוצבו מחדש (התנהגות)#

סקריפטי האתחול ונתיב ביצוע הסקריפטים pyexec עוצבו מחדש. ההתנהגות זהה ברובה, אך הסדר/הטיפול של האתחול וביצוע הסקריפט השתנו — בדקו מחדש סקריפטים שהסתמכו על רצף האתחול הקודם (לדוגמה, התנהגות מותאמת אישית של boot.py או טיפול מדויק בחריגות בסוף הסקריפט).

Commit: b0c7a3272

רשימת תיוג להגירה#

בדקו מחדש כל התנהגות מותאמת אישית של boot.py / תלוית-אתחול מול רצף האתחול/הביצוע המעוצב מחדש (שינוי האתחול/הביצוע). כל שאר הסקריפטים רצים ללא שינוי.
