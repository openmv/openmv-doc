תחילת העבודה עם CamScripter#

סקירה כללית#

מה זה?#

ה-CamScripter הוא כלי לסייע לכם בכתיבה ובניפוי באגים של סקריפטים מותאמים אישית עבור ה-CMUcam3. הוא מאפשר לכתוב סקריפטים באמצעות שפת התכנות Lua. באמצעות שפה זו וה-Lua API המסופק עבור ה-CMUcam3, תוכלו להתאים את המצלמה כמעט לכל שימוש שתוכלו לחשוב עליו. במדריך זה נלווה אתכם בתהליך ההתקנה ולאחר מכן נפתח סקריפט לדוגמה שירוץ על המצלמה.

לפני שתתחיל#

מדריך זה מניח:

  • כבר יש לך CMUcam3

  • כבר הפעלת את המצלמה, באמצעות מדריך התחלה מהירה או מדריך התקנת ה-SDK

  • יש לך כל מה שנחוץ כדי לחבר את המצלמה למחשב שלך

התקנה, חלק 1: Lua, תקשורת ביציאה טורית#

Lua#

ה-CamScripter מאפשר לכם לתכנת את ה-CMUcam3 באמצעות שפת התכנות Lua. לפני שנוכל לעשות זאת, נצטרך להתקין שפה זו במחשב שלכם.

  1. הורד את קובץ ה-ZIP הבינארי המתאים. (עבור Windows: Lua-Windows (לא נשמר בארכיון) או עבור משתמשים אחרים http://luabinaries.luaforge.net/download.html)

  2. חלץ את הבינארי של Lua אל C:\lua

  3. פתחו שורת פקודה (ב-Windows, בחרו ”Run…“ מתפריט Start, והקלידו »cmd«).

  4. cd אל היכן שחילצת את קובצי ה-Lua. אצלי, אני מקליד »cd C:lua«

  5. הקלד »luax.x.exe -v« כאשר x הוא גרסת ה-Lua

בהנחה שהכול עבר חלק בהתקנה, אתה אמור לראות את מספר הגרסה של Lua מודפס על המסך שלך כמו בצילום המסך למטה (לא נשמר בארכיון).

אם אפשר, כדאי שתתקין את lua מאתר האינטרנט של ה-CMUcam3. אם ברצונך להדר אותו בעצמך, עליך תחילה לתקן את קוד המקור הרגיל של lua כך שלא ישתמש בנקודה צפה (lua-5.1-no-fp.patch).

ב-windows תחת cygwin, השתמש בשורות הבאות כדי להדר:

cd src
make LUAC_T=luac.exe MYCFLAGS=-mno-cygwin MYLDFLAGS=-mno-cygwin luac.exe
...
make LUA_T=lua.exe MYCFLAGS=-mno-cygwin MYLDFLAGS=-mno-cygwin lua.exe

סביבת הרצה של Java (JRE)#

לצורך ה-CamScripter נדרשת סביבת הרצה של Java (גרסה 6 ומעלה). אם עדיין אין לך JRE מותקן, ניתן להוריד ולהתקין אחד כאן.

RXTX#

ה-CamScripter משתמש בספריית התקשורת הטורית RXTX כדי לתקשר עם ה-CMUcam3 דרך היציאה הטורית של המחשב שלכם. ספרייה זו תצטרך להיות מותקנת במחשב שלכם. ראשית, הורידו את RXTX (לפחות גרסה 2.1). גרסה 2.1.7 נמצאת כאן: ftp://ftp.qbang.org/pub/rxtx/rxtx-2.1-7-bins-r2.zip.

כדי להתקין (Windows בלבד - עבור מערכות הפעלה אחרות, עקוב אחר ההוראות בקובץ rxtx INSTALL):

  1. חלץ את הורדת ה-RXTX, ונווט אל התיקייה שנוצרה

  2. לחץ פעמיים על Windows

  3. לחץ פעמיים על i368-mingw32

  4. עכשיו תצטרכו להבין היכן ממוקמת סביבת זמן הריצה של Java שלכם. סביר להניח שהיא ב-C:\Program Files\Java\jre<version number>. שלי, למשל, ממוקמת ב-C:\Program Files\Java\jre1.6.0_01. העתיקו את הקבצים rxtxSerial.dll ו-rxtxParallel.dll מתיקיית RXTX i368-mingw32 לתוך התיקייה jre/bin. אז בדוגמה שלי, שמתי את שני הקבצים האלה בתיקייה C:\Program Files\Java\jre1.6.0_01\bin.

התקנה, חלק 2: Eclipse#

אוקיי, אנחנו כמעט מוכנים לצאת לדרך! נותרו רק כמה דברים לעשות.

אם עדיין אין לך אותו, התקן את Eclipse (גרסה 3.2 או חדשה יותר). ניתן להתקין אותו מ-http://eclipse.org/downloads. Eclipse Classic הוא בחירה בטוחה, אם כי כל אחת מהחבילות האחרות המפורטות אמורה לעבוד גם כן. כדי להתקין, פשוט חלץ את החבילה שבחרת אל C:\Program Files\eclipse.

עכשיו כאשר Eclipse מותקן, עלינו להתקין את ה-CamScripter כתוסף של Eclipse:

  1. פתחו את Eclipse על ידי לחיצה כפולה על eclipse.exe ב-C:\Program Files\eclipse\

  2. לחצו על Enter כדי לקבל את סביבת העבודה (workspace) של ברירת המחדל

  3. פתחו את אשף ההתקנה/עדכון על ידי בחירה ב-Help -> Software Updates -> Find and Install

  4. בחרו בלחצן הרדיו ”Search for new features to install“, ולחצו על Next.

  5. לחץ על הלחצן ”New Remote Site…“

  6. עבור name, הזן Camscripter, ועבור URL הזן http://cmucam.org/~juanpablo/. לחץ OK.

  7. ודא שהתיבה ליד Camscripter מסומנת, ולחץ על הלחצן ”Finish“.

  8. בחרו ”Camscripter“ כתכונה להתקנה, ולחצו על Next.

  9. קבל את הסכם הרישוי, ולחץ Next

  10. לחץ Finish - Eclipse יוריד עבורך את התוסף

  11. לחץ ”Install“ כדי להתקין את התוסף Camscripter.

  12. לחץ Yes כדי להפעיל מחדש את פלטפורמת Eclipse

לאחר מכן Eclipse יופעל מחדש, כאשר תוסף Camscripter מותקן עבורך. כדי לוודא שהוא מותקן, בחר Window -> Show View -> Other…

לחץ על סימן הפלוס ליד CMUcam3, בחר ”CamScripter“, ולחץ OK. אתה אמור לראות את תצוגת CamScripter בתחתית Eclipse (האזור המוקף באדום בתמונה למטה, לא נשמר בארכיון). ניתן להרחיב את האזור התחתון אם הוא קטן מדי.

  1. תצוגת CamScripter

  2. חלונית הקונסולה שבה מוצגות הודעות טקסט

  3. חלונית המשוב שבה מוצגות הודעות תמונה וגרפיקה

  4. מריץ קוד Lua. ניתן להריץ קובץ שלם או רק שורות קוד נבחרות.

  5. מתקין קוד Lua בכרטיס ה-SD/MMC של ה-CMUcam3

הגדרה#

התקנו הכול! רק כמה שלבי הגדרה ונתחיל לתכנת את ה-CamScripter.

טען את פרויקט camscripter אל המצלמה#

עכשיו כאשר הכל מותקן, עלינו לטעון למצלמה את פרויקט ה-CamScripter שלנו. הריצו make על פרויקט camscripter הממוקם ב-projects/camscripter (בעץ קוד המקור שהוצאתם ב-מדריך התחלה מהירה), ולאחר מכן טענו את קובץ ה-hex למצלמה. כבו את המצלמה לאחר שהקובץ נטען.

הגדירו את העדפות ה-CamScripter#

עכשיו חזרו ל-Eclipse, ופתחו את מסך העדפות ה-CamScripter: Window –> Preferences –> CamScripter Setup.

  • הגדירו את מיקום ה-Luac כך שיצביע על קובץ ה-luacx.x.exe שלכם (כאשר x היא גרסת Lua) (שימו לב ל-»c«) בכל מקום שבו התקנתם את Lua ב-Installation, Part 1: Lua, Serial Port Communication.

    • למי שמעוניין, luac פשוט מהדר את קוד ה-lua, בעוד luax.x.exe למעשה מפרש ומריץ את הקוד. אנו מהדרים את הקוד ב-Eclipse, ואז מריצים אותו על המצלמה – ומכאן, luac.

  • ספקו את שם היציאה הטורית שאליה אתם מחברים את המצלמה. תוכלו ללחוץ על כפתור ”See Serial Ports“ כדי לראות רשימה של יציאות טוריות זמינות.

צילום המסך של העדפות ה-CamScripter לא נשמר בארכיון.

Hello, World!#

עכשיו כאשר הכל מותקן ומוגדר כראוי, בואו ניגש לתוכנית Hello, World המתבקשת. ראשית, תצטרכו ליצור פרויקט חדש. בחרו File –> New –> Project… תקבלו תיבה המאפשרת לכם לבחור את סוג הפרויקט שברצונכם ליצור (ראו למטה). לחצו לחיצה כפולה על ”General“, בחרו ”Project“, ולחצו על כפתור Next.

תנו לפרויקט שלכם את השם ”Hello“, ולחצו על כפתור Finish. לאחר מכן צרו קובץ חדש על ידי לחיצה על File –> New –> File (ב-Eclipse 3.3 זה File –> New –> Other –> General –> File). בתיבת שם הקובץ, הזינו ”hello_world.lua“, כאשר Hello היא תיקיית האב. לחצו על כפתור Finish.

ודא שוב שהמצלמה שלך מחוברת ליציאה הטורית של המחשב שלך. לפני שאתה מפעיל את המצלמה חשוב להבין את שני המצבים שהמצלמה תומכת בהם:

  • עצמאי: המצלמה אינה מצפה לאינטראקציה כלשהי עם המחשב כלל. כברירת מחדל המצלמה מתחילה במצב זה.

  • אינטראקטיבי: המצלמה מקבלת קוד Lua מהודר מ-CamScripter ב-Eclipse ומחזירה משוב. כדי להכניס את המצלמה למצב האינטראקטיבי, הפעל אותה וכאשר תראה את נורית ה-LED הכתומה דולקת, לחץ על לחצן המצלמה (יש לך 3 שניות ללחוץ על הלחצן). באותו רגע, נורית ה-LED הכתומה תהבהב מספר פעמים כדי להודיע לך שאנו נכנסים למצב אינטראקטיבי. אם הכול עבר כשורה, תראה את ההודעה Welcome to CamScripter Interactive mode. Click the execute button to run a script.

בוא נפעיל את המצלמה במצב האינטראקטיבי כדי שנוכל להריץ את תוכנית ה-Lua הראשונה שלנו.

בחלון העורך, הזן את שורת הקוד:

print("Hello, World!");

שמרו את הקובץ ולחצו על כפתור Execute בתצוגת ה-CamScripter. והנה! הקוד שלכם יעבור בחוט אל המצלמה, שביצעה אותו, ושלחה בחזרה את ההודעה שהדפסתם.

עכשיו בואו ננסה משהו מעניין קצת יותר. הוסיפו שלוש שורות קוד נוספות, כך שהקובץ hello_world.lua שלכם יכיל:

print("Hello, World!");
take_picture("pic_name");
print_picture("pic_name");
print("It's a picture!");

עכשיו לחצו שוב על כפתור Execute. הפעם ייקח יותר זמן לבצע, מכיוון ששליחת תמונות דרך היציאה הטורית גוזלת די הרבה זמן. אך ברגע שזה יסתיים, אמורות להופיע שתי ההודעות שלכם באזור ה-Console של ה-CamScripter, והתמונה שצילמתם עם המצלמה באזור ה-Feedback (הסרתם את מכסה העדשה, נכון?)

תבחין שזו תמונה ברזולוציה די נמוכה. המצלמה מוגדרת לצלם תמונות ברזולוציה נמוכה כברירת מחדל ב-CamScripter, מכיוון שלוקח קצת יותר זמן לשלוח תמונות ברזולוציה גבוהה דרך היציאה הטורית. עם זאת, תוכל לשנות את המצלמה לרזולוציה גבוהה באמצעות הפונקציה camera_set_resolution.

צילומי המסך של פרויקט Eclipse, קובץ, camera-on ו-Hello-World עבור מקטע זה לא נשמרו בארכיון.

קישורים שימושיים#

ה-CamScripter מותקן ומוגדר במלואו כעת, וראיתם דוגמה קטנה המציגה את הפונקציונליות שלו. אם ברצונכם לראות דוגמאות גדולות יותר המשתמשות ב-Camscripter, הסתכלו על התוכניות לדוגמה הכלולות בתיקיית camscripter/examples. כמה הערות וקישורים אחרים שעשויים לעזור לכם:

ה-CamScripter Lua API מתאר את פונקציות ה-CMUcam3 שתוכלו להשתמש בהן בתוכניות Lua שלכם כדי לשלוט בפעולות ה-CMUcam3.

הנה כמה קישורים שיעזרו לך להתחיל להשתמש ב-Lua. שים לב שספריות חיצוניות כגון ספריות ה-string וה-math אינן כלולות בגרסת ה-Lua המותקנת ב-CMUcam3. היה צורך להסיר אותן כדי שמפרש ה-Lua ייכנס לשטח המוגבל שב-CMUcam3.
