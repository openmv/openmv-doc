Kullanıcı tanımlı sınıflar için __del__ özel metodu uygulanmamıştır¶

Örnek kod:

import gc


class Foo:
    def __del__(self):
        print("__del__")


f = Foo()
del f

gc.collect()

__init_subclass__ isn’t automatically called.¶

Cause: MicroPython does not currently implement PEP 487.

Workaround: Manually call __init_subclass__ after class creation if needed. e.g.:

class A(Base):
    pass
A.__init_subclass__()

Örnek kod:

class Base:
    @classmethod
    def __init_subclass__(cls):
        print(f"Base.__init_subclass__({cls.__name__})")


class A(Base):
    pass

CPython output:

MicroPython output:

 
Base.__init_subclass__(A)

__init_subclass__ isn’t an implicit classmethod.¶

Cause: MicroPython does not currently implement PEP 487. __init_subclass__ is not currently in the list of special-cased class/static methods.

Workaround: Decorate declarations of __init_subclass__ with @classmethod.

Örnek kod:

def regularize_spelling(text, prefix="bound_"):
    # for regularizing across the CPython "method" vs MicroPython "bound_method" spelling for the type of a bound classmethod
    if text.startswith(prefix):
        return text[len(prefix) :]
    return text


class A:
    def __init_subclass__(cls):
        pass

    @classmethod
    def manual_decorated(cls):
        pass


a = type(A.__init_subclass__).__name__
b = type(A.manual_decorated).__name__

print(regularize_spelling(a))
print(regularize_spelling(b))
if a != b:
    print("FAIL")

CPython output:

MicroPython output:

 
method
method

function
method
FAIL

MicroPython doesn’t support parameterized __init_subclass__ class customization.¶

Cause: MicroPython does not currently implement PEP 487. The MicroPython syntax tree does not include a kwargs node after the class inheritance list.

Workaround: Use class variables or another mechanism to specify base-class customizations.

Örnek kod:

class Base:
    @classmethod
    def __init_subclass__(cls, arg=None, **kwargs):
        cls.init_subclass_was_called = True
        print(f"Base.__init_subclass__({cls.__name__}, {arg=!r}, {kwargs=!r})")


class A(Base, arg="arg"):
    pass


# Regularize across MicroPython not automatically calling __init_subclass__ either.
if not getattr(A, "init_subclass_was_called", False):
    A.__init_subclass__()

CPython output:

MicroPython output:

 
Base.__init_subclass__(A, arg='arg', kwargs={})

Traceback (most recent call last):
  File "<stdin>", line 16, in <module>
TypeError: function doesn't take keyword arguments

__init_subclass__ can’t be defined a cooperatively-recursive way.¶

Cause: MicroPython does not currently implement PEP 487. The base object type does not have an __init_subclass__ implementation.

Workaround: Omit the recursive __init_subclass__ call unless it’s known that the grandparent also defines it.

Örnek kod:

class Base:
    @classmethod
    def __init_subclass__(cls, **kwargs):
        cls.init_subclass_was_called = True
        super().__init_subclass__(**kwargs)


class A(Base):
    pass


# Regularize across MicroPython not automatically calling __init_subclass__ either.
if not getattr(A, "init_subclass_was_called", False):
    A.__init_subclass__()

CPython output:

MicroPython output:

 
Traceback (most recent call last):
  File "<stdin>", line 22, in <module>
  File "<stdin>", line 13, in __init_subclass__
AttributeError: 'super' object has no attribute '__init_subclass__'

Metot Çözümleme Sırası (MRO) CPython ile uyumlu değildir¶

Neden: Derinlik öncelikli ve kapsamlı olmayan metot çözümleme sırası

Geçici çözüm: Çoklu kalıtım ve karmaşık metot geçersiz kılmalarına sahip karmaşık sınıf hiyerarşilerinden kaçının. Pek çok dilin çoklu kalıtımı hiç desteklemediğini göz önünde bulundurun.

Örnek kod:

class Foo:
    def __str__(self):
        return "Foo"


class C(tuple, Foo):
    pass


t = C((1, 2, 3))
print(t)

CPython output:

MicroPython output:

 
Foo

(1, 2, 3)

Özel sınıf üyeleri için ad değiştirme (name mangling) uygulanmamıştır¶

Neden: MicroPython derleyicisi, özel sınıf üyeleri için ad değiştirmeyi uygulamamaktadır.

Geçici çözüm: Özel sınıf üyesi adına elle benzersiz bir önek ekleyerek genel adlarla çakışmadan kaçının.

Örnek kod:

def __print_string(string):
    print(string)


class Foo:
    def __init__(self, string):
        self.string = string

    def do_print(self):
        __print_string(self.string)


example_string = "Example String to print."

class_item = Foo(example_string)
print(class_item.string)

class_item.do_print()

CPython output:

MicroPython output:

 
Example String to print.
Traceback (most recent call last):
  File "<stdin>", line 26, in <module>
  File "<stdin>", line 18, in do_print
NameError: name '_Foo__print_string' is not defined. Did you mean: '__print_string'?

Example String to print.
Example String to print.

Yerel türler devralınırken __init__(self, ...) içinde super().__init__() çağrısından önce bir metot çağrılması AttributeError hatasına yol açar (MICROPY_BUILTIN_METHOD_CHECK_SELF_ARG etkin değilse segfault oluşur).¶

Neden: MicroPython, yerel türlerde ayrı __new__ ve __init__ metotlarına sahip değildir.

Geçici çözüm: super().__init__() çağrısını önce yapın.

Örnek kod:

class L1(list):
    def __init__(self, a):
        self.append(a)


try:
    L1(1)
    print("OK")
except AttributeError:
    print("AttributeError")


class L2(list):
    def __init__(self, a):
        super().__init__()
        self.append(a)


try:
    L2(1)
    print("OK")
except AttributeError:
    print("AttributeError")

CPython output:

MicroPython output:

 
OK
OK

AttributeError
OK

Alt sınıfta super() getter özelliğinin çağrılması değeri değil, property nesnesini döndürür¶

Exceptions¶

Throwing a derived exception class instance in its __init__ without first calling super().__init__ is a TypeError¶

Cause: In MicroPython, an object is incompletely constructed if it does not call its superclass init function or return normally from its __init__. This prevents its usage in some circumstances.

Workaround: Call the superclass __init__ method before raising the exception.

Örnek kod:

class C(Exception):
    def __init__(self):
        raise self


class C1(Exception):
    def __init__(self):
        super().__init__()
        raise self


try:
    C()
except Exception as e:
    print(type(e).__name__)

try:
    C1()
except Exception as e:
    print(type(e).__name__)

CPython output:

MicroPython output:

 
C
C1

TypeError
C1

Fonksiyonlar¶

Metotlara ilişkin hata iletileri beklenmedik bağımsız değişken sayısı gösterebilir¶

Neden: MicroPython, “self”i bir bağımsız değişken olarak sayar.

Geçici çözüm: Hata iletilerini yukarıdaki bilgiyi göz önünde bulundurarak yorumlayın.

Örnek kod:

try:
    [].append()
except Exception as e:
    print(e)

CPython output:

MicroPython output:

 
list.append() takes exactly one argument (0 given)

function takes 2 positional arguments but 1 were given

Fonksiyon nesneleri __module__ özelliğine sahip değildir¶

Neden: MicroPython, küçük kod boyutu ve düşük RAM kullanımı için optimize edilmiştir.

Geçici çözüm: Yerleşik olmayan modüller için sys.modules[function.__globals__['__name__']] kullanın.

Örnek kod:

def f():
    pass


print(f.__module__)

CPython output:

MicroPython output:

 
__main__

Traceback (most recent call last):
  File "<stdin>", line 13, in <module>
AttributeError: 'function' object has no attribute '__module__'

Fonksiyonlar için kullanıcı tanımlı özellikler desteklenmemektedir¶

Neden: MicroPython, bellek kullanımı için yüksek düzeyde optimize edilmiştir.

Geçici çözüm: Harici sözlük kullanın, örn. FUNC_X[f] = 0.

Örnek kod:

def f():
    pass


f.x = 0
print(f.x)

  File "<stdin>", line 13, in <module>

Üreteç¶

Tamamlanmadan sonlanan bir üretecte bağlam yöneticisinin __exit__() çağrılmaz¶

Örnek kod:

class foo(object):
    def __enter__(self):
        print("Enter")

    def __exit__(self, *args):
        print("Exit")


def bar(x):
    with foo():
        while True:
            x += 1
            yield x


def func():
    g = bar(0)
    for _ in range(3):
        print(next(g))


func()

Yerel değişkenler locals() sonucuna dahil edilmez¶

Neden: MicroPython sembolik yerel ortam tutmaz; bir slot dizisine optimize edilmiştir. Bu nedenle yerel değişkenlere ada göre erişilemez.

Örnek kod:

def test():
    val = 2
    print(locals())


test()

{'test': <function test at 0x7f23a9205260>, '__name__': '__main__', '__file__': '<stdin>'}

eval() fonksiyonunda çalışan kod yerel değişkenlere erişemez¶

Neden: MicroPython sembolik yerel ortam tutmaz; bir slot dizisine optimize edilmiştir. Bu nedenle yerel değişkenlere ada göre erişilemez. Sonuç olarak MicroPython’da eval(expr), eval(expr, globals(), globals()) ile eşdeğerdir.

Örnek kod:

val = 1


def test():
    val = 2
    print(val)
    eval("print(val)")

f-strings¶

f-string’ler, bitişik değişmez değerler küme ayracı içeriyorsa bunlarla birleştirmeyi desteklemez¶

Neden: MicroPython, kod alanı için optimize edilmiştir.

Geçici çözüm: Her ikisi de f-string olmayan değişmez dizeler arasında + operatörünü kullanın

Örnek kod:

x, y = 1, 2
print("aa" f"{x}")  # works
print(f"{x}" "ab")  # works
print("a{}a" f"{x}")  # fails
print(f"{x}" "a{}b")  # fails

f-string’ler, dengesiz iç içe küme ayraçlarını ve parantezleri çözmek için ayrıştırma gerektiren ifadeleri destekleyemez¶

Geçici çözüm: f-string içindeki ifadelerde her zaman dengeli küme ayraçları ve parantezler kullanın

import¶

MicroPython’da bir paketin __path__ özelliği farklı bir türe sahiptir (dize listesi yerine tek dize)¶

Neden: MicroPython, dosya sistemi genelinde bölünmüş ad alanı paketlerini desteklemez. Bunun ötesinde, MicroPython’un içe aktarma sistemi minimum bellek kullanımı için yüksek düzeyde optimize edilmiştir.

Geçici çözüm: İçe aktarma işlemenin ayrıntıları doğası gereği uygulamaya bağlıdır. Taşınabilir uygulamalarda bu tür ayrıntılara güvenmeyin.

Örnek kod:

import modules

print(modules.__path__)

CPython output:

MicroPython output:

 
['/home/runner/work/openmv-doc/openmv-doc/micropython/tests/cpydiff/modules']

../tests/cpydiff/modules

MicroPython, dosya sistemi genelinde bölünmüş ad alanı paketlerini desteklemez.¶

Neden: MicroPython’un içe aktarma sistemi, basitlik, minimum bellek kullanımı ve minimum dosya sistemi arama yükü için yüksek düzeyde optimize edilmiştir.

Geçici çözüm: Aynı ad alanı paketine ait modülleri farklı dizinlere yüklemeyin. MicroPython için en fazla 3 bileşenli modül arama yoluna sahip olmanız önerilir: geçerli uygulamanız için, kullanıcı başına (yazılabilir) ve sistem genelinde (yazılamaz).

Örnek kod:

import sys

sys.path.append(sys.path[1] + "/modules")
sys.path.append(sys.path[1] + "/modules2")

import subpkg.foo
import subpkg.bar

print("Two modules of a split namespace package imported")

CPython output:

MicroPython output:

 
Two modules of a split namespace package imported

Traceback (most recent call last):
  File "<stdin>", line 14, in <module>
ImportError: no module named 'subpkg.bar'
