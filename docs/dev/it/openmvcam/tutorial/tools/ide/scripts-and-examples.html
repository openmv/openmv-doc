13.1.2. Script, esempi e la cartella documenti¶

13.1.2.1. Nuovi file¶

Crea un nuovo file e l’IDE apre un buffer untitled_N.py precaricato con un ciclo di cattura minimale – resetta il sensore, configura il formato dei pixel e la dimensione del frame, quindi esegue lo snapshot e stampa il frame rate all’infinito. Lo script iniziale è adattato alla scheda connessa, quindi funziona così com’è creato. Il buffer risiede solo in memoria: puoi modificarlo ed eseguirlo ripetutamente senza mai salvarlo, il che rende i nuovi file il modo più economico per testare un’idea. Salvalo su disco e diventa un normale script.

13.1.2.2. Il menu degli esempi¶

File → Examples contiene una libreria categorizzata di script di esempio che copre essenzialmente ogni funzionalità della camera – una cartella per argomento, dagli snapshot di base fino al tracciamento del colore, al machine learning e alle periferiche della scheda. Aprire un esempio lo carica in un buffer di memoria come un nuovo file, così puoi modificarlo ed eseguirlo liberamente senza toccare la copia installata.

Per impostazione predefinita il menu è filtrato in base alla scheda connessa e al suo sensore: gli esempi che richiedono hardware di cui la tua camera non dispone sono nascosti, e il menu richiede una connessione prima di mostrare qualsiasi cosa. Tools → Filter examples by board and sensor type disattiva il filtro quando vuoi sfogliare l’intero set.

Anche gli esempi vengono adattati man mano che si aprono. Gli script sono scritti per un tipico sensore a colori, quindi quando la camera connessa è diversa – un sensore termico o a eventi, una scheda con una risoluzione nativa differente – l’IDE adatta le righe del formato dei pixel e della dimensione del frame a valori supportati dal tuo hardware durante il caricamento, e l’esempio funziona così com’è aperto.

Lavorare attraverso il menu degli esempi è il modo più rapido per imparare cosa può fare la camera; la maggior parte delle applicazioni nasce come un esempio che è cresciuto.

13.1.2.3. La cartella documenti¶

Per i tuoi script personali, l’IDE monitora una directory OpenMV all’interno della tua cartella documenti (Documents/OpenMV). Tutto ciò che memorizzi lì compare sotto File → Documents Folder, incluse le sottodirectory, così la tua raccolta di script è a un menu di distanza. Il menu legge la directory in tempo reale – i file che aggiungi al di fuori dell’IDE compaiono la volta successiva che il menu viene aperto.

L’IDE viene eseguito come istanza singola: apri un file .py dal tuo file manager, oppure trascinane uno sulla finestra dell’IDE, e si aprirà nella finestra già in esecuzione anziché avviare una seconda copia.

13.1.2.4. Compilazione incrociata di uno script¶

Tools → MicroPython Tools → Copy/Convert Python File copia un file .py in una destinazione – il drive della camera, quando ne è connessa una – e ti permette di scegliere cosa succede lungo il percorso: copiarlo invariato, rimuovere spazi e/o commenti per ridurne le dimensioni, oppure compilarlo in un file bytecode .mpy con il compilatore mpy-cross corrispondente alla scheda di destinazione. Un file .mpy si importa esattamente come il .py da cui proviene, si carica più velocemente e non distribuisce il proprio sorgente – la forma da usare quando si distribuisce un modulo di libreria al filesystem della camera. Con una camera connessa l’IDE seleziona automaticamente la scheda di destinazione; altrimenti la finestra di dialogo chiede per quale scheda compilare. La scelta della scheda è importante perché imposta l’architettura del processore a cui mira il compilatore: un file .mpy è abbinato alla camera su cui verrà eseguito, e uno compilato per una famiglia di schede diversa non è garantito che si carichi.
