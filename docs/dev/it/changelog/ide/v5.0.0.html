v5.0.0#

v5.0.0 è una release importante. Introduce l”OpenMV Viewer — una build separata e di sola lettura per mostrare un prodotto finito — un nuovo OpenMV Cam Settings Editor, il protocollo di debug della camera v2 per le camere v5.0.0 e un ampio insieme di miglioramenti ai flussi di lavoro di editor, terminale, firmware e machine vision. È basata su Qt Creator 14.0.2.

Il salto di versione alla 5.0.0 segna l’arrivo della generazione di camere v5.0.0 e degli strumenti che la supportano; i progetti e i flussi di lavoro v4 esistenti continuano a funzionare senza modifiche.

In evidenza#

  • OpenMV Viewer — una nuova variante di build installata separatamente che esegue l’IDE come visualizzatore di telemetria bloccato e di sola lettura. Mostra il frame buffer della camera, il terminale seriale e l’istogramma e può eseguire uno script fisso, ma nasconde gli strumenti di authoring, di AI/modelli e di impostazioni, così un cliente può dimostrare un prodotto costruito su OpenMV senza l’ambiente di sviluppo completo.

  • OpenMV Cam Settings Editor — un editor grafico per il file di configurazione delle impostazioni a bordo della camera, così puoi visualizzare e modificare le impostazioni della scheda da un modulo invece di modificare il file a mano.

  • Protocollo di debug della camera v2 — le camere v5.0.0 usano un nuovo protocollo di debug più robusto con validazione CRC dell’intestazione, recupero dei salti di sequenza e una visualizzazione del frame rate suddivisa che riporta separatamente gli FPS sulla camera e gli FPS di visualizzazione nell’IDE.

  • Caricamento di firmware personalizzato da un .zip — l’installer del firmware accetta ora direttamente il bundle .zip di output di build di una scheda, semplificando il flashing di un’immagine firmware personalizzata o compilata localmente.

  • Logging di debug del protocollo nel Serial Terminal — un interruttore nel Serial Terminal registra il protocollo di debug della camera con righe leggibili ed etichettate con emoji per diagnosticare i problemi di connessione.

  • Menu Aiuto e documentazione nell’editor rielaborati — un sottomenu OpenMV Docs sostituisce i vecchi collegamenti piatti, F1 apre la documentazione per il simbolo Python sotto il cursore, passando il mouse su un nome di API ne mostra la documentazione inline e la finestra di dialogo Informazioni è stata modernizzata.

Nuove funzionalità#

  • Variante di build OpenMV Viewer. Una build in modalità viewer forzata, con marchio e installata come OpenMV Viewer, con un proprio id eseguibile, archivio delle impostazioni e bundle id così da coesistere con l’IDE completo. Si presenta come un visualizzatore di telemetria di sola lettura: Run/Stop e lo stop alla connessione funzionano ed eseguono uno script dal disco (tramite -auto_run), i menu File/Edit e le Preferences rimangono e il recupero di firmware/ROMFS è mantenuto, mentre gli strumenti di authoring, gli strumenti di machine vision per sviluppatori, le categorie di impostazioni per sviluppatori e le risorse di AI/modelli/documentazione sono rimossi dalla build e dall’interfaccia. ROMFS diventa di solo caricamento e viene spostato nel menu Tools, e le finestre di dialogo di completamento dell’aggiornamento del firmware sono rese generiche (07722e78a8b, 5cd12ee260e, b97bf5447e4, 952b86de391, 1103f7988ed, f0f60261485, c743d5d67b9, b4bcdc2d71a, a37c032b6ff, 480e5668d29, 11c064cac93).

  • OpenMV Cam Settings Editor — un editor GUI per il file di configurazione delle impostazioni della camera, aperto dal menu Tools (issue #205) (f55a0d7fa58, 2d7f80b03cf).

  • Protocollo di debug della camera v2 per le camere v5.0.0, con validazione CRC dell’intestazione prima che qualsiasi campo dell’intestazione sia considerato affidabile, recupero dei salti di sequenza che avanza oltre i pacchetti persi invece di bloccarsi, e una lettura del frame rate suddivisa tra la frequenza sulla camera e la frequenza di visualizzazione dell’IDE (c377bdbaea8, a3f0488ddcb, 29108dd40f7, c15e6c02789).

  • Load Custom Firmware accetta un bundle .zip — seleziona lo zip di output di build di una scheda e l’IDE esegue il flashing del firmware (e del ROMFS) che contiene (55e4eb2d900).

  • Logging di debug del Serial Terminal — un pulsante attiva/disattiva un logging di debug a livelli del protocollo della camera, con ogni riga etichettata con emoji e resa nel font emoji per la leggibilità (09990c62810, 42e23c3c11e, 997705548c7, d2208de77d4).

  • Copy/Convert Current Script — un’azione del menu Tools che copia o converte lo script corrente, con una protezione contro il salvataggio di un main.py/boot.py compilato (81a954f33cd).

  • Guida alla documentazione con F1 sul simbolo Python sotto il cursore, che condivide il risolutore dei simboli con i tooltip al passaggio del mouse (c341330bc16).

  • Tooltip di documentazione nell’editor — passando il mouse su un nome di API OpenMV ne mostra la documentazione inline, ricavata dagli stub .pyi inclusi e dalla documentazione del modulo stesso, così puoi leggere cosa fa una funzione, un metodo, una costante o un modulo senza lasciare il tuo script (7ebde92d2a6, f225824ba13, 59ad01a364a, 02c1e61a2a6).

  • Assegna un nome a una camera — un alias locale all’IDE che puoi assegnare a una camera connessa, mostrato al posto del nome della porta seriale così da distinguere camere identiche (c600158ca34).

  • Pagina delle scorciatoie da tastiera — la pagina delle scorciatoie nelle Preferences è ora mostrata e filtrata sulle azioni effettivamente raggiungibili, con provenienza e disposizione più chiare (4f5eae52e8e, 2d472fe5b40).

Miglioramenti a editor, terminale e interfaccia#

  • Menu Help ristrutturato in un sottomenu OpenMV Docs (eliminando i vecchi link piatti dei pinout), con link alle note di rilascio aggiunti ai percorsi di versione delle release ufficiali (e1337ff67c4, 9eacc0994e6).

  • Finestra di dialogo About modernizzata — anno di copyright automatico, una tabella dei loghi dei partner e un link alle note di rilascio su una sola riga (d89fb74138d).

  • Il pannello Frame Buffer mostra ora la lettura della risoluzione e un’etichetta della modalità del frame buffer accanto al pulsante JPG fisso; l’etichetta della modalità è resa grigia così da non sembrare cliccabile (cb735f82337, 376b17d4e77, 39c6ffc3b26).

  • Nel pannello dell’istogramma i grafici sono smussati con spline, il pannello può comprimersi (troncando le etichette e i grafici comprimibili) e l’etichetta del canale è posta accanto alle statistiche (327324ef7e9, 9f8c7035483, d204e93d5e4).

  • Divisori dei pannelli accoppiati — trascinando un divisore con Ctrl/Cmd si spostano insieme i separatori editor/terminale e frame-buffer/istogramma, e le maniglie dei mini-separatori ottengono un alone al passaggio del mouse con un’area di presa più ampia (379dee250d6, 96b83984342).

  • Serial Terminal — lo scorrimento automatico non viene più riagganciato a ogni blocco di output, così lo scorrimento verso l’alto durante un output veloce resta fermo; il terminale può eseguire un file dal disco quando non c’è un editor (af67a076e95, bc132aef536).

  • I comandi della documentazione sono denominati in base alla pagina invece che all’indice (a3201e9a4b4).

  • Ridimensionamento High-DPI / frazionario — icone, overlay e la schermata iniziale vengono visualizzati correttamente con fattori di scala frazionari (125%, 150% e simili), e la finestra principale ricorda la propria geometria e lo schermo tra un riavvio e l’altro nelle configurazioni multi-monitor (c60711e4950, 60901b6e13f, f241d2ea8ce, 54a1bc31fbd).

Firmware, schede e strumenti#

  • Una scheda può fissare l’ultima versione firmware distribuita, così una scheda che non riceve più firmware più recente non viene segnalata come obsoleta (bd987d32f42).

  • Il testo di stato del firmware obsoleto è ora colorato di rosso per risaltare (9b18d96bac5).

  • Suggerimento per il recupero del bootloader — una mancata corrispondenza di bootloader.bin suggerisce ora di entrare nella modalità di recupero del bootloader (d46b998c76e).

  • La riproduzione video è più robusta: il percorso di FFmpeg viene mostrato quando gli strumenti non riescono ad avviarlo, il messaggio è più chiaro quando un’architettura non supporta la riproduzione, i formati di input vengono interrogati con ffmpeg -demuxers e sono stati aggiunti fallback per il terminale Linux (0cc0e74a438, a75ec4898d0, 28e1ef10326, 51651780723).

  • Dataset Capture Script aggiornato al modulo csi (41a781651e4).

  • L’aspetto della procedura guidata dell’installer di Windows è stato migliorato (250599d9d9e).

Correzioni di bug#

  • L’installazione del firmware di sviluppo per le schede .lst (AE3) prepara ora nella directory di flash i binari a cui fa riferimento il listato, e l’installazione dev/personalizzata prepara l’immagine ROMFS del bundle eliminando prima eventuali romfs*.img obsoleti, così viene effettuato il flashing del ROMFS scaricato invece di un dump residuo dell’editor (3cf56e98668, 43bedc8c591).

  • Gli avvisi di libjpeg per i frame troncati/corrotti sono filtrati dal log così che uno stream con perdita non intasi il terminale (6e2a8bed6cf, a5166abbdc1).

  • L’elenco degli esempi non mostra più l”index.json interno (14ac8ca48a7).

  • I pop-up di aggiornamento del firmware e di onboarding sono soppressi in modalità viewer così che una demo di prodotto non chieda mai di sovrascrivere il firmware del cliente (7b8528c3ad6).

  • Le scorciatoie da tastiera pendenti e solo per l’authoring rimaste dai menu rimossi sono state disabilitate (be3bee8bb58, 27d1afc8705).

Localizzazione#

  • Quattro nuove lingue — indonesiano, rumeno, thailandese e vietnamita — portando l’IDE a 37 lingue tradotte, e ogni catalogo è stato aggiornato, tradotto e ripulito in tutte le lingue (fcf04090843, c830e6c4de3, 5e22679923f).

Riga di comando#

  • Gli alias di opzione ``–word`` in stile GNU sono ora accettati insieme alle forme con un solo trattino, e l’output di -help segnala l’equivalenza (66414213dd9, 768c5d0e159).

Supporto per piattaforme e strumenti#

  • Base Qt Creator: 14.0.2.

  • OpenMV IDE aggiornato alla versione 5.0.0.

  • Novità: la variante di build OpenMV Viewer installata separatamente.

  • Supporto Python 3.14 — il parser parso incluso è stato aggiornato alla versione 0.8.7, mantenendo il completamento del codice e il language server funzionanti con la più recente sintassi Python (578a4e39d97).

Modifiche che rompono la compatibilità#

Nessuna per i progetti esistenti. Il protocollo di debug v2 si applica alle camere v5.0.0; le camere di generazione v4 continuano a usare il protocollo esistente e gli script e i flussi di lavoro esistenti non sono interessati.
