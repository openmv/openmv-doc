v5.0.0#

v5.0.0 este o versiune majoră. Introduce OpenMV Viewer — o compilare separată, doar-în-citire, pentru prezentarea unui produs finit — un nou OpenMV Cam Settings Editor, protocolul de depanare a camerei v2 pentru camerele v5.0.0 și un set amplu de îmbunătățiri ale fluxului de lucru pentru editor, terminal, firmware și viziune artificială. Este construită pe Qt Creator 14.0.2.

Saltul de versiune la 5.0.0 marchează sosirea generației de camere v5.0.0 și a instrumentelor care o susțin; proiectele și fluxurile de lucru v4 existente continuă să funcționeze neschimbate.

Repere#

  • OpenMV Viewer — o variantă de compilare nouă, instalată separat, care rulează IDE-ul ca un vizualizator de telemetrie blocat, doar-în-citire. Afișează frame buffer-ul camerei, terminalul serial și histograma și poate rula un script fix, dar ascunde instrumentele de creare, AI/model și de setări, astfel încât un client să poată demonstra un produs construit pe OpenMV fără mediul complet de dezvoltare.

  • OpenMV Cam Settings Editor — un editor grafic pentru fișierul de configurare a setărilor de pe cameră, astfel încât să poți vizualiza și modifica setările plăcii dintr-un formular în loc să editezi fișierul manual.

  • Protocolul de depanare a camerei v2 — camerele v5.0.0 folosesc un protocol de depanare nou, mai robust, cu validare CRC a antetului, recuperare a golurilor de secvență și o afișare divizată a ratei de cadre care raportează separat FPS-ul de pe cameră și FPS-ul de afișare al IDE-ului.

  • Încarcă firmware personalizat dintr-un .zip — programul de instalare a firmware-ului acceptă acum direct pachetul .zip rezultat din compilarea unei plăci, facilitând scrierea unei imagini de firmware personalizate sau construite local.

  • Jurnalizarea de depanare a protocolului în Serial Terminal — un comutator din Serial Terminal jurnalizează protocolul de depanare a camerei cu linii lizibile, etichetate cu emoji, pentru diagnosticarea problemelor de conectare.

  • Meniu Ajutor și documentație în editor refăcute — un submeniu OpenMV Docs înlocuiește vechile legături simple, F1 deschide documentația pentru simbolul Python de sub cursor, trecerea cu mouse-ul peste un nume de API afișează documentația acestuia inline, iar dialogul Despre a fost modernizat.

Funcții noi#

  • Varianta de compilare OpenMV Viewer. O compilare cu mod-vizualizator forțat, marcată și instalată ca OpenMV Viewer, cu propriul id de executabil, magazin de setări și id de pachet, astfel încât coexistă cu IDE-ul complet. Se prezintă ca un vizualizator de telemetrie doar-în-citire: Run/Stop și oprire-la-conectare funcționează și rulează un script de pe disc (prin -auto_run), meniurile File/Edit și Preferences rămân, iar recuperarea firmware/ROMFS este păstrată, în timp ce instrumentele de creare, instrumentele de viziune artificială pentru dezvoltatori, categoriile de setări pentru dezvoltatori și resursele AI/model/docs sunt eliminate din compilare și din interfață. ROMFS devine doar-de-încărcare și este promovat în meniul Tools, iar dialogurile de finalizare a actualizării firmware-ului sunt generalizate (07722e78a8b, 5cd12ee260e, b97bf5447e4, 952b86de391, 1103f7988ed, f0f60261485, c743d5d67b9, b4bcdc2d71a, a37c032b6ff, 480e5668d29, 11c064cac93).

  • OpenMV Cam Settings Editor — un editor GUI pentru fișierul de configurare a setărilor camerei, deschis din meniul Tools (issue #205) (f55a0d7fa58, 2d7f80b03cf).

  • Protocolul de depanare a camerei v2 pentru camerele v5.0.0, cu validare CRC a antetului înainte ca vreun câmp din antet să fie considerat de încredere, recuperare a golurilor de secvență care avansează peste pachetele pierdute în loc să se blocheze, și o afișare a ratei de cadre divizată în rata de pe cameră și rata de afișare a IDE-ului (c377bdbaea8, a3f0488ddcb, 29108dd40f7, c15e6c02789).

  • Încarcă firmware personalizat acceptă un pachet .zip — alegi arhiva zip rezultată din compilarea unei plăci, iar IDE-ul scrie firmware-ul (și ROMFS-ul) pe care le conține (55e4eb2d900).

  • Jurnalizarea de depanare în Serial Terminal — un buton comută jurnalizarea de depanare pe niveluri a protocolului camerei, fiecare linie fiind etichetată cu emoji și randată în fontul emoji pentru lizibilitate (09990c62810, 42e23c3c11e, 997705548c7, d2208de77d4).

  • Copiază/Convertește scriptul curent — o acțiune din Tools care copiază sau convertește scriptul curent, cu o protecție împotriva salvării unui main.py/boot.py compilat (81a954f33cd).

  • Ajutor de documentație F1 pentru simbolul Python de sub cursor, partajând rezolvatorul de simboluri cu sfaturile de la trecerea cu mouse-ul (c341330bc16).

  • Sugestii de documentație în editor — trecerea cu mouse-ul peste un nume de API OpenMV afișează documentația acestuia inline, obținută din fișierele stub .pyi livrate și din documentația proprie a modulului, astfel încât poți citi ce face o funcție, o metodă, o constantă sau un modul fără a-ți părăsi scriptul (7ebde92d2a6, f225824ba13, 59ad01a364a, 02c1e61a2a6).

  • Denumește o cameră — un alias local în IDE pe care îl poți da unei camere conectate, afișat în locul numelui portului serial, astfel încât camerele identice să poată fi deosebite (c600158ca34).

  • Pagina Scurtături de tastatură — pagina de scurtături din Preferences este acum afișată și filtrată la acțiunile pe care le poți accesa efectiv, cu o sursă și un aspect mai clare (4f5eae52e8e, 2d472fe5b40).

Îmbunătățiri ale editorului, terminalului și interfeței#

  • Meniul Help restructurat într-un submeniu OpenMV Docs (renunțând la vechile legături plate de pinout), cu legături către notele de versiune adăugate la căile versiunilor de lansare oficiale (e1337ff67c4, 9eacc0994e6).

  • Dialog About modernizat — an de copyright automat, un tabel cu logo-uri de parteneri și o legătură pe un singur rând către notele de versiune (d89fb74138d).

  • Panoul Frame Buffer afișează acum rezoluția și o etichetă a modului frame buffer lângă butonul JPG fix; eticheta modului este estompată pentru a nu părea că poate fi apăsată (cb735f82337, 376b17d4e77, 39c6ffc3b26).

  • Panoul Histogramă — graficele sunt netezite prin spline, panoul se poate comprima (eliminând etichete și grafice comprimabile), iar eticheta canalului se află lângă statistici (327324ef7e9, 9f8c7035483, d204e93d5e4).

  • Separatoare de panouri cuplate — tragerea unui separator cu Ctrl/Cmd mută împreună separatoarele editor/terminal și frame-buffer/histogramă, iar mânerele mini-separatoarelor primesc un halou la trecerea cu mouse-ul cu o zonă de prindere mai largă (379dee250d6, 96b83984342).

  • Serial Terminal — derularea automată nu mai este re-fixată la fiecare bloc de ieșire, astfel încât derularea în sus în timpul unei ieșiri rapide rămâne pe loc; terminalul poate rula un fișier de pe disc când nu există editor (af67a076e95, bc132aef536).

  • Comenzile Docs sunt denumite după pagină în loc de după index (a3201e9a4b4).

  • Scalare High-DPI / fracționară — pictogramele, suprapunerile și ecranul de întâmpinare sunt redate corect la factori de scalare fracționari (125%, 150% și altele similare), iar fereastra principală își reține geometria și ecranul între reporniri pe configurațiile cu mai multe monitoare (c60711e4950, 60901b6e13f, f241d2ea8ce, 54a1bc31fbd).

Firmware, plăci și instrumente#

  • O placă își poate fixa ultima versiune de firmware livrată, astfel încât o placă ce nu mai primește firmware mai nou să nu fie marcată ca depășită (bd987d32f42).

  • Firmware depășit — textul de stare este acum colorat în roșu pentru a ieși în evidență (9b18d96bac5).

  • Sugestie de recuperare a bootloader-ului — o nepotrivire de bootloader.bin sugerează acum intrarea în modul de recuperare a bootloader-ului (d46b998c76e).

  • Redarea video este mai robustă: calea FFmpeg este afișată când instrumentele nu o pot lansa, mesajul este mai clar când o arhitectură nu acceptă redarea, formatele de intrare sunt interogate cu ffmpeg -demuxers și au fost adăugate soluții alternative de terminal pentru Linux (0cc0e74a438, a75ec4898d0, 28e1ef10326, 51651780723).

  • Scriptul Dataset Capture actualizat la modulul csi (41a781651e4).

  • Programul de instalare pentru Windows — aspectul asistentului a fost îmbunătățit (250599d9d9e).

Corectări de erori#

  • Instalarea firmware-ului de dezvoltare pentru plăcile .lst (AE3) pregătește acum în directorul de flash binarele la care face referire listarea, iar instalarea dev/personalizată pregătește imaginea ROMFS a pachetului ștergând mai întâi orice romfs*.img învechit, astfel încât ROMFS-ul descărcat să fie scris în loc de o rămășiță din editor (3cf56e98668, 43bedc8c591).

  • Avertismentele libjpeg pentru cadrele trunchiate/corupte sunt filtrate din jurnal, astfel încât un flux cu pierderi să nu inunde terminalul (6e2a8bed6cf, a5166abbdc1).

  • Lista de exemple nu mai afișează fișierul intern index.json (14ac8ca48a7).

  • Ferestrele pop-up de actualizare a firmware-ului și de întâmpinare sunt suprimate în modul vizualizator, astfel încât o demonstrație de produs să nu ceară niciodată suprascrierea firmware-ului clientului (7b8528c3ad6).

  • Scurtăturile de tastatură orfane și cele exclusiv pentru creare rămase din meniurile eliminate au fost dezactivate (be3bee8bb58, 27d1afc8705).

Localizare#

  • Patru limbi noi — indoneziană, română, thailandeză și vietnameză — aducând IDE-ul la 37 de limbi traduse, iar fiecare catalog a fost actualizat, tradus și curățat în toate limbile (fcf04090843, c830e6c4de3, 5e22679923f).

Linia de comandă#

  • Aliasurile de opțiuni de tip GNU ``–word`` sunt acum acceptate alături de formele cu o singură liniuță, iar ieșirea -help notează echivalența (66414213dd9, 768c5d0e159).

Suport pentru platformă și instrumente#

  • Bază Qt Creator: 14.0.2.

  • OpenMV IDE — versiunea a fost ridicată la 5.0.0.

  • Nou: varianta de compilare OpenMV Viewer instalată separat.

  • Suport Python 3.14 — parserul parso inclus a fost actualizat la 0.8.7, menținând funcționale completarea codului și serverul de limbaj cu cea mai recentă sintaxă Python (578a4e39d97).

Modificări incompatibile#

Niciuna pentru proiectele existente. Protocolul de depanare v2 se aplică camerelor v5.0.0; camerele din generația v4 continuă să folosească protocolul existent, iar scripturile și fluxurile de lucru existente nu sunt afectate.
