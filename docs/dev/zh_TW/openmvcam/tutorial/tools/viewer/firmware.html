13.2.6. 韌體更新與復原#

Viewer 保留了相機的韌體管理員，因此無需完整的 IDE 也能在現場復原或重新配置裝置。它為每種開發板隨附相符的韌體版本，並為每個開發板系列的 bootloader 隨附載入器，而且會自動偵測開發板並使用正確的通訊協定，無需人工指定。

與 IDE 不同，Viewer 不會反覆催促更新：因此展示絕不會被過時提示打斷，產品出貨時所搭載的任何韌體都會維持不變。當你確實需要時，可從 Tools 選單依需求使用韌體操作。

13.2.6.1. 第三方主機板和車隊更新#

檢視器支援 third-party repositories，因此在自己的檢視器中運送自己的電路板的公司可以在現場保持一組當前的攝影機。這是無煩人規則的一個例外：第三方板 does 收到過時提示並透過檢視器進行更新，因為更新是供應商自己的 - 從儲存庫清單中的 URL 提供，而不是從 OpenMV 提供。 OpenMV自己的板子仍然沒有提示。檢視器在啟動時檢查儲存庫的韌體通道，並在供應商發佈時提供更新的韌體；它不會下拉儲存庫也可能攜帶的範例、模型或存根，因為檢視器沒有它們的介面。

13.2.6.2. 載入韌體#

Tools → Load Custom Firmware 會從磁碟燒錄韌體二進位檔——或開發板的建置輸出 .zip 套件——這是將產品自有韌體放到替換相機上的方法。該對話框提供與 IDE 相同的選項：抹除內部快閃檔案系統，以及重設 ROM 檔案系統。

13.2.6.3. 載入 ROM 檔案系統#

Tools → Load ROMFS onto OpenMV Cam 會將 .img ROM 檔案系統映像檔——產品的模型與資產——寫入相機，因此 Viewer 除了韌體之外也能還原裝置的資料。

13.2.6.4. 復原變磚的相機#

韌體損壞到不再顯示為序列埠的相機仍然有救，因為 bootloader 位於韌體之下，除了硬體損壞以外幾乎任何情況都能存活。當連線找不到相機時，Viewer 會詢問是否接上了變磚的相機，並引導你完成復原：你選擇開發板類型，Viewer 驅動該開發板的 bootloader 介面，隨附的發行韌體便會寫入。Tools → Force enter OpenMV Cam bootloader 則會為仍能連線的相機明確執行重新開機進入 bootloader 的步驟。
