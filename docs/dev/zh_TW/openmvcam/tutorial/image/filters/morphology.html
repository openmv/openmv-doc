5.15. 形態學運算¶

形態學 運算作用於二值影像——也就是閾值化與邊緣偵測所產生的遮罩。每個運算都會走過與平滑濾波器相同的那種滑動鄰域，但它在每個位置提出的問題是是/否：鄰域中的 每個 像素都是開啟的嗎？鄰域中是否有 任何 像素是開啟的？開/關的圖樣看起來如何？這些答案能以平均濾波器辦不到的方式擴張區域、收縮區域，並重新切割它們的邊界。

形態學介於 初始 二值遮罩（閾值化、邊緣偵測或其他分類器的輸出）與管線其餘部分可用的 乾淨 二值遮罩之間。原始的閾值輸出通常會有散佈在真正前景區域中的孤立雜訊像素、在原本完整的區域中打出的小孔洞，以及閾值切在物體邊緣附近時所產生的鋸齒狀邊界。形態學能去除這些缺陷。

5.15.1. 四種經典運算¶

兩個基本運算，加上它們的兩種組合，構成了形態學工具組：

dilate() 擴張 每個前景區域。其規則是：任何在其 (2 * size + 1) 視窗中至少有一個前景鄰居的像素都會變成前景。可見的效果是前景區域在每個方向上擴大 size 個像素，而其內部的 孔洞 則縮小（或消失）相同的量。

erode() 則相反。任何在其視窗中並非 每個 鄰居都屬於前景的像素都會變成背景。前景區域在每個方向上縮小 size 個像素，孤立的前景像素（沒有任何前景鄰居）完全消失，而較大區域之間的細小連接也會被切斷。

open() 是 先 erode 後 dilate。經過 erode 的影像已移除所有孤立的雜訊像素，但同時也在每個方向上縮小了 size 個像素。在 erode 之後接上相同大小的 dilate，可將真正的前景區域大致還原到原本的邊界，同時讓雜訊維持已被移除的狀態。正是這種組合讓 open 成為經典形態學中標準的「移除雜訊」運算：孤立的前景像素消失，而真正的區域則毫髮無傷地回復。

close() 是其鏡像——先 dilate 後 erode。Dilate 填滿前景區域內部的小孔洞，並連接被小縫隙分開的區域；erode 則將結果收縮回原本的外部邊界，同時讓填補後的內部維持實心。close 是標準的「填滿小縫隙」運算。

binary_mask.open(1)       # remove single-pixel noise
binary_mask.close(2)      # fill small holes and gaps

size 參數的意義與亮度濾波器中相同：size=1 代表 3×3 的鄰域，size=2 代表 5×5，依此類推。較大的 size 代表更強烈的清理——以及更長的每像素成本。

5.15.2. 頂帽與黑帽¶

另外兩種組合值得了解，因為它們擷取的 正是 open 與 close 所移除的特徵：

top_hat() 回傳原始影像與其經 open 處理後版本之間的 差異——也就是 open 本來會移除的那些前景像素。這實際上就是一張遮罩，標示出雜訊像素、孤立的小型前景區域，以及 open 運算無法保留的細薄前景結構。當這些小型前景特徵正是應用程式所關心的對象，而非想要移除的目標時，這對於 擷取 它們很有用。

black_hat() 回傳影像經 close 處理後版本與原始影像之間的 差異——也就是 close 本來會填補的那些背景像素。這是一張遮罩，標示出前景區域內部的小孔洞，以及 close 運算本來會橋接的區域間狹窄縫隙。

這兩者都不如四種基本運算常用，但這個模式值得記住——當應用程式需要擷取標準清理流程所 移除 的小型或細薄特徵時，頂帽與黑帽正是把它們找回來的自然方法。

5.15.3. 閾值模式¶

四種基本形態學運算都接受一個整數 threshold 關鍵字，用來軟化每個位置的開/關測試。沒有它時，運算的行為如上述描述：erode() 要求 每個 鄰居都開啟，dilate() 則要求 至少有一個。設定 threshold 後，每個運算可容忍那麼多個鄰居投下相反的票。對 erode 而言，threshold 是一個像素可以擁有且仍能存活的 背景 鄰居數量：threshold=4 會保留任何八個鄰居中至少有四個開啟的像素（在 3×3 視窗中，中心像素有八個鄰居），因此它的侵蝕沒那麼強烈。對 dilate 而言，threshold 是一個背景像素必須多擁有的 前景 鄰居數量，才會被開啟：threshold=2 要求至少有三個前景鄰居而非一個，因此它的擴張沒那麼強烈。

閾值形式對於在不改變視窗大小的情況下調整形態學流程的強度很有用，因為改變視窗大小也會改變它所作用特徵的 尺度。大多數應用程式維持預設行為；閾值形式則是為了那些預設行為只是稍微太多或太少的情況而存在。
