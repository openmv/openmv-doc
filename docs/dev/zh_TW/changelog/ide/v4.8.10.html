v4.8.10¶

v4.8.10 新增了對 OpenMV AE3 板子的支援，並更新了隨附的 AI 工具鏈，將 Ethos-U Vela NPU 編譯器更新至 5.0.0，並將 ST Edge AI 工具更新至 4.0.0。本次發行仍以 Qt Creator 14.0.2 為基礎，未進行重新定基（rebase），且不含任何破壞性變更。

重點摘要¶

  • OpenMV AE3 Alif Ensemble E3 板子現已獲得支援，包括 alif_tools 開機載入程式/DFU 燒錄以及逐核心的 Ethos-U55 NPU 加速。

  • Ethos-U Vela NPU 模型編譯器由 4.2.0 更新至 5.0.0，並針對 Python 3.13 ABI 重新建置。

  • ST Edge AI（stedgeai）模型轉換工具在所有平台上由 3.0.0 更新至 4.0.0。

  • 更快的連線退回： IDE 現在會在 2 次重新連線嘗試後放棄失敗的相機連線，而非 10 次。

新功能¶

  • OpenMV AE3 Alif Ensemble E3 板子支援（內部代號「AE3 classic」）。新增了 AE3 / OPENMV AE3 SBL2 板子類型，IDE 顯示名稱為 OpenMV AE3 SBL2（VID:PID 37C5:16E3），具備 alif_tools 開機載入程式/DFU 燒錄、雙核心（HP/HE）romfs 配置，以及逐核心的 Ethos-U55 Vela NPU 加速組態（高效能核心 ethos-u55-256、高效率核心 ethos-u55-128）。該板子是一個透過其開機載入程式 VID:PID 對應關係抵達的隱藏/DFU 燒錄目標，而非通常列出的可選連線目標（adbed3825）。

其他變更與改進¶

  • 將自動相機重新連線嘗試的最大次數由 10 次減少為 2 次，使 IDE 在連線失敗時更快放棄（2ddecec53）。

  • 在 Windows、Linux 與 macOS arm64 上，將隨附的 Ethos-U Vela NPU 模型編譯器由 4.2.0 更新至 5.0.0，並針對 Python 3.13 ABI 重新建置，重新編譯了 regor 與 mlw_codec 二進位檔，且將隨附的 numpy 在所有平台上更新至 2.4.4（be9d29d9d、04c73f187、93260af21、a58458cfb、a6e1ac7ed）。

  • 在 Windows、Linux 與 macOS arm64 上，將隨附的 ST Edge AI（stedgeai）模型轉換工具由 3.0.0 更新至 4.0.0（1a727567c、8dcfd8472、27aabc740）。

平台與工具支援¶

  • Qt Creator 基礎版本： 14.0.2。

  • 新板子： OpenMV AE3 SBL2（Alif Ensemble E3），一個透過 alif_tools 燒錄的隱藏/DFU 燒錄目標，具備雙核心 Ethos-U55 NPU 加速。

  • Ethos-U Vela NPU 編譯器：4.2.0 -> 5.0.0（Windows、Linux、macOS arm64），針對 Python 3.13 ABI 重新安裝。

  • ST Edge AI（stedgeai）工具：3.0.0 -> 4.0.0（Windows、Linux、macOS arm64）。

  • 隨附 numpy（Vela 環境）：在所有平台上更新至 2.4.4。

破壞性變更¶

無。本次發行未引入任何使用者可見的工作流程或行為破壞，且既有專案可繼續正常運作。
