v5.0.0#

v5.0.0 是一次重大發行。它引入了 OpenMV Viewer — 一個獨立、唯讀的建置版本，用於展示已完成的產品 — 全新的 OpenMV Cam Settings Editor、供 v5.0.0 相機使用的 v2 相機除錯協定，以及大量的編輯器、終端機、韌體與機器視覺工作流程改進。它以 Qt Creator 14.0.2 為基礎建置。

版本躍升至 5.0.0 標誌著 v5.0.0 相機世代及其支援工具的到來；現有的 v4 專案與工作流程可繼續照常運作。

重點摘要#

  • OpenMV Viewer — 一個全新、獨立安裝的建置變體，會將 IDE 以鎖定、唯讀的遙測檢視器方式執行。它會顯示相機的畫面緩衝區、序列終端機與長條圖，並可執行固定的腳本，但會隱藏編寫、AI／模型與設定工具，讓客戶不需完整的開發環境即可展示以 OpenMV 打造的產品。

  • OpenMV Cam Settings Editor — 一個圖形化編輯器，用於編輯相機板載的設定組態檔，讓你可以透過表單檢視與變更開發板設定，而不必手動編輯檔案。

  • v2 相機除錯協定 — v5.0.0 相機採用全新、更穩健的除錯協定，具備標頭 CRC 驗證、序號間隙復原，以及分離式的畫面速率顯示，可分別回報相機端 FPS 與 IDE 顯示 FPS。

  • 從 .zip 載入自訂韌體 — 韌體安裝程式現在可直接接受開發板的建置輸出 .zip 套件，讓你能輕鬆燒錄自訂或本機建置的韌體映像。

  • 序列終端機協定除錯記錄 — 序列終端機中的一個開關可將相機除錯協定以可讀、附有 emoji 標記的行記錄下來，用於診斷連線問題。

  • 重新設計的說明選單與編輯器內文件 — 新的 OpenMV Docs 子選單取代了舊有的扁平連結，F1 會開啟游標下 Python 符號的文件，將滑鼠停留在 API 名稱上會內嵌顯示其文件，並且關於對話框也已現代化。

新功能#

  • OpenMV Viewer 建置變體。 一個強制檢視器模式的建置，以 OpenMV Viewer 為名進行品牌化與安裝，具有自己的執行檔識別碼、設定儲存區與套件識別碼，因此可與完整版 IDE 並存。它以唯讀遙測檢視器的形式呈現：執行／停止與連線時停止皆可運作並會從磁碟執行腳本（透過 -auto_run），檔案／編輯選單與偏好設定保留，且韌體／ROMFS 復原功能保留，而編寫工具、開發者機器視覺工具、開發者設定類別，以及 AI／模型／文件資源則從建置與 UI 中移除。ROMFS 變為僅供載入，並移至工具選單，韌體更新完成對話方塊也已一般化 (07722e78a8b, 5cd12ee260e, b97bf5447e4, 952b86de391, 1103f7988ed, f0f60261485, c743d5d67b9, b4bcdc2d71a, a37c032b6ff, 480e5668d29, 11c064cac93).

  • OpenMV Cam Settings Editor — 一個用於編輯相機設定組態檔的 GUI 編輯器，可從工具選單開啟（issue #205） (f55a0d7fa58, 2d7f80b03cf).

  • v2 相機除錯協定，供 v5.0.0 相機使用，會在信任任何標頭欄位之前先進行標頭 CRC 驗證，具備可略過遺失封包向前推進而非停滯的序號間隙復原，以及分為相機端速率與 IDE 顯示速率的畫面速率讀數 (c377bdbaea8, a3f0488ddcb, 29108dd40f7, c15e6c02789).

  • 載入自訂韌體可接受 .zip 套件 — 選擇開發板的建置輸出 zip，IDE 便會燒錄其中包含的韌體（與 ROMFS） (55e4eb2d900).

  • 序列終端機除錯記錄 — 一個按鈕可切換相機協定的分層除錯記錄，每一行都附有 emoji 標記並以 emoji 字型算繪以提升可讀性 (09990c62810, 42e23c3c11e, 997705548c7, d2208de77d4).

  • 複製／轉換目前腳本 — 一個工具動作，用於複製或轉換目前的腳本，並具備避免儲存已編譯 main.py／boot.py 的防護 (81a954f33cd).

  • F1 文件說明，作用於游標所在的 Python 符號，與停留提示共用符號解析器 (c341330bc16).

  • 編輯器內文件工具提示 — 將滑鼠停留在 OpenMV API 名稱上會內嵌顯示其文件，內容取自隨附的 .pyi 存根與模組本身的文件，因此您無需離開指令碼即可閱讀函式、方法、常數或模組的功能說明（7ebde92d2a6、f225824ba13、59ad01a364a、02c1e61a2a6）。

  • 為相機命名 — 你可以為已連線的相機指定一個僅存於 IDE 本機的別名，顯示時會取代序列埠名稱，讓相同型號的相機可以區分開來 (c600158ca34).

  • 鍵盤快速鍵頁面 — 偏好設定中的快速鍵頁面現在會顯示，並篩選為你實際可存取的動作，來源與版面也更清楚 (4f5eae52e8e, 2d472fe5b40).

編輯器、終端機與 UI 改進#

  • 說明選單已重整**為 **OpenMV Docs 子選單（移除舊有的扁平接腳圖連結），並在官方發行版本路徑中加入發行說明連結 (e1337ff67c4, 9eacc0994e6).

  • 關於對話方塊已現代化 — 自動的版權年份、合作夥伴標誌表格，以及單行的發行說明連結 (d89fb74138d).

  • 畫面緩衝區面板現在會在固定的 JPG 按鈕旁顯示解析度讀數與畫面緩衝區模式標籤；該模式標籤呈灰色，因此看起來不像可點按 (cb735f82337, 376b17d4e77, 39c6ffc3b26).

  • 長條圖窗格的繪圖經過雲形線平滑處理，窗格可壓縮（省略標籤與可擠壓的繪圖），且通道標籤位於統計數據旁 (327324ef7e9, 9f8c7035483, d204e93d5e4).

  • 連動的面板分隔線 — 按住 Ctrl／Cmd 拖曳分隔線會讓編輯器／終端機與畫面緩衝區／長條圖的分隔器一起移動，且迷你分隔器把手在滑鼠停留時會出現光暈並提供更寬的抓取區域 (379dee250d6, 96b83984342).

  • 序列終端機 — 自動捲動不再於每次輸出區塊時重新固定，因此在快速輸出期間向上捲動能維持位置；當沒有編輯器時，終端機可從磁碟執行檔案 (af67a076e95, bc132aef536).

  • 文件指令改以頁面名稱命名，而非依索引命名 (a3201e9a4b4).

  • High-DPI／分數縮放 — 圖示、疊加層與啟動畫面在分數縮放倍率（125%、150% 等）下都能正確算繪，且在多螢幕設定中，主視窗會在重新啟動後記住其幾何位置與所在螢幕（c60711e4950、60901b6e13f、f241d2ea8ce、54a1bc31fbd）。

韌體、開發板與工具#

  • 開發板可固定其最後出貨的韌體版本，如此一來，不再收到較新韌體的開發板便不會被標示為過期 (bd987d32f42).

  • 過期韌體的狀態文字現在以紅色顯示以突顯 (9b18d96bac5).

  • 開機載入程式復原提示 — bootloader.bin 不相符時現在會提示進入開機載入程式復原模式 (d46b998c76e).

  • 影片播放更為穩健：當工具無法啟動 FFmpeg 時會顯示其路徑、當某架構不支援播放時訊息更清楚、以 ffmpeg -demuxers 查詢輸入格式，並新增了 Linux 終端機的後備方案 (0cc0e74a438, a75ec4898d0, 28e1ef10326, 51651780723).

  • 資料集擷取腳本已更新為使用 csi 模組 (41a781651e4).

  • Windows 安裝程式的精靈外觀已改進 (250599d9d9e).

錯誤修正#

  • 開發韌體安裝對於 .lst 開發板（AE3）現在會將清單所參照的二進位檔暫存至燒錄目錄，且開發／自訂安裝會暫存套件的 ROMFS 映像，同時先清除任何過時的 romfs*.img，如此便會燒錄下載的 ROMFS，而非殘留的編輯器傾印 (3cf56e98668, 43bedc8c591).

  • libjpeg 警告（針對截斷／損毀的畫面）會從記錄中過濾掉，如此有損的串流便不會灌爆終端機 (6e2a8bed6cf, a5166abbdc1).

  • 範例清單不再顯示內部的 index.json (14ac8ca48a7).

  • 韌體升級與導覽彈出視窗在檢視器模式下會被抑制，因此產品展示絕不會提示覆寫客戶的韌體 (7b8528c3ad6).

  • 從已移除選單中殘留的無效與僅供編寫用的鍵盤快速鍵已停用 (be3bee8bb58, 27d1afc8705).

在地化#

  • 四種新語言 — 印尼文、羅馬尼亞文、泰文與越南文 — 使 IDE 的翻譯語言達到 37 種，且所有語言的每個目錄都經過更新、翻譯與清理 (fcf04090843, c830e6c4de3, 5e22679923f).

命令列#

  • GNU 風格的 ``--word`` 選項別名現在可與單破折號形式並用，且 -help 輸出會註明兩者等效 (66414213dd9, 768c5d0e159).

平台與工具支援#

  • Qt Creator 基礎版本：14.0.2。

  • OpenMV IDE 版本提升至 5.0.0。

  • 新增：**獨立安裝的 **OpenMV Viewer 建置變體。

  • Python 3.14 支援 — 隨附的 parso 剖析器已更新至 0.8.7，讓程式碼自動完成與語言伺服器能持續支援最新的 Python 語法（578a4e39d97）。

破壞性變更#

對現有專案而言無。v2 除錯協定適用於 v5.0.0 相機；v4 世代相機會繼續使用現有協定，且現有腳本與工作流程不受影響。
