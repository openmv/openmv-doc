Pixy (CMUcam5)#

備註

Pixy 網站已搬遷至 pixycam.com，Pixy 及其後繼產品在該處仍受支援。本頁面保存了原始的 CMUcam5 概觀。

Pixy 是一套小巧、快速、易於使用、低成本且容易取得的視覺系統：

  • 學習偵測您所教導它的物體

  • 每秒 50 次輸出它所偵測到的內容

  • 透過隨附的傳輸線連接至 Arduino，也可與 Raspberry Pi、BeagleBone 及類似的控制器搭配使用

  • 提供適用於 Arduino、Raspberry Pi 等的所有函式庫

  • 支援 C/C++ 與 Python

  • 透過數種介面之一進行通訊：SPI、I2C、UART、USB 或類比/數位輸出

  • 設定工具程式可在 Windows、macOS 與 Linux 上執行

  • 所有軟體與韌體皆為開放原始碼，採用 GNU 授權

  • 提供所有硬體文件，包括原理圖、材料清單、PCB 佈局等等

Pixy 的起源#

Pixy (CMUcam5) 是卡內基美隆機器人研究所與 Charmed Labs 之間的合作成果。Pixy 出自源遠流長的 CMUcam 系列，但 Pixy 真正的起點是一場 Kickstarter 集資活動。它於 2014 年 3 月首次開始出貨，並迅速成為有史以來最受歡迎的視覺系統之一。Pixy 完全透過銷售來取得資金。

視覺作為一種感測器#

如果您希望您的機器人執行諸如撿起物體、追逐球，或定位充電站等工作，而且您想要用單一感測器來協助完成所有這些工作，那麼視覺就是您的感測器。視覺（影像）感測器之所以實用，是因為它們非常靈活：只要有正確的演算法，影像感測器幾乎可以感測或偵測任何東西。

但影像感測器有兩個缺點。首先，它們會輸出大量資料——每秒數十 MB——其次，處理如此大量的資料可能會使許多處理器不堪負荷。而且，即使處理器能夠跟上資料，其大部分的處理能力也將無法用於其他工作。

Pixy 透過將一顆強大的專用處理器與影像感測器搭配，來解決這些問題。Pixy 處理來自影像感測器的影像，並只將有用的資訊（例如「在 x=54, y=103 處偵測到紫色恐龍」）傳送給您的微控制器，而且它以畫格速率（50 Hz）進行此作業。

Pixy 仍是一項持續推出的產品。其目前的說明文件、軟體與支援都位於 pixycam.com。
