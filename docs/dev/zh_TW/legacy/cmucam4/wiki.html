典型用途#

CMUcam4 可用於追蹤色彩或收集基本的影像統計資料。當存在對比強烈且鮮明的色彩時，可達到最佳效能。舉例來說，它可以輕鬆地在白色背景上追蹤一顆紅球，但在光線變化的情況下，要區分不同深淺的棕色就會很困難。追蹤彩色物體可用於定位地標、循線，或追逐移動的信標。運用色彩統計，CMUcam4 可以監測場景、偵測特定色彩，或進行基本的動作偵測。如果 CMUcam4 偵測到劇烈的色彩變化，那麼場景中很可能有東西改變了。運用「line mode」，CMUcam4 可以產生彩色物體的低解析度二值影像。這可用於進行更複雜的影像處理，包括具分支偵測的循線，甚至是簡單的形狀辨識。這些較進階的操作需要自訂演算法來後處理由 CMUcam4 傳送的二值影像。就如同一般數位相機的情況，這類處理可能需要一台電腦，或至少一顆快速的微控制器。

典型組態#

CMUcam4 最常見的組態是讓它透過標準 TTL 序列埠與主處理器通訊。這個「主處理器」可以是電腦（透過 USB 或 RS232）、Arduino、Basic Stamp、PIC 或類似的微控制器。CMUcam4 夠小，足以為那些負擔不起標準電腦視覺系統之尺寸或功耗的嵌入式系統加入簡單的視覺功能。它的通訊協定被設計成即使是最慢的處理器也能相容。CMUcam4 支援各種鮑率以相容於較慢的處理器。對於更慢的處理器，CMUcam4 可以在「poll mode」下運作。在此模式下，主處理器可以只向 CMUcam4 索取單一封包的資料。這讓較慢的處理器能夠更容易地與資料保持同步。也可以使用「delay mode」命令在個別序列資料字元之間加入延遲。由於通訊延遲，poll mode 與 delay mode 都會降低一秒內可處理的畫格總數。
