13.1.20. Conclusão¶

O IDE num parágrafo: os scripts são editados num editor profissional que conhece a API da câmara, executados na câmara com um botão, e observados através de três instrumentos em tempo real – o visualizador do buffer de fotograma para o que a câmara vê, o histograma para os números por detrás disso, e o terminal série para o que o script diz. Em torno desse ciclo situam-se as ferramentas de manutenção que mantêm em ordem o firmware, o sistema de ficheiros e o ROMFS de uma câmara; as ferramentas de visão por computador que produzem os artefactos que os scripts consomem – tuplos de limiar, descritores tratados, etiquetas impressas, modelos convertidos, conjuntos de dados com etiquetas; e as ferramentas avançadas que analisam o desempenho em profundidade.

Para onde ir a partir daqui:

  • O menu de exemplos é a resposta padrão a «como faço X na câmara» – quase todas as funcionalidades da biblioteca têm um exemplo executável.

  • A referência da biblioteca documenta cada módulo que o popup de conclusão oferece.

  • O pacote Python openmv controla uma câmara a partir de scripts Python do lado do anfitrião – o protocolo de depuração do IDE sem o IDE – para bancadas de teste, automação e frontends de ambiente de trabalho personalizados.

  • O capítulo de produção retoma onde o passo de implementação do IDE fica: integrar scripts no firmware, distribuir recursos em ROMFS e preparar uma câmara para condições de campo.
