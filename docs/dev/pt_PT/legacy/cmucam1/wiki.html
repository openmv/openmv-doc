Utilizações Típicas#

A CMUcam1 pode ser utilizada para seguir ou monitorizar cores. O melhor desempenho pode ser conseguido quando existem cores altamente contrastantes e intensas. Por exemplo, pode facilmente seguir uma bola vermelha sobre um fundo branco, mas seria difícil diferenciar entre diferentes tons de castanho com luz variável. Seguir objetos coloridos pode ser utilizado para localizar pontos de referência, seguir linhas ou perseguir balizas em movimento. Utilizando estatísticas de cor, é possível à CMUcam1 monitorizar uma cena, detetar uma cor específica ou fazer deteção de movimento primitiva. Se a CMUcam1 detetar uma mudança drástica de cor, então é provável que algo na cena tenha mudado. Utilizando o «line mode», a CMUcam1 pode gerar imagens binárias de baixa resolução de objetos coloridos. Isto pode ser utilizado para fazer um processamento de imagem mais sofisticado que inclui deteção de ramificações, ou mesmo reconhecimento simples de formas. Estas operações mais avançadas requerem algoritmos personalizados para pós-processar as imagens binárias enviadas pela CMUcam1. Tal como acontece com uma câmara digital normal, este tipo de processamento poderá requerer um computador ou pelo menos um microcontrolador rápido.

Configuração Típica#

A configuração mais comum para a CMUcam1 é fazê-la comunicar com um processador mestre através de uma porta série RS232 padrão. Este «processador mestre» pode ser um computador, PIC, Basic Stamp, Handy Board, Brainstem ou microcontrolador semelhante. A CMUcam1 é suficientemente pequena para adicionar visão simples a sistemas incorporados que não podem suportar o tamanho ou a potência de um sistema de visão padrão baseado em computador. O seu protocolo de comunicação foi concebido para acomodar até os processadores mais lentos. Se o seu dispositivo não tiver uma porta série totalmente com conversão de nível, também pode comunicar com a CMUcam1 através da sua porta série TTL. Esta é igual a uma porta série normal, exceto que os dados são transmitidos utilizando lógica não invertida de 0 a 5 volts. A CMUcam1 suporta vários baud rates para acomodar processadores mais lentos. Para processadores ainda mais lentos, a CMUcam1 pode funcionar em «poll mode». Neste modo, o processador anfitrião pode pedir à CMUcam1 apenas um único pacote de dados. Isto dá aos processadores mais lentos a capacidade de se manterem mais facilmente sincronizados com os dados. Também é possível adicionar um atraso entre caracteres individuais de dados série utilizando o comando «delay mode». Devido aos atrasos de comunicação, tanto o poll mode como o delay mode reduzirão o número total de frames que podem ser processados num segundo. As resoluções dos frames são afetadas pelos atrasos série e pelo baud rate.
