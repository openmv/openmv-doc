Pixy (CMUcam5)#

Nota

O site do Pixy mudou-se para pixycam.com, onde o Pixy e os seus sucessores continuam a ser suportados. Esta página preserva a visão geral original da CMUcam5.

O Pixy é um sistema de visão pequeno, rápido, fácil de usar, de baixo custo e prontamente disponível:

  • Aprende a detetar objetos que lhe ensina

  • Produz o que deteta 50 vezes por segundo

  • Liga-se ao Arduino com o cabo incluído, e também funciona com Raspberry Pi, BeagleBone e controladores semelhantes

  • Todas as bibliotecas para Arduino, Raspberry Pi, e assim por diante, são fornecidas

  • C/C++ e Python são suportados

  • Comunica através de uma de várias interfaces: SPI, I2C, UART, USB, ou saída analógica/digital

  • O utilitário de configuração funciona em Windows, macOS e Linux

  • Todo o software e firmware é de código aberto, licenciado sob GNU

  • Toda a documentação de hardware, incluindo esquemas, lista de materiais, layout de PCB, e assim por diante, é fornecida

Como o Pixy começou#

O Pixy (CMUcam5) é uma parceria entre o Carnegie Mellon Robotics Institute e a Charmed Labs. O Pixy vem de uma longa linhagem de CMUcams, mas teve o seu verdadeiro início como uma campanha no Kickstarter. Começou a ser enviado em março de 2014 e rapidamente se tornou num dos sistemas de visão mais populares alguma vez feitos. O Pixy é financiado exclusivamente através das vendas.

Visão como um sensor#

Se quiser que o seu robô execute uma tarefa como apanhar um objeto, perseguir uma bola ou localizar uma estação de carregamento, e quiser um único sensor para ajudar a realizar todas estas tarefas, então a visão é o seu sensor. Os sensores de visão (imagem) são úteis porque são muito flexíveis: com o algoritmo certo, um sensor de imagem pode detetar praticamente qualquer coisa.

Mas existem dois inconvenientes com os sensores de imagem. Primeiro, produzem muitos dados – dezenas de megabytes por segundo – e segundo, processar esta quantidade de dados pode sobrecarregar muitos processadores. E se o processador conseguir acompanhar os dados, grande parte da sua capacidade de processamento não estará disponível para outras tarefas.

O Pixy resolve estes problemas ao emparelhar um poderoso processador dedicado com o sensor de imagem. O Pixy processa as imagens do sensor de imagem e envia apenas a informação útil (por exemplo, «dinossauro roxo detetado em x=54, y=103») para o seu microcontrolador, e faz isto à taxa de fotogramas (50 Hz).

O Pixy continua a ser um produto ativo. A sua documentação, software e suporte atuais encontram-se em pixycam.com.
