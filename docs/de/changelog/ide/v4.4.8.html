v4.4.8¶

v4.4.8 ist ein Wartungs-Release, das die mitgelieferte Firmware und Tools aktualisiert. Es baut weiterhin auf Qt Creator 14.0.2 auf, ohne Rebase in diesem Bereich. Die mitgelieferte OpenMV-Firmware wechselt auf v4.6.0, erstmals wird STM32N6-Firmware (OpenMV Cam N6) mitgeliefert, neue thermische FLIR-Boson-Sensoren werden erkannt, und das Alif Security Toolkit wird aktualisiert. Es gibt keine für Nutzer sichtbaren inkompatiblen Änderungen, und bestehende Nutzerprojekte sind nicht betroffen.

Höhepunkte¶

  • Mitgelieferte Firmware v4.6.0: Die mitgelieferte OpenMV-Firmware und die Beispiele werden von v4.5.9 auf v4.6.0 aktualisiert (66bfd486f).

  • OpenMV Cam N6-Firmware mitgeliefert: STM32N6-Firmware (OpenMV Cam N6) wird nun erstmals im Bundle ausgeliefert (66bfd486f).

  • Thermische FLIR-Boson-Sensoren: Die thermischen Sensoren BOSON, BOSON-320/BOSON-640 sowie BOSON-320+/BOSON-640+ (Boson Plus) werden nun von der IDE erkannt (8b795053c, d5068b286).

  • Alif Security Toolkit v1.101.0: Das mitgelieferte Windows-Toolkit wird auf v1.101.0 aktualisiert (7d51b00b1).

  • OpenMV Cam AE3 SBL-Flashen behoben: Das AE3-Board führt nun beim Flashen den korrekten Secure Bootloader aus (ed7c95357).

Neue Funktionen¶

  • FLIR-Boson-Beispiele: Mitgelieferte Beispielskripte für FLIR Boson Graustufen- und Ironbow-Streaming sowie Farbverfolgung wurden unter 01-Camera/05-Thermal-Cameras/02-FLIR-Boson hinzugefügt, ebenso ein GenX320-Graustufen-LED-Tracking-Beispiel unter 01-Camera/03-Event-Cameras/02-Genx320 (66bfd486f).

Weitere Änderungen und Verbesserungen¶

  • Der mitgelieferte Beispielbaum wurde neu organisiert: Die FLIR-Lepton-Beispiele wurden in eine neue Kategorie 01-Camera/05-Thermal-Cameras/01-FLIR-Lepton neben die neuen Boson-Beispiele verschoben, und die Event-Kamera-Beispiele wurden in die Unterordner 01-Camera/03-Event-Cameras/01-Frogeye2020 und 02-Genx320 aufgeteilt (66bfd486f).

  • Der ungenutzte, verwaiste Firmware-Ordner OPENMV4_PRO wurde aus dem Bundle entfernt; kein Board-Eintrag verwies darauf, sodass kein flashbares Board betroffen ist (66bfd486f).

  • Die mitgelieferte OpenMV AE3 SBL-Firmware, der Bootloader und die zum Flashen verwendeten signierten Images wurden aktualisiert (b8c06a8ab).

  • Das mitgelieferte Alif Security Toolkit (Windows) wurde auf v1.101.0 aktualisiert, einschließlich des überarbeiteten Benutzerhandbuchs (AUGD0005) und der signierten SP-Images (7d51b00b1).

Fehlerbehebungen¶

  • Das OpenMV Cam AE3 SBL-Flashen wurde behoben, sodass der korrekte Secure Bootloader ausgeführt wird, indem die Board-Architektur-Zeichenkette auf OPENMV AE3 SBL gesetzt wurde (ed7c95357).

  • Das blhost-Flash-Lösch-Timeout wurde beim Programmieren von i.MX RT-Boards von 60 s auf 120 s erhöht, um Timeouts beim Löschen und Flashen größerer Bereiche zu verhindern (b48e139e4).

Plattform- und Tool-Unterstützung¶

  • Qt Creator-Basis: 14.0.2.

  • Mitgelieferte OpenMV-Firmware: aktualisiert von 4.5.9 auf 4.6.0 (66bfd486f).

  • OpenMV Cam N6: STM32N6-Firmware (firmware.bin, network.bin, openmv.bin) wird nun unter firmware/OPENMV_N6 mitgeliefert (66bfd486f).

  • OpenMV AE3 SBL: Firmware- und Bootloader-Binärdateien aktualisiert (b8c06a8ab).

  • Alif Security Toolkit (Windows): aktualisiert von v1.100.0 auf v1.101.0 (7d51b00b1).

  • Neu erkannte Sensoren: FLIR BOSON, BOSON-320, BOSON-640 (8b795053c), FLIR BOSON-320+ und BOSON-640+ (d5068b286) sowie der PixArt PS5520 (66bfd486f).

Inkompatible Änderungen¶

Keine. Es gibt keine Änderungen an der Scripting-API, und bestehende Nutzerprojekte sind nicht betroffen. Der entfernte Firmware-Ordner OPENMV4_PRO war ungenutzt und wurde von keinem Board referenziert, sodass seine Entfernung keinen Flash-Workflow unterbricht.
